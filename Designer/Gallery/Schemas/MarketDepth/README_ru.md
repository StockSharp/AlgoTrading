# Пример обработки стакана заявок в StockSharp Strategy Designer

## Обзор

Данный пример иллюстрирует схему работы со стаканом заявок в StockSharp Strategy Designer. Стакан заявок (market depth), или «биржевой стакан», содержит информацию о ордерах на покупку и продажу по разным ценовым уровням для конкретного инструмента. Он необходим для стратегий, требующих анализа динамики спроса и предложения на различных ценовых уровнях в реальном времени.

## Описание схемы

Схема состоит из нескольких взаимосвязанных компонентов, предназначенных для получения, обработки и отображения данных стакана заявок:

1. **Узел инструмента**: этот узел представляет [инструмент](https://doc.stocksharp.com/topics/designer/strategies/using_visual_designer/elements/data_sources/variable.html) (например, акцию, фьючерс или другой финансовый инструмент), для которого будет получен стакан заявок. Это ключевой элемент, определяющий, какой рынок или инструмент анализируется.

2. **Узел TimeFrameCandle**: обрабатывает [данные свечей](https://doc.stocksharp.com/topics/designer/strategies/using_visual_designer/elements/data_sources/candles.html) по инструменту, агрегированные за заданный таймфрейм (в примере — 5 минут). Может использоваться для сопоставления изменений в стакане заявок с ценовыми движениями во времени.

3. **Узлы стакана заявок**: предназначены для захвата изменений [стакана заявок](https://doc.stocksharp.com/topics/designer/strategies/using_visual_designer/elements/market_depths/order_book.html) в реальном времени и реагирования на них. Включают настройки обработки входящих данных стакана для получения актуальной информации о ордерах на покупку и продажу.

4. **Узел панели графика**: предполагает визуализацию данных свечей на [графике](https://doc.stocksharp.com/topics/designer/strategies/using_visual_designer/elements/common/chart.html). Помогает трейдерам и алгоритмам лучше понять рыночную ситуацию и принимать обоснованные решения.

5. **Узел панели стакана заявок**: специально предназначен для отображения данных стакана в [специальной панели](https://doc.stocksharp.com/topics/designer/strategies/using_visual_designer/elements/market_depths/order_book_panel.html) с такими функциями, как подсветка лучших цен спроса и предложения, визуализация глубины рынка и другими возможностями.

## Рабочий процесс

- **Узел инструмента** поставляет данные, которые используются как входные данные для **узла TimeFrameCandle** и **узла стакана заявок**.
- **Узел TimeFrameCandle** обрабатывает эти данные для формирования свечей за заданный таймфрейм, которые могут применяться для анализа тренда или других видов технического анализа.
- **Узел стакана заявок** обрабатывает стакан в реальном времени для указанного инструмента. Может использоваться для принятия торговых решений на основе определённых условий, например, значительного дисбаланса ордеров на покупку и продажу на определённых ценовых уровнях.
- Визуализация осуществляется через **узел панели графика** и **узел панели стакана заявок**, обеспечивая доступность данных как для торговых алгоритмов, так и для просмотра трейдером.

## Практическое применение

Данная схема может использоваться в различных торговых стратегиях, включая:
- **Высокочастотный трейдинг (HFT)**, где незначительные изменения в динамике ордербука могут сигнализировать о потенциально прибыльных сделках.
- **Арбитражные стратегии**, предполагающие сравнение стаканов заявок на нескольких биржах для использования ценовых расхождений.
- **Маркет-мейкинговые стратегии**, где понимание обеих сторон стакана заявок критически важно для выставления соответствующих ордеров на покупку и продажу.

## Заключение

Схема, представленная в файле JSON, демонстрирует комплексный подход к обработке данных стакана заявок в StockSharp Strategy Designer. Интеграция обработки данных в реальном времени с продвинутыми инструментами визуализации помогает трейдерам и алгоритмам принимать быстрые решения на основе данных о состоянии ордербука. Этот пример служит надёжной основой для разработки более сложных торговых стратегий, требующих глубокого понимания рыночной динамики.
