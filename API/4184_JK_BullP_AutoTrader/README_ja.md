# JK BullP AutoTrader 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
JK BullP AutoTrader は、もともと MetaTrader 4 用に書かれた勢いに従うエキスパート アドバイザーです。エルダー ブルズ パワーを監視します。
インジケーターを示し、強気の圧力が弱まるかマイナスに転じたときに反応します。 StockSharp ポートは、
オリジナルでありながら、明示的なパラメータ、詳細なトレーリング管理、プラットフォームに優しいリスク制御を提供します。

## 取引ロジック
1. この戦略は、設定可能なローソク足シリーズ (デフォルトでは 1 時間足のローソク足) をサブスクライブし、13 期間の指数関数を計算します。
移動平均 (EMA) はブルズパワーのベースラインを再現します。
2. 完成したローソクごとに、ブルズパワーはローソクの高値と EMA の値の差として測定されます。
3. 2 つの連続するブルズパワーの読み取り値が比較されます。
   - 以前の値が最新の値を上回り、最新の値がプラスのままの場合、ストラテジーはショート ポジションをオープンします。
   - 最新のブルズパワー値がゼロを下回った場合、戦略はロングポジションをオープンします。
4. 一度にアクティブにできるポジションは 1 つだけで、取引が開かれている間に新しい注文をブロックした元の MQL エキスパートを反映しています。

## リスク管理と出口
- **最初のストップロス/テイクプロフィット:** 距離はピップ単位で設定され、証券価格ステップを使用して価格単位に変換されます。
どちらの保護も、StockSharp の `StartProtection` ヘルパーを通じて有効になり、動作を MetaTrader 入力に近づけます。
- **トレーリングストップ:** 変動利益が指定されたトレーリング距離を超えると、ストップレベルがローソクごとに移動します。
(MetaTrader のように) 既存のストップ注文を変更する代わりに、ポートは価格が上昇したときにポジションを決済する成行注文を発行します。
末尾のしきい値を超えて終了します。これにより、会場が保護命令に対応していない場合でも、適時退場が保証されます。

## パラメーター
| 名前 | 説明 | デフォルト |
| --- | --- | --- |
| `OrderVolume` | エントリーに使用される成行注文のサイズ。 | 8.5 |
| `TakeProfitPips` | 利益確定距離 (pips 単位) (価格単位に変換)。 | 500 |
| `StopLossPips` | ピップス単位のストップロス距離。 | 20 |
| `TrailingStopPips` | トレーリングストップを有効にして維持するピップ単位の利益距離。 | 10 |
| `EmaPeriod` | Bulls Power の計算で使用される EMA の長さ。 | 13 |
| `CandleType` | 戦略を推進するローソク足のデータ タイプ (デフォルトの 1 時間の時間枠)。 | 1時間キャンドル |

## 実装メモ
- 元のスクリプトの未使用の入力 (`Patr`、`Prange`、`Kstop`、`kts`、`Vts`) は、意図的に省略されました。
MetaTrader ロジックには影響しません。
- ピップ距離は機器 `PriceStep` に依存します。ステップデータが利用できない場合、値 `1` が保守的なデフォルトとして使用されます。
- この戦略は、StockSharp の高レベルの `Bind` API を使用し、終了したキャンドルのみを処理し、内部状態 (`_previousBullsPower`) を保持します。
MT4 シフトベースの計算と一致するようにします。
- 新しいポジションがオープンされたときにストップレベルが古くならないように、トレーリングロジックは各終了後に自動的にリセットされます。
