# トレーリングストップ戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、オリジナルのMQLスクリプト`TRAILING.mq4`からトレーリングストップロジックを実装したものです。既存のオープンポジションを管理し、市場が指定した利益目標に達するかストップロスに触れた場合にポジションを決済します。トレーリングパラメーターが有効な場合、ストップレベルが価格に追随して利益を確定します。

## パラメーター
- **TakeProfit** – エントリー価格からの絶対価格単位での利益距離。
- **StopLoss** – エントリー価格からの最大許容逆行距離。
- **Trailing** – ストップレベルの動的トレーリングに使用する距離。
- **CandleType** – 価格更新の取得に使用するローソク足シリーズ。

## 仕組み
1. 戦略は選択したローソク足シリーズを購読します。
2. 各完了ローソク足の後、現在のポジションを評価します。
3. ロングポジションの場合、利益が*TakeProfit*を超えるか損失が*StopLoss*を超えるとポジションを決済します。
4. *Trailing*がゼロより大きい場合、ストップレベルが価格とともに上昇します。価格がトレーリングストップを下回るとポジションが決済されます。
5. ショートポジションは同じロジックを逆方向に適用します。
6. エントリー価格は最初の約定から記録され、ポジション決済時にリセットされます。

## 注記
- この戦略はローソク足処理に`Bind`を使った高レベルAPIを使用します。
- 自ら新しいポジションを開かず、すでに開かれたポジションのみを管理します。
- パラメーターは`StrategyParam`を介して公開されており、最適化できます。
