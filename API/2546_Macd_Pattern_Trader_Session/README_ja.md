# MACD PatternTrader セッション戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、MetaTraderエキスパートアドバイザー **MacdPatternTraderAll0.01** の直接変換です。6つの異なるMACDベースのエントリーパターン、オプションの取引時間フィルタリング、部分的な利益確定、およびマーチンゲールポジションサイジングオプションを使用して単一の銘柄を取引します。すべての計算は、設定された`CandleType`によって提供される完成したローソク足で実行されます。

## 取引ロジック
1. すべての完成したローソク足において、戦略は6つのMACDインジケーターを更新します（各パターンには独自の速いEMA、遅いEMAの長さと1期間のシグナルラインがあります）。
2. 取引時間フィルタリングが有効な場合、新しい取引は`SessionStart`と`SessionEnd`の間でのみ評価されます。リスク管理は常にアクティブです。
3. 各MACDパターンは、現在のMACD値と前の2つの値の間の非常に特定の値の関係をチェックして、モメンタムリバーサルを検出します。パターンがトリガーされると、対応する方向に成行注文を送信し、内部ストップロスとテイクプロフィットレベルを設定します。
4. ストップロスは、設定可能なルックバックの最近の極値（ショートの最高値、ロングの最低値）に価格ステップで測定されたオフセットを加減したものとして計算されます。テイクプロフィットは、元のエキスパートの再帰的なターゲット検索を再現するためにブロック内の古いローソク足グループをスキャンします。
5. 一度に管理されるのは1つのネットポジションのみです。反対方向に新しいシグナルが現れると、現在のポジションがクローズされ、マーチンゲール調整済みボリュームで逆ポジションがオープンされます。
6. アクティブなポジションは`ManageActivePosition`によって監視されます。ロジックはオリジナルの部分クローズルーティンをエミュレートします：
   - ロングの場合：利益が`ProfitThreshold`（5通貨単位）を超え、前の終値が中期EMAを上回ると、ポジションの3分の1が売却されます。利益が続き、前の高値が長期SMAと非常に遅いEMAの平均を上回ると、残りのポジションの半分がクローズされます。
   - ショートの場合：対称ルールが、利益目標と移動平均フィルターが満たされたときに、3分の1、次に残りの半分をクローズします。
7. リスク管理は取引ウィンドウに関係なくすべてのローソク足で実行されます。価格がローソク足内（高値/安値に基づく）で保存されたストップロスまたはテイクプロフィットレベルを突破すると、ポジション全体が突破価格でフラット化されます。
8. 取引が完全にクローズされた後、実現PnLが評価されます。`UseMartingale`が有効な場合、損失取引は次の注文ボリュームを2倍にし、一方で有益な出口はボリュームをベース`LotSize`にリセットします。

## 主要パターン
- **パターン1:** `Pattern1MaxThreshold`を超えて下落し始めるMACDスパイクと、`Pattern1MinThreshold`を下回ってバウンスするドロップを検出します。
- **パターン2:** 最小限の偏差でのゼロライン周辺のMACD交差を探します。
- **パターン3:** 2段階の閾値（`Pattern3MaxThreshold`、`Pattern3SecondaryMax`、`Pattern3MinThreshold`、`Pattern3SecondaryMin`）を使用して、両側での3段階のロールオーバーを検出します。また、元の`bars_bup`の累積を模倣するために、二次最大値を超えた連続バーをカウントします。
- **パターン4:** MACDが主要閾値を超えるが前のバーがより狭い二次範囲内にある場合に取引し、リバーサルを予期します。
- **パターン5:** `Pattern5PrimaryMax/Min`と二次制限によって定義される狭い範囲内でのMACDの素早いフリップに応答します。
- **パターン6:** 取引をトリガーする前に複数の連続したMACD偏差を要求するために、カウンター（`Pattern6MaxBars`、`Pattern6MinBars`、`Pattern6CountBars`）を使用します。

## リスク管理
- 内部ストップロスとテイクプロフィットターゲットは各エントリーに対して再計算されます。ストップは価格ステップで測定されたオフセットに加えた価格の極値を使用します。テイクプロフィットは極値が改善しなくなるまでローソク足の連続ブロックを検索し、MQLエキスパートの再帰的ロジックを再現します。
- 部分エグジットは元の最小ロットサイズ（0.01）を尊重し、各方向で何回の部分クローズが実行されたかを追跡します。
- 戦略はブローカー側の保護注文を配置しません；代わりに、設定された価格でポジションをクローズするためにローソク足の高値と安値を監視します。

## パラメーター
| パラメーター | 説明 | デフォルト |
| --- | --- | --- |
| `CandleType` | インジケーターと取引シグナルに使用するローソク足シリーズ。 | 1時間足 |
| `LotSize` | マーチンゲール調整前のベース取引ボリューム。 | 0.1 |
| `UseTimeFilter` | `SessionStart`と`SessionEnd`の間のみ取引を有効にします。 | true |
| `SessionStart` / `SessionEnd` | 取引ウィンドウ（地元の取引所時間）。 | 07:00 / 17:00 |
| `UseMartingale` | 損失取引後に`LotSize`を2倍にします。 | true |
| `Ema1Period`, `Ema2Period`, `SmaPeriod`, `Ema3Period` | 部分エグジットに使用する移動平均。 | 7, 21, 98, 365 |
| パターン固有パラメーター | 各パターンは、元のエキスパートの入力に合致する独自の有効化フラグ、ストップロス/テイクプロフィットルックバック、オフセット、EMA長さ、および閾値値を持ちます。 | コンストラクターのデフォルト値を参照 |

すべての閾値とEMA長さは`StrategyParam`オブジェクトを通じて公開され、最適化またはファインチューニングが可能です。

## 注意事項
- 戦略は、オフセットと利益を口座通貨に変換するために銘柄が`PriceStep`と`PriceStepCost`を提供することを前提としています。利用できない場合は価格差が直接使用されます。
- ストップとターゲットは内部でシミュレートされます；バーのクローズ時に評価されます。リアルタイムでのイントラバー実行はMetaTraderの動作と異なる場合があります。
- マーチンゲールメカニズムは損失連続後にエクスポージャーを急速に増加させる可能性があります—注意して使用してください。
