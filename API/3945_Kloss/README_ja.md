# クロス MQL/8186 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

**Kloss MQL/8186 戦略**は、MetaTrader 4 エキスパート アドバイザー `Kloss.mq4` を直接変換したものです。これは、商品チャネル指数 (CCI)、Stochastic オシレーター、シフトされた典型的な価格フィルターを組み合わせて、単一ポジションの反転を時間指定します。 StockSharp バージョンは、高レベルのキャンドル サブスクリプション API を使用しながら、元のエントリーしきい値、ストップロスとテイクプロフィットの距離、ボリューム ロジック (固定ロット サイズまたはパーセンテージ ベースのサイジング) を維持します。

## 取引ロジック

- **データ**: 設定された時間枠 (デフォルトは 5 分) の完了したローソク足。インジケーターは同じ系列で計算されます。
- **指標**:
  - CCI、期間は 10。絶対値は `±CciThreshold` (デフォルトは 120) と比較されます。
  - `%K=5`、`%D=3` を備えた Stochastic オシレーター、`=3` を平滑化します。メインの `%K` ラインは、売られすぎ/買われすぎのバンドに対してチェックされます。
  - 典型的な価格 ((高値 + 安値 + 終値) / 3 ) は、エキスパートアドバイザーからのシフトされた LWMA を複製するために、完成した 5 本のローソク足によって遅延されます。
- **ロングエントリー**:
  - CCI <= `-CciThreshold`。
  - Stochastic %K < `StochasticOversold` (デフォルトは 30)。
  - 以前のローソク足の始値 > 5 つ前のローソク足の通常価格。
  - 既存のロングポジション (`Position <= 0`) はありません。オープンなショートはすべてクローズされ、単一の成行注文でロングに反転されます。
- **簡単なエントリー**:
  - CCI >= `CciThreshold`。
  - Stochastic %K > `StochasticOverbought` (デフォルトは 70)。
  - 前回のローソク足の終値 < 5 つ前のローソク足の通常価格。
  - 既存のショートポジション (`Position >= 0`) はありません。オープンしているロングはすべてクローズされ、1 つの成行注文でショートに反転されます。
- **ポジション管理**: StockSharp の `StartProtection` は、指定されたポイント距離を使用してストップロス注文と利食い注文を自動的に発行します。それ以外の場合、戦略は常に単一のポジション (フラット、ロング、またはショート) を保持します。

## ポジションサイズ

- **固定ボリューム**: `FixedVolume > 0` の場合、ストラテジーは常にその正確なボリュームを取引します (商品の `VolumeStep` および `MinVolume` に合わせた後)。
- **リスク パーセント**: `FixedVolume = 0` の場合、ストラテジーは口座価値の `RiskPercent` (デフォルト 0.2) を最新の終値で割って注文サイズを推定します。ボリュームは `MaxVolume` (デフォルトは 5) によってクランプされ、楽器のステップに丸められます。
- **安全策**: 口座情報が欠落している場合、または計算された値が正でない場合、このメソッドは最小取引可能量に戻ります。

## パラメーター

| 名前 | 説明 | デフォルト |
| ---- | ----------- | ------- |
| `CciPeriod` | 商品チャネル指数の計算に使用されるローソク足の数。 | 10 |
| `CciThreshold` | エントリをトリガーする絶対的な CCI レベル。 | 120 |
| `StochasticKPeriod` | Stochastic オシレーターの %K 周期。 | 5 |
| `StochasticDPeriod` | %D 平滑化期間。 | 3 |
| `StochasticSmooth` | 追加の平滑化が信号の前の %K に適用されます。 | 3 |
| `StochasticOversold` | 長いエントリを確認するための %K しきい値。 | 30 |
| `StochasticOverbought` | 短いエントリを確認するための %K しきい値。 | 70 |
| `StopLossPoints` | 保護停止の価格ポイントの距離。 | 48 |
| `TakeProfitPoints` | 利益目標の価格点の距離。 | 152 |
| `FixedVolume` | 正の値を指定すると、取引量が固定されます。 | 0 |
| `RiskPercent` | `FixedVolume` がゼロの場合に、ポートフォリオの割合を体積に変換します。 | 0.2 |
| `MaxVolume` | 許可される最大取引量。 | 5 |
| `CandleType` | インジケーター計算のローソク足の種類/時間枠。 | 5分の時間枠 |

## 実行メモ

- **単一ポジション**: 1 つのポジションのみがオープンのままです。反転では、既存のポジションをクローズし、単一の成行注文で新しいポジションをオープンします。
- **インジケーターの同期**: 価格シフトには、最後に完了した 5 つのローソク足が使用されます。最初の取引が表示される前に、少なくとも 6 つのローソク足が処理される必要があります。
- **ストップ/ターゲット**: `StartProtection` は、商品の `PriceStep` を使用して、ポイントベースの距離を絶対価格オフセットに変換します。 `PriceStep` が不明な場合は、生のポイント値が適用されます。
- **データ要件**: OHLC キャンドルを提供するあらゆる機器で動作します。ボリューム アライメントは、利用可能な場合は `MinVolume` と `VolumeStep` を尊重します。
- **MT4 との違い**: MetaTrader の証拠金の計算は、アカウントの資本 (`Portfolio.CurrentValue`) によって概算されます。株式データが利用できない場合、戦略は最小取引可能量に戻ります。

## 使用のヒント

1. `CandleType` を、MetaTrader で使用されるマーケット セッションに合わせて調整します (元のテンプレートでは M5)。
2. ティックサイズに応じた停止距離を確認します。ポイントから価格への変換は自動的に行われますが、非外国為替商品に合わせて値を調整する必要がある場合があります。
3. 固定契約サイズの場合は、`FixedVolume` を希望のロットに設定し、`RiskPercent` をゼロに設定します。
4. 新しいシンボルの戦略を調整するときに、インジケーターのしきい値の最適化を有効にします。
