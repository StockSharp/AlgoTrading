## 概要
**PosNegDiCrossoverStrategy** は、MetaTrader エキスパート `_HPCS_PosNegDIsCrossOver_Mt4_EA_V01_WE` の StockSharp 移植です。の
元のシステムは、平均方向指数 (ADX) の +DI ラインと -DI ラインの間のクロスオーバーをリッスンし、すぐに
新しいリーダーの方向にポジションをオープンします。各ポジションは対称的なストップロスとテイクプロフィットによって保護されます
しきい値はピップ単位で測定され、トレードに負けるとマーチンゲール形式の回復ループがトリガーされ、ボリュームが倍増して再び入ります。
一定の試行回数に達するか、有利な終了が発生するまで。

## 取引ロジック
1. **シグナル検出** – 完成したローソク足が新しい ADX 値を提供すると、戦略は現在の +DI と -DI を比較します。
以前のものと合わせて読みます。 +DIが-DIを上回った場合に強気のシグナルが発生し、次の場合に弱気のシグナルが発生します。
+DI が -DI を下回ります。バーごとに最初のエントリーは 1 つだけ、重複取引を防止する MQL ガードをミラーリングできます。
同じキャンドル。
2. **時間フィルター** – エントリは、ユーザー定義の日次ウィンドウ内でのみ許可されます。窓の外では戦略はうまくいき続ける
アクティブなポジション（仮想ストップと利益確定）はありますが、新しいサイクルを開始したり、マーチンゲール数列を継続したりすることはありません。
3. **注文の発注** – 成行注文は、設定された基本ボリュームで検出された方向に送信されます。埋めた後、
戦略は、商品ステップを使用して、`TakeProfitPips` と `StopLossPips` を絶対価格に変換します (10 倍の乗数は
小数点以下 3 桁または 5 桁で引用される金融商品に適用され、手動終了チェックのためにそれらのレベルを保存します。
4. **保護の取り扱い** – 完成したキャンドルはそれぞれ検査されます。安値がストップを突き抜けた場合、または、
高い目標に到達します。ショートポジションは対称条件を使用します。手仕舞いは成行注文で実行されるため、サイクルは
次のステップを決定する前に結果を評価できます。
5. **Martingale ループ** – 損失後、戦略は現在のボリュームを `MartingaleMultiplier` 倍し、サイクルを増分します。
カウンターし、すぐに同じ方向に再エントリーします（取引ウィンドウを考慮して）。収益性の高いExitが発生したとき、または
試行回数が `MartingaleCycleLimit` に達すると、サイクルはベース ボリュームにリセットされ、次の ADX クロスオーバーを待ちます。

## パラメーター
| 名前 | デフォルト | 説明 |
| ---- | ------- | ----------- |
| `CandleType` | 15分の時間枠 | ADX の計算とストップ/ターゲットのモニタリングに使用されるローソク足シリーズ。 |
| `AdxPeriod` | 14 | 平均方向指数インジケーターの長さ。 |
| `UseTimeFilter` | `true` | 毎日の取引ウィンドウを有効にします。 |
| `StartTime` | 00:00 | 取引セッションの開始（為替時間）。 |
| `StopTime` | 23:59 | 取引セッションの終了（為替時間）。 |
| `OrderVolume` | 0.1 | 各サイクルの初期成行注文量。 |
| `TakeProfitPips` | 10 | 利益目標までの距離 (pips 単位) (商品ステップを使用して価格に変換)。 |
| `StopLossPips` | 10 | ピップ単位での保護停止までの距離。 |
| `MartingaleMultiplier` | 2 | マーチンゲール ループ中の各損失取引の後に適用される出来高乗数。 |
| `MartingaleCycleLimit` | 5 | 同じ信号に対して許可されるマーチンゲール再エントリの最大数。 |
