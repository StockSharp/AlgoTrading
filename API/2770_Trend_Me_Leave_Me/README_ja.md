# Trend Me Leave Me 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**Trend Me Leave Me** 戦略は、Yury Reshetov による古典的な MQL5 エキスパートアドバイザーの直接移植です。静かな価格
アクションの期間を辛抱強く待ち、パラボリック SAR が示す支配的な方向に参加し、利益のある出口の後にトレード方向を交互
に切り替えます。トレードがストップされた場合、戦略は同じ方向を再度試み、元の「trend me, leave me」の動作を再現します。
この C# 実装は StockSharp 高レベル API を使用し、ソースシステムの完全な意思決定フローを維持しながら、すべての数値入力
を設定可能なパラメーターとして公開します。

## 主要なアイデア
### 静市場フィルター
- `AdxPeriod` 長さの Average Directional Index (ADX) が方向の強さを測定します。
- ADX の移動平均が `AdxQuietLevel` を下回った場合のみ、戦略は新規エントリーを許可し、EA の低ボラティリティ押し目への
  フォーカスを模倣します。

### タイミングのための SAR アライメント
- パラボリック SAR のポイントが方向ガイドとして機能します。ロングシグナルはローソク足のクローズが SAR ドットを上回る
  必要があり、ショートシグナルはドットを下回るクローズが必要です。
- SAR パラメーター `SarStep` と `SarMax` は MQL バージョンの加速設定に一致し、必要に応じて最適化できます。

### 方向スケジューラー
- `TradeDirections` フラグは元の `cmd` 変数を表します。*買い*状態から始まります。
- **テイクプロフィット**出口後、フラグは反対側に切り替わり、リバーサルトレードを誘います。
- **ストップロス**（またはブレークイーブン）出口後、フラグは同じ側に留まり、次の機会が前の方向を再試行します。

## トレード管理
- `StopLossPips` と `TakeProfitPips` は平均約定価格からの固定距離を定義します。いずれかのパラメーターを `0` に設定
  すると、対応する保護が無効になります。
- `BreakevenPips` は、市場が指定された pip 距離だけ有利な方向に動いた後、ストップをエントリー価格に移動します。価格
  が後でエントリーレベルに戻った場合、トレードはほぼゼロの利益でクローズされ、次のシグナルが同じ側に留まります。
- ストップ/テイクロジックは、完成した各ローソク足で高値と安値の両方を使用してバー内のヒットを近似し、バー駆動環境で
  EA のティックバイティック動作をできるだけ忠実に保ちます。

## ポジションサイジング
- 注文数量はベース `Strategy.Volume` プロパティで制御されます。例では MQL スクリプトの固定リスク資金管理オブジェクト
  を含まないシンプルなリスクモデルを維持します。より高度なサイジングが必要な場合は `Volume` を調整するか、戦略を
  オーバーライドしてください。

## パラメーター
| パラメーター | 説明 | デフォルト |
|------------|------|----------|
| `StopLossPips` | エントリー価格と保護ストップ間の pips 距離。 | `50` |
| `TakeProfitPips` | エントリー価格とターゲット間の pips 距離。 | `180` |
| `BreakevenPips` | この数の有利な動きの pips 後にストップをエントリーに移動します。 | `5` |
| `AdxPeriod` | ADX フィルターの平滑化期間。 | `14` |
| `AdxQuietLevel` | 静かな市場と判断される最大 ADX 読み取り値。 | `20` |
| `SarStep` | パラボリック SAR の加速ステップ。 | `0.02` |
| `SarMax` | パラボリック SAR の最大加速係数。 | `0.2` |
| `CandleType` | 計算に使用される時間軸。 | `1h` ローソク足 |

## 実装上の注意
- pip 計算は EA のデジット調整に従います：インストゥルメントが 3 または 5 桁の小数点を使用する場合、価格ステップを
  10 で乗算してブローカーのティックサイズを標準 pip に変換します。
- インジケーターのバインディングは StockSharp 高レベル API に依存し、すべてのトレードアクションは S# の規約に準拠
  するために `BuyMarket`/`SellMarket` を使用します。
- Python 翻訳はまだ含まれていません。要求通り `PY/` ディレクトリは意図的に不在です。
- StockSharp がサポートするシンボルに戦略を適用してください。戦略を開始する前に `Volume` を設定し、インストゥルメント
  のボラティリティに合わせてパラメーターを調整してください。
