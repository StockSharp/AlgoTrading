# 価格インパルス戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

価格インパルス戦略は生のLevel1気配値をスキャンし、最良買い気配と最良売り気配の間の急激な変化に反応します。設定可能な数のティックにわたる価格ジャンプを監視し、移動がポイントベースの閾値を超えた場合に市場へ参入することで、オリジナルのMetaTrader 5エキスパートアドバイザーを再現します。保護用のストップロスとテイクプロフィットのオフセットは、高レベルのヘルパー`StartProtection`を通じて自動的に適用されます。

このアプローチは市場中立です。売り気配が古い気配値と比較して上昇した場合にロングポジションを建て、買い気配が以前の値を下回った場合にショートポジションを建てます。設定可能なクールダウンにより、指定されたスリープ間隔を待機するMQL実装と同様に、取引後すぐに再エントリーすることを防ぎます。

## 仕組み

- Level1データを購読し、最良買い気配と最良売り気配の価格のローリング履歴を保存します。
- 最新の気配値と`HistoryGap`ティック前に到着した気配値との価格差を計算します（`ExtraHistory`で定義される追加バッファを含む）。
- 売り気配が`ImpulsePoints * PriceStep`を超えて上昇し、ロングエクスポージャーが存在しない場合にロングポジションを建てます。
- 買い気配が同じ閾値を超えて下落し、ショートエクスポージャーが存在しない場合にショートポジションを建てます。
- 価格ポイントで表された固定のテイクプロフィットとストップロスレベルを適用し、注文間に`CooldownSeconds`の一時停止を強制します。

## パラメータ

- **OrderVolume** – 各成行注文で送信されるボリューム。ソースロボットに合わせるためデフォルトは`0.1`ロットですが、他の銘柄向けに最適化できます。
- **StopLossPoints** – エントリー価格から保護ストップまでの距離、銘柄ポイントで測定されます。`0`の値はストップを無効にします。
- **TakeProfitPoints** – テイクプロフィット目標までの距離、同じくポイントで測定されます。`0`の値は目標を無効にします。
- **ImpulsePoints** – エントリーをトリガーするために、現在の気配値と`HistoryGap`ティック前の気配値の間で超えなければならない最小価格インパルス（ポイント）。
- **HistoryGap** – 現在価格と比較ベースラインを分けるLevel1更新の数。値が高いほど大きなルックバックが必要で、ノイズを平滑化しますがエントリーが遅れます。
- **ExtraHistory** – コールバック間に複数のティックが到着した場合の気配値の急増を吸収するためにローリングバッファに保持される追加のLevel1サンプル。履歴配列をオーバーサンプリングするMT5実装と一貫した動作を維持します。
- **CooldownSeconds** – 別のエントリーを配置できるようになるまでの取引後の最小待機時間。MQLエキスパートの`InpSleep`パラメータを戦略が反映することを保証し、急速なフリップフロッピングを防ぎます。

## 注意事項

- ポイント距離パラメータは`Security.PriceStep`（またはフォールバックとして`Security.MinPriceStep`）を使用して自動的に変換されるため、同じ設定が異なるティックサイズに適応します。
- 戦略がオンライン状態で、履歴バッファに十分なデータが含まれ、インパルス条件が満たされた場合にのみ取引が開始されます。
- 決定は生の気配値更新で行われるため、信頼性の高いLevel1フィードを持つ流動性の高い銘柄で最もよく機能します。
- この戦略にはPythonポートはありません。ユーザーの要求に応じてC#バージョンのみが提供されます。
