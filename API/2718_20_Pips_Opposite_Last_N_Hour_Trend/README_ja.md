# 直近N時間トレンドへの反対方向20ピップス戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

このStockSharp戦略はMetaTraderエキスパートアドバイザー
**«20 Pips Opposite Last N Hour Trend»**の高レベル移植版です。時間足ローソク足を観察し、
直前の`N`時間の価格の動きを測定し、設定された取引時間が終了したときに
反対方向にポジションをオープンします。取引は
固定20pipsテイクプロフィットターゲットと時間制限で管理され、
連続した損失後にはマーチンゲールスタイルのボリュームスケールが適用されます。

実装はStockSharpのローソク足サブスクリプション、パラメーターシステム、
注文ヘルパー（`BuyMarket`, `SellMarket`）を使用しているため、
Designer、API、Runner、Shellのいずれでも変更なしに実行できます。

## トレードロジック

- 戦略は選択されたローソク足タイプ（デフォルト：1時間バー）をサブスクライブします。
- 各完成したローソク足について、終値を内部履歴に保存します。
- `OpenTime.Hour == TradingHour`のローソク足が完成し、十分な
  履歴が利用可能な場合：
  - `HoursToCheckTrend`バー前に発生した終値と
    前の終値（1バー前）を比較します。
  - その窓で価格が下落した場合（弱気ドリフト）、戦略は買います；
    価格が上昇した場合（強気ドリフト）は売ります。同じ終値の場合は取引をスキップします。
- 1日に1つの取引のみがオープンされ、設定された取引時間のみに限られます。
  他のすべてのローソク足は純粋に管理のためのみに使用されます。

## ポジション管理

- 20pipsターゲット（3/5桁シンボルに対して調整済み）はエントリー直後に
  計算されます。完成したローソク足の高値/安値がターゲットに触れたことを示すと、
  ポジションはそのレベルで閉じられます。
- ターゲットが次の時間中に達成されなければ、ポジションは
  オーバーナイトのエクスポージャーを避けるため、次のローソク足の終わりに閉じられます。
- 新しい取引日が始まると日次カウンターが自動的にリセットされ、
  次の資格のあるシグナルが次のセッションで発動できるようになります。

## 資金管理

- `Volume`はベース注文サイズを設定します。`MaxVolume`はマーチンゲールステップの結果サイズを上限にします。
- 損失後の退場に続いて、戦略は適切な乗数によって次のポジションを増加させます：
  最初の損失 → `FirstMultiplier`、2番目の損失 →
  `SecondMultiplier`など。5回を超える連続損失は5番目の
  乗数を再使用します。いかなる利益またはトントンのクローズもシーケンスをリセットします。
- ボリューム計算は最後に実行されたポジション価格に依存するため、
  完全なブローカーPnLデータなしでも利益/損失検出が決定論的に維持されます。

## パラメーター

| パラメーター | デフォルト値 | 説明 |
|-----------|------------|------|
| `MaxPositions` | 9 | 1日に許可される最大取引数。0に設定すると取引が無効になります。 |
| `Volume` | 0.1 | 連続の最初の取引のベースボリューム。 |
| `MaxVolume` | 5 | 乗数後の調整されたボリュームのハードキャップ。 |
| `TakeProfitPips` | 20 | pipsでのテイクプロフィット距離。0でTPを無効にします。 |
| `TradingHour` | 7 | ポジションをオープンする資格がある時刻（0-23）。 |
| `HoursToCheckTrend` | 24 | 直前のトレンドを測定するために使用される時間足終値の数。 |
| `FirstMultiplier` | 2 | 最初の連続損失後に適用される乗数。 |
| `SecondMultiplier` | 4 | 2番目の連続損失後に適用される乗数。 |
| `ThirdMultiplier` | 8 | 3番目の連続損失後に適用される乗数。 |
| `FourthMultiplier` | 16 | 4番目の連続損失後に適用される乗数。 |
| `FifthMultiplier` | 32 | 5番目以降の連続損失から適用される乗数。 |
| `CandleType` | H1 | シグナル生成と管理に使用されるローソク足データタイプ。 |

## 追加の注意事項

- pip サイズは`Security.PriceStep`と小数点以下の桁数から計算されるため、
  20pipsターゲットは4桁および5桁のFXシンボルで正しく動作します。
- `StartProtection()`は戦略開始時に呼び出され、StockSharpの組み込み
  保護（無制限ポジションの自動ストップ、ポートフォリオリセット）を有効にします。
- ロジックは完成したローソク足のみを使用し、インジケーター値を
  直接読み取ることはなく、`AGENTS.md`のガイドラインに適合しています。

> **リスク免責事項：** マーチンゲールスタイルのポジションサイジングは
> 実質的なドローダウンにつながる可能性があります。ライブ取引に展開する前に、
> 常にヒストリカルデータでパラメーターをテストし、慎重なリスク制限を使用してください。
