# 過去 N 時間のトレンド戦略とは反対の 10 ピップス
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

この戦略は、MetaTrader のエキスパート **10pipsOnceADayOppositeLastNHourTrend** を忠実に移植したものです。設定可能な時間に 1 日に 1 回正確に取引され、最後の *N* 回の完了した時間足ローソク足で観察された価格変化の反対側を意図的に取得します。このロジックは 5 桁の価格設定の通貨ペア向けに設計されていますが、C# バージョンでは商品の `PriceStep` と小数点以下の桁数を使用してピップ サイズが自動的に調整されます。

選択した取引時間に、戦略は `HoursToCheckTrend` 時間前の終値を検査し、それを最後に完了した時間足ローソク足の終値と比較します。

- 古い終値が**高い**場合、市場は下落している(弱気)ため、この戦略は**ロング**ポジションをオープンします。
- それ以外の場合、市場は上昇 (強気) しているため、**ショート** ポジションをオープンします。

ポジションは、保護ストップ、毎日の時間ベースの決済、または市場が取引ウィンドウの外にあるときに手動で決済されます。

## お金の管理

ポジションのサイジングは、元のエキスパートのマーチンゲール ラダーを反映しています。

1. 基本ボリュームは `FixedVolume` から取得されます。ゼロに設定すると、戦略は小数点第 1 位に四捨五入された `Portfolio.CurrentValue * MaximumRisk / 1000` を使用したリスクベースのサイジングに戻ります。
2. ボリュームは、`MinimumVolume`、`MaximumVolume`、商品のボリューム制限、および `Portfolio.CurrentValue / 1000` ロットに等しいソフト キャップによって制限されます。
3. 各取引が終了すると、結果が保存されます (最後の 5 回の取引まで)。新しいエントリーを準備するとき、戦略はその履歴をスキャンし、`FirstMultiplier` … `FifthMultiplier` シーケンスを使用して、見つかった最初の損失に応じてロットサイズを乗算します。これにより、MQL バージョンのネストされた `OrderSelect` チェックが再現されます。

## リスク管理

- `StopLossPips`、`TakeProfitPips`、および `TrailingStopPips` は pip 単位で動作します。ポートは、外国為替シンボルの標準的な 3/5 10 進数の乗数を使用してピップ サイズを再計算します。
- トレーリングストップはロングポジションとショートポジションで対称的です。元の MQL コードでは、符号エラーのため短辺トレイルはトリガーされませんでした。 C# バージョンではこの問題が修正され、両方向が同じように動作します。
- `OrderMaxAge` は、設定された期間 (デフォルトでは 21 時間) を超えて存続するポジションを決済します。
- 許可された取引時間外では、この戦略は未処理のエクスポージャーを清算して、次のセッションまで横ばいを維持します。
- `MaxOrders` は、新しいシグナルが評価されるときにオープンポジションやアクティブな注文がないことを要求することで、偶発的な再エントリーを防ぎます。

## 詳細なワークフロー

1. 時間足ローソク足を購読します (時間枠は `CandleType` で変更できます)。
2. 完成した各ローソク足の終値を小さなローリング バッファーに収集します。
3. 許可された時間に最初に完成したキャンドルについて:
   - ポートフォリオ/接続状態をチェックし、オープンなポジションがないことを確認します。
   - 比較する過去のローソク足が少なくとも `HoursToCheckTrend` 個あることを確認してください。
   - 現在の終値と `HoursToCheckTrend` バー前の終値を比較して方向を決定します。
   - 上記の資金管理ルーチンを使用してロットサイズを計算し、成行注文を送信します。
4. ポジションがオープンの間、戦略は次のとおりです。
   - ローソク足の高値/安値を使用して、ストップロス、テイクプロフィット、およびトレーリングレベルを評価します。
   - 新しい高値 (ロングの場合) または安値 (ショートの場合) の後にトレーリング ストップを更新します。
   - エントリのタイムスタンプを追跡して、`OrderMaxAge` を強制できるようにします。
   - マーチンゲール乗数に供給するために、取引が終了したときに実現利益/損失を記録します。

## パラメーター

| パラメータ | 説明 | デフォルト |
|-----------|-------------|---------|
| `FixedVolume` | 固定ロットサイズ。リスクベースのサイジングを使用するには、`0` に設定します。 | `0.1` |
| `MinimumVolume` | 注文量の厳密な下限。 | `0.1` |
| `MaximumVolume` | 注文量の厳密な上限。 | `5` |
| `MaximumRisk` | `FixedVolume = 0` の場合に使用される資本の割合。 | `0.05` |
| `MaxOrders` | 最大同時注文/ポジション。 | `1` |
| `TradingHour` | 新しい取引が許可される時刻 (0 ～ 23)。 | `7` |
| `HoursToCheckTrend` | 傾向を比較するための時間単位のルックバック ウィンドウ。 | `30` |
| `OrderMaxAge` | ポジションの最大存続期間。 | `21h` |
| `StopLossPips` | ピップス単位のストップロス距離。 | `50` |
| `TakeProfitPips` | 利益確定距離 (pips)。 | `10` |
| `TrailingStopPips` | トレーリングストップの距離 (pips)。 | `0` (無効) |
| `FirstMultiplier` … `FifthMultiplier` | それぞれの深度で最新の負け取引が見つかった場合に適用されるロット乗数。 | `4`, `2`, `5`, `5`, `1` |
| `CandleType` | キャンドルのサブスクリプションの期間。 | `1 hour` |

## 元の MQL 専門家との違い

- Martingale のサイジング、注文エージング、取引ウィンドウのロジックは 1 対 1 で再現されます。唯一の意図的な変更は、元のスクリプトの符号のバグを修正するために対称的な短辺のトレーリング ストップです。
- 高レベルのヘルパーを使用する場合、StockSharp ストラテジーでは個別のストップ/リミット注文が登録されないため、すべての保護レベルは次に終了したローソク足で成行注文で実行されます。これは、ストップ注文がトリガーされたときの元のエキスパートの動作と一致します。
- アカウントの資本は `Portfolio.CurrentValue` から読み取られます。アダプターがこのフィールドを提供しない場合、戦略はベースの `Volume` (デフォルトは `1`) に戻ります。
- 許可される取引時間のリストは、`0…23` の元の配列を反映しています。取引を特定の日に制限するには、コンストラクター内で `_tradingDayHours` を編集します。

## 使用上の注意

- `PriceStep` ×10 ヒューリスティックを使用したピップ サイズ計算が有効な時間単位の外国為替データで最適に機能します。
- 戦略でロットサイズを正しく調整できるように、`Security.VolumeStep`、`VolumeMin`、および `VolumeMax` がコネクタによって設定されていることを常に確認してください。
- エントリーは完成したローソク足ごとに 1 回だけ評価されるため、その日の最初のシグナルを見逃さないように、選択した取引時間前に戦略を開始する必要があります。
