# MACD マルチタイムフレーム・エキスパート戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、StockSharp フレームワーク内でオリジナルの MetaTrader ロボット「MACD Expert」を再現します。5分、15分、1時間、4時間の4つの時間軸でMACD トレンドを同期し、すべての時間軸が同じ方向を指している場合のみ新しいポジションを許可します。目標は、高スプレッドの期間をフィルタリングしながら、マルチタイムフレームのモメンタムの一致を捉えることです。

## データとインジケーター
- **ロウソク足**: 5m（実行）、15m、1h、4h の確認。すべてのロウソク足は終値を使用し、完成したバーのみ。
- **インジケーター**: デフォルト 12/26/9 の `MovingAverageConvergenceDivergenceSignal`。各時間軸は独自の MACD インスタンスを持ち、シグナルが干渉しないようにします。
- **レベル1クォート**: 取引を開く前にライブスプレッドを監視するために最良の売値/買値クォートを使用します。

## 取引ロジック
1. 4つすべての MACD インスタンスが完成した値を発行するのを待ちます。
2. 各時間軸での MACD ラインとシグナルラインの関係を計算します。
3. 価格ポイント（価格ステップ）で測定された最大スプレッドフィルターを適用します。
4. 一度に最大1つのポジションのみを開きます；新しい注文が許可される前に既存のポジションはストップロスまたはテイクプロフィットで終了しなければなりません。

### ロングセットアップ
- **すべての**監視される時間軸で MACD シグナルラインが MACD ラインより上にある。
- スプレッドが `MaxSpreadPoints` を超えない。
- 最新の5分ロウソク足の終値で `OrderVolume` ロットのロングポジションが開かれます。

### ショートセットアップ
- **すべての**監視される時間軸で MACD シグナルラインが MACD ラインより下にある。
- スプレッドが `MaxSpreadPoints` を超えない。
- 最新の5分ロウソク足の終値で `OrderVolume` ロットのショートポジションが開かれます。

### ポジション管理
- ロング取引は論理的なターゲットをエントリーより `TakeProfitPoints` 上に、ストップをエントリーより `StopLossPoints` 下に設定します。
- ショート取引は論理的なターゲットをエントリーより `TakeProfitPoints` 下に、ストップをエントリーより `StopLossPoints` 上に設定します。
- 完成した5分ロウソク足のインtraバーの高値/安値がそれぞれのターゲットまたはストップレベルに触れると決済がトリガーされます。
- ポジション中、戦略は反対のシグナルを無視します；オリジナルの MQL ロジックと一致するように、ストップまたはテイクプロフィットで取引が閉じられるまで再び反応するのを待ちます。

## パラメーター
| 名前 | デフォルト | 説明 |
| --- | --- | --- |
| `OrderVolume` | 0.1 | ロット単位のポジションサイズ（MQL バージョンの `Lots` 入力を反映）。 |
| `StopLossPoints` | 200 | 価格ポイントでの保護ストップまでの距離。 |
| `TakeProfitPoints` | 400 | 価格ポイントでの利益目標までの距離。 |
| `MaxSpreadPoints` | 20 | エントリーがスキップされる前の価格ポイントでの最大許容スプレッド。 |
| `FastPeriod` | 12 | 各 MACD インスタンス内の高速 EMA の長さ。 |
| `SlowPeriod` | 26 | 各 MACD インスタンス内の低速 EMA の長さ。 |
| `SignalPeriod` | 9 | 各 MACD インスタンス内のシグナル EMA の長さ。 |
| `FiveMinuteCandleType` | 5分ロウソク足 | 主要な実行時間軸。 |
| `FifteenMinuteCandleType` | 15分ロウソク足 | 最初の確認時間軸。 |
| `HourCandleType` | 1時間ロウソク足 | 2番目の確認時間軸。 |
| `FourHourCandleType` | 4時間ロウソク足 | 3番目の確認時間軸。 |

## 実装上の注意
- プロジェクトのガイドラインに従って、`GetValue` を呼び出さずに強く型付けされた MACD 値を読み取るために `BindEx` を使用します。
- 共有ヘルパーが MACD/シグナルの関係を `{-1, 0, 1}` フラグに変換して確認チェックを簡素化します。
- スプレッド検証は最良の売値から最良の買値を `Security.PriceStep` で割り、しきい値が MetaTrader の「ポイント」動作に一致するようにします。
- 取引イベントはデバッグを支援するために `LogInfo` で記録され、Designer または Runner でテストするときに役立ちます。
- タスク要件に従って Python 変換は提供されておらず、C# バージョンのみが含まれています。
