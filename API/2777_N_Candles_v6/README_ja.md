# N Candles v6 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**N Candles v6** 戦略は最新の完成したローソク足を監視し、同じ方向の連続を探します。市場が `N` 本の強気ローソク足を連続して印刷すると、戦略はロングポジションを開き、一連の `N` 本の弱気ローソク足はショートエントリーを生成します。ロジックは MetaTrader のエキスパートアドバイザー *N Candles v6.mq5* からインスピレーションを得て、StockSharp 高レベル API に適応されています。

アルゴリズムは標準的な時間ベースのローソク足を提供する任意のシンボルに対して設計されています。設定可能な取引ウィンドウにより、望ましいセッション外で戦略を非アクティブに保ちますが、アクティブなトレーリングと出口ロジックは、ブロックされた時間中でもオープンポジションを保護し続けます。

## 取引ロジック
1. 設定されたローソク足タイプをサブスクライブし、完成したローソク足のみを処理します。
2. 連続した強気（`Close > Open`）と弱気（`Close < Open`）のローソク足を数えます。ドージはカウンターをリセットします。
3. `CandlesCount` 本の強気ローソク足が現れた場合：
   - 予測されるネットポジションが `MaxPositionVolume` を下回ることを確認します。
   - 成行買い注文を送ります。ショートポジションが存在する場合、1 つのトレードでポジションをロングにフリップするために注文サイズが増加されます。
4. `CandlesCount` 本の弱気ローソク足が現れた場合：
   - 新しいショートエクスポージャーが `MaxPositionVolume` を超えないことを確認します。
   - 成行売り注文を送り、ロングポジションをクローズする必要がある場合は注文を拡大します。
5. 最新のローソク足が連続を破った場合（「黒い羊」）：
   - 選択された `ClosingMode` を適用して、すべて、反対方向、または同じ方向のポジションを一度クローズします。
6. トレーリングと保護的な出口は各ローソク足で実行されます：
   - ストップロスとテイクプロフィットのレベルは pip 距離とインストゥルメントの価格ステップから導出されます。
   - トレーリングストップは価格が `TrailingStopPips + TrailingStepPips` 分動いた後に活性化し、有利な方向にのみラチェットします。
   - ストップ、テイクプロフィット、またはトレーリングレベルの違反は即座に全ポジションをクローズします。

## リスク管理
- **ストップロス（pips）** – シンボルの価格ステップを使用して pip 距離を絶対価格オフセットに変換します（5 桁および 3 桁の商品は自動的にスケーリングされます）。
- **テイクプロフィット（pips）** – 指定したサイズの有利な動きの後にポジションをクローズします。
- **トレーリングストップ / ステップ（pips）** – トレードが設定された利益閾値に達したら動的な保護を有効にします。トレーリングがアクティブなときはステップはゼロ以外でなければなりません。
- **最大ポジション数量** – 絶対ネットポジションを制限します。上限を超えるシグナルは無視されます。
- **クローズモード** – 非適合ローソク足が現れた時の反応を決定します：
  - `All` – 全ポジションをフラットにします。
  - `Opposite` – 連続の方向に対して反対のポジションをクローズします（例：強気の連続が破れた後のショートをクローズ）。
  - `Unidirectional` – 連続の方向のポジションのみをクローズします。
- **取引ウィンドウ** – ローソク足の開始時間の時間が `StartHour` と `EndHour` の間（包含）にある場合にのみ新しいトレードを開きます。保護的な出口は新しいトレードがブロックされていても動作し続けます。

## パラメーター
| 名前 | デフォルト | 説明 |
|------|---------|-----|
| `CandlesCount` | 3 | シグナルに必要な同一ローソク足の数。 |
| `OrderVolume` | 0.01 | ベース成行注文サイズ。反対のエクスポージャーは新しいトレードを確立する前にクローズされます。 |
| `TakeProfitPips` | 50 | pips 単位のテイクプロフィット距離。`0` はターゲットを無効にします。 |
| `StopLossPips` | 50 | pips 単位のストップロス距離。`0` はストップを無効にします。 |
| `TrailingStopPips` | 10 | pips 単位のトレーリングストップ距離。`0` はトレーリングを無効にします。 |
| `TrailingStepPips` | 4 | トレーリングレベルが移動する前の最小価格改善。トレーリングが有効なときは > 0 でなければなりません。 |
| `MaxPositionVolume` | 2 | 最大絶対ネットポジション。 |
| `UseTradingHours` | true | 取引ウィンドウフィルタリングを有効にします。 |
| `StartHour` | 11 | 取引セッションの開始（0-23）。 |
| `EndHour` | 18 | 取引セッションの終了（0-23）。 |
| `ClosingMode` | All | 黒い羊ローソク足が現れた時の動作。 |
| `CandleType` | 1 時間足 | シグナル生成に使用されるデータタイプ。 |

## 注意事項
- pip 変換はインストゥルメントの `PriceStep` に基づいています。5 桁および 3 桁の引用については、戦略は伝統的な pip 定義に合わせるためにステップを 10 倍します。
- StockSharp のセーフガードサービス（停止時キャンセル、再接続の安全性など）を有効にするために、起動時に `StartProtection()` を呼び出します。
- ロジックはネットポジション（`Strategy.Position`）を使用するため、ネッティング口座で正しく動作します。高い `MaxPositionVolume` を設定することでヘッジングスタイルの動作をエミュレートできます。
