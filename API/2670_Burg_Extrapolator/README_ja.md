# Burg Extrapolator 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

Burg Extrapolator戦略は、StockSharpの高レベルAPIを使用してMetaTraderエキスパートアドバイザー「Burg Extrapolator」を再現します。システムはBurg法で解かれた自己回帰（AR）モデルを適用して将来の始値を予測します。取引の判断は予測パスの振幅によって決まります：将来の高値と安値の間の予測距離が設定されたしきい値を超えると、戦略はポジションを開くか閉じます。

## コアロジック

1. **データ準備**
   - 完了した各ローソク足で`PastBars`の始値を収集します。
   - オプションで系列を対数モメンタムまたは変化率の値に変換します。
   - 生の価格を使用するとき、ローリング平均を引くことで価格を正規化します。
2. **自己回帰モデリング**
   - `ModelOrderFraction`によって決まる次数でBurg法によりAR係数を推定します。
   - 数ステップ先まで外挿し（予測ホライゾン = `PastBars - order - 1`）、価格予測を再構築します。
3. **シグナル生成**
   - 予測された最大価格と最小価格を追跡します。
   - 予測スイングが`MinProfitPips`を超えると、それぞれの方向にエントリーシグナルを生成します。
   - 予測スイングが`MaxLossPips`を超えると、既存ポジションに対して決済シグナルを発行します。
4. **注文実行**
   - ポジションはリスクベースで計算されたボリュームを使用した成行注文で開かれます。
   - ストップまたは逆方向シグナルが発生すると、戦略は成行注文でポジションを閉じます。

## パラメーター

| パラメーター | 説明 |
|------------|------|
| `RiskPercent` | 取引ごとにリスクにさらす資本のパーセンテージ。ストップロス距離が利用可能な場合に注文のサイズを計算するために使用します。 |
| `MaxPositions` | 方向ごとに許可される注文サイズの倍数として表された最大累積ボリューム。 |
| `MinProfitPips` | 新しいポジションを開くために必要な最小予測利益スイング（pips単位）。 |
| `MaxLossPips` | ポジション決済をトリガーする最大許容予測ドローダウン（pips単位）。 |
| `TakeProfitPips` | 静的テイクプロフィット距離（pips単位）。無効にするにはゼロに設定。 |
| `StopLossPips` | 静的ストップロス距離（pips単位）。リスクサイジングに必要。 |
| `TrailingStopPips` | トレーリングストップ距離（pips単位）。ストップロスが有効な場合にのみ機能します。 |
| `PastBars` | Burgモデルへの入力として使用する過去バーの数。 |
| `ModelOrderFraction` | AR次数を定義する`PastBars`の分数（整数切り捨て）。 |
| `UseMomentum` | 対数モメンタム前処理（`log(p[i]/p[i-1])`）を有効にします。 |
| `UseRateOfChange` | モメンタムが無効のとき、変化率前処理（`p[i]/p[i-1]-1`）を有効にします。 |
| `OrderVolume` | リスクベースのサイジングが計算できないときのフォールバック注文サイズ。 |
| `CandleType` | 計算に使用するローソク足のデータタイプ（時間軸）。 |

## 取引ルール

- **エントリー**: 予測パスが`MinProfitPips`より大きいスイングを示すとき、最も高い予測価格が最初に来る場合はロングポジションを開くか、最低の予測が最初に現れる場合はショートポジションを開きます。
- **エグジット**: 予測スイングが`MaxLossPips`を超えるか、逆方向のエントリーシグナルが検出されたときにポジションを閉じます。
- **保護**: pipsから導出された絶対価格単位でオプションのストップロス、テイクプロフィット、トレーリングストップを設定するために`StartProtection`を使用します。
- **ポジションサイジング**: `StopLossPips`と`RiskPercent`の両方が正の場合、取引ボリュームは`risk_amount / (stop_distance)`として計算されます。それ以外の場合は`OrderVolume`が使用されます。

## 実装メモ

- 先読みバイアスを避けるために完成したローソク足のみで動作します。
- `Bind`コールバック内で値を直接処理することでインジケーター`GetValue`コールを避けます。
- リスク管理のために`SubscribeCandles`と`StartProtection`を使用して、StockSharp高レベルAPIの慣例を守ります。
- トレーリングロジックはプラットフォーム管理のトレーリングストップを有効にすることで元のEAを反映します。

## 使用のヒント

- `PastBars`と`ModelOrderFraction`を慎重に選んでください; 高い次数はオーバーフィッティングや不安定な予測につながる可能性があります。
- 予測ホライゾンは`PastBars - order - 1`に等しいです; `ModelOrderFraction`を1未満に保つことでホライゾンが少なくとも数バーであることを確認してください。
- モメンタムとROCモードは正の価格を必要とします。ゼロを越える可能性のある銘柄は生価格モードを使用する必要があります。
- 分数pipsのある市場では、戦略は銘柄の小数点以下桁数を使用してpipサイズを自動的にスケールします（3または5桁の場合は×10）。

## 制限事項

- ARモデルは定常性を前提とします; 強いトレンドやレジームシフトは精度を低下させる可能性があります。
- 予測ベースのシグナルはノイズに敏感です — ライブ取引で使用する場合は追加フィルターと組み合わせることを検討してください。
- 正確なリスクサイジングにはポートフォリオ評価と有効なストップロス距離が必要です; そうでなければデフォルトボリュームが使用されます。
