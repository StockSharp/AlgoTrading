# E Skoch 指値注文戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

**E Skoch 指値注文戦略**は、新しいバーを待機してトレーディング時間軸と日足時間軸の両方で最新の2つの高値と安値を分析し、ブレイクアウト注文を置くオリジナルのMetaTraderエキスパートアドバイザーを再現したものです。目標は、日足トレンドで確認された短期プルバック後に市場が前のバーを突破する際のモメンタムを捉えることです。

StockSharpの実装はオリジナルのアイデアを維持しつつ、ローソク足サブスクリプション、自動保護注文、戦略パラメーターなどの高レベルAPI機能を使用しています。C#バージョンは`CS/`フォルダーに保存されており、Pythonポートはまだ提供されていません。

## 取引ロジック

1. 各完成ローソク足で、戦略は作業時間軸の前の2本のローソク足と前の2本の日足ローソク足の高値と安値を取得します。
2. 最後の日足高値が2日前の高値より低い**かつ**前の日中高値がその前より低い場合、戦略は最新の日中高値に設定可能なバッファーを加えた位置に**バイストップ**を置きます。
3. 最後の日足安値が2日前の安値より高い**かつ**前の日中安値がその前より高い場合、戦略は最新の日中安値から設定可能なバッファーを引いた位置に**セルストップ**を置きます。
4. 各指値注文は個別のストップロスとテイクプロフィットレベルを設定します。エントリーが発動すると、戦略はオープンポジションに対してすぐに保護的なストップおよびリミット注文を送信します。
5. ポジションも注文もアクティブでない場合、戦略は現在の資産をベースラインとして記録します。口座の資産がそのベースラインから設定されたパーセンテージ分成長した場合、すべてのポジションが閉じられ保護注文がキャンセルされます。
6. オプションのブロッキング（`CheckExistingTrade`）は、ポジションがオープンしている間は新しいエントリーを防ぎ、オリジナルの入力パラメーター「CheckTrade」を反映します。

## パラメーター

| パラメーター | 説明 |
| --- | --- |
| `CandleType` | シグナルに使用するメインの時間軸。デフォルト：1時間足ローソク足。 |
| `TakeProfitBuyPips` / `StopLossBuyPips` | ロング側の利益と損失のオフセット（pips単位）。 |
| `TakeProfitSellPips` / `StopLossSellPips` | ショート側の利益と損失のオフセット（pips単位）。 |
| `IndentHighPips` / `IndentLowPips` | ストップ注文を置くために使用する最新の高値または安値からの距離（pips）。 |
| `CheckExistingTrade` | trueの場合、ポジションがオープンしている間は新しい注文がスキップされます。 |
| `PercentEquity` | すべてのポジションを終了するために必要な資産のパーセンテージ増加。 |
| `Volume` | 注文サイズ（オリジナルのエキスパートアドバイザーに合わせてデフォルト0.01ロット）。 |

## リスク管理

- バイストップ注文はエントリー価格の下にストップロスを、上にテイクプロフィットを置きます。
- セルストップ注文はエントリー価格の上にストップロスを、下にテイクプロフィットを置きます。
- 保護注文はポジションが閉じられるか新しい保護セットが作成された時に自動的にキャンセルされます。
- 資産成長チェックは、取引が再開される前に利益を確保するグローバルな「サーキットブレーカー」として機能します。

## 注記

- 戦略はトレーディング時間軸と日足ローソク足の両方を必要とするため、Designerまたはバックテスト中に両方のサブスクリプションのデータが利用可能であることを確認してください。
- pip変換は、価格ステップを10倍にすることで分数pip価格（3または5桁の小数点）を使用するシンボルに自動的に調整されます。
- ロジックは単一の集約ポジションを想定しており、`CheckExistingTrade`が有効の場合、同時のロングとショートのエクスポージャーは意図的に避けられます。
