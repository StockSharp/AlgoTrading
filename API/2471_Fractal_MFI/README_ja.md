# Fractal MFI戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略はエキスパートアドバイザー `Exp_Fractal_MFI.mq5` の翻訳です。マネーフローインデックス（MFI）インジケーターを使用し、オシレーターが事前定義された上限・下限レベルを交差する時に取引シグナルを生成します。

## 仕組み
- 設定可能な期間でMFIを計算します。
- 前回のMFI値が**低レベル**より上にあり、現在の値がそれを下回ると、シグナルが生成されます。
  - **Direct**モードではロングポジションを開き、オプションでショートを決済します。
  - **Against**モードではショートポジションを開き、オプションでロングを決済します。
- 前回のMFI値が**高レベル**より下にあり、現在の値がそれを上回ると、別のシグナルが生成されます。
  - **Direct**モードではショートポジションを開き、オプションでロングを決済します。
  - **Against**モードではロングポジションを開き、オプションでショートを決済します。

完成したローソク足のみが処理されます。ロングまたはショートポジションの開閉を個別に有効/無効にできます。

## パラメーター
- `MfiPeriod` – Money Flow Index計算の期間。
- `HighLevel` – MFIの上限しきい値。
- `LowLevel` – MFIの下限しきい値。
- `CandleType` – 計算に使用するローソク足の時間軸。
- `Trend` – インジケーターの方向で取引する`Direct`または、シグナルを反転する`Against`を選択。
- `BuyPosOpen` / `SellPosOpen` – ロングまたはショートポジションの開始を許可。
- `BuyPosClose` / `SellPosClose` – 逆シグナル時に既存ポジションの決済を許可。

## 注記
このC#バージョンは高レベルAPIの使用に焦点を当てており、元のMQLコードの資金管理ルールやストップレベルは実装していません。
