# Exp タイムゾーンピボット・オープンシステム Tm Plus 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は エキスパートアドバイザー **Exp_TimeZonePivotsOpenSystem_Tm_Plus** の StockSharp 高レベルポートです。日次セッションオープン周辺に2つのブレイクアウトゾーンを投影し、ブレイクアウト後の押し戻しを取引する独自の *TimeZonePivotsOpenSystem* インジケーターを再現します。元のスクリプトのすべてのコンポーネント—シグナル遅延、タイムフィルター、非対称な決済ロジック、資金管理プリセット—が明示的なパラメーターにマッピングされ、MQL5 実装と一貫した動作を維持します。

## 取引ロジック

1. 設定された `StartHour` 時に、戦略はセッションのオープン価格を記録します。その後、そのアンカーの上下 `OffsetPoints` (ポイント単位) に2つの動的レベルが描画されます。
2. 完了したローソク足が上限レベルより **上** でクローズするたびに、戦略は:
   - 現在のバーがもはやバンドより上にない場合のみ、次のローソク足でロングエントリーを実行するようスケジュールします (`SignalBar` の遅延を尊重)。
   - `SellPosClose` が有効な場合、開いているショートポジションを即座にクローズします。
3. 完了したローソク足が下限レベルより **下** でクローズするたびに、戦略は:
   - 現在のバーがもはやバンドより下にない場合に、次のローソク足のショートエントリーをスケジュールします。
   - `BuyPosClose` が有効な場合、開いているロングポジションを即座にクローズします。
4. エントリーは `TryExecutePendingEntries` により次のローソク足の最初の更新時に実行されます。これは注文を新しいバーが始まるまで遅らせる元のエキスパートと一致します。

シグナル遅延パラメーター `SignalBar` は元の `CopyBuffer` シフトを再現します。値 `0` は最近クローズしたバーに反応し、`1` は行動する前に追加のバーを待って追加の確認を与えます。

## 注文管理

* **ストップロス / テイクプロフィット** – 距離はポイントで設定 (`StopLossPoints`、`TakeProfitPoints`) され、インストゥルメントのステップを使って価格に変換されます。両方のレベルはローソク足の高値/安値を使用して監視されるため、バー内のタッチも決済をトリガーします。
* **時間ベースの決済** – `TimeTrade` が true の場合、MQL5 コードの `nTime` タイマーを反映して `HoldingMinutes` 分後にポジションが強制クローズされます。
* **手動クローズ** – 反対方向のブレイクアウトシグナルは、対応する `BuyPosClose` または `SellPosClose` フラグが有効な場合、実行中の取引をクローズします。

## 資金管理

`MoneyMode` パラメーターは `MarginMode` 列挙型を再現します:

- `Lot` – `MoneyManagement` に等しい固定ボリューム。
- `Balance` と `FreeMargin` – 口座のエクイティまたは空きマージンの倍数を使用 (`MoneyManagement * エクイティ / 価格`)。
- `LossBalance` と `LossFreeMargin` – 希望する資本割合をストップ距離で割るリスクベースのサイジング。

`StopLossPoints` がゼロに設定されると、リスクモードは価格ベースのサイジングに優雅にフォールバックします。

## パラメーター

| パラメーター | 説明 | デフォルト |
|------------|------|----------|
| `MoneyManagement` | `MoneyMode` に応じてポジションサイズを決定するベース係数。 | `0.1` |
| `MoneyMode` | ポジションサイジングモデル (`Lot`、`Balance`、`FreeMargin`、`LossBalance`、`LossFreeMargin`)。 | `Lot` |
| `StopLossPoints` | 執行価格からのストップロス距離 (ポイント)。 | `1000` |
| `TakeProfitPoints` | 執行価格からのテイクプロフィット距離 (ポイント)。 | `2000` |
| `DeviationPoints` | エキスパートから保持された情報パラメーター (ポイント単位のスリッページ設定)。 | `10` |
| `BuyPosOpen` / `SellPosOpen` | ロングとショートのエントリーを有効化または無効化。 | `true` |
| `BuyPosClose` / `SellPosClose` | 反対のブレイクアウトがポジションを強制クローズすることを許可。 | `true` |
| `TimeTrade` | 最大保有時間フィルターを有効化。 | `true` |
| `HoldingMinutes` | 任意のポジションの最大存続時間 (分)。 | `720` |
| `OffsetPoints` | セッションオープンからのピボットバンドの距離 (ポイント)。 | `200` |
| `SignalBar` | シグナル評価を遅延させるバー数 (0 = 最後のクローズしたバー)。 | `1` |
| `CandleType` | インジケーターの計算に使用するメインの時間軸。 | `TimeSpan.FromHours(1).TimeFrame()` |
| `StartHour` | セッションオープン価格を定義する時刻 (0-23)。 | `0` |

## 使用上の注意

- 戦略はセキュリティが有効な `PriceStep` を提供することを前提としています。インストゥルメントにそのメタデータがない場合、`0.0001` のフォールバックが使用されます。
- エントリーは新しいローソク足の最初の更新でトリガーされるため、実際の執行価格はその時点の市場に従い、エキスパートと同様、急速な市場では理論的なオープン価格と異なる場合があります。
- 元のインジケーターオーバーレイを再現するには、MQL5 スクリプトが時間足以下の期間でのみ動作するため、バックテストの時間軸を H1 以下に保ってください。
- より反応的な動作には `SignalBar` を `0` に設定するか、ブレイクアウト後に1バー余分に待つために `1` (デフォルト) に設定します。
