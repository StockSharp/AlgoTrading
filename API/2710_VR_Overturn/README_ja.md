# VR Overturn 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- StockSharp の高レベル API を使用して MetaTrader エキスパート「VR---Overturn」を再現します。
- 常に一つのポジションのみを保有し、前のポジションが閉じられると即座に次の取引を評価します。
- マーチンゲールまたはアンチマーチンゲールのサイジングによる自動ポジション反転を望む裁量トレーダー向けに構築されています。

## トレードロジック
1. **初期ポジション** – 戦略は設定された方向（`FirstPositionDirection`）でベースボリューム（`BaseVolume`）を使って最初の取引を開始します。
2. **ストップロス / テイクプロフィット** – 保護的な決済注文は `StopLossPips` と `TakeProfitPips` を使用して自動的に添付されます。エンジンはインスツルメントの価格ステップを分析してpipsを絶対価格オフセットに変換します（3桁および5桁のインスツルメントはオリジナルのエキスパートと同様に10倍の調整が適用されます）。
3. **ポジションクローズ処理** – いずれかの保護注文によってポジションが閉じられると、戦略は以下を記録します：
   - クローズされた取引のサイド（ロングまたはショート）。
   - 約定数量。
   - 実現PnL（エントリー価格とエグジット価格の差）。
4. **次のエントリーサイジング** – 保存された結果が次の注文のサイドとロットサイズを決定します。
   - 勝ちトレードは同じ方向を維持し、負けトレードは方向を反転します。
   - マーチンゲールモードは損失後にポジションサイズを乗算し、勝利後にベースボリュームにリセットします。
   - アンチマーチンゲールモードは勝利後にポジションサイズを乗算し、損失後にベースボリュームにリセットします。
5. **ロット丸め** – 計算されたサイズは、市場注文が送信される前にインスツルメントの最も近いボリュームステップに丸められます。

## パラメーター
| パラメーター | 説明 | デフォルト値 |
|-----------|------|------------|
| `FirstPositionDirection` | 最初の取引の方向（Buy/Sell）。 | Buy |
| `Mode` | サイジングモード：Martingale（損失後に増加）またはAntiMartingale（利益後に増加）。 | Martingale |
| `BaseVolume` | 初期ポジションボリューム。シーケンスがリセットされるときに使用されます。 | 0.1 |
| `StopLossPips` | ストップロスまでの距離（pips）。 | 30 |
| `TakeProfitPips` | テイクプロフィットまでの距離（pips）。 | 90 |
| `LotMultiplier` | 拡張ステップ中に適用される乗数（マーチンゲールでは損失後、アンチマーチンゲールでは利益後）。 | 1.6 |

## リスク管理
- すべてのエントリーに対してストップロスとテイクプロフィット注文を添付するために `StartProtection` を使用します。
- ストップとターゲットの距離は、設定されたpip値から導出される絶対価格オフセットです。
- 追加のトレーリングロジックは適用されないため、リスクは完全に保護注文とポジション反転ルールによって制御されます。

## 運用上の注意
- この戦略はローソク足やインジケーターに依存しません；取引確認（`OnOwnTradeReceived`）のみに反応します。
- 保護注文が部分約定した場合、戦略はポジションがフラットになるまで残りの量を累積してから再び動作します。
- コミッションとスワップの値はStockSharpの取引では利用できないため、利益比較は価格差のみを使用します。ブローカーが大きな手数料を請求する場合は、ストップや乗数を広げることを検討してください。
- 価格ステップとボリュームステップのメタデータを提供するインスツルメントで動作します；本番環境に展開する前に両方を確認してください。
