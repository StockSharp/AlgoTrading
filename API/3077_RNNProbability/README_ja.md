# RNN Probability 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
RNN Probability戦略は、MetaTraderエキスパート *RNN (barabashkakvn's edition)* の変換です。元のロボットはRSI期間で区切られた3つのRSIスナップショットを収集し、リカレントニューラルネットワークをエミュレートする手作りの確率格子に入力します。StockSharpポートは、ハイレベルなローソク足サブスクリプションAPIを使ってこの動作を複製し、MetaTraderのロット、価格ステップ、ストップ/ターゲット距離を自動的にStockSharpの概念に変換します。

最新の確定ローソク足のRSI値が利用可能になると、戦略はRSI期間1つ分と2つ分遡って3点履歴を構築します。これらの正規化された読み取り値は、8つのMetaTraderの重み（`Weight0` … `Weight7`）と組み合わせられ、市場が下落すべき確率を生成します。確率は `[-1; 1]` の範囲に再マッピングされ、符号がロングまたはショートポジションを開くかどうかを決定します。元のExpert Advisorに合わせ、一度に1つのポジションのみ維持されます。

## トレードロジック
1. 設定されたローソク足シリーズをサブスクライブし、選択した `AppliedPrice` 入力（デフォルトはOpen）を使用して `RelativeStrengthIndex` インジケーターを手動で処理します。
2. 1つと2つの完全な期間前のRSI読み取りにアクセスするのに十分な大きさのローリングバッファに確定RSI値を保存します。
3. 3つのRSI値を `[0; 1]` の範囲に正規化し、ニューラルネットワーク格子を評価します：
   - 最初のブランチ（`Weight0`、`Weight1`、`Weight2`、`Weight3`）は現在のRSIが下半分にある場合（50未満）を処理します。
   - 2番目のブランチ（`Weight4`、`Weight5`、`Weight6`、`Weight7`）は現在のRSIが上半分にある場合を処理します。
4. 結果の確率を `-1` と `+1` の間のトレードシグナルに変換します。
5. ポジションが開いておらずシグナルが負の場合、`TradeVolume` ロットを買います。シグナルが非負の場合、代わりに `TradeVolume` ロットを売ります。
6. オプションでpipsで表した対称的なストップロスとテイクプロフィットレベルを設定します。戦略はpips距離を絶対価格オフセットに自動変換し、3桁および5桁のforexシンボルのMetaTrader追加桁調整を含みます。
7. RSI入力、確率、結果のシグナルを各決定に記録し、ソースエキスパートの詳細な動作を反映します。

## パラメーター
| 名前 | タイプ | デフォルト | 説明 |
| --- | --- | --- | --- |
| `CandleType` | `DataType` | 1時間時間軸 | インジケーター更新とシグナル生成に使用するプライマリローソク足シリーズ。 |
| `TradeVolume` | `decimal` | `1` | 各市場注文で送信するロットサイズ。 |
| `RsiPeriod` | `int` | `9` | RSIインジケーターの長さ。過去RSIサンプル間の距離も定義します。 |
| `AppliedPrice` | `AppliedPriceType` | `Open` | RSIに転送する価格コンポーネント（Open、Close、High、Low、Median、Typical、Weighted）。 |
| `StopLossTakeProfitPips` | `decimal` | `100` | ストップロスとテイクプロフィット両方のpips距離。ゼロに設定すると保護注文を無効にします。 |
| `Weight0` … `Weight7` | `decimal` | `6, 96, 90, 35, 64, 83, 66, 50` | 8つの格子ブランチに適用される確率の重み。各値は0から100のパーセンテージを表します。 |

## 元のMetaTraderエキスパートとの違い
- メール通知が削除されました。StockSharpのログはSMTPサーバーに依存せず同じ洞察を提供します。
- ポジションサイジングは単一の `TradeVolume` に固定されています。部分的なクローズや段階的なスケーリングは、ソースコードのワンポジション設計に合わせるために意図的に省略されています。
- インジケーターデータはStockSharpのハイレベルローソク足サブスクリプションを通じて配信され、手動の `CopyBuffer` 呼び出しとポインタ算術を排除します。
- pips変換は銘柄の `PriceStep` を使用し、ハードコードされたティックサイズに依存するのではなく、3/5桁のforexシンボルを自動的に補正します。

## 使用上のヒント
- 戦略を起動する前に `TradeVolume` を銘柄の最小ロットステップに合わせます；コンストラクターも `Strategy.Volume` に値を反映します。
- 異なる市場にニューラルネットワーク格子を適応させるために最適化中に8つの重みを調整します。すべての重みは最適化パラメーターとして公開されています。
- 広いスプレッドを持つシンボルで取引する場合や裁量的なエグジットを使用する場合は、`StopLossTakeProfitPips` を減らすかゼロに設定します。
- ニューラルネットワーク出力の検証を容易にするために、ローソク足、RSI、実行されたトレードを視覚化するためにチャートに戦略を追加します。

## インジケーター
- 選択された適用価格から計算された1つの `RelativeStrengthIndex`。
