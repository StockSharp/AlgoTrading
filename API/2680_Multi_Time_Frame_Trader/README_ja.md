# マルチ時間軸トレーダー戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、StockSharp のハイレベル API を使用してオリジナル MQL の "Multi Time Frame Trader" ロジックを再現します。3つの
多項式回帰チャンネル（M1、M5、H1）を組み合わせ、時間足の傾きが示す方向において、低い時間軸がチャンネルの極値をテストする
ときにのみ取引します。

システムは完了した各ローソク足で回帰チャンネルの上部、中間、下部バンドを継続的に再計算します。時間足の上部バンドが減少する
と、バイアスはベア（弱気）；上昇するとバイアスはブル（強気）になります。M5 と M1 のローソク足が対応するバンドに達し、
方向フィルターが一致するとエントリーが発動します。

## コアワークフロー

- **サブスクリプション**：戦略は1分、5分、1時間のローソク足を同時に監視します。
- **回帰チャンネル**：各サブスクリプションは `Bars` ポイントで多項式回帰ライン（次数1〜3）を構築し、`StdMultiplier` 標準偏差
  分オフセットして抵抗と支持のバンドを取得します。
- **傾きの推定**：チャンネルの傾きは現在の上部バンドと `Bars` ローソク足前の上部バンドの差から導出され、`i-Regr` インジケーター
  の動作を反映します。
- **方向フィルター**：H1 の傾きが、ショートのみ（負の傾き）かロングのみ（正の傾き）が許可されるかを定義します。

## エントリーロジック

### ショートトレード

1. 時間足の傾きが負。
2. 最後の5分足の高値が5分足回帰の抵抗に触れるか突破する。
3. 最後の1分足の高値が1分足の抵抗に触れるか突破する。
4. 既存のショートポジションが開いていない（`Position >= 0`）。
5. マーケット売り注文が送られ、ストップロスはエントリーの上にチャンネル幅の半分の位置に設定され、ターゲットは M5 の中間ラインと
   等しくなります。

### ロングトレード

1. 時間足の傾きが正。
2. 最後の5分足の安値が5分足回帰のサポートに触れるか突破する。
3. 最後の1分足の安値が1分足のサポートに触れるか突破する。
4. 既存のロングポジションが開いていない（`Position <= 0`）。
5. マーケット買い注文が送られ、ストップロスはエントリーの下にチャンネル幅の半分の位置に設定され、ターゲットは M5 の中間ラインと
   等しくなります。

## 決済ルール

- ストップとターゲットは内部に保存され、完了した M1 ローソク足ごとに評価されます。ローソク足のレンジが保存されたストップレベルを
  越えた場合、ポジションは即座に閉じられます。
- ストップの前に利益目標が達成された場合も、ポジションは閉じられます。
- 決済後は追跡されたレベルがリセットされ、遅延なしに新しいシグナルを評価できます。

## パラメーター

| パラメーター | デフォルト | 説明 |
|------------|-----------|------|
| `Degree` | 1 | 回帰チャンネルの多項式次数（1=線形、2=放物線、3=立方）。 |
| `StdMultiplier` | 2.0 | バンド幅を定義する標準偏差の乗数。 |
| `Bars` | 250 | 回帰フィッティングと傾きのルックバックに使用するローソク足の数。 |
| `Shift` | 0 | 回帰評価点の水平シフト（0から `Bars - 1` の間に制限）。 |
| `UseTrading` | true | `false` に設定するとすべての注文生成が無効になりますが、チャンネルは更新を続けます。 |

## 追加メモ

- StockSharp のマーケット注文は自動的に SL/TP レベルを添付しないため、戦略はストップとターゲットレベルをローカルに保存します。
- 分足と時間足のローソク足をサポートするあらゆるインストゥルメントで動作しますが、元のロジックは外国為替ペア向けに設計されました。
- 取引するインストゥルメントのボラティリティに合わせて `Bars` を調整してください。小さな値は早く反応し、大きな値はより滑らかな
  チャンネルを生成します。
- 直線の回帰チャンネル（古典的な線形バージョンに最も近い）には `Degree` を 1 に設定し、MQL インジケーターの多項式モードを
  エミュレートするには高い次数を使用してください。
