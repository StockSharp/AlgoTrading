# オープンクローズ2アンプStochastic戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- MetaTrader 4 エキスパート *open_close2ampnstochastic_strategy* のポートは、StockSharp のハイレベル API の上に再構築されました。
- 古典的な Stochastic オシレーター (長さ 9、スムージング 3/3) と 2 本のバーの価格アクション フィルターを併用します。注文が送信される前に、現在のローソク足は前のローソク足の方向を継続する必要があります。
- シングルポジション取引向けに設計されています。デフォルトのローソク足ソースは 1 時間ですが、`CandleType` パラメータを通じて任意の時間枠を接続できます。

## 信号ロジック
1. **エントリーガード** – 一度にオープンできるポジションは 1 つだけです。戦略がフラットの場合、最後に完全に形成されたローソク足を評価します。
   - **ロングエントリー** Stochastic メインラインがシグナルラインより上にあり、*かつ* 最新のローソク足の始値と終値の両方が以前の値を下回っている場合 (下降圧力の継続とそれに続くオシレーターの強さ)。
   - **ショートエントリー** Stochastic メインラインがシグナルラインの下にあり、 *かつ* ローソク足が前の値よりも高い始値と終値を示している場合 (弱気オシレーターの確認で上向きに押し上げられる)。
2. **Exit ルール** – ポジションが存在する間、同じ条件が逆に反映されます。
   - **メインラインがシグナルラインを下回り、新しいローソク足がより高い始値/終値を印刷する場合、**ロングクローズ**。
   - **メインラインがシグナルラインを上回り、新しいローソク足が始値/終値を下げると**ショートで終了**。
3. **ドローダウン ガード** – MT4 の緊急出口を再現します。浮動損失の大きさ (実現損益 + 現在のローソクベースの推定値) が `MaximumRisk × account_margin` に達すると、ストラテジーはポジションを直ちに清算します。 StockSharp は MetaTrader の *AccountMargin* を公開しないため、ポートは `Portfolio.BlockedValue` 経由で近似し、ブロックされたマージンが利用できない場合は `Portfolio.CurrentValue` にフォールバックします。

## 資金管理
- **BaseVolume** は元の `Lots` 入力をミラーリングし、アカウント情報が利用できない場合に使用されます。
- ポートフォリオ評価が存在する場合、生の注文サイズは `Portfolio.CurrentValue × MaximumRisk / 1000` になり、元の `AccountFreeMargin` ベースのサイズ設定と一致します。
- トレードに負けるたびに、次のポジションは `losses / DecreaseFactor` ずつ減ります。ストリークカウンターは利益のある取引後にリセットされます。結果のサイズは、`MinimumVolume` を下回ることはできません。デフォルトは、MQL スクリプトと同様に 0.1 ロットです。
- 計算されたすべての出来高は、成行注文を送信する前に商品制限 (`VolumeStep`、`MinVolume`、`MaxVolume`) に合わせて調整されます。

## パラメーター
| 名前 | 種類 | デフォルト | 説明 |
| --- | --- | --- | --- |
| `BaseVolume` | 10進数 | `0.1` | リスクベースのサイジングを計算できない場合のフォールバック注文サイズ。 |
| `MaximumRisk` | 10進数 | `0.3` | 動的サイジングとドローダウンガードの両方に使用される資本の割合。リスク計算を無効にするには、`0` に設定します。 |
| `DecreaseFactor` | 10進数 | `100` | 連続負け後に適用される除数。値を大きくすると、減少が遅くなります。 |
| `MinimumVolume` | 10進数 | `0.1` | 計算された体積の絶対床。 |
| `StochasticLength` | 整数 | `9` | Stochastic オシレーターのルックバック期間。 |
| `StochasticKLength` | 整数 | `3` | %K ラインの平滑化期間。 |
| `StochasticDLength` | 整数 | `3` | %D 信号線の平滑化期間。 |
| `CandleType` | `DataType` | `TimeFrame(1h)` | インジケーターと価格フィルターを駆動するために使用されるローソク足のソース。 |

## 実装メモ
- 非常口に必要な変動損益は、最新のローソク足の終値と `Strategy.PositionPrice` で推定されます。これは、MetaTrader の `AccountProfit` の意図を反映していますが、実際のブローカー側の計算は異なる場合があります。
- ブロックされた証拠金もポートフォリオ値もコネクタによって公開されていない場合、戦略が `BaseVolume` を使用して取引を行っている間、ドローダウン ガードはアイドル状態のままになります。
- `StartProtection()` は起動時に有効になるため、StockSharp の保護メカニズム (ルーティングの停止/取得、再接続) は、MQL バージョンに存在するセーフティ ネットを反映します。

## オリジナルエキスパートとの違い
- MetaTrader ロットの丸めは、StockSharp を通じて利用可能な商品メタデータを使用してエミュレートされます。ポジションのサイジングがブローカーの制約と一致するように、取引された証券の `VolumeStep`/`MinVolume` の値を確認します。
- MT4 コードは、`Volume[0]` でガードしながらティックごとに評価しました。ポートは完了したローソク足のみを処理するため、シグナルの重複が防止され、StockSharp 戦略に推奨されるパターンです。
- アカウントの指標は近似値です。厳格な証拠金制限に依存している場合は、ブローカーの正確な計算式と一致するように `MaximumRisk` を調整するか、ガードをオーバーライドしてください。
