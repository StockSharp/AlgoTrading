# Blau C-Momentum戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、MetaTraderエキスパートアドバイザー**Exp_BlauCMomentum**のStockSharpへのポートです。設定可能な時間軸のキャンドルを使用して単一のインストゥルメントで取引し、Blauの3重平滑化モメンタムを2つのモードのいずれかで解釈します：

* **ブレイクダウンモード** – モメンタムラインがゼロレベルをクロスすることに反応します。
* **ツイストモード** – 平滑化されたモメンタムの傾きの方向変化に反応します。

インジケーターは外部の時間軸で計算され、モメンタム計算にオプションで異なる適用価格を使用できます。ポジションは成行注文で開かれ、組み込みのストップロスとテイクプロフィットモジュールを使用して保護できます。

## 機能の仕組み
1. 選択した時間軸のキャンドルにサブスクライブします。
2. Blau C-Momentumを計算します：
   * 生のモメンタムは、`MomentumLength`バーで区切られた2つの適用価格の差です。
   * 生のモメンタムは選択した移動平均メソッドで3回平滑化され、価格ステップにスケーリングされます（×100/Point）。
3. `SignalBar`で定義されたバーシフトのために平滑化されたインジケーター履歴を保存します。
4. シグナルを生成します：
   * **ブレイクダウン** – 前のバーがゼロより上にあり、シグナルバーがゼロ以下の場合、ロングを開く/フリップ；前のバーがゼロより下にあり、シグナルバーがゼロ以上の場合、ショートを開く/フリップ。オプションのエグジットフラグは、前のバーがゼロラインをクロスする際に反対側を閉じます。
   * **ツイスト** – 2つの前のバーを比較；モメンタムが上方に加速する場合（前 &lt; 古い）とシグナルバーが確認する場合、ロングを開く/フリップ；モメンタムが下方に加速する場合（前 &gt; 古い）とシグナルバーが確認する場合、ショートを開く/フリップ。オプションのエグジットフラグは同じ条件で反対側を閉じます。
5. ポジションサイジングに`MoneyManagement`と`MarginModes`を使用します。負の値は固定ボリュームを意味します；正の値はポートフォリオ価値の一部をリスクにさらすか割り当てます。シンプルな時間ロックは同じキャンドル内での即時再エントリーを防ぎます。

## パラメーター
| グループ | 名前 | 説明 |
|-------|------|-------------|
| 取引 | `MoneyManagement` | ポジションサイジングのための資本の割合。負の値 = 固定ボリューム。 |
| 取引 | `MarginModes` | マネーマネジメントの解釈（`FreeMarginShare`、`BalanceShare`、`FreeMarginRisk`、`BalanceRisk`）。リスクモードはストップロス距離と`StepPrice`を使用します。 |
| リスク | `StopLossPoints` | インストゥルメントの価格ステップでのストップロス距離（無効にするには`0`に設定）。 |
| リスク | `TakeProfitPoints` | インストゥルメントの価格ステップでのテイクプロフィット距離（無効にするには`0`に設定）。 |
| 取引 | `SlippagePoints` | 許容スリッページ（互換性のために保持、注文配置には使用されません）。 |
| 取引 | `EnableLongEntry`、`EnableShortEntry` | ロング/ショートポジションの開設を許可。 |
| 取引 | `EnableLongExit`、`EnableShortExit` | インジケーターに従った既存ポジションのクローズを許可。 |
| ロジック | `EntryModes` | `Breakdown`または`Twist`。 |
| データ | `CandleType` | インジケーター計算に使用する時間軸（デフォルト4h）。 |
| インジケーター | `SmoothingMethod` | 移動平均メソッド：`Simple`、`Exponential`、`Smoothed`、`LinearWeighted`、`Jurik`、`TripleExponential`、`Adaptive`。 |
| インジケーター | `MomentumLength` | 生のモメンタム平均化の深さ（2つの価格値の間のバー数）。 |
| インジケーター | `FirstSmoothLength`、`SecondSmoothLength`、`ThirdSmoothLength` | 3つの平滑化ステージの長さ。 |
| インジケーター | `Phase` | Jurikフェーズパラメーター（平滑化メソッドが`Jurik`のときに使用）。 |
| インジケーター | `PriceForClose`、`PriceForOpen` | モメンタムに使用される適用価格（数式についてはコードコメントを参照）。 |
| ロジック | `SignalBar` | シグナルに使用するバーインデックス（0 = 現在の閉じたバー、1 = 前のバーなど）。 |

## 使用上の注意
* 戦略をインストゥルメントに接続し、キャンドルシリーズを設定します。取引時間軸はインジケーター時間軸と同じです。
* ストップ/テイクプロフィット値が正の場合、高レベルAPIの保護モジュールが自動的に有効になります。
* StockSharpはMetaTraderスタイルのバランス/フリーマージンを公開しないため、マージンモードは近似値です。リスクベースのモードは`StopLossPoints`と`Security.StepPrice`に依存します。
* 元のライブラリの高度な平滑化メソッド（Parabolic、VIDYA、JurX）は最も近い利用可能なStockSharpインジケーターにマッピングされます（`TripleExponential` ≈ T3、`Adaptive` ≈ KAMA）。
* スリッページパラメーターは完全性のために保持されますが、成行注文が使用されるため、値は情報提供目的です。

## はじめに
1. StockSharp環境で接続、ポートフォリオ、インストゥルメントを設定します。
2. `BlauCMomentumStrategy`のインスタンスを作成し、`Security`、`Portfolio`、および希望のパラメーターを割り当てます。
3. `Start()`を呼び出します；戦略はキャンドルにサブスクライブし、インジケーターを計算し、自動的に取引します。
4. 開かれた/閉じられたポジションとインジケーターの状態に関する情報についてログを監視します。
