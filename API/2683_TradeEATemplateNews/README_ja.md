# TradeEATemplateNews 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
TradeEATemplateNews 戦略は、MetaTrader 4 エキスパートアドバイザー "Trade EA Template for News" の C# 変換です。元のシステムは外部ウェブサイトからダウンロードされた予定された経済イベントの前後に取引を一時停止していました。この StockSharp ポートはコアのアイデアを維持しながらハイレベル API に適応させます：

- 設定された時間軸（デフォルトは H1）の完了したローソク足を使用します。
- 口座がフラットなときにのみ取引し、ゼロのオープン注文を必要とした MQL テンプレートとまったく同じです。
- 重要度に応じてイベントの前後のエントリーをブロックする手動の経済ニュースブラックアウトを適用します。
- 執行価格から 100 ポイント離れた保護的なストップロスとテイクプロフィットブラケットを自動作成します（銘柄ステップを通じて変換）。

## トレーディングロジック
1. 各完了したローソク足がニューススケジュールの再計算をトリガーします。戦略は次のバーが前の始値に対して終値を比較できるように、前のローソク足の始値を保存します。
2. 現在時刻が設定されたブラックアウトウィンドウ内に入った場合、戦略は保留中の注文をキャンセルし、新しい取引を開きません。
3. ポジションが開いておらず、取引が許可されている場合：
   - 最後のローソク足が前のローソク足の始値より高く閉じた場合、ロングポジションが開かれます。
   - 最後のローソク足が前のローソク足の始値より低く閉じた場合、ショートポジションが開かれます。
4. ストップロスとテイクプロフィットのレベルはポイントで表現され（`TakeProfitPoints` と `StopLossPoints`）、銘柄の `Step` 値を使用して絶対価格オフセットに変換されます。

## 手動ニューススケジュール
元のエキスパートは investing.com または DailyFX からデータをダウンロードしていました。移植性のため、StockSharp バージョンは `NewsEventsDefinition` パラメーターを通じて提供される手動でキュレーションされたカレンダーを期待します。フォーマットはセミコロンまたは改行で区切られたエントリーのリストを受け入れます。各エントリーには少なくとも3つのカンマ区切りフィールドが含まれている必要があります：

```
YYYY-MM-DD HH:MM,通貨,重要度[,タイトル]
```

- `YYYY-MM-DD HH:MM` — UTC でのイベント開始。オプションの `TimeZoneOffsetHours` パラメーターはすべての解析された時刻を要求された量だけシフトします（例えば UTC+3 には `3` を設定）。
- `通貨` — `USD`、`EUR`、`EUR/USD` などの通貨コードまたはインストゥルメント識別子。複数のコードは `/`、`,`、`;`、`|` またはスペースで区切れます。
- `重要度` — 重要度キーワード。認識される値：`Low`、`Medium`、`Mid`、`Midle`、`Moderate`、`High`、`NFP`、`Nonfarm` または `Non-farm` を含む文字列。
- `タイトル` — ログメッセージに印刷されるオプションのフリーテキスト説明。

例：

```
2024-03-01 13:30,USD,High,Nonfarm Payrolls;2024-03-01 15:00,USD,Low,Factory Orders
```

### ブラックアウトウィンドウ
- `UseLowNews`、`UseMediumNews`、`UseHighNews`、`UseNfpNews` がどのイベントを考慮するかを切り替えます。
- `LowMinutesBefore/After`、`MediumMinutesBefore/After`、`HighMinutesBefore/After`、`NfpMinutesBefore/After` がイベント周囲で取引を無効にする分数を決定します。
- `OnlySymbolNews` は現在の銘柄の通貨コードと一致するエントリーのみにブラックアウトを制限します（例：`EURUSD` は `{EUR, USD}` のペアになります）。すべてのイベントで取引を一時停止するには無効にします。
- 戦略は任意の時点で最も高い重要度のイベントのみをアクティブに保ちます。情報ログメッセージが現在の状態の理由と次の予定されたリリースを知らせます。

## パラメーター
| パラメーター | 説明 | デフォルト |
|------------|------|----------|
| `CandleType` | サブスクライブするローソク足データタイプ。デフォルトは1時間。 | `1h` |
| `UseLowNews` | 低重要度のイベントを有効にします。 | `true` |
| `LowMinutesBefore` / `LowMinutesAfter` | エントリーをブロックする低インパクトニュースの前/後の分数。 | `15 / 15` |
| `UseMediumNews` | 中程度の重要度のイベントを有効にします。 | `true` |
| `MediumMinutesBefore` / `MediumMinutesAfter` | 中インパクトニュースの前/後の分数。 | `30 / 30` |
| `UseHighNews` | 高重要度のイベントを有効にします。 | `true` |
| `HighMinutesBefore` / `HighMinutesAfter` | 高インパクトニュースの前/後の分数。 | `60 / 60` |
| `UseNfpNews` | Non-farm Payrolls フラグを有効にします。 | `true` |
| `NfpMinutesBefore` / `NfpMinutesAfter` | NFP イベントの前/後の分数。 | `180 / 180` |
| `OnlySymbolNews` | 現在の銘柄の通貨コードでカレンダーをフィルタリングします。 | `true` |
| `NewsEventsDefinition` | 手動経済カレンダー説明文字列。 | 空 |
| `TimeZoneOffsetHours` | 各解析されたイベントに適用されるオフセット（デフォルトでは UTC）。 | `0` |
| `TakeProfitPoints` | 保護的テイクプロフィット注文のポイント距離。 | `100` |
| `StopLossPoints` | 保護的ストップロス注文のポイント距離。 | `100` |

`Volume` は `Strategy` から継承され、希望するポジションサイズに従って設定する必要があります。

## MQL バージョンとの違い
- 自動 HTTP ダウンロードなし — ユーザーがニュースリストを手動で提供し、外部依存関係を回避して変換を確定的に保ちます。
- チャートラベルと垂直線はアクティブまたは予定されたイベントを説明するログメッセージに置き換えられます。
- MQL エキスパートは固定ロットサイズ `0.01` で注文を開いていました；StockSharp ではポジションサイズは `Volume` プロパティから来ます。
- テンプレートのニュース対応動作を維持しながら、すべてのロジックがハイレベルローソク足サブスクリプション API で実装されています。

## デプロイメントの注意事項
1. 戦略を開始する前に `NewsEventsDefinition` を入力するか、更新して停止・再起動してスケジュールを再ロードします。
2. `TimeZoneOffsetHours` と分前/後パラメーターを取引セッションに合わせて調整します。
3. UI またはコードで `Volume`、ポートフォリオ、銘柄を設定してから戦略を開始します。
4. ブラックアウトロジックを確認するために、"Trading paused due to high news" または "Next scheduled news" などのメッセージについて戦略ログを確認します。

Python 翻訳はリクエスト通り意図的に省略されています。
