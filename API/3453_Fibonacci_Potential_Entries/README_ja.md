# Fibonacci の潜在的なエントリー戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

この戦略は、元の **EA_PUB_FibonacciPotentialEntries** エキスパート アドバイザーの動作を再現します。 50% と 61% の Fibonacci リトレースメント レベルで 2 つの指値注文を出し、高レベルの StockSharp API を使用してライフサイクルを管理します。

## 取引ロジック

1. **初期配置**
   - 買値/売値が利用可能になるとすぐに、戦略は現在のスプレッドを計算し、2 つの指値注文を送信します。
     - オーダー #1: 61% レベルより下 (ショートの場合は上) の保護ストップを備えた 50% レベルに配置されます。
     - オーダー #2: 61% レベルで配置され、100% レベルの途中に保護ストップが配置されます。
   - 最初の取引ではポートフォリオの 0.7% のリスクが発生し、2 番目の取引では `RiskPercent` パラメータの残りの部分のリスクが発生するようにボリュームのサイズが設定されます。

2. **ターゲットの処理**
   - 価格が `TargetPrice` レベルに達すると、戦略は成行注文を使用して各約定ポジションの半分をクローズします。
   - 部分的にエグジットした後、残りのボリュームは損益分岐点（エントリー価格）で保護されます。市場がそのレベルに戻ると、残りのポジションは自動的にクローズされます。

3. **方向**
   - `IsBullish = true` は購入制限を作成します (元の強気テンプレート)。
   - `IsBullish = false` は、売り指値と逆ストップ/ターゲット チェックを使用した動作を反映しています。

## パラメーター

| 名前 | 説明 |
|------|-------------|
| `PriceOn50Level` | 最初の指値注文の価格レベル。 |
| `PriceOn61Level` | 2 番目の指値注文の価格レベル。 |
| `PriceOn100Level` | 2 番目の取引ストップを計算するために使用される基準レベル。 |
| `TargetPrice` | 両方のポジションの共通利益目標。 |
| `RiskPercent` | 両方のエントリーにわたるポートフォリオ資本のリスクの合計パーセンテージ。 |
| `IsBullish` | 長いセットアップと短いセットアップから選択します。 |

## 変換メモ

- リポジトリ ガイドラインの要求どおりに、高レベルのヘルパー (`SubscribeLevel1`、`BuyLimit`、`SellLimit`、`BuyMarket`、`SellMarket`) のみが使用されます。
- 部分的なエグジットと損益分岐点調整は成行注文で再現され、低レベルの注文変更呼び出しに依存することなく、MQL ロボットの動作に一致します。
- ポジションボリュームは、StockSharp の規則との一貫性を保つために、楽器のボリュームステップに正規化されます。
