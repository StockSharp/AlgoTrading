# MA Shift Puria Method戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
MA Shift Puria Method戦略は、StockSharpの高レベルAPIに適応させたクラシックな「Puria」エキスパートアドバイザーの実装です。アルゴリズムは複数の指数移動平均（EMA）をMACDフィルターとオプションのフラクタルベースのトレーリングロジックと組み合わせます。シグナルは完成した足のみで評価されます。ポジション管理には固定のストップロスとテイクプロフィットレベル、設定可能なトレーリングストップ、および確認されたスイングポイントが現れたときに目標付近で利益を確保するオプションのフラクタルトレーリングモードが含まれます。

## インジケーターと計算
- **ファストEMA（デフォルト14）** – 短期モメンタムを捉え、ファスト平均の傾きを定義します。
- **スローEMA（デフォルト80）** – より広い市場方向を表します。ファストとスローEMA間の距離はシグナルを検証するためにユーザー定義のピップ閾値を超える必要があります。
- **MACD（ファスト11、スロー102、シグナル9）** – メインラインがトレード方向のゼロ軸を交差することを要求し、3本前の足では反対側にいたことでモメンタム方向を確認します。
- **フラクタルウィンドウ（5本の足）** – フラクタルトレーリングが有効な場合に使用されます。戦略はローリングする5本の足バッファからスイング高値と安値を導出し、MetaTraderのフラクタル定義（中央の足は両側の2本の足と比較してローカル極値）に合致します。

## エントリーロジック
戦略が取引を許可されており、最新の完成した足で以下の条件が真の場合のみ新しいポジションが開かれます：

### ロングエントリー
1. ファストEMAがスローEMAより上にある。
2. スローEMAが3本前の足の値と比較して上昇傾向にある。
3. ファストEMAが上昇傾斜を持つ（現在値が前の値より上）。
4. MACDメインラインがゼロより上にあり、3本前の足ではゼロより下だった。
5. ファストEMAが最新の2本の足の間で設定された**Shift Minimum**（ピップス）より増加し、さらに加速しているかまたは前の増分が非正だった。

### ショートエントリー
1. ファストEMAがスローEMAより下にある。
2. スローEMAが3本前の足の値と比較して下降傾向にある。
3. ファストEMAが下降傾斜を持つ（現在値が前の値より下）。
4. MACDメインラインがゼロより下にあり、3本前の足ではゼロより上だった。
5. ファストEMAが**Shift Minimum**閾値より減少し、さらに加速しているかまたは前の増分が非負だった。

戦略は選択されたモードに応じて、固定増分（手動ボリューム）またはポートフォリオリスクに基づく動的サイジングの単位でポジションを開きます。反対方向のポジションが開いている場合、アルゴリズムはそれを閉じ、単一のマーケット注文で現在の方向に新しいポジションを開きます。

## エグジットとリスク管理
- **ストップロス** – エントリー価格から相対的なピップスで設定されます。足の安値/高値が保護レベルに触れると、ポジションは即座にクローズされます。
- **テイクプロフィット** – ピップスでも表現されます。目標に達すると、ポジション全体がクローズされます。
- **トレーリングストップ** – 有効な場合、利益がトレーリング距離にトレーリングステップを加えた分を超えた後、ストップレベルは設定された距離で価格を追跡します。ロジックは元のMQLエキスパートを反映し、ストップが少なくともトレーリングステップ分動けるときのみ更新します。
- **フラクタルトレーリング** – オプション。価格がテイクプロフィット距離の95%をカバーすると、ストップは5本の足フラクタルパターンで識別された最新のスイング安値（ロング）またはスイング高値（ショート）に移動でき、ブレイクアウトの余地を残しながらリスクを絞ります。
- **リスクベースのサイジング** – 手動ボリュームが無効の場合、戦略はトレードごとにポートフォリオの固定割合をリスクにさらします。リスク資本を金銭的なストップ距離で割り、結果をexchangeの制限内で最も近い許容ボリュームステップに丸めます。

## パラメーター
| 名前 | 説明 | デフォルト値 |
|------|-------------|---------|
| `UseManualVolume` | 固定ボリュームとリスクベースのサイジングを切り替えます。 | `true` |
| `ManualVolume` | 手動サイジングがアクティブな場合のトレードごとのボリューム。 | `0.1` |
| `RiskPercent` | トレードごとのリスク資本割合（`UseManualVolume`がfalseの場合に使用）。 | `9` |
| `StopLossPips` | ストップロス距離（ピップス）。 | `45` |
| `TakeProfitPips` | テイクプロフィット距離（ピップス）。 | `75` |
| `TrailingStopPips` | トレーリングストップ距離（ピップス）。 | `15` |
| `TrailingStepPips` | トレーリングストップを更新する前の最小ピップ移動。 | `5` |
| `MaxPositions` | 一つの方向に蓄積できるポジション単位の最大数。 | `1` |
| `ShiftMinPips` | 有効なシグナルに必要な最小EMA傾き（ピップス）。 | `20` |
| `FastLength` | ファストEMAの長さ。 | `14` |
| `SlowLength` | スローEMAの長さ。 | `80` |
| `MacdFast` | MACDファスト期間。 | `11` |
| `MacdSlow` | MACDスロー期間。 | `102` |
| `UseFractalTrailing` | フラクタルトレーリングストップ調整を有効/無効にします。 | `false` |
| `CandleType` | 計算に使用する足タイプ（時間軸）。 | `15分` |

## 実装メモ
- 戦略は一つの足ストリームをサブスクライブし、`SubscribeCandles().Bind(...)`経由でEMAとMACDインジケーターをバインドし、手動バッファクエリなしにシグナルハンドラーでインジケーター値が受信されることを保証します。
- 内部状態は元のロジックで必要なMQLの`shift`インデックスを模倣するために最後の3つのEMAとMACD値を追跡します。
- フラクタルはMetaTraderの動作に対応するために5本のローリングウィンドウを使用してローカルに計算され、インジケーターで`GetValue`を呼び出しません。
- ストップとテイクプロフィットの管理は価格レベルが違反されたときにマーケットエグジットで実行され、元のポジション変更の効果を反映します。
- `StartProtection()`の呼び出しは予期しない切断時の回復力のためにStockSharpの組み込みポジション監視を有効にします。

## 使用上の推奨事項
1. 元のPuriaセットアップを反映するために適切な足タイプ（例: 主要なFXペアの15分バー）を選択してください。
2. ピップベースのパラメーターをインストゥルメントのポイント値に合わせて調整してください。ヘルパーは自動的に5桁の見積もりにスケールしますが、エキゾチックなインストゥルメントはカスタム調整が必要な場合があります。
3. リスクベースのサイジングを有効にする場合、計算されたボリュームが取引可能であることを確認するためにポートフォリオ評価とボリュームステップの制約を確認してください。
4. 必要に応じてポートフォリオレベルの資金管理やセッションフィルターと組み合わせてください。戦略は元のMQLエキスパートのシグナルとトレーリングロジックに厳密に焦点を当てています。
