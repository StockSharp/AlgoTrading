# Blau TStoch インジケーター戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- MetaTrader 5 エキスパートアドバイザー `Exp_BlauTStochI` の StockSharp 高レベル API へのポート。
- 設定可能な時間軸で Blau トリプルストキャスティックインデックス（William Blau）を取引します。
- 2 つの実行モードをサポート：**Breakdown**（ゼロラインブレイクアウト）と **Twist**（傾き反転）。
- ポジション権限は元のエキスパートアドバイザーのフラグを再現します（ロング/ショート取引のオープン/クローズの独立したトグル）。

## インジケーターの構築
- `MomentumLength` バーにわたって `適用価格 - 最低値` としてモメンタムシリーズを計算し、その範囲 `最高値 - 最低値` を計算します。
- 分子と分母の両方に 3 つの連続したスムージングステージを適用します。
- サポートされているスムージング方法：指数 (EMA)、シンプル (SMA)、スムージング/ランニング (SMMA)、線形加重 (LWMA)。
- 元の MQL オプション（JJMA、JurX、ParMA、T3、VIDYA、AMA）は**再現されません**；`Phase` パラメーターは互換性のために保持されますが無視されます。
- 適用価格オプションは MQL 列挙に一致します（終値、始値、高値、安値、中央値、典型的価格、加重価格、シンプル価格、四分値、トレンドフォロー変形、DeMark）。
- 最終インジケーター値：`100 * スムーズドStoch / スムーズドRange - 50`。

## 取引ルール
### Breakdown モード
- `SignalBar` で定義されたバーのインジケーターを検査します（デフォルト 1、つまり最後に閉じたローソク足）。
- **ロングエントリー：** 前の値（`SignalBar+1`）がゼロより上 **かつ** 現在の値（`SignalBar`）がゼロ以下にクロス。
- **ショートエントリー：** 前の値がゼロより下 **かつ** 現在の値がゼロ以上にクロス。
- **ロング決済：** 前の値がゼロより下でロング決済が許可されている。
- **ショート決済：** 前の値がゼロより上でショート決済が許可されている。

### Twist モード
- **ロングエントリー：** インジケーター上昇中（`value[SignalBar+1] < value[SignalBar+2]`）かつ最新値が前の値以上。
- **ショートエントリー：** インジケーター下降中（`value[SignalBar+1] > value[SignalBar+2]`）かつ最新値が前の値以下。
- **ロング決済：** インジケーターの傾きが下向きに転換（`value[SignalBar+1] > value[SignalBar+2]`）。
- **ショート決済：** インジケーターの傾きが上向きに転換（`value[SignalBar+1] < value[SignalBar+2]`）。

### ポジション管理
- エントリーは設定された `Volume` に絶対ポジションサイズを加算して既存の逆ポジションを反転します。
- 決済は成行注文で既存のポジション全体をクローズします。
- 取引処理は完成したローソク足でのみ、インジケーターが完全に形成された後に実行されます。

## リスク管理
- 価格ステップで測定されたオプションのストップロスとテイクプロフィット（`StopLossPoints`、`TakeProfitPoints`）。
- 両方とも `StartProtection` を通じて実装されており、距離をゼロに設定することで無効化できます。

## パラメーター
| パラメーター | 説明 | デフォルト値 |
|------------|------|------------|
| `CandleType` | 計算用のデータタイプ/時間軸。 | 4 時間ローソク足 |
| `Smoothing` | スムージング方法 (EMA/SMA/SMMA/LWMA)。 | EMA |
| `MomentumLength` | 最高値/最低値検出のルックバック。 | 20 |
| `FirstSmoothing` | スムージングステージ 1 の長さ。 | 5 |
| `SecondSmoothing` | スムージングステージ 2 の長さ。 | 8 |
| `ThirdSmoothing` | スムージングステージ 3 の長さ。 | 3 |
| `Phase` | 互換性のために保持（無視）。 | 15 |
| `PriceType` | 適用価格定数。 | Close |
| `SignalBar` | シグナル評価に使用するバーシフト (>= 1)。 | 1 |
| `Mode` | 取引モード (Breakdown/Twist)。 | Twist |
| `AllowLongEntries` | ロングエントリーを有効化。 | true |
| `AllowShortEntries` | ショートエントリーを有効化。 | true |
| `AllowLongExits` | ロング取引のクローズを有効化。 | true |
| `AllowShortExits` | ショート取引のクローズを有効化。 | true |
| `TakeProfitPoints` | ステップでのテイクプロフィット距離（0 で無効化）。 | 2000 |
| `StopLossPoints` | ステップでのストップロス距離（0 で無効化）。 | 1000 |

## MT5 エキスパートとの違い
- SmoothAlgorithms.mqh の高度なスムージングアルゴリズムは実装されていません；EMA/SMA/SMMA/LWMA から選択してください。
- マネーマネジメント（ロットサイジング）は簡素化されており、戦略は StockSharp の `Volume` プロパティに依存します。
- シグナル評価は完成したローソク足でのみ行われます；バー内実行はありません。

## 使用上の注意
- `SignalBar` が少なくとも 1 のままであることを確認してください；実装は十分なインジケーター履歴を自動的に維持します。
- スムージング長を増やすと、各ステージがウィンドウ全体を完了する必要があるため、形成時間が増加します。
- 高い時間軸でのリバーサルトレードには、ストップ/テイク距離を広げるか、権限を通じて一方を無効にすることを検討してください。
