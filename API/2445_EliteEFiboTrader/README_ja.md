# Elite eFibo Trader 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**Elite eFibo Trader 戦略**は、MQL5エキスパートアドバイザー「Elite eFibo Trader」の移植版です。フィボナッチベースの平均化グリッドを実装し、最初の成行ポジションを開いて固定距離に追加のストップ注文を積み重ねます。戦略はティックデータで動作し、グリッドが拡張するにつれてトレーリングストップを自動的に管理します。

## 動作の仕組み
1. アクティブなポジションも未決注文もなく、トレードが許可されている場合、戦略は選択した方向（買いまたは売り）で新しいサイクルを開始します。
2. 最初の注文は`LotsLevel1`に設定されたボリュームで成行送信されます。さらに13の追加ストップ注文が現在価格から`LevelDistance`の倍数の位置に置かれます。そのボリュームは`LotsLevel2`…`LotsLevel14`で定義されたフィボナッチ数列に従います。
3. 各執行済み注文はエントリー価格から`StopLossPoints`離れた個別のストップレベルを設定します。これらのストップの最高値（ロングポジションの場合）または最低値（ショートポジションの場合）が、すべてのオープンポジションのアクティブなトレーリングレベルになります。
4. 価格がトレーリングレベルに達すると、ポジション全体がクローズされ、残りの未決注文がすべてキャンセルされます。
5. 未実現利益は口座通貨で監視されます。`MoneyTakeProfit`に達するとグリッドがクローズされます。`TradeAgainAfterProfit`の設定に応じて、戦略は自動的に再起動するか、手動再有効化を待ちます。

戦略は`SubscribeTrades()`を通じてティックレベルの市場データを必要とし、一度に一方向（`OpenBuy` xor `OpenSell`）のみが有効であることを前提とします。

## パラメーター
- `OpenBuy` – グリッドのロングのみバージョンを有効化。
- `OpenSell` – グリッドのショートのみバージョンを有効化。
- `TradeAgainAfterProfit` – 利益確定後に自動的に新しいサイクルを開始。
- `LevelDistance` – 未決注文間のスペーシング（銘柄の価格ステップ単位）。
- `StopLossPoints` – 各エントリーからのストップロス距離（価格ステップ単位）。
- `MoneyTakeProfit` – 口座通貨で表された未実現利益目標。
- `LotsLevel1`…`LotsLevel14` – 各グリッドレベルの個別ボリューム。デフォルト値はフィボナッチ数列（1, 1, 2, 3, 5, …, 377）に従います。

## トレードロジックの詳細
- 価格オフセットは銘柄の`PriceStep`で計算されます。ゼロの場合、戦略は注文を置きません。
- 一度にアクティブなトレードサイクルは1つのみです。すべての未決注文はサイクル開始時に作成され、執行されるか明示的にキャンセルされるまで残ります。
- トレーリングストップは新しいグリッドレベルが約定されるか、ポジションの一部がクローズされるたびに再計算されます。これにより、すべての注文が利用可能な最良の保護レベルを共有することが保証されます。
- 利益管理は`Position`、`PositionPrice`、`PriceStep`、`StepPrice`から導出されたフローティングPnLに基づきます。
- `TradeAgainAfterProfit`が無効の場合、パラメーターが手動で再び切り替えられるまで、金銭目標達成後に戦略は非アクティブのままです。

## 使用上の注意
- 開始前に正しい方向を設定してください（ロングまたはショート）。両方向を同時に有効にするとグリッドの起動が妨げられます。
- 銘柄のボラティリティとコントラクトサイズに応じてレベル距離とボリュームを調整してください。大きなフィボナッチボリュームは積極的なスケーリングを生み出すため、過去データで慎重にテストする必要があります。
- トレーディング口座とブローカーが計算された価格レベルでのストップ注文をサポートしていることを確認してください。そうでない場合、注文が拒否される可能性があります。
