# Last ZZ50戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Last ZZ50戦略は、Vladimir KarputovのMetaTrader向けエキスパートアドバイザー「Last ZZ50」を再現します。
ZigZagインジケーターを使用して最近の3つの転換点を追跡し、最後の2つのZigZag脚の中間点に保留注文を配置します。
このアプローチは最新のスイングからのブレイクアウトに参加しようとし、ZigZag構造が変化するたびに注文をキャンセルまたは再配置します。

## トレードロジック
- **ピボット検出** – ZigZagインジケーター（デフォルト: 深さ12、偏差5、バックステップ3）がA（最新）、B、Cとラベル付けされた最新ピボットを提供する。
- **BC脚注文** – ピボットCがBと異なり、新しいピボットAが脚の方向を無効にしない場合、戦略は`(B + C) / 2`に保留注文を配置する。
  - BC脚が上昇している場合、注文はロング、そうでない場合はショートになる。
  - 中間点に対する現在価格の位置に応じてリミットまたはストップタイプが選択される。
- **AB脚注文** – 同じ中間点ロジックがAB脚に適用され、現在価格に応じてリミットまたはストップ注文が使用される。
- **セッションフィルター** – 取引は設定可能な曜日とイントラデイウィンドウに限定される（デフォルト: 月曜日09:01から金曜日21:01）。ウィンドウ外では、戦略は保留注文をキャンセルし、オプションで任意のポジションをフラット化できる。
- **トレーリングエグジット** – ポジションがトレーリングストップとトレーリングステップの閾値の合計以上の利益を得たら、利益を確保するために保護ストップ注文が価格の後ろに追従する。

## リスク管理
- 保留注文のボリュームは、乗数パラメーターとインストゥルメントの最小取引可能ボリュームを掛けたものに等しい。
- ABとBCの両注文は、ZigZagピボットが変化するたびにキャンセルされ再作成され、古い注文が板に残らないようにする。
- トレーリングストップは、ポジションが余裕を持って利益状態になった後にのみ有効化され、乱れた相場での早期エグジットを減らす。

## パラメーター
- `LotMultiplier` – 注文送信時に最小取引可能ボリュームに適用される乗数。
- `ZigZagDepth`, `ZigZagDeviation`, `ZigZagBackstep` – ZigZagインジケーターの設定値。
- `TrailingStopPips`, `TrailingStepPips` – pipsで測定されるトレーリングストップの距離と有効化閾値。
- `StartDay`, `EndDay`, `StartTime`, `EndTime` – 取引セッションの境界。
- `CloseOutsideSession` – 時間フィルターが非アクティブな場合にポジションをフラット化するかどうか。
- `CandleType` – ZigZag計算に使用されるローソク足シリーズ（デフォルト1時間）。

## インジケーター
- **ZigZag** – 注文配置と構造検証を駆動するピボットポイントを提供する。
