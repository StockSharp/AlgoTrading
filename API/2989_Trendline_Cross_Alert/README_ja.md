# トレンドライン・クロス・アラート戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、手動で描かれた水平線とトレンドラインの価格クロスを監視した元のMetaTraderエキスパートアドバイザーの動作を再現します。完了したローソク足を継続的に監視し、ローソク足の実体が登録されたレベルをまたいだかどうかを確認し、クロスが最初に発生したときにアラートを生成します。デフォルトでは自動注文は送信されません；モジュールは裁量的なレベルを追跡し、オペレーターに通知することに焦点を当てています。

## 変換のハイライト
- *監視色*の値でタグ付けされた線のみが考慮され、色でオブジェクトをフィルタリングした元のEAを反映しています。
- クロスが検出されると、その後のローソク足が重複したアラートを発しないように、ラインは内部的にフラグが立てられます。これはMetaTraderの `CrossedColor` 入力にオブジェクトを再色付けすることを反映しています。
- StockSharpはターミナルからチャートオブジェクトを公開しないため、レベルはテキストパラメーターを通じて定義されます。水平エントリーは `Name|Color|Price` ブロックから解析され、トレンドラインは `Name|Color|StartTime|StartPrice|EndTime|EndPrice` を使用し、2つのアンカーポイント間の無限の線として評価されます。
- アラート、プッシュ通知、メールのオプションは、プラットフォーム固有の通知チャンネルなしでもワークフローが透明性を保てるように、情報ログエントリーにマッピングされます。

## パラメーター
| パラメーター | タイプ | 説明 |
| --- | --- | --- |
| `MonitoringColor` | `string` | 監視されるためにラインが一致しなければならない色ラベル。大文字小文字を区別しません。 |
| `CrossedColor` | `string` | ラインがクロスされたことを示すアラートメッセージで使用されるラベル。 |
| `HorizontalLevelsInput` | `string` | セミコロンで区切られた水平レベルのリスト。各エントリーは `Name|Color|Price`；色が省略された場合、監視色が仮定されます。 |
| `TrendlineDefinitions` | `string` | セミコロンで区切られたトレンドラインのリスト。各エントリーは `Name|Color|StartTime|StartPrice|EndTime|EndPrice`。時刻はISO 8601形式で、取引カレンダーのタイムゾーンを使用する必要があります。 |
| `EnableAlerts` | `bool` | `true` の場合、戦略はクロスを説明する情報ログエントリーを書き込みます。 |
| `EnableNotifications` | `bool` | プッシュ通知をエミュレートする2番目のログエントリーを追加します。 |
| `EnableEmails` | `bool` | メールアラートをエミュレートする3番目のログエントリーを追加します。 |
| `CandleType` | `DataType` | 市場を監視するために使用されるローソク足シリーズ。 |

## 定義フォーマット
1. 複数のエントリーをセミコロン（`;`）で区切ります。
2. 水平レベルは名前または色を省略できます：
   - `1.1050` → 監視色を使用して価格 `1.1050` で `Horizontal 1` として監視されます。
   - `Resistance|1.1180` → 監視色を使用した状態でのカスタム名。
   - `Breakout|Blue|1.1225` → カスタム色でも追跡されるためには `MonitoringColor` と一致しなければなりません。
3. トレンドラインにはISO 8601タイムスタンプ（`2024-03-15T10:00:00Z`）を持つ2つのアンカーポイントが必要です。欠落した色の値は監視色にデフォルト設定されます。ラインはMetaTraderのトレンドラインとまったく同様にアンカーを超えて外挿されます。

## 実行フロー
1. `OnStarted` 中に、テキスト定義は強く型付けされた構造体に解析され、メモリに保存されます。
2. 設定されたサブスクリプションからの完了したローソク足が `ProcessCandle` を発動させます。
3. メソッドはローソク足がレベルの一方の側で開き、他方の側で閉じたかどうかを確認します。そうであれば、ラインはクロスとしてマークされ、メッセージが生成されます。
4. メッセージにはクロスの方向、理論的なラインの価格、終値が含まれ、裁量的なトレーダーが手動で反応できます。

## 通知
StockSharpの戦略はUIポップアップの代わりにログメッセージを発します。有効化された各通知チャンネルは別々のログエントリーを生成し、必要に応じてホストアプリケーションがそれらを実際のアラートシステムにルーティングできます。

## 使用チェックリスト
1. 楽器と時間軸を選択し、それに応じて `CandleType` を設定します。
2. MetaTraderのワークスペースで描かれたライン（またはカスタム値）で `HorizontalLevelsInput` と `TrendlineDefinitions` を入力します。
3. 目的のアラートチャンネルに合わせて通知ブール値を調整します。
4. 戦略を開始します。希望する場合、チャートサブシステムを使用してラインを手動でプロットできます；このモジュールは検出に焦点を当てています。

## 設定例
```
MonitoringColor = "Yellow"
CrossedColor = "Green"
HorizontalLevelsInput = "DailyPivot|Yellow|1.1025;WeeklyHigh|Yellow|1.1100"
TrendlineDefinitions = "UpperChannel|Yellow|2024-03-14T08:00:00Z|1.0950|2024-03-14T16:00:00Z|1.1080"
EnableAlerts = true
EnableNotifications = true
EnableEmails = false
CandleType = 15 minute candles
```
この設定は2つの静的レベルと1つの上昇トレンドラインを監視します。`Price crossed horizontal line 'DailyPivot' upward at 1.10250 ...` のようなメッセージが各レベルを終値が通過する最初の時に書き込まれます。

## リスク管理と拡張
- 戦略はポジションを変更しません。自動取引が必要な場合は、別の実行ロジックと組み合わせてください。
- アラートをリセットするには、戦略を停止して再起動するか、定義文字列を調整します。`HashSet` 状態の永続化は元のEA動作に近い状態を保つために意図的に避けられています。
- セッションフィルターやボラティリティチェックなどの追加の保護手段は、`ProcessCandle` メソッドを拡張することで重ね合わせることができます。
