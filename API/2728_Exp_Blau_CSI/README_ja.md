# Exp Blau CSI戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略はMetaTrader 5のエキスパートアドバイザー`Exp_BlauCSI`のC#変換です。選択したローソク足シリーズに対して計算されたBlau Candle Stochastic Index（CSI）で取引します。戦略はゼロラインのブレイクダウンまたはインジケーターの方向変化のいずれかに反応でき、価格ステップで測定された設定可能なストップロスとテイクプロフィットレベルをサポートします。

## 取引ロジック

Blau CSIはモメンタムコンポーネントを最近のローソク足の高値-安値レンジと比較します。両方の部分は選択した移動平均タイプを使用して3回平滑化されます。

* **ブレイクダウンモード** – インジケーターがゼロを下回って交差し、前の値が正だった間のショートをクローズするときにロングポジションを開きます。前の値が負だった間のロングをクローズしながらゼロを超えて交差するときにショートポジションを開きます。
* **ツイストモード** – インジケーターが上向きに転換するとき（下落後に前のバーと比較して値が上昇）にロングポジションを開きます。インジケーターが下向きに転換するときにショートポジションを開きます。前のバーの方向は常に反対側の既存ポジションをクローズするために使用されます。

シグナルは完全にクローズしたローソク足での確認を保証するために設定可能な過去バー（`Signal Bar`）で評価されます。

## パラメーター

| パラメーター | 説明 |
|-------------|------|
| `Entry Mode` | `Breakdown`または`Twist`ロジックを選択する。 |
| `Smoothing Method` | Blau CSI内で使用される移動平均タイプ（単純、指数、平滑化、LinearWeighted、またはJurik）。 |
| `Momentum Length` | モメンタムとレンジコンポーネントを計算するために使用されるバー数。 |
| `First/Second/Third Smoothing` | モメンタムとレンジに適用される3つの平滑化ステージの深さ。 |
| `Smoothing Phase` | Jurik平滑化のフェーズパラメーター（他の方法では無視される）。 |
| `Momentum Price` / `Reference Price` | 先行および遅行モメンタム値に適用される価格定数（終値、始値、高値、安値、中央値、典型値、加重値、単純値、四半期、トレンドフォロー、またはDemark）。 |
| `Signal Bar` | Blau CSIバッファを評価する際のオフセット（バー単位）。デフォルト`1`は前のクローズしたバーを意味する。 |
| `Stop Loss (pts)` | 価格ステップでのストップロス距離（`0`で無効）。 |
| `Take Profit (pts)` | 価格ステップでのテイクプロフィット距離（`0`で無効）。 |
| `Allow Long/Short Entries` | 各方向のポジション開設を有効または無効にする。 |
| `Allow Long/Short Exits` | 既存ポジションの出口シグナルを有効または無効にする。 |
| `Candle Type` | サブスクリプションのデータタイプ（デフォルトは4時間時間軸）。 |
| `Start Date` / `End Date` | 取引活動の日付フィルター。 |
| `Order Volume` | 成行注文ボリューム。 |

## リスク管理

新しいポジションが開かれると、戦略はインストゥルメントの`PriceStep`を使用してストップロスとテイクプロフィットレベルを計算します。インストゥルメントがステップを提供しない場合、ストップは自動的に無効になります。トレーリングは実行されません；各ポジションはシグナルによってまたはターゲットに達してクローズされるまで初期保護レベルを維持します。

## 使用ノート

1. 選択した`Candle Type`のローソク足データを提供するインストゥルメントに戦略を適用する。
2. 取引計画に従ってインジケーターモードと平滑化パラメーターを選択する。
3. ストップロスまたはテイクプロフィット距離を使用する場合、インストゥルメントに有効な`PriceStep`があることを確認する。
4. オプションで`Start Date`と`End Date`を使用して取引を時間範囲に制限する。

## オリジナルMT5バージョンとの違い

* 実装はMetaTraderの取引関数の代わりにStockSharpインジケーターとC#戦略APIを使用する。
* ロットサイズ管理は簡略化されています：注文ボリュームは`Order Volume`パラメーターから直接取得されます。
* StockSharpが提供する平滑化方法のみがサポートされます（単純、指数、平滑化、LinearWeighted、Jurik）。サポートされていないMT5モードは指数平滑化にフォールバックします。
* 取引方向トグルとストップ管理はオリジナルの動作と互換性があります。

この戦略はStockSharp Designer、Shell、Runner、またはカスタムStockSharpホストアプリケーション内でバックテストの準備ができています。
