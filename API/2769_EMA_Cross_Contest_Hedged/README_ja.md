# EMA クロス・コンテスト・ヘッジ戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- StockSharp 高レベル API を使用して MetaTrader 戦略「EMA Cross Contest Hedged」を再現します。
- 一対の指数移動平均（EMA）でトレードし、オプションで MACD メインラインで確認します。
- 各エントリー後に保留中のストップオーダーの「ヘッジ」レベルの階段を構築し、強いトレンドにスケールインします。
- pips で表された静的なストップロス/テイクプロフィットレベルと、最小利益後に活性化するトレーリングストップを適用します。
- シグナルに現在の完成ローソク足または前のクローズしたローソク足のどちらを使用するかを選択できます。

## インジケーターとデータ
- 設定可能な長さの短い EMA（デフォルト 4）。
- 設定可能な長さの長い EMA（デフォルト 24）；短い期間は長い期間より小さくなければなりません。
- MACD (4, 24, 12) メインライン はオプションの確認フィルターとして使用されます。
- `CandleType` パラメーターが提供する任意の時間軸で動作します（デフォルトは 15 分足）。

## エントリー条件
1. 設定された時間軸の完成したローソク足を待ちます。
2. 速い EMA と遅い EMA の値を計算します。`TradeBar` に応じて、以下のどちらかを使用してクロスオーバーを判定します：
   - 最新の完成したローソク足と前のローソク足（`Current`）。
   - 前のローソク足とさらに前のローソク足（`Previous`、デフォルト）。
3. 速い EMA が遅い EMA を上抜けたときにロングシグナルを生成します。`UseMacdFilter` が有効な場合、同じバーの MACD 値が非負でなければなりません。
4. 速い EMA が遅い EMA を下抜けたときにショートシグナルを生成します。MACD フィルターが有効な場合、MACD 値は非正でなければなりません。
5. 露出がない場合のみ（以前のすべてのトレードがフラット）新規ポジションを開きます。
6. サイズ `OrderVolume` の成行注文を実行します。エントリー後、戦略は：
   - 約定価格から `StopLossPips` と `TakeProfitPips` だけオフセットされたストップロスとテイクプロフィットのレベルを保存します。
   - トレーリングストップ状態をリセットします。
   - トレード方向に `HedgeLevelPips` の間隔で 4 つのヘッジングストップオーダーを作成します。各保留注文は同じストップロス/テイクプロフィット距離を継承し、価格が早くそこに到達しない限り `PendingExpirationSeconds` 秒後に期限切れになります。

## エグジット管理
- **ストップロス / テイクプロフィット：** 戦略はバー内の高値と安値を監視します。価格が保存されたストップまたはターゲットに触れた場合、全ポジションがクローズされます。
- **トレーリングストップ：** 利益が `TrailingStopPips + TrailingStepPips` を超えると、ストップは最後のクローズから `TrailingStopPips` 後ろに追従します。ロングポジションは上方に、ショートポジションは下方に追従します。
- **逆クロスオーバー：** `CloseOppositePositions` が有効な場合、逆の EMA クロスオーバーが検出されるとすぐにポジションがクローズされます。
- **保留中の階段：** 各ヘッジング注文は、価格がストップレベルを越えると追加の成行注文に変換されます。新しい約定は平均エントリー価格を調整し、保護レベルを適切に調整します。

## パラメーター
| 名前 | デフォルト | 説明 |
| --- | --- | --- |
| `OrderVolume` | 0.1 | すべての成行注文またはストップ注文に使用される注文サイズ。 |
| `StopLossPips` | 140 | pips 単位のストップ距離。0 に設定すると無効になります。 |
| `TakeProfitPips` | 120 | pips 単位のテイクプロフィット距離。0 に設定すると無効になります。 |
| `TrailingStopPips` | 30 | pips 単位のトレーリングストップ距離。0 に設定するとトレーリングを無効にします。 |
| `TrailingStepPips` | 1 | トレーリングストップが再び締まる前の最小追加利益（pips）。 |
| `HedgeLevelPips` | 6 | 段階的なヘッジングストップ注文間の距離。 |
| `CloseOppositePositions` | false | 逆のクロスオーバーが現れたときにアクティブなポジションをクローズします。 |
| `UseMacdFilter` | false | MACD 確認を要求します（ロングは >= 0、ショートは <= 0）。 |
| `PendingExpirationSeconds` | 65535 | 各ヘッジングストップ注文の有効期間（秒）。 |
| `ShortMaPeriod` | 4 | 短い EMA の長さ。`LongMaPeriod` より小さくなければなりません。 |
| `LongMaPeriod` | 24 | 長い EMA の長さ。 |
| `TradeBar` | Previous | クロスオーバーを検出するために使用するバーのペアを決定します。 |
| `CandleType` | 15 分 | データプロバイダーからリクエストされる時間軸。 |

## 補足事項
- pips は `Security.PriceStep` を乗算して変換され、MetaTrader の pip 規約に合わせるために 3 桁および 5 桁の小数点を持つ商品には自動的に 10 の係数が適用されます。
- 保留中のヘッジング注文は戦略内部でシミュレートされ、ローソク足の範囲がそのレベルに触れるとすぐに実行されます。
- `StartProtection()` は StockSharp の組み込みポジション保護サービスを有効にするために呼び出されます。
- 戦略は元のヘッジされた実装を反映するために、ロングおよびショートポジションに別個のトレーリングストップロジックを維持します。
