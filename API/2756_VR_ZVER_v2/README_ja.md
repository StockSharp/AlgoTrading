# VR-ZVER v2 ストラテジー
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

VR-ZVER v2 ストラテジーは、クラシックなMetaTraderのエキスパートアドバイザーのStockSharpポートです。元のスクリプトのトリプル確認のアイデアを維持しています：すべてのトレードは移動平均、ストキャスティクスオシレーター、RSIによってサポートされなければなりません。すべての有効なフィルターが一致した時のみ、ストラテジーは成行注文を出します。

## トレードロジック

- シグナルはローソク足が閉じるときに評価されます。イントラバーの変動はストップやターゲットの検証にのみ使用されます。
- MAフィルターが有効な場合、3つの指数移動平均（高速、低速、非常に低速）は同じ順序でスタックされてトレンドを検証する必要があります。
- ストキャスティクスフィルターは、設定可能な上限と下限バンドの近くで%K/%Dのクロスオーバーを待ちます。
- RSIフィルターは、オシレーターが中立ゾーンを離れることを要求します（ロングの場合は下限バンドを下回り、ショートの場合は上限バンドを上回る）。
- シグナルは、すべての有効なフィルターが同じ方向に投票した場合のみ受け入れられます。いずれかのフィルターが同意しない場合、取引は行われません。
- ストラテジーは一度に1つのポジションを開きます。ヘッジやグリッドの構築は行いません；フラットの場合、次の整合されたシグナルを待ちます。

## ポジション管理

- テイクプロフィットとストップロスはpipsで表されます。初期ストップは設定された距離の3分の2に設定され、元のEA動作を再現します。
- ブレークイーブントリガー（pips単位）は、取引が指定された距離を得た後にストップをエントリー価格に移動します。
- トレーリングストップは距離と追加ステップを使用します。ステップにより、ストップはすべての小さな上昇で更新されるのを防ぎ、MT5のトレーリングロジックに対応します。
- ロングとショートトレードは同じ管理ルールを共有し、ローソク足の高値/安値に対して対称的に反応します。

## ポジションサイジング

- `FixedVolume`がゼロより大きい場合、各注文を固定サイズで開きます。
- `FixedVolume`がゼロに設定されている場合、ストラテジーは`RiskPercent`、現在のポートフォリオ価値、ストップ距離からボリュームを計算します。価格ステップとステップ価格は、pips距離を金銭的リスクに変換するために使用されます。
- ボリュームは、インスツルメントの`VolumeMin`、`VolumeMax`、`VolumeStep`の制約を尊重するために丸められます。計算されたサイズが小さすぎる場合、注文はスキップされます。

## パラメーター

| 名前 | 説明 |
| ---- | ---- |
| `CandleType` | シグナル生成に使用する時間軸（デフォルト15分ローソク足）。 |
| `FixedVolume`, `RiskPercent` | 固定またはリスクベースのサイジングを選択。 |
| `StopLossPips`, `TakeProfitPips` | pips単位の基本保護距離。 |
| `TrailingStopPips`, `TrailingStepPips`, `BreakevenPips` | トレード管理の閾値。 |
| `AllowLongs`, `AllowShorts` | 個別の方向を有効または無効化。 |
| `UseMovingAverageFilter`, `FastMaPeriod`, `SlowMaPeriod`, `VerySlowMaPeriod` | トリプルEMAトレンドフィルター。 |
| `UseStochastic`, `StochasticKPeriod`, `StochasticDPeriod`, `StochasticSmooth`, `StochasticUpperLevel`, `StochasticLowerLevel` | ストキャスティクス確認設定。 |
| `UseRsi`, `RsiPeriod`, `RsiUpperLevel`, `RsiLowerLevel` | RSI確認バンド。 |

## 注意事項

- pip変換は元のEAをエミュレートします：5桁と3桁のシンボルは、pip値を計算する前に価格ステップを10倍にします。
- StockSharpポートは成行注文のみを使用します。MetaTraderバージョンのロックと保留注文機能は、実装を高レベルAPIと一貫して保つために意図的に省略されています。
- EMA、ストキャスティクス、RSIのオーバーレイを表示したい場合は、ストラテジーをチャートに添付してください；チャートエリアが利用可能な場合に自動的に描画されます。
