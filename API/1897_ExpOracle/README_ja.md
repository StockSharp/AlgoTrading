# Exp Oracle 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略はMetaTraderエキスパートアドバイザー **Exp_Oracle** のC#移植版です。相対力指数（RSI）と商品チャネル指数（CCI）を組み合わせて数バー先の市場の方向を予測するカスタム *Oracle* インジケーターに依存しています。インジケーターは2本のラインを生成します。

- **Oracleライン** – CCIとRSI極値の生の組み合わせ。
- **シグナルライン** – Oracleラインの平滑化移動平均。

戦略はこれらのラインを解釈するための3つのトレードモードを提供します。

1. **Breakdown** – シグナルラインがゼロレベルをクロスしたときにポジションをオープンします。
2. **Twist** – シグナルラインの局所的な転換点に反応します。
3. **Disposition** – シグナルラインとOracleラインのクロスで取引します。

## パラメーター

- `OraclePeriod` – RSIとCCI計算の期間。
- `Smooth` – シグナルラインを平滑化するために使用するバーの数。
- `Mode` – トレードシグナルの生成に使用するアルゴリズム（`Breakdown`、`Twist`、または `Disposition`）。
- `CandleType` – 受信ローソク足の時間軸。
- `AllowBuy` – ロングエントリーを有効にします。
- `AllowSell` – ショートエントリーを有効にします。
- `Volume` – ベースクラス `Strategy` から継承された戦略ボリューム。

## エントリーとエグジットのルール

### Breakdown
- シグナルラインがゼロを上抜けしたときに**買い**。
- シグナルラインがゼロを下抜けしたときに**売り**。

### Twist
- シグナルラインが下落後に上向きに転換したときに**買い**。
- シグナルラインが上昇後に下向きに転換したときに**売り**。

### Disposition
- シグナルラインがOracleラインを上抜けしたときに**買い**。
- シグナルラインがOracleラインを下抜けしたときに**売り**。

反対のシグナルが現れると既存のポジションは決済され反転されます。戦略はシンプルさのために成行注文を使用します。

## インジケーターのロジック

各バーについて:
1. 指定された `OraclePeriod` でRSIとCCIを計算します。
2. 最近のCCIとRSI値の差を使用して4つのダイバージェンス値を構築します。
3. OracleラインはダイバージェンスのMaxとMinの合計です。
4. シグナルラインはOracleラインの `Smooth` バーにわたる単純移動平均です。

このアプローチはモメンタム（RSI）とチャネル（CCI）の情報を組み合わせることで短期的な価格動向を予測しようとします。

## 注意事項

- 戦略は完成したローソク足のみで動作します。
- 保護ストップは実装されていません。必要に応じて外部のリスク管理を使用してください。
