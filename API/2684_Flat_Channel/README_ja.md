# フラットチャンネル戦略 (2684)
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は MetaTrader 5 エキスパートアドバイザー *Flat Channel (barabashkakvn 版)* の C# 変換です。標準偏差インジケーターを使用して低ボラティリティ期間（「フラット」チャンネル）を検出し、チャンネルの境界にブレイクアウトのストップ注文を配置します。価格がフラットレンジから抜け出すと対応するストップ注文が発動し、市場の両側に挟まれることを避けるために反対方向の注文がキャンセルされます。

## コアロジック

1. **ボラティリティフィルター** – 戦略はローソク足をサブスクライブし、中間価格の標準偏差を計算します。値が少なくとも `FlatBars` 連続ローソク足で下落し続けるとフラットフェーズが確認されます。
2. **チャンネル構築** – フラットフェーズが確認されると、フラットレンジの最高値と最安値が追跡されます。チャンネル幅は `ChannelMinPips` と `ChannelMaxPips` の間（インストゥルメントのティックサイズで価格単位に変換）に維持される必要があります。
3. **エントリー注文** – 価格がチャンネル内で取引されている間、戦略は以下を配置します：
   - チャンネル高値にバイストップ、ストップロスはエントリーの `2 × チャンネル幅` 下、テイクプロフィットは `1 × チャンネル幅` 上。
   - チャンネル安値にセルストップ、対称的なストップロス/テイクプロフィット距離で。
4. **注文の有効期限** – 保留中のストップ注文は `OrderLifetimeSeconds` 後に期限切れになります。タイムアウトが経過するとキャンセルされ、フラット状態が続く場合は再作成できます。
5. **ポジション管理** – エントリー注文が執行された後、反対方向のストップ注文がキャンセルされ、新しい保護注文（ストップロスとテイクプロフィット）が登録されます。オプションのブレークイーブンロジックは、価格がテイクプロフィット目標に向かってフィボナッチ比率（`FiboTrail`）の距離を移動したときにストップロスをエントリー価格に移動します。
6. **取引ウィンドウ** – `UseTradingHours` フィルターは曜日と月曜日/金曜日の特定の時間帯でアクティビティを制限し、元の EA のスケジュール制御をエミュレートします。

## インジケーター

- **StandardDeviation**（中間価格、長さ = `StdDevPeriod`） – ボラティリティの低下を検出します。
- **DonchianChannels**（長さ = `FlatBars`） – フラットチャンネルの初期高値/安値の境界を提供します。

## リスクと資金管理

- `UseMoneyManagement` が無効の場合、`FixedVolume` がロットサイズを定義します。
- `UseMoneyManagement` が有効の場合、ポジションサイズは `PriceStep` と `StepPrice` を使用して金額で表されたストップロス距離で割った現在のポートフォリオ価値の `RiskPercent` から推定されます。
- 損失取引の後、次のポジションは `FixedVolume × 4` を使用し、元の EA の回復動作を再現します。

## パラメーター

| パラメーター | 説明 |
|------------|------|
| `UseTradingHours` | 曜日/時間スケジュールフィルターを有効または無効にします。 |
| `TradeTuesday`、`TradeWednesday`、`TradeThursday` | 週半ばの個別の日の取引を許可します（月曜日と金曜日は常に許可されますが、時間制限で制御されます）。 |
| `MondayStartHour`、`FridayStopHour` | 月曜日の開始時間と金曜日の終了時間（24時間制）。 |
| `UseMoneyManagement`、`RiskPercent`、`FixedVolume` | 上記の資金管理オプション。 |
| `OrderLifetimeSeconds` | 保留中のエントリー注文の有効期限（0 = 有効期限なし）。 |
| `StdDevPeriod`、`FlatBars` | フラットフェーズ検出を制御するインジケーター設定。 |
| `ChannelMinPips`、`ChannelMaxPips` | pips で表された許容チャンネル幅（インストゥルメントのティックサイズを使用して変換）。 |
| `UseBreakeven`、`FiboTrail` | ブレークイーブンロジックを有効にし、ストップ調整を発動するフィボナッチ乗数を設定します。 |
| `CandleType` | 計算に使用するローソク足データタイプまたは時間軸。 |

## 注意事項

- 戦略は、pips ベースのしきい値を実際の価格に変換できるように `PriceStep` と `StepPrice` を公開するシンボルを期待します。
- 保留中の注文はボラティリティが下落し続ける場合にのみ再作成されます。ボラティリティが上昇するとフラット状態がリセットされ、すべてのエントリー注文がキャンセルされます。
- 保護的なストップとテイクプロフィット注文はポジションが閉じると自動的にキャンセルされます。
