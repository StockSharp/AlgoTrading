# RRS もつれた EA 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**RRS Tangled EA 戦略** は、MetaTrader 4 エキスパート アドバイザー「RRS Tangled EA」の StockSharp 移植です。オリジナルのシステムは、取引の方向とシンボルをランダムに選択する一方で、同時注文の数を制限し、トレーリングストップと厳格なリスク制限を通じて変動利益を保護します。変換されたバージョンは現在選択されている商品に焦点を当てており、高レベルの StockSharp API を使用してランダム エントリー、トレーリング、およびリスク管理動作を再現します。

## 中核ロジック
1. 設定されたキャンドル シリーズをサブスクライブし、完成したキャンドルを待ちます。
2. 各バーについて:
   - 既存のロングバスケットとショートバスケットのトレーリングストップレベルを更新します。
   - ローソク足の高値と安値を使用して、ストップロスとテイクプロフィットの距離を確認します。
   - すべてのオープンエントリーの変動利益を評価します。マネー・アット・リスクのしきい値を超えた場合はすべてを閉じます。
   - 取引が許可され、スプレッドが許容され、エントリー数が制限を下回っている場合は、`[0, 3]` にランダムな整数を描画します。
   - 構成された境界間のランダムなボリュームを使用して、ランダム値が `1` の場合は新しいロングを開き、値が `2` の場合は新しいショートを開きます。
3. トレーリングストップは、価格がアクティブ化距離だけ変動すると最良の買値/売値に従い、価格がトレーリングギャップによってリトレースした場合に利益を確定します。
4. リスク管理は、固定通貨モードまたは経常口座残高の割合として機能します。フローティングロスが設定された量を超えると、すべてのポジションが即座にフラット化されます。

## パラメーター
| 名前 | 説明 |
|------|-------------|
| `MinVolume` | ランダムに生成された取引量の下限。 |
| `MaxVolume` | ランダムな取引量の上限。 |
| `TakeProfitPips` | バスケットの平均エントリー価格に適用されるピップ単位のターゲット距離。 |
| `StopLossPips` | 平均エントリー価格から測定されたピップ単位のプロテクティブストップ距離。 |
| `TrailingStartPips` | トレーリングロジックがアクティブになる前に必要な利益距離。 |
| `TrailingGapPips` | トレーリングストップと最良買値/売値との間に維持されるギャップ。 |
| `MaxSpreadPips` | 新しいランダムエントリーを開く前に許可される最大スプレッド。 |
| `MaxOpenTrades` | 両方向の同時エントリの最大数。 |
| `RiskManagementMode` | 固定金額リスク処理と残高比率リスク処理を切り替えます。 |
| `RiskAmount` | 変動損益に対して監視されるリスク量 (通貨または割合)。 |
| `TradeComment` | 簿記のためのオプションのコメント。ソース EA との互換性のために保持されています。 |
| `Notes` | 戦略ステータス文字列内に表示される情報テキスト。 |
| `CandleType` | 意思決定に使用するキャンドルシリーズ。 |

## MQL バージョンとの違い
- 取引は、MetaTrader マーケットウォッチからシンボルをランダムに選択するのではなく、戦略に割り当てられた商品で実行されます。これにより、実装は StockSharp の単一セキュリティ戦略との互換性が保たれます。
- 注文管理は、元の EA が同じマジック ナンバーでポジションをグループ化した方法を反映して、集約されたロング/ショート バスケットに対して実行されます。
- スプレッド制御は、MetaTrader の `MarketInfo` 呼び出しではなく、オーダーブックからの最新の最良買値/売値に依存します。

## 使用上の注意
- スプレッドとトレーリングの計算が正確に保たれるように、接続されたブローカーまたはシミュレーターが買い気配と売り気配の両方を提供していることを確認してください。
- `MinVolume` と `MaxVolume` を楽器の許容音量範囲内に設定します。この戦略は、ランダムなボリュームをシンボルのボリューム ステップと制限に自動的にスナップします。
- リスク管理ロジックは、変動損失が設定されたしきい値を超えると、「すべて」の取引を直ちに終了します。次のローソク足まで新しいポジションはオープンされません。
