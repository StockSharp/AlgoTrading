# 適応型Perceptron戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

この戦略はMetaTrader 5エキスパートアドバイザー*Perceptron.mq5*のStockSharpへのポートです。  
5つの離散的なインジケーターシグナルが2層パーセプトロンによって組み合わされます。各取引はインジケーターの状態を記録し、ポジションが閉じられると、達成された利益に応じてシナプス荷重が強化または罰則を受けます。この動作はStockSharpの高レベルローソク足APIを活用しながらオリジナルEAの自己学習ループを模倣します。

## インジケーター層

| コード | 説明 | シグナルロジック |
| --- | --- | --- |
| `IND1` | 高速/低速単純移動平均クロスオーバー | 前のバーで高速MAが低速MAを上回った時+1、下向きクロスで−1、それ以外は0。 |
| `IND2` | 相対力指数（RSI） | RSIが売られ過ぎゾーンを離れた時（30を上回るクロス）+1、RSIが買われ過ぎゾーンを離れた時（70を下回るクロス）−1。 |
| `IND3` | 商品チャンネル指数（CCI） | −100を上回るクロスで+1、+100を下回るクロスで−1。 |
| `IND4` | 短期単純移動平均の傾き | 前の2本のバー間で短期MAが増加した場合+1、減少した場合−1。 |
| `IND5` | Awesome Oscillatorモメンタム色 | ヒストグラムが前の値より増加した場合（強気色）+1、減少した場合−1。 |

すべてのインジケーターは閉じたローソク足で評価されます。MQL5スクリプトで使用される`CopyBuffer`ウィンドウ処理を複製するために、内部で履歴バッファが維持されます。

## パーセプトロンアーキテクチャ

- 5つの隠れニューロン（`NN1`…`NN5`）がそれぞれ4つのインジケーターを組み合わせ、EAの配線を反映します。
- 各ニューロンはシナプス荷重の独自の辞書にバイアス荷重（`NNS1`…`NNS5`）を加えたものを持ちます。
- 最終的な活性化`brainReturn`はニューロン出力の加重和です。  
  - `brainReturn > 0` → ロングエントリーを要求（前の取引もロングでなかった場合）。  
  - `brainReturn < 0` → ショートエントリーを要求（前の取引もショートでなかった場合）。
- ポジションはアクティブなポジションがない場合にのみ成行注文で開かれます。

## ポジション管理

- エントリー価格、方向、インジケーター/ニューロンの状態が各約定で記録されます。
- テイクプロフィットとストップロスのオフセットは絶対価格単位で適用されます（例：5桁のFXペアの4ポイントに0.0004）。  
  エントリー後に新しいローソク足が開くと：
  - ロングの場合、まず高値をテイクプロフィット価格と比較し、次に安値をストップロスと比較します。  
  - ショートの場合、まず安値をテイクプロフィット価格と比較し、次に高値をストップロスと比較します。  
  - 同じローソク足内で両方のレベルを超えた場合、オリジナルEAの楽観的な動作に合わせてテイクプロフィットが優先されます。
- 決済が検出されると、戦略は成行注文でポジションを閉じ、対応するTP/SLレベルを使用して実現損益を計算します。

## 適応的荷重更新

取引が閉じると、記録されたインジケーターとニューロンの状態が再生されます：

1. `directionSign`（ロングで−1、ショートで+1）と`outcomeSign`（実現PnLの符号）が決定されます。
2. バイアス荷重は`[SinMin, SinMax]`内で調整されます：
   - `sign(neuronOutput) * directionSign`が正の場合、バイアスは取引結果に従います（利益で増加、損失で減少）。
   - それ以外の場合、バイアスは結果と反対方向に動きます。
3. シナプス荷重も同様に動作しますが無制限のまま：ポジション方向に揃ったシグナルは利益で強化を受け、損失でペナルティを受け、逆のシグナルはその逆を行います。
4. 保存されたシグナルは偶発的な再使用を避けるためにクリアされます。

これにより、EAから1,500行以上の条件付きシナプス管理をコンパクトな強化ルーチンに一般化します。

## パラメーター

| パラメーター | デフォルト | 説明 |
| --- | --- | --- |
| `CandleType` | 1分の時間軸 | 戦略が使用するローソク足サブスクリプション。 |
| `FastMaLength` | 5 | クロスオーバーシグナルの高速SMA期間。 |
| `SlowMaLength` | 9 | 低速SMAの期間。 |
| `RsiLength` | 14 | RSI計算期間。 |
| `CciLength` | 14 | CCI計算期間。 |
| `SlopeMaLength` | 5 | 傾き検出に使用するMAの期間。 |
| `AoShortLength` | 5 | Awesome Oscillatorの短期間。 |
| `AoLongLength` | 34 | Awesome Oscillatorの長期間。 |
| `StopLossOffset` | 0.001 | 絶対価格単位でのストップロス距離（0でストップを無効化）。 |
| `TakeProfitOffset` | 0.0004 | 絶対価格単位でのテイクプロフィット距離（0で目標を無効化）。 |
| `SinMax` | 5 | ニューロンバイアス荷重の上限。 |
| `SinMin` | 0 | ニューロンバイアス荷重の下限。 |
| `SinPlusStep` | 0.03 | 正の強化増分。 |
| `SinMinusStep` | 0.03 | 負の強化減分。 |

すべての数値パラメーターは`StrategyParam<T>`として公開されており、StockSharp Designerで最適化できます。

## 実装上の注記

- マルチインジケーターバインディングを持つ高レベルローソク足サブスクリプションAPIを使用します。
- シナプスを更新する際に実現価格が既知となるよう、手動の取引管理が採用されています。
- インジケーター履歴はnullフィールドで保存され、完全な形成後にのみシグナルが発動することを保証します。
- EAのAwesome Oscillatorカラーバッファは、現在と前のヒストグラム値を比較することで近似されます。
- チャート出力はローソク足シリーズと高速・低速移動平均を描画します。取引マーカーはリアルタイムで適応的な動作を示します。

## 制限と前提

- ストップと目標は完成したローソク足ごとに1回評価されます。バー内のイベント順序は不明なため、両方の閾値が達した場合は利益目標が優先されます。
- インジケーター荷重はオリジナルEAと同様に無制限であり、長期強化サイクル中に大きく成長する可能性があります。
- オリジナルEAの`LastTradeType`はリセットされませんでした。このポートでは各決済後にクリアされ、同じ方向の連続取引が可能になります。
