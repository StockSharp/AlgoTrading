# SMC Hilo MaxMin ブレイクアウト戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、MetaTrader エキスパート *SMC MaxMin at 1200* の動作を再現します。指定された終了時間に、
前のローソクの高値を上回る買いストップ注文と、前のローソクの安値を下回る売りストップ注文。未決注文が水増しされる
ブローカーの最小ストップ距離によって、ピップから商品価格単位に変換されます。ブレイクアウトが発生したら、逆の順序で
はキャンセルされ、オープンポジションは固定ストップ、利益目標、およびオプションのトレーリングストップを通じて管理されます。

元の MQL4 コードとの主な違い:

- StockSharp の順序プリミティブ (`BuyStop`、`SellStop`、`BuyLimit`、`SellLimit`) は、直接の `OrderSend` 呼び出しを置き換えます。
- 最小停止距離、ストップロス、テイクプロフィットの入力はピップ単位で表され、`Security.PriceStep` を通じて次のように変換されます。
実際の楽器のティックサイズを尊重してください。
- トレーリングストップ管理は、トレーリングバッファーよりも大きな収益性の高い距離が達成された場合にのみストップ注文を移動します。
- すべてのロジックは高レベルのローソク足サブスクリプション API によって駆動されるため、直接の履歴スキャンや手動のインジケーター バッファーは使用されません。

## 取引ルール
1. **時間の設定** – 終了時間が `SetHour` に等しい場合、以前に完了したローソク足を参照として使用します。
2. **ロングエントリー** – `previous_high + min_stop_distance + price_step`に買いストップを置きます。
3. **ショートエントリー** – `previous_low - min_stop_distance - price_step`に売りストップを置きます。
4. **相互排他性** – いずれかのストップが約定した場合、反対側の未決注文は直ちにキャンセルされます。
5. **ストップロス** – ロングストップは `previous_low - StopLossPips`、ショートストップは `previous_high + StopLossPips` (両方とも変換されます)
価格単位にします）。
6. **利益確定** – ロングポジションは、`entry + TakeProfitPips` で売り指値を使用します。ショートポジションは買い指値を使用します
`entry - TakeProfitPips`。
7. **トレーリングストップ** – ポジションが `TrailingStopPips` を超えて利益を上げている場合、同じピップを維持するためにストップがトレーリングされます
現在の買値/売値からの距離。
8. **注文タイムアウト** – セットアップから 2 時間後 (`SetHour + 2`)、満たされていない保留中のストップはすべてキャンセルされます。

## パラメーター
| 名前 | 説明 | デフォルト |
| ---- | ----------- | ------- |
| `Volume` | 両方のエントリー注文に使用される注文量。 | `0.1` |
| `SetHour` | ブレイクアウト ストラドルが作成される終了時間 (0 ～ 23)。 | `15` |
| `TakeProfitPips` | 利益目標距離 (pips)。テイクプロフィット注文を無効にするには、`0` に設定します。 | `500` |
| `StopLossPips` | ピップ単位の保護停止距離。初期停止を無効にするには、`0` に設定します。 | `30` |
| `TrailingStopPips` | トレーリングストップまでの距離 (pips)。静的な停止を維持するには、`0` に設定します。 | `30` |
| `MinStopDistancePips` | エントリー価格をパッドするために使用されるブローカーの最小ストップ距離。 | `0` |
| `CandleType` | 時間単位のセッションを定義するローソク足のタイプ。デフォルトは 1 時間の時間枠です。 | `1h` |

## 使用上の注意
- この戦略では、トレーリング ストップを管理し、距離計算のために最新の買値/売値を維持するためにレベル 1 データが必要です。
- 基礎となる金融商品のティック サイズが標準以外の場合 (例: 0.01 ピップの円クロス)、`TakeProfitPips` を調整します。
それに応じて `StopLossPips` と `TrailingStopPips` も同様です。
- `TakeProfitPips` または `StopLossPips` がゼロの場合、それぞれの注文は送信されませんが、次の場合にはトレーリング ストップが有効になる可能性があります。
末尾のパラメータは正の値です。
- 構成された `SetHour` が受信データ フィードのブローカー サーバー時刻と一致していることを確認してください。
