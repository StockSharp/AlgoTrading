# 効率的なポーランドレイヤー Expert Advisor システム
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、MQL4 エキスパート アドバイザ「Polish Layer Expert Advisor System Efficient」を直接移植したものです。これは日中チャート用に設計されており (オリジナルの作成者は 5 分または 15 分のローソク足を推奨しています)、取引は一度に 1 つのポジションに制限されています。トレンドの方向は、2 つの平滑化された RSI フィルターと高速価格移動平均と低速価格移動平均の調整によって定義されます。実際のエントリーでは、一般的なトレンド内で発生する極端な状況からの反転を捉えるために、Stochastic オシレーター、ディマーカー、および Williams %R インジケーターからの 3 つの確認が必要です。

## 取引ロジック
1. **トレンド フィルター。** 終値の 9 期間の単純移動平均 (SMA) は、ロングを許容するには 45 期間の線形加重移動平均 (LWMA) を上回っており、ショートを許容するには下回っている必要があります。同時に、RSI の 9 期間 SMA は、RSI の 45 期間 SMA より上（ロングの場合）または下（ショートの場合）でなければなりません。価格と RSI フィルタの間に不一致がある場合、新しい注文はブロックされます。
2. **Stochastic トリガー。** トレンド フィルターが強気の場合、戦略は Stochastic %K ラインが売られ過ぎのしきい値 (デフォルト 19) を上向きに交差し、同時に %D を上抜けるのを待ちます。弱気のセットアップの場合、%K は買われすぎのしきい値 (デフォルト 81) を下回り、%D を下回る必要があります。速度低下要因は、MQL4 スクリプトから保存されます。
3. **モメンタムの確認** ロングシグナルでは、DeMarker が 0.35 まで上向きにクロスする必要があり、Williams %R が現在完了しているローソク足の -81 まで上向きにクロスする必要があります。短いシグナルは、それぞれ 0.63 と -19 を通過する下向きのクロスを要求します。すべての交差は、以前に終了したローソク足と現在のローソク足の間で評価されます。
4. **ポジション管理** 成行注文のみが発行され、保護的なストップまたはターゲットが取引を終了するまで戦略はフラットのままです。プロテクションレベルは、商品価格ステップを使用してピップベースのパラメーターから再計算されます。価格ステップが利用できない場合、保護は無効になります。

## リスク管理
* **ストップロス/テイクプロフィット** 距離はピップ単位で設定されます。正の場合、値は `Security.PriceStep` (1 pip = 1 価格ステップ) を使用して実際の価格オフセットに変換され、エントリー直後に適用されます。パラメータを `0` に設定すると、対応する保護レベルが無効になります。
* **単一ポジション。** 元の EA はピラミッド化されていないため、ポジションがすでに存在する場合、ポートは入力を拒否します。

## パラメーター
| パラメータ | デフォルト | 説明 |
|-----------|---------|-------------|
| `Volume` | `0.1` | ロット単位での注文量。ブローカーの契約規模に応じて調整します。 |
| `CandleType` | `TimeSpan.FromMinutes(15).TimeFrame()` | インジケーターの計算に使用されるローソク足のタイプ。元の EA をミラーリングするには、5 分または 15 分の時間枠に設定します。 |
| `RsiPeriod` | `14` | 基本の RSI 計算のルックバック長。 |
| `ShortPricePeriod` | `9` | トレンドフィルターで使用される高速価格 SMA の期間。 |
| `LongPricePeriod` | `45` | トレンドフィルターで使用される低価格LWMAの期間。 |
| `ShortRsiPeriod` | `9` | RSI 値に適用される高速な SMA の長さ。 |
| `LongRsiPeriod` | `45` | RSI 値に適用される低速な SMA の長さ。 |
| `StochasticKPeriod` | `5` | Stochastic オシレーターの基本 %K 周期。 |
| `StochasticDPeriod` | `3` | %D ラインの平滑化期間。 |
| `StochasticSlowing` | `3` | %K に適用される追加の平滑化係数。 |
| `DemarkerPeriod` | `14` | DeMarker インジケーターの平均期間。 |
| `WilliamsPeriod` | `14` | Williams %R のルックバック期間。 |
| `StochasticOversoldLevel` | `19` | ロングエントリーを可能にするために %K が上方に超える必要がある売られ過ぎのしきい値。 |
| `StochasticOverboughtLevel` | `81` | ショートエントリーを可能にするために、%K が下向きに超える必要がある買われすぎのしきい値。 |
| `DemarkerBuyLevel` | `0.35` | 長いエントリ (下からのクロス) に必要な DeMarker の最小値。 |
| `DemarkerSellLevel` | `0.63` | 短いエントリ (上からのクロス) に許可される DeMarker の最大値。 |
| `WilliamsBuyLevel` | `-81` | Williams %R レベル交差によりロングエントリが確認されました。 |
| `WilliamsSellLevel` | `-19` | Williams %R 交差レベルでショートエントリーが確認されました。 |
| `StopLossPips` | `7777` | ピップス単位のストップロス距離。非常に大きなデフォルトでは、構成されていない限り、停止が事実上無効になります。 |
| `TakeProfitPips` | `17` | 利益確定距離 (pips)。固定ターゲットを無効にするには、`0` に設定します。 |

## 注意事項
* `Security.PriceStep`、`Security.MinVolume`、および `Security.VolumeStep` が正しく構成されていることを確認してください。この戦略では、リスクパラメータを変換するときに、1 ピップが 1 価格ステップに等しいと想定します。
* エントリーフィルターは、連続する完了したローソク足間のインジケーターのクロスオーバーに依存します。履歴データをインポートするときは、結果を再現するためにバーの配置を元の時間枠と同一に保ちます。
