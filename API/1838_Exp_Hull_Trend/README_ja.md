# Exp Hullトレンド戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

Exp Hullトレンド戦略はHull Moving Average（HMA）インジケーターに基づいています。アルゴリズムは中間Hull計算値と平滑化されたHull移動平均を比較します。速いHull線が遅い平滑化された線を上抜けすると、戦略はロングポジションをオープンします。速い線が平滑化された線を下抜けすると、戦略はショートポジションをオープンします。

## 戦略ロジック

1. 期間 **Length / 2** で終値の加重移動平均（WMA）を計算します。
2. 期間 **Length** で終値の別のWMAを計算します。
3. 中間Hull値を構築します：`fast = 2 * WMA(Length/2) - WMA(Length)`。
4. 期間 `sqrt(Length)` のWMAで中間値を平滑化し、最終Hull値 `slow` を取得します。
5. シグナルを生成します：
   - **ロングエントリー** – `fast` が `slow` を上抜けしたとき。
   - **ショートエントリー** – `fast` が `slow` を下抜けしたとき。
6. 反対のシグナルが出るとポジションは反転されます。保護注文は `StartProtection` を通じて処理されます。

## パラメーター

| 名前 | 説明 |
|------|------|
| `Hull Length` | Hull計算の基本期間。両方のWMAの感度を決定します。 |
| `Candle Type` | インジケーター計算に使用するローソク足の時間軸。 |

## 注意事項

- 戦略は完了したローソク足のみで動作します。
- インジケーター値は手動データコレクションを避けるために高レベルAPIを介してバインドされます。
- ボリュームは戦略設定から取得されます。シグナルの方向が変わると、ポジションが反転されます。
