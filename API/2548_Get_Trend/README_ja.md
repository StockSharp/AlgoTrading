# トレンド取得戦略 (Get Trend)
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

この戦略は、MetaTraderのエキスパートアドバイザー **"Get trend"** をStockSharpに移植したものです。元々はH1確認フィルター付きのM15取引向けに設計されています。アルゴリズムは、スムーズ移動平均とストキャスティクスオシレーターを組み合わせて、上位時間軸のトレンドに沿った平均回帰エントリーのタイミングを計ります。

## トレードロジック

- **主要時間軸:** 15分足ローソク足はシグナル生成と注文執行に使用されます。
- **確認時間軸:** 1時間足ローソク足は、支配的なトレンドを検証するための上位時間軸のスムーズ移動平均と終値を提供します。
- **トレンドフィルター:** M15とH1の両方の終値が、それぞれのスムーズ移動平均の同じ側にある必要があります。さらに、M15の終値が移動平均から設定可能な距離内に収まり、プルバックエントリーを確保する必要があります。
- **モメンタムトリガー:** ロングトレードは、ストキャスティクスの%Kラインが売られ過ぎ領域（20未満）で%Dを上抜けることが必要です。ショートトレードは、買われ過ぎ領域（80超）での逆クロスが必要です。
- **注文管理:** ポジションは、価格ポイントで定義された固定のストップロスとテイクプロフィットレベルで保護されます。オプションのトレーリングストップは、価格がトレードに有利な方向に十分進んだ後、エグジットを締め付けます。

## エントリー条件

### ロングセットアップ
1. M15の終値がM15のスムーズ移動平均を下回っています。
2. H1の終値がH1のスムーズ移動平均を下回っています。
3. M15の終値とM15の平均との距離が **Price Threshold**（ポイント/ティック単位）を超えていません。
4. ストキャスティクスの%Kと%Dが両方とも20未満です。
5. 前の%K値が%Dを下回っており、現在の%K値が%Dを上抜けました。
6. 既存のロングポジションがありません（ショートポジションがある場合はクローズして反転します）。

### ショートセットアップ
1. M15の終値がM15のスムーズ移動平均を上回っています。
2. H1の終値がH1のスムーズ移動平均を上回っています。
3. M15の終値とM15の平均との距離が **Price Threshold** を超えていません。
4. ストキャスティクスの%Kと%Dが両方とも80超です。
5. 前の%K値が%Dを上回っており、現在の%K値が%Dを下抜けました。
6. 既存のショートポジションがありません（ロングポジションがある場合はクローズして反転します）。

## エグジットルール

- **ストップロス:** エントリー価格からの絶対的な価格ポイントで設定されます。
- **テイクプロフィット:** エントリー価格からの絶対的な価格ポイントで設定されます。
- **トレーリングストップ:** 有効化されると、価格がトレーリング距離を超えて動いた時点でストップが引き寄せられ、設定されたトレーリングオフセットを守りながら利益を確定します。

## パラメーター

| パラメーター | 説明 | デフォルト値 |
|------------|------|------------|
| `M15CandleType` | シグナル生成に使用するローソク足タイプ。 | 15分時間軸 |
| `H1CandleType` | 確認に使用するローソク足タイプ。 | 1時間時間軸 |
| `MaM15Length` | M15ローソク足のスムーズMAの長さ。 | 99 |
| `MaH1Length` | H1ローソク足のスムーズMAの長さ。 | 184 |
| `StochasticLength` | ストキャスティクスオシレーターの%K期間。 | 27 |
| `StochasticSignalLength` | %Dスムージング期間。 | 3 |
| `ThresholdPoints` | エントリーを許可するための価格とM15 MAの最大距離（ポイント単位）。 | 10 |
| `TakeProfitPoints` | テイクプロフィット距離（ポイント単位）。 | 540 |
| `StopLossPoints` | ストップロス距離（ポイント単位）。 | 90 |
| `TrailingStopPoints` | トレーリングストップ距離（ポイント単位）。 | 20 |
| `TradeVolume` | 新規トレードを開く際の基本注文数量。 | 0.1 |

すべてのポイントベースのパラメーターは、絶対的な価格増分に変換するために、銘柄の `PriceStep` で乗算されます。

## 実装メモ

- 戦略はStockSharpの高レベルAPIを使用し、ローソク足サブスクリプションとインジケーター連携（`BindEx`）によって手動バッファ管理を回避します。
- トレーリングストップのロジックはMetaTraderバージョンを反映しています：未実現利益がトレーリング距離を超えた時点で起動し、継続的にストップを価格方向に引き締めます。
- ポジションを反転する前にアクティブな注文がキャンセルされ、板上の相反する注文を防ぎます。
- チャートエリアにはM15ローソク足とスムーズ移動平均が表示され、ビジュアル診断のための専用ストキャスティクスパネルが備わっています。

## 使用上のヒント

- データプロバイダーに合わせてローソク足タイプを設定します（同じDataTypeコンセプトを公開している場合、ボリュームベースのローソク足に置き換えることができます）。
- 異なるボラティリティやティックサイズの資産を取引する際は、閾値とストップパラメーターを調整してください。
- 最良の結果を得るには、移動平均へのプルバックが一般的なトレンド銘柄に戦略を適用してください。
