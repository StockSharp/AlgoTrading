# Up3x1動的ポジションサイジング戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- MetaTrader 5エキスパートアドバイザー`up3x1.mq5`をStockSharp高レベルAPIに変換。
- 三重指数移動平均（EMA）クロスオーバーによる取引とストップロス、テイクプロフィット、トレーリングストップ管理。
- バーごとに1つの決定を強制した元の`iTickVolume(0) > 1`ガードをエミュレートするために完成したローソク足のみを処理します。
- デフォルトのローソク足シリーズは1時間ですが、時間軸は`CandleType`パラメーターで設定可能です。

## 取引ロジック
1. **インジケーター**
   - 速いEMA（`FastPeriod`、デフォルト24）。
   - 中間EMA（`MediumPeriod`、デフォルト60）。
   - 遅いEMA（`SlowPeriod`、デフォルト120）。
2. **ロングエントリー**
   - 前のバー：速いEMAが中間EMAより下で、中間が遅いEMAより下（`EMAfast₍t-1₎ < EMAmedium₍t-1₎ < EMAslow₍t-1₎`）。
   - 現在のバー：速いEMAが遅いEMAより下のままで中間EMAが速いEMAより下（`EMAmedium₍t₎ < EMAfast₍t₎ < EMAslow₍t₎`）。
3. **ショートエントリー**
   - 前のバー：速いEMAが中間EMAより上で、中間が遅いEMAより上（`EMAfast₍t-1₎ > EMAmedium₍t-1₎ > EMAslow₍t-1₎`）。
   - 現在のバー：中間EMAが速いEMAを上方クロスし、両方が遅いEMAより上に留まる（`EMAmedium₍t₎ > EMAfast₍t₎ > EMAslow₍t₎`）。
4. **両方向のエグジットロジック**
   - 価格がエントリーから`TakeProfitOffset`前進したらテイクプロフィット（ロングはローソク足の高値、ショートは安値を使用）。
   - 価格がエントリーから`StopLossOffset`戻ったらストップロス（ロングはローソク足の安値、ショートは高値を使用）。
   - トレーリングストップはポジションが`TrailingStopOffset`より有利に動いたら起動し、その後ローソク足の極値で評価された固定距離で価格に追従します。
   - 速いEMAが遅いEMAより上に留まりながら中間EMAより下にクロスバックしたらフォールバックエグジット（MQLバージョンの`ma_one_1 > ma_two_1 > ma_three_1`チェックを反映）。

## ポジションサイジングとリスク管理
- `RiskFraction`（デフォルト0.02）は現在のポートフォリオ価値に乗算して元の`FreeMargin * 0.02 / 1000`ロットサイジングを近似します。
- `BaseVolume`（デフォルト0.1）はポートフォリオデータが利用できないか計算されたサイズが正でない場合のフォールバックとして機能します。
- 1回以上の損失エグジット後、ボリュームは`volume * losses / 3`で削減され、スクリプトの累積`losses`カウンターを模倣します（カウンターは元のコードと同様に利益トレードの後にリセットされません）。
- ボリュームは`Security.VolumeStep`に切り捨てられ、`Security.MinVolume` / `Security.MaxVolume`で制限され、インストゥルメントの最小値が満たせない場合はゼロになります。

## パラメーター
| パラメーター | デフォルト | 説明 |
|------------|----------|------|
| `FastPeriod` | 24 | 最も速いEMAの長さ。 |
| `MediumPeriod` | 60 | 中間EMAの長さ。 |
| `SlowPeriod` | 120 | 長期トレンドフィルターとして使用される遅いEMAの長さ。 |
| `TakeProfitOffset` | 0.015 | テイクプロフィット注文の絶対価格距離（インストゥルメントの見積もりに適応）。 |
| `StopLossOffset` | 0.01 | ストップロス注文の絶対価格距離。 |
| `TrailingStopOffset` | 0.004 | 価格が十分に前進したら利益を確保するトレーリング距離。無効にするには0に設定。 |
| `BaseVolume` | 0.1 | 動的サイジングが計算できない場合のフォールバック取引サイズ。 |
| `RiskFraction` | 0.02 | 動的サイジング式に適用されるポートフォリオ価値の割合。 |
| `CandleType` | 1時間時間軸 | インジケーター計算と意思決定に使用されるローソク足シリーズ。 |

## 変換に関する注意事項
- トレーリングストップと保護的エグジットは高レベルAPIが完成したローソク足を処理するため、生のティックではなくローソク足の高値/安値を使用します。これによりバックテストとライブ実行全体で動作が決定論的に保たれます。
- ストップロスとテイクプロフィットは、個別の保護注文を配置するのではなく、評価されたしきい値での成行フラット化コマンドで実行され、高レベル戦略フローとの互換性を確保します。
- 動的ポジションサイジングは`Portfolio.CurrentValue`に依存します。利用できない場合、戦略は元の`LotCheck`の手動`Lots`入力へのフォールバックと同様に`BaseVolume`にフォールバックします。
- `losses`カウンターはMQL実装に従うため意図的に累積（勝ちトレードでリセットされない）です。
- すべてのコメントはプロジェクトのガイドラインに従い英語です。

## 使用のヒント
1. 戦略をインストゥルメントとポートフォリオに付加し、MT5からエミュレートしたいチャート解像度に合わせて`CandleType`を設定します。
2. インストゥルメントのティックサイズを反映するように価格オフセットを確認してください（例：5桁のForexペアの場合、0.015はソースエキスパートと同様に150ポイントに相当）。
3. アカウントに対して現実的なポジションサイズを達成するために`RiskFraction` / `BaseVolume`を調整します。
4. オプション：`TrailingStopOffset`をゼロに設定してトレーリングを無効にします。
5. MetaTraderの`Print`診断を反映する「Enter long」や「Exit short」などのメッセージについてログを監視してください。
