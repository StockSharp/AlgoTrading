# BandOsMa戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**BandOsMa 戦略** は、MetaTrader 5 "BandOsMA" エキスパート アドバイザーを StockSharp 戦略に変換します。ヒストグラム値に直接構築された Bollinger バンドを使用して、MACD ヒストグラム (OsMA) を評価します。バンドの上または下のブレイクアウトはエントリーシグナルを作成し、ヒストグラムの追加の移動平均がシグナルの出口を管理します。

この戦略は、ユーザーが選択した単一のシンボルと時間枠で動作します。インジケーターの値は、StockSharp の高レベルのキャンドル サブスクリプションを使用して、完成したキャンドルで計算されます。

## 取引ロジック
1. **指標**
   - `MovingAverageConvergenceDivergenceSignal` は、MACD ヒストグラム (OsMA) を提供します。
   - `BollingerBands` は、極端な逸脱を検出するために OsMA シーケンスに適用されます。
   - 構成可能な移動平均はヒストグラムを平滑化し、出口フィルターとして機能します。
2. **エントリー**
   - **ロングシグナル**は、前のバーが下位バンドより上に留まりながら、現在の OsMA が下位バンドの下で終了したときに表示されます。
   - **ショートシグナル**は、前のバーが上部バンドよりも下に留まっていたときに、現在の OsMA が上部バンドより上で終了したときに表示されます。
3. **終了**
   - ヒストグラムが移動平均を反対方向に横切ると、シグナルがクリアされます。
   - オープンポジションがアクティブシグナルと一致しなくなった場合、そのポジションは直ちにクローズされます。
   - 各ポジションにはピップベースのストップロスが付加されます。このストップは、同じ距離と `StopLossPoints / 50` に等しいトレーリング ステップを持つトレーリング ストップとしても機能します (MetaTrader ヘルパー クラスをミラーリングします)。

## ポジション管理
- **ストップロスとトレーリング**: ストップ距離は MetaTrader ポイントで表され、商品の `PriceStep` を使用して価格単位に変換されます。同じ距離がトレーリング ストップにも使用され、終値が少なくともトレーリング ステップ分改善すると、トレーリング ストップが前進します。
- **一度に 1 つのポジション**: 維持されるネット ポジションは 1 つだけです。反対のシグナルは、新しいエントリーを検討する前に現在のポジションを閉じます。

## 実装メモ
- ローソク足の処理では、`WhenCandlesFinished` を使用して、最後のバーのみがロジックを駆動するようにします。
- インジケーター値は、MetaTrader スタイルのバッファー シフトをエミュレートするために履歴バッファーに保存されます。負のシフトはサポートされていません。元のエキスパートのデフォルトと同様に、ゼロまたは正の値を使用します。
- トレーリングストップはティックごとの更新ではなく、ローソク足の終値に依存します。正確なティックレベルのトレーリングが必要な場合は、ピップ距離を調整します。

## 使用法
1. StockSharp で目的のシンボルとタイムフレームを選択します。
2. パラメータ、特に `CandleType`、`LotSize`、インジケーター期間を設定します。
3. 戦略を開始します。ローソク足を購読し、インジケーターを計算し、記述されたロジックに従って取引を実行します。
