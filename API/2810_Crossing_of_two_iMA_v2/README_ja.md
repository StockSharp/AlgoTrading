# 2つのiMAクロス v2 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は StockSharp の高レベル API を使用して MetaTrader エキスパートアドバイザー「Crossing of two iMA v2」を再現します。2 つのシフトされた移動平均がクロスオーバーシグナルを生成し、オプションで 3 番目の移動平均によってフィルタリングされます。保護ストップ、固定またはパーセンテージベースのポジションサイジング、バー・バイ・バーのトレーリングストップが元のロボットの動作を模倣しながら、StockSharp のベストプラクティスに準拠した実装を維持します。

## 指標と入力
- **第 1 移動平均** – 設定可能な期間、シフト、平滑化方法、適用価格。
- **第 2 移動平均** – 同じオプションセットを持つ独立した設定。
- **第 3 移動平均フィルター** – オプションのトレンドフィルター。第 1 MA がフィルターを下回っている場合にのみロングトレードを維持し、第 1 MA がフィルターを上回っている場合にショートトレードを維持。
- **ローソク足タイプ** – データサブスクリプションが提供するタイムフレーム/シリーズを制御。

## トレードロジック
### ステップ 1 – 即時クロスオーバー
1. 完成した各ローソク足で、戦略は選択した適用価格を使用してすべての移動平均を更新します。
2. 前のバーと現在のバーの間で第 1 MA が第 2 MA を**上回るようにクロス**したとき、**ロング**エントリーが引き起こされます。
3. 前のバーと現在のバーの間で第 1 MA が第 2 MA を**下回るようにクロス**したとき、**ショート**エントリーが引き起こされます。
4. フィルターが有効な場合、ロングシグナルは第 1 MA がフィルター MA の**下に**留まることを要求し、ショートシグナルは第 1 MA がフィルター MA の**上に**留まることを要求します。

### ステップ 2 – 遅延確認
ステップ 1 でシグナルが発生しない場合、戦略は 2 バー前に始まったが今も有効なクロスオーバーをチェックします。これは最近の履歴で見逃したクロスを検索する元の EA の動作を反映しています。繰り返しの約定を避けるため、シグナルは最後のトレードから少なくとも 3 バー経過した場合にのみ有効になります。

### 注文実行
- エントリーは成行価格で実行されます。新しい方向に開く前に反対のポジションが閉じられます。
- 現在のローソク足でストップロス、テイクプロフィット、またはトレーリングストップレベルに触れたときにエグジットが発生します。保護レベルが侵害されると、成行注文でトレードが閉じられます。

## リスク管理
- **ストップロス**と**テイクプロフィット**の距離はピップで設定されます。銘柄の `PriceStep` を使用して価格オフセットに変換されます（利用できない場合は `1` をデフォルト）。
- **トレーリングストップ**はエントリー価格から始まり、有利な価格の動きに従います。ベスト価格が前のトレーリングレベルを少なくとも `TrailingStepPips` ピップ超えるたびにストップが更新されます。
- 固定ストップとトレーリングストップの両方がアクティブな場合、戦略はより保守的なレベルを使用します（ロングポジションでは高い方、ショートポジションでは低い方）。

## ポジションサイジング
- `UseRiskPercent` が **true** の場合、ボリュームは `Equity * RiskPercent / (StopLossPips * PipValue)` に等しくなります。ストップが定義されていない場合、戦略は固定ボリュームにフォールバックします。
- `UseRiskPercent` が **false** の場合、トレードサイズは常に `FixedVolume` です。
- `PipValue` は取引する銘柄の 1 ロット/契約あたりの 1 ピップの金銭的価値を反映する必要があります。

## 実装上の注意
- StockSharp の実装は閉じたローソク足のみで動作し、未決注文を登録しません。ストップまたはリミットエントリーが必要なユーザーは、それに応じて戦略を拡張できます。
- 第 3 の移動平均フィルターは、すべてのクロスオーバーを取引するために無効にでき、EA のオプション `InpFilterMA = false` に対応します。
- 正確なリスク管理のために、ローソク足タイプ、価格ステップ、ピップ値のパラメーターが取引される銘柄と一致していることを確認してください。

## パラメーター
| 名前 | 説明 | デフォルト |
| --- | --- | --- |
| `FirstPeriod` | 第 1 移動平均の期間。 | 5 |
| `FirstShift` | 第 1 移動平均の出力に適用されるシフト（バー）。 | 3 |
| `FirstMethod` | 第 1 移動平均の平滑化方法（`Simple`、`Exponential`、`Smoothed`、`Weighted`）。 | `Smoothed` |
| `FirstAppliedPrice` | 第 1 移動平均の適用価格（`Close`、`Open`、`High`、`Low`、`Median`、`Typical`、`Weighted`）。 | `Close` |
| `SecondPeriod` | 第 2 移動平均の期間。 | 8 |
| `SecondShift` | 第 2 移動平均の出力に適用されるシフト（バー）。 | 5 |
| `SecondMethod` | 第 2 移動平均の平滑化方法。 | `Smoothed` |
| `SecondAppliedPrice` | 第 2 移動平均の適用価格。 | `Close` |
| `UseFilter` | 第 3 移動平均方向フィルターを有効にします。 | `true` |
| `ThirdPeriod` | 第 3 移動平均フィルターの期間。 | 13 |
| `ThirdShift` | 第 3 移動平均の出力に適用されるシフト（バー）。 | 8 |
| `ThirdMethod` | 第 3 移動平均フィルターの平滑化方法。 | `Smoothed` |
| `ThirdAppliedPrice` | 第 3 移動平均フィルターの適用価格。 | `Close` |
| `UseRiskPercent` | 固定ボリュームとパーセンテージベースのポジションサイジングを切り替えます。 | `true` |
| `FixedVolume` | 固定サイジングがアクティブな場合のトレードサイズ。 | 0.1 |
| `RiskPercent` | トレードごとにリスクにさらされる口座資産の割合。 | 5 |
| `PipValue` | 1 ロット/契約あたりの 1 ピップの金銭的価値。 | 1 |
| `StopLossPips` | ピップでのストップロス距離。 | 50 |
| `TakeProfitPips` | ピップでのテイクプロフィット距離。 | 50 |
| `TrailingStopPips` | ピップでのトレーリングストップ距離。 | 10 |
| `TrailingStepPips` | トレーリングストップを進めるために必要な最小ピップ増分。 | 4 |
| `CandleType` | 戦略が使用するローソク足データタイプ / タイムフレーム。 | 1 分ローソク足 |
