# クロスシンプル戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

**Kloss Simple Strategy** は、MetaTrader 4 エキスパート アドバイザー `Kloss_.mq4` を直接変換したものです。 StockSharp の高レベルの API を使用して元の取引アイデアを再構築し、加重終値に基づいて計算された指数移動平均 (EMA)、商品チャネル指数 (CCI)、および Stochastic オシレーターというインジケーター セットを同一に保ちます。シグナルは、MQL バージョンの 1 バー シフト ロジックを反映して、以前に完了したローソク足から生成されます。ポジションサイジングは、元のロット計算ルールと同様に、固定注文量またはポートフォリオ値のリスクパーセンテージのいずれかに依存できます。

## コアアイデア

1. 中立レベル付近の **CCI** および **Stochastic** のしきい値を使用して、運動量のコンテキストを監視します。
2. 加重終値の短期的な **EMA** で勢いシグナルを確認します。
3. 前のローソク足がすべてのシグナル条件を満たしている場合にのみポジションを入力し、不完全な市場データでの早まった取引を防ぎます。
4. MT4 スクリプトの「MaxOrders」パラメータをエミュレートして、構成可能な制限まで同じ方向の複数のエントリを許可します。

## インジケーターの構成

- **EMA (MaPeriod)**: 加重終値 `(Close * 2 + High + Low) / 4` を使用して、MetaTrader からの `PRICE_WEIGHTED` と一致します。短期トレンドフィルターとして機能します。
- **CCI (CciPeriod)**: 平均価格からの勢いの偏差を評価します。しきい値 `±CciLevel` は、積極的なエントリと保守的なエントリを定義します。
- **Stochastic (StochasticKPeriod / DPeriod / Smooth)**: メインの %K ラインを使用して、ニュートラルな 50 レベルを基準とした買われすぎまたは売られすぎの状態を検出します。 50 からの偏差は `StochasticLevel` によって制御されます。

すべてのインジケーターは、`CandleType` で定義された主要なローソク足シリーズで動作します。この戦略は、完成したローソク足のインジケーター値のみを更新し、安定したバックテストと実際の動作を保証します。

## 取引ロジック

### 長いセットアップ

1. 前回のローソク足の終値は、前の EMA の値を上回っています。
2. 以前の CCI の値は `-CciLevel` を下回っており、売られすぎの勢いを示しています。
3. 以前の Stochastic %K 値は `50 - StochasticLevel` を下回っており、売られ過ぎのオシレーションを確認しています。
4. 条件が満たされると、既存のロング注文の数が `MaxOrders` を下回っている限り、ショート エクスポージャーはクローズされ、新しいロング ポジションがオープンされます。

### 簡単なセットアップ

1. 前回のローソク足の終値は、前の EMA の値を下回っています。
2. 以前の CCI の値は `+CciLevel` を上回っており、買われすぎの勢いを示しています。
3. 以前の Stochastic %K 値は `50 + StochasticLevel` を上回っており、買われ過ぎのオシレーションを確認しています。
4. 条件が満たされると、長時間露光はクローズされ、`MaxOrders` の制限に従って新しいショート ポジションがオープンされます。

### 出口管理

- **ストップロス/テイクプロフィット**: オプションの商品ポイントの絶対距離。いずれかの値がゼロより大きい場合、StockSharp の組み込み位置保護が有効になります。
- **反対シグナル**: 反対方向にオープンする前に、元のエキスパートアドバイザーを模倣するために現在のポジションがクローズされます。

## ポジションサイズ

- **OrderVolume**: MT4 から `Lots` パラメータを複製するデフォルトの固定サイズ。
- **RiskPercentage**: ゼロより大きい場合、ストラテジーは取引サイズをポートフォリオ値のパーセンテージとして計算します。利用可能な場合は商品証拠金要件を使用し、利用できない場合は価格ベースのサイジングに戻り、MQL コードの `Lots == 0` の動作を再現します。
- **MaxOrders**: 最大 `MaxOrders * OrderVolume` 件の露出を許可することで、方向ごとの累積数量を制限します。

## パラメーター

| パラメータ | 説明 |
|-----------|-------------|
| `OrderVolume` | `RiskPercentage` がゼロの場合に使用される基本注文サイズ。 |
| `MaPeriod` | 加重終値に基づいて構築された EMA の長さ。 |
| `CciPeriod` | CCI の計算に使用されるバーの数。 |
| `CciLevel` | 信号生成の絶対 CCI しきい値。 |
| `StochasticKPeriod` | Stochastic %K 行をルックバックします。 |
| `StochasticDPeriod` | %D ラインの移動平均期間。 |
| `StochasticSmooth` | 追加の平滑化が %K に適用されました。 |
| `StochasticLevel` | 買われすぎ/売られすぎの検出に使用される 50 からの偏差。 |
| `MaxOrders` | 方向ごとに許可されるエントリの最大数。 |
| `StopLossPoints` | 価格ポイントでのオプションのストップロス距離。 |
| `TakeProfitPoints` | 価格ポイントでのオプションのテイクプロフィットディスタンス。 |
| `RiskPercentage` | 動的なポジションサイジングのためのポートフォリオの割合。 |
| `CandleType` | すべての計算に使用されるローソク足シリーズ。 |

## 実用上の注意

- 短期オシレーターが価格変動に素早く反応する日中データに最適です。
- 加重終値により、ローソク足の高値/安値範囲を組み込みながら、EMA の応答性が維持されます。
- すべての決定は前のローソク足に依存するため、この戦略はバー内の再描画を回避し、ヒストリカル テストにおいて決定性を維持します。
- `OrderVolume` と `MaxOrders` が実行可能な取引サイズに対応するように、リスク管理をブローカーの契約仕様に合わせる必要があります。
