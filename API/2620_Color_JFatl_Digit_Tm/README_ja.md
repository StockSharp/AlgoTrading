# Color JFATL Digit TM 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**Color JFATL Digit TM 戦略**は、Jurikフィルタリングされたジャパニーズ版FATLとJurik MATMを組み合わせたMetaTrader 5のオリジナルエキスパートアドバイザーのポートです。スムーズなFATLラインの傾きを監視し、各バーは強気（色 = 2）、弱気（色 = 0）、中立（色 = 1）として分類されます。これらの色の状態の変化がエントリー、エグジット、ポジション管理をトリガーし、設定可能なセッション時間、ストップロス、テイクプロフィット距離を尊重します。

## ロジック
1. **カスタムインジケーターの複製**
   - FATL値は、選択した適用価格を元の39係数の重みテーブルで畳み込み計算することで算出されます。
   - 結果はStockSharpの `JurikMovingAverage` を使用して平滑化されます。ライブラリが `Phase` プロパティを公開している場合、MT5の入力を反映するためにリフレクションで設定されます。
   - 平滑化された値は価格ステップに `10^DigitRounding` を掛けることで楽器の精度に丸められ、MQL5の `Digit` パラメーターを再現します。
   - 現在の丸められた値と前の値の差がバーの色を定義します（`2 = 上昇`、`0 = 下降`、`1 = 変化なし / 継承`）。

2. **シグナル評価**
   - リングバッファが最新の色コードを保持します。`SignalBar` パラメーターは、スキップする完了バーの数を選択します（デフォルト = 1、つまり前の閉じたバー）。
   - 前の色が強気（`2`）で最新の色が強気以外（`< 2`）の場合、**ロングエントリー**がトリガーされます。
   - 前の色が弱気（`0`）で最新の色が弱気以外（`> 0`）の場合、**ショートエントリー**がトリガーされます。
   - 前の色が弱気（`0`）になると**ロングエグジット**が発生します。
   - 前の色が強気（`2`）になると**ショートエグジット**が発生します。
   - ポジションが既に存在する場合はエントリーがスキップされ、MT5エキスパートの単一ポジション動作が複製されます。

3. **セッション制御と保護**
   - オプションのセッションフィルタリング（`EnableTimeFilter`）は開始時刻が終了時刻より大きい場合の夜間セッションを含むMT5の時間/分ロジックを反映します。
   - 取引が許可された窓の外にある場合、すべてのオープンポジションが即座に清算され、オリジナルのエキスパートに対応します。
   - ポイントで表されたストップロスとテイクプロフィット距離は、楽器の価格ステップを使用して価格単位に変換され、`StartProtection` に渡されます。

## パラメーター
- `OrderVolume` – 注文ごとのボリューム（買いと売りエントリーの両方に使用）。
- `EnableTimeFilter`、`StartHour`、`StartMinute`、`EndHour`、`EndMinute` – セッションウィンドウの設定。
- `StopLossPoints`、`TakeProfitPoints` – ポイント単位の保護距離（0 で各レッグを無効化）。
- `BuyOpenEnabled`、`SellOpenEnabled`、`BuyCloseEnabled`、`SellCloseEnabled` – ロング/ショートエントリーとエグジットを個別に有効または無効にします。
- `SignalCandleType` – カスタムインジケーターとトレードシグナルに使用する時間軸（デフォルトは4時間足）。
- `JmaLength`、`JmaPhase` – Jurik平滑化設定（基礎となるインジケーターが公開する場合、phaseが適用されます）。
- `AppliedPriceMode` – MT5バージョンと同一の適用価格列挙（終値、始値、中値、典型値、TrendFollowバリアント、Demark等）。
- `DigitRounding` – MQLインジケーターの `Digit` 入力を模倣する丸め乗数。
- `SignalBar` – 色の変化を評価する際に遡る閉じたバーの数（デフォルト 1）。

## 注意事項
- 戦略はStockSharpの変換ガイドラインで推奨される通り、`SubscribeCandles` と高レベルの注文ヘルパー（`BuyMarket`、`SellMarket`）を使用します。
- Jurikのphaseはリフレクションで適用されます。ランタイム実装が `Phase` プロパティを公開しない場合、デフォルトの動作が自動的に使用されます。
- 丸めには有効な `Security.PriceStep` が必要です。利用できない場合、インジケーター値は丸められないままになります。

## 使用方法
1. 設定された `SignalCandleType` を提供できる楽器と接続に戦略を接続します。
2. 適用価格、Jurikパラメーター、セッション時刻、マネーマネジメントの入力値を希望通りに設定します。
3. 戦略を開始します。単一ポジションを管理し、ストップロス/テイクプロフィット保護と上述の色ベースのシグナルを尊重します。
