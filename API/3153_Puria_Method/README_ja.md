# Puria Method戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Puria Method戦略は、もともとMetaTrader用に設計されたトレンドフォローシステムです。3つの移動平均とMACDトレンドフィルターを組み合わせて、Momentumブレイクアウトを検出します。StockSharpへの変換では元のエントリーロジックを維持しつつ、部分的な利益確定や自動トレーリングストップなどの最新のリスク管理機能を追加しています。

## トレードロジック
- 設定可能な平滑化手法と価格ソースを使用して3つの移動平均を計算します。
- 前のバーにおける、遅い基準MAと2つの速いMAの差を評価します。強気シグナルは両方の速いMAが基準より少なくとも0.5ポイント上にある必要があります。弱気シグナルは基準が同じマージンでリードする必要があります。
- MACD主線で市場の方向性を確認します。ロングトレードは前のMACD値が正で、設定されたバー数で直近のMACDヒストリーが非減少であることが必要です。ショートトレードは逆の条件が必要です。
- エントリーが発動すると、戦略は反対のポジション（あれば）を決済し、シグナル方向に新しいネットポジションを開きます。

## リスク管理
- **ストップロス / テイクプロフィット:** 価格はエントリーからpips距離で計算され、インストゥルメントの価格ステップに正規化されます。
- **トレーリングストップ:** ポジションがトレーリング閾値にステップを加えた値を超えると、追加のトレーリングステップごとにストップが前進します。
- **部分決済:** 価格が最小利益距離を超えると、利益を確保するために設定可能な割合のポジションが決済されます。
- **ポジション管理:** アルゴリズムはエントリー後の最高値（ロング）または最安値（ショート）を追跡し、ローソク足がそのレベルを突破したときにストップまたは利益ルールを発動します。

## パラメーター
| 名前 | 説明 |
| ---- | ----------- |
| `StopLossPips` | pips単位のストップロス距離。 |
| `TakeProfitPips` | pips単位のテイクプロフィット距離。 |
| `TrailingStopPips` | pips単位のトレーリングストップ距離。 |
| `TrailingStepPips` | トレーリングストップが更新される前の最小利益前進量。 |
| `MinProfitStepPips` | 部分利益確定前の最小距離（pips）。 |
| `MinProfitFraction` | 最小利益ステップ達成時に決済するポジションの割合。 |
| `CandleType` | 戦略が使用するプライマリローソク足シリーズ。 |
| `Ma0Period`, `Ma1Period`, `Ma2Period` | 3つの移動平均の期間。 |
| `Ma0Shift`, `Ma1Shift`, `Ma2Shift` | 各移動平均に適用されるオプションのバーシフト。 |
| `Ma0Method`, `Ma1Method`, `Ma2Method` | 移動平均の平滑化手法（単純、指数、平滑化、線形加重）。 |
| `Ma0Price`, `Ma1Price`, `Ma2Price` | 移動平均のローソク足価格ソース。 |
| `MacdFastPeriod`, `MacdSlowPeriod`, `MacdSignalPeriod` | MACDの設定。 |
| `MacdTrendBars` | MACDの単調トレンドを確認するために使用するバー数（最小3）。 |
| `MacdPrice` | MACD計算のローソク足価格ソース。 |

## 注意事項
- 戦略は未完成のローソク足データへの依存を避けるため、MAとMACDの比較には前の完成したバーを使用します。
- pipsサイズはインストゥルメントの価格ステップと小数精度から自動的に導出されます。
- トレーリングと部分決済機能はゼロ以外の設定値が必要です。そうでない場合、対応するブロックは非アクティブのままです。
- 変換版は完成したローソク足（`CandleStates.Finished`）のみに依存し、元のチャートのタイムフレームと一致するローソク足シリーズと組み合わせて使用する必要があります。
