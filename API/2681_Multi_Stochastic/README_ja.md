# マルチ Stochastic 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
マルチ Stochastic 戦略は、MetaTrader 5 エキスパートアドバイザー "Multi Stochastic (barabashkakvn's edition)" の StockSharp ハイレベル実装です。最大4つの通貨ペアを同時に監視し、Stochastic オシレーター読み値（5、3、3）からの同期シグナルに基づきます。売られすぎまたは買われすぎのクロスオーバーが発生するとシンボルごとに単一のマーケットポジションを開き、固定 pip ベースのストップロスとテイクプロフィット目標で取引を閉じます。

## トレーディングロジック
- 設定された各シンボルは独自の Stochastic オシレーター（長さ5、%K 平滑化3、%D 平滑化3）を受け取ります。
- ロングシグナルは、現在の %K が OversoldLevel（デフォルト20）を下回り、前のバーの %K が %D を下回り、現在のバーが %K を %D より上でクローズするときに生成されます。
- ショートシグナルは、現在の %K が OverboughtLevel（デフォルト80）を上回り、前のバーの %K が %D を上回り、現在のバーが %K を %D より下でクローズするときに生成されます。
- インストゥルメントごとに許可されるオープンポジションは1つのみです。既存のポジションが閉じるまで追加のシグナルは無視されます。

## リスク管理
- ストップロスとテイクプロフィットの値は pips で表されます。戦略は銘柄の価格ステップに乗算し、3桁または5桁の外国為替クォートに調整することで、pips を絶対価格距離に自動変換します（これらのインストゥルメントでは pip = ステップ × 10）。
- ロングポジションは、ローソク足の安値がストップロスレベルに触れるか、ローソク足の高値がテイクプロフィットレベルに達したときに閉じます。
- ショートポジションは、ローソク足の高値がストップロスレベルに触れるか、ローソク足の安値がテイクプロフィットレベルに達したときに閉じます。

## パラメーター
- `CandleType` – サブスクライブされたすべてのローソク足に使用する時間軸（デフォルト：1時間）。
- `StochasticLength` – Stochastic オシレーターの基本長さ（デフォルト：5）。
- `StochasticKPeriod` – %K の平滑化期間（デフォルト：3）。
- `StochasticDPeriod` – %D の平滑化期間（デフォルト：3）。
- `OversoldLevel` – 売られすぎ状態を検出するために使用するしきい値（デフォルト：20）。
- `OverboughtLevel` – 買われすぎ状態を検出するために使用するしきい値（デフォルト：80）。
- `StopLossPips` – 保護ストップへの距離（pips、デフォルト：50）。
- `TakeProfitPips` – 利益目標への距離（pips、デフォルト：10）。
- `UseSymbol1` … `UseSymbol4` – 各シンボルスロットの取引を有効にします（デフォルト：true）。
- `Symbol1` … `Symbol4` – 各スロットで取引される銘柄。Symbol 1 は指定がない場合、戦略のメイン銘柄にフォールバックします。

## 実装上の注意
- 各シンボルのサブスクリプションは独立しています。それぞれは `SubscribeCandles` と `BindEx` を使用してローソク足データと共に `StochasticOscillatorValue` の更新を受け取ります。
- MT5 のクロスオーバー検出ロジックをエミュレートするため、前の %K と %D の値がシンボルごとにキャッシュされます。
- リスクパラメーターは各エントリーで再計算され、ポジションが閉じた後またはポジションが存在しない場合にストップ/テイクレベルがリセットされます。
- 注文は共有の `Volume` プロパティを使用して `BuyMarket`/`SellMarket` で送信され、元のエキスパートの単一ポジション制約に対応します。

## MT5 バージョンとの違い
- StockSharp バージョンは、手動レート更新呼び出しの代わりにハイレベルサブスクリプションを活用します。
- pip サイズの検出は `Security.PriceStep` と `Security.Decimals` に基づいています。メタデータが利用できない場合、不正なリスク計算を防ぐためにストップとターゲットは無効のままです。
- ログ記録とチャート描画フックは拡張可能な状態ですが、コア動作には必要ありません。

## 使用のヒント
1. シンボルスロットに希望の銘柄を割り当て、取引の時間軸に合わせてローソク足の時間軸を調整してください。
2. ストップロスとテイクプロフィットの距離がインストゥルメントのティックサイズと互換性があることを確認し、即座の閉鎖を避けてください。
3. 少ないインストゥルメントを監視する際のリソース消費を削減するために、未使用のシンボルスロットを無効にしてください。
