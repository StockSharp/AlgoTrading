# OzFx Accelerator Stochastic戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- MetaTraderエキスパートアドバイザー*OzFx（barabashkakvn版）*をStockSharpの高レベル戦略APIに変換したもの。
- Acceleration/Decelerationオシレーター（AC）とStochasticの閾値を組み合わせてトレンドに段階的にエントリーする。
- 注文がロット単位でサイジングされ、保護がpipsで表現される裁量型FX取引向けに設計されている。

## トレードロジック
1. Awesome Oscillatorとその5期間SMAの差としてAcceleration/Decelerationオシレーターを計算する。
2. 設定可能な`%K`、`%D`、スローイング期間でStochasticオシレーターをサブスクライブする。
3. 新しいローソク足が閉じると、最新の2つのAC値とStochasticレベルを合わせて評価する:
   - **ロングセットアップ**: `%K`が設定レベルを上抜け、現在のACが正で上昇し、前の値が負だった場合。
   - **ショートセットアップ**: `%K`がレベルを下抜け、現在のACが負で下落し、前の値が正だった場合。
4. 有効なシグナルで最大5つの等量成行注文を開く。最初のレイヤーはストップ/ターゲットなしで起動するオリジナルEAを反映し、残りのレイヤーは設定されたストップロスと段階的なテイクプロフィットを継承する。
5. エグジット管理はオリジナルの`modok`フラグ動作をエミュレートする:
   - トレーリングストップが無効な場合、戦略は収益性のあるエグジット後にのみストップをブレイクイーブンに締め、Stochastic/ACの組み合わせがポジションに対して反転するとすべてのレイヤーをクローズする。
   - トレーリングストップが有効な場合、動きが*TrailingStop + TrailingStep*を超えるとストップが価格に追従し、同じモメンタムの反転がスタックをクローズする。

## ポジションスケーリングとターゲット
- ロングポジションは`i = 1..4`に対して`entry + TakeProfit * i`でテイクプロフィットを持つ4つの追加レイヤーを配置する。ショートは価格より下に対称的にこれを反映する。
- ストップロス（設定されている場合）は最初のレイヤーを除くすべてのレイヤーに取り付けられ、MT5スクリプトとまったく同じになる。
- 部分的なテイクプロフィットは内部フラグを更新し、次のキャンペーンが即座に"modok = true"状態で開始され、初期レイヤーのブレイクイーブン保護がアンロックされるようにする。

## リスク管理
- `StopLossPips`と`TakeProfitPips`はpipsで定義される。戦略はインストゥルメントのティックサイズと桁精度（`5`または`3`小数点ペアは分数pipsとしてカウント）を使用してこれらを変換する。
- `TrailingStopPips = 0`はトレーリングロジックを無効にし、テイクプロフィット後のブレイクイーブン調整のみを有効にする。任意の正の値は上記のトレーリングブロックを有効化する。
- すべてのエグジットは、ローソク足範囲が保存されたストップまたはターゲットレベルを超えたときに成行注文で実行され、ブローカー側の保護注文に依存したオリジナルエキスパートの動作と一致する。

## パラメーター
| 名前 | 説明 | デフォルト |
| --- | --- | --- |
| `OrderVolume` | レイヤーごとのロットサイズ。 | `0.1` |
| `StopLossPips` | 保護ストップ注文の距離（pips）。 | `100` |
| `TakeProfitPips` | 段階的テイクプロフィット間の基本距離（pips）。 | `50` |
| `TrailingStopPips` | トレーリングストップ距離（pips）（0でトレーリング無効）。 | `50` |
| `TrailingStepPips` | トレーリングストップを進める前の追加距離。 | `5` |
| `KPeriod` | Stochastic `%K`ルックバック。 | `5` |
| `DPeriod` | Stochastic `%D`平滑化。 | `3` |
| `SmoothingPeriod` | `%K`に適用される最終平滑化。 | `3` |
| `StochasticLevel` | 強気/弱気レジームを分ける閾値。 | `50` |
| `CandleType` | 計算用のソースローソク足シリーズ。 | `4時間足` |

## 実装上の注意事項
- シグナル、トレーリング更新、保護エグジットは完了したローソク足で処理され、新しいバーでトリガーするEAと一致した状態を保つ。
- ACインジケーターはAwesome OscillatorとバインドしてそのSMA5期間を引くことで再現され、低レベルインジケーターバッファへのアクセスはない。
- Pip変換は4/5桁FXシンボルに自動的に適応し、ティックサイズメタデータが欠落している場合は合理的なデフォルトにフォールバックする。
- 戦略は段階的エントリーの内部台帳を維持し、部分的なテイクプロフィットとストップ調整がMetaTraderバージョンのポジションごとのロジックと一致するようにする。
- StockSharpがエグジットを成行注文で実行するため、ローソク足の高値/安値が保存されたストップまたはターゲットレベルを突破したときに取引がフラット化され、ブローカー側のトリガーを待つのではなく即座に実行される。
