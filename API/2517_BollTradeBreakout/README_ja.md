# BollTradeブレイクアウト戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、設定可能なpipバッファと残高ベースのオプションのポジションサイジングを使用してボリンジャーバンドのブレイクアウトを取引することで、オリジナルの**BollTrade**エキスパートアドバイザーを再現します。注文は完了したローソク足のみで開かれ、固定のストップロスとテイクプロフィットレベルで管理されます。

## コンセプト

- 設定可能な主要時間軸を購読し、指定された期間と偏差でボリンジャーバンドエンベロープを計算します。
- 早期エントリーを減らすために、上部バンドの上と下部バンドの下にpipで測定した追加オフセット（`Band Offset`）を加えます。
- ローソク足の終値がオフセットを引いた下部バンドを下回ると**ロング**ポジションを開きます。
- ローソク足の終値がオフセットを加えた上部バンドを上回ると**ショート**ポジションを開きます。
- どの時点でも1つのポジションのみがアクティブになれます。戦略は新しいエントリーを評価する前に現在の取引が終了するまで待機します。

## 取引管理

- ストップロスとテイクプロフィットレベルはエントリー直後に設定されます。これらはpipの倍数で表され、完成したすべてのローソク足で評価されます。価格がいずれかのレベルに触れると、ポジションはマーケットでクローズされます。
- `Scale Volume`が有効になっている場合、取引ボリュームは口座残高に伴って増加（または減少）します。スケーリングの基準値は初期ポートフォリオの価値をベースロットサイズで割ったもので、オリジナルのMQL実装を模倣しています。ボリュームはソースコードと同様にリスクをコントロールするために500ロットに制限されています。
- pipサイズはインストゥルメントの価格ステップから導出されます。非常に小さなステップ（forex型シンボル）の場合、コードはMetaTraderバージョンの動作に合わせて、小数pip単位のステップを標準pipに変換するためにステップを10倍します。

## パラメーター

| 名前 | 説明 | デフォルト値 |
| ---- | ---- | ------------ |
| `Candle Type` | シグナルローソク足に使用される時間軸。 | 15分時間軸 |
| `Bollinger Period` | ボリンジャーバンド計算のバー数。 | 4 |
| `Bollinger Deviation` | ボリンジャーバンドの幅乗数。 | 2 |
| `Band Offset` | シグナルを発動する前に両バンドの外側に追加される追加pipオフセット。 | 3 |
| `Take Profit (pips)` | pip単位の利益目標までの距離。 | 3 |
| `Stop Loss (pips)` | pip単位の保護ストップまでの距離。 | 20 |
| `Base Volume` | スケーリングが無効の場合のデフォルトロット単位ボリューム。 | 1 |
| `Scale Volume` | 有効にすると、口座残高に合わせてポジションサイズをスケーリングします。 | 有効 |

## 使用上の注意

- pipベースのオフセットが明確なブレイクアウトレベルを提供するforexまたはCFDシンボルで最も効果的ですが、`PriceStep`が設定されていれば先物や株式でも実行できます。
- 戦略は完成したローソク足のみを処理するため、バークローズ前に戻るイントラバースパイクはエントリーを発動しません。
- エグジットは固定のストップとターゲットで処理されるため、選択した時間軸と銘柄のボラティリティに適した距離を確保してください。
- オリジナルEAはブローカー側のストップに依存していました。このポートはStockSharp内で同じ保護動作をエミュレートするためにローソク足の極値を監視します。
