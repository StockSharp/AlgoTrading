# 振り子スイング戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**振り子スイング戦略**は、MetaTrader エキスパート アドバイザー *振り子 1_01* の StockSharp 移植です。元のシステムは、現在の価格付近で 2 つの保留中のストップ注文を保持し、約定するたびにその量を徐々に増やします。この C# バージョンは、高レベルの StockSharp ヘルパーを使用して同じ「スイング」動作を再現します。

重要なアイデア:

- 最後に閉じたローソク足から設定可能な距離を置いて、対称的な買いストップ注文と売りストップ注文を維持します。
- 各フィルの後、同じ側の次のストップの音量が増加し、MQL ソースからのマーチンゲール スタイルの進行が実装されます。
- 短期ピップ目標に達したとき、または口座資本がグローバル損益基準を超えたときにポジションを閉じます。

## 仕組み
1. 戦略が開始されると、ユーザー定義のローソク足シリーズ (デフォルト: 15 分) と、オプションで毎日のローソク足をサブスクライブします。日次範囲は、市場価格と保留中のストップの間の距離を制御します。
2. 終了した取引ローソクごとに、アルゴリズムは次のようになります。
   - グローバルな株式ベースの制限を更新します。
   - 現在のポジションがローカル利益目標に達したかどうかを検証します。
   - 最新の日次レンジまたは手動ピップ入力からストップ距離を計算し、買いストップ注文と売りストップ注文を発注/アップグレードします。
3. ストップ注文が約定されると、対応する進行レベルが進むため、その側の次のストップでは乗算されたボリュームが使用されます。 `MaxLevels` に達すると、ポジションがゼロに戻るまで、その方向に対して新しい注文は作成されません。
4. グローバルテイクプロフィット/ストップロスチェックはローソク足ごとに実行され、設定された資本しきい値に違反した場合はポートフォリオを清算します。

## パラメーター
| 名前 | 種類 | デフォルト | 説明 |
| ---- | ---- | ------- | ----------- |
| `BaseVolume` | `decimal` | `0.1` | 最初の保留中のストップのボリューム。 |
| `VolumeMultiplier` | `decimal` | `2` | 同じ側の各塗りつぶしレベルの後に適用される係数。 |
| `MaxLevels` | `int` | `8` | 一方向に許可されるフィルの最大数。 |
| `ManualStepPips` | `int` | `50` | 日次範囲が利用できない場合の停止距離 (ピップス)。 |
| `UseDynamicRange` | `bool` | `true` | 有効にすると、最後に終了した毎日のローソク足からステップが導出されます。 |
| `RangeFraction` | `decimal` | `0.2` | 基本停止距離として使用される日次範囲の割合。 |
| `TakeProfitPips` | `int` | `10` | 現在のポジションをクローズするローカルピップターゲット。 `0` を無効に設定します。 |
| `SlippagePips` | `int` | `3` | MetaTrader の滑りを模倣するために保留距離に追加のバッファーが追加されました。 |
| `UseGlobalTargets` | `bool` | `true` | 株式ベースの清算チェックを有効にします。 |
| `GlobalTakePercent` | `decimal` | `1` | 世界的な利益確定を引き起こす株価の伸び（パーセント）。 |
| `GlobalStopPercent` | `decimal` | `2` | グローバルストップロスをトリガーする株式ドローダウン（パーセント）。 |
| `CandleType` | `DataType` | `15m` キャンドル | 主要な取引ロジックに使用される時間枠。 |

## 注意事項
- 位置のサイジングは、楽器の音量ステップ、最小および最大音量設定を考慮します。
- ストップ価格は商品の価格ステップにスナップし、価格許容範囲を尊重することで定期的な注文の置き換えを回避します。
- グローバル ターゲットは `Portfolio.CurrentValue` (またはフォールバックとして `BeginValue`) に依存するため、選択したポートフォリオはこの情報を公開する必要があります。
- この戦略では、`StartProtection()` を使用して、起動時に StockSharp の組み込みポジション保護を有効にします。

## 変換の違い
- 元の MQL スクリプトの UI ラベル描画とアカウント残高テーブルは省略されています。
- グローバルなテイクプロフィットレベルは、MQL で使用される生のティック値の計算ではなく、パーセンテージベースの株式しきい値に従い、ブローカー間で動作の一貫性を保ちます。
- `OrderModify` などの MetaTrader 固有の関数は、StockSharp の注文キャンセルおよび再送信ルーチンに置き換えられます。
