# Breakdown Catcher 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Breakdown Catcher戦略は、MetaTraderのエキスパートアドバイザー「Breakdown catcher」からポートされたバーごとのブレイクアウトシステムです。完了した各ローソク足の後、戦略は前の高値の上と前の安値の下に仮想ブレイクアウトレベルを配置します（オプションでインデントだけシフト）。次のローソク足がこれらのレベルの一方を突破すると、戦略はブレイクアウト方向でポジションを開き、ピップスで表されたストップロス、テイクプロフィット、およびオプションのトレーリング保護を即座に設定します。

## トレードロジック
1. 各ローソク足のクローズ時に、完了したバーの高値と安値が次の期間の参照レンジになります。
2. 買いブレイクアウトレベル = 前の高値 + インデント（ピップス）。売りブレイクアウトレベル = 前の安値 − インデント。
3. 現在のローソク足がポジションなしの状態で買いレベルを突破した場合、戦略は成行でロングポジションを開き、ショートコンテキストを削除し、保護レベルを保存します。
4. 現在のローソク足がフラットの状態で売りレベルを突破した場合、戦略は成行でショートポジションを開きます。
5. ストップロスとテイクプロフィットの距離は、楽器の価格ステップと3/5小数点楽器のための古典的なMetaTrader調整を使用してピップスから絶対価格に変換されます。
6. トレードが少なくとも`TrailingStop + TrailingStep`ピップス有利方向に動いた後、トレーリングストップが保護価格を締める可能性があります。トレーリングステップは、十分な追加移動後にのみストップが動くMetaTraderロジックを模倣します。
7. 同じローソク足内で両方のブレイクアウトレベルに達した場合、曖昧な実行順序を避けるためにその棒のトレードをスキップします。
8. 現在のビッドアスクスプレッドが設定された`AllowedSpreadPoints`を超えるたびに、スプレッドフィルターが新規エントリーをブロックします。

## マネーマネジメント
* 戦略は注文サイズにベースの`Strategy.Volume`を使用します。ポジションを反転する際、完全なフリップを確保するために現在のポジションの絶対値だけボリュームが増加します。
* ストップロス、テイクプロフィット、トレーリングストップは、価格レンジが保護レベルを含む場合に成行の出口注文を発行することで内部的に管理されます。

## パラメーター
| パラメーター | 説明 | デフォルト |
|-----------|-------------|---------|
| `StopLossPips` | ピップスでのストップロス距離。 | `30` |
| `TakeProfitPips` | ピップスでのテイクプロフィット距離。 | `90` |
| `TrailingStopPips` | ピップスでのトレーリングストップ距離。トレーリングを無効にするには`0`に設定。 | `30` |
| `TrailingStepPips` | トレーリングストップが動く前に必要な追加進捗。トレーリングが有効な場合は正でなければならない。 | `5` |
| `IndentPips` | ブレイクアウトレベルに適用される追加オフセット。 | `0` |
| `AllowedSpreadPoints` | 生のポイント（`PriceStep`単位）で測定された最大スプレッド。 | `5` |
| `CandleType` | ブレイクアウト検出に使用するローソク足シリーズ。 | `1時間タイムフレーム` |

## 注記と制限
* ピップスの変換は元のEAと同じ桁調整に従います：楽器に3または5小数点がある場合、1ピップは10価格ステップに相当します。
* StockSharpの高レベルAPIはローソク足イベントで動作するため、単一のローソク足内で両方のブレイクアウトレベルがどの順序で達するかを判定できません；そのため戦略はそのようなバーをスキップします。
* 保護注文は成行出口でモデル化され、ブローカー側のストップ注文に依存せずに戦略が自己完結していることを保証します。
