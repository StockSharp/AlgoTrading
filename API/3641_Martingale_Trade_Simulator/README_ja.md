# Martingale 貿易シミュレーター戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

`MartingaleTradeSimulatorStrategy` は、StockSharp フレームワーク内で MetaTrader の「Martingale Trade Simulator」エキスパート アドバイザーを再作成します。この戦略は、トレーダーが追加の自動化スクリプトを作成することなく、即時成行注文を送信し、マーチンゲール スタイルの平均を適用し、トレーリング保護を管理できる手動取引パネルです。パラメータの切り替えにリアルタイムで反応するため、オリジナルの MQL ロボットと同様に、ストラテジー テスターの実験に適しています。

## 仕組み

### 手動マーケットボタン
- `Buy` および `Sell` パラメータは仮想ボタンとして機能します。いずれかのパラメータが `true` に設定されている場合、ストラテジーは出来高 `Order Volume` の成行注文を送信し、パラメータを自動的に `false` にリセットします。
- 未決注文は使用されません。この戦略は完全に市場執行で機能し、MetaTrader のビジュアル テスター内のシミュレーターの動作を反映しています。

### Martingale の平均化
- `Enable Martingale` を有効にすると、`Martingale` パラメータが `true` に切り替えられたときにパネルで平均注文を行うことができます。
- 戦略はアクティブなポジションをチェックします。
  - **ロングポジション:** 現在の売値が最低約定買値より少なくとも `Martingale Step (points)` 低い場合、新しい買い注文が送信されます。
  - **空売りポジション:** 現在の入札価格が約定した最高売価格を少なくとも `Martingale Step (points)` 上回る場合、新しい売り注文が発行されます。
- 各平均注文量は `Order Volume × Martingale Multiplier^N` に等しくなります。ここで、`N` は現在の方向の連続エントリの数です。
- マーチンゲールがアクティブな場合、累積ドローダウンをカバーするために、加重平均エントリー価格プラス/マイナス `Martingale TP Offset (points)` にテイクプロフィットターゲットが再計算されます。

### トレーリングストップモジュール
- `Enable Trailing` は、最新の最良価格に従う保護トレーリング ストップを有効にします。
- トレーリングストップは市場価格から離れた `Trailing Stop (points)` で始まり、価格が少なくとも `Trailing Step (points)` 上昇した後にのみ前に進みます。
- 市場価格がトレーリングレベルを横切ると、この戦略は反対の成行注文でポジション全体を即座にクローズします。

### ストップロスとテイクプロフィット
- `Stop Loss (points)` と `Take Profit (points)` は、元のエキスパート アドバイザーの基本的なリスク管理を再現します。
- ロングポジションの場合、ストップは平均エントリー価格よりも下に配置され、テイクプロフィットは上に配置されます。ショートポジションの場合、両方のレベルがミラーリングされます。
- 保護的出口は成行注文で実行されるため、この戦略は StockSharp でサポートされているコネクタとの互換性を維持します。

## パラメーター

| パラメータ | 説明 | デフォルト |
|-----------|-------------|---------|
| `Order Volume` | 手動成行注文の基本サイズ。 | `1` |
| `Stop Loss (points)` | 保護停止までの距離。ゼロはストップロスを無効にします。 | `500` |
| `Take Profit (points)` | 保護対象までの距離。ゼロはテイクプロフィットを無効にします。 | `500` |
| `Enable Trailing` | トレーリングストップモジュールのオン/オフを切り替えます。 | `true` |
| `Trailing Stop (points)` | 価格とトレーリングストップ間の距離。 | `50` |
| `Trailing Step (points)` | トレーリングストップを前進させるために必要な最小限の有利な動き。 | `20` |
| `Enable Martingale` | `Martingale` ボタンで制御される注文の平均化を許可します。 | `true` |
| `Martingale Multiplier` | 追加の平均取引ごとに使用されるボリューム乗数。 | `1.2` |
| `Martingale Step (points)` | 平均化注文が許可される前に必要な逆方向の動き。 | `150` |
| `Martingale TP Offset (points)` | 平均化されたテイクプロフィットレベルに適用される追加のオフセット。 | `50` |
| `Buy` | 成行買い注文を送信するには、`true` に設定します (自動リセット)。 | `false` |
| `Sell` | 成行売り注文を送信するには、`true` に設定します (自動リセット)。 | `false` |
| `Martingale` | 平均注文を評価して発注するには、`true` に設定します (自動リセット)。 | `false` |

## 使い方のヒント

1. ストラテジをインストゥルメントにアタッチし、`Order Volume` を設定して、テスター モードまたはライブ モードで開始します。
2. `Buy` / `Sell` トグルを使用して、MetaTrader パネルからのボタンのクリックをシミュレートします。
3. 最初の取引後、価格がポジションに対して変動するたびに、`Martingale` トグルをトリガーします。この戦略は価格距離を検証し、条件が満たされた場合に取引量を増やします。
4. トレーリング パラメータとリスク パラメータを調整して、元の EA の動作を再現するか、別の設定を試してください。

## 注意事項

- この戦略は、レベル 1 データ (最良の買値/売値および最後の取引) に基づいて市場の状況を評価します。
- C# コード内のすべてのコメントは英語であり、リポジトリのガイドラインとの一貫性が保たれています。
