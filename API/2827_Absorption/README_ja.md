# 吸収戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略はMetaTrader用のAbsorptionエキスパートアドバイザーを再現します。前のバーのレンジを吸収し、短いルックバック内で極値を形成する「エングルフィング」ローソク足を探します。そのような吸収バーが現れると、アルゴリズムは市場の両側にストップ注文を配置し、固定目標、ブレークイーンロジック、トレーリングストップを組み合わせて結果のポジションを管理します。

## 取引ロジック

1. **パターン検出**
   - 最後の2本の完成したローソク足を検査します。
   - ローソク足の高値が前のローソク足の高値を上回り、安値が前のローソク足の安値を下回る場合に*吸収バー*として扱います。
   - バーを検証するため、その高値または安値が過去`MaxSearch`本のローソク足内で最も極端な値であるかどうかを確認します。
   - 古いローソク足（2本前）に優先権が与えられます。両方のバーが吸収条件を満たす場合、古いバーが使用されます。そうでない場合、最も最近のバーがセットアップをトリガーすることができます。
2. **注文配置**
   - バー高値に設定された`Indent`を加えた位置に買いストップ注文が配置されます。
   - バー安値から同じ`Indent`を引いた位置に売りストップ注文が配置されます。
   - 両方の注文は共通の戦略ボリュームを使用します。
   - 各保留注文は独自の保護ストップレベルとオプションのテイクプロフィット目標を保存します。注文は未執行のまま`OrderExpirationHours`経過後に自動的に失効します。
3. **ポジション管理**
   - 一方の側が執行されると、反対の保留注文がキャンセルされます。
   - 初期ストップは、インデントを差し引いた/加えた吸収ローソク足の反対側にあります。
   - オプションの固定テイクプロフィットは、設定された価格ステップの距離に達すると取引をクローズします。
   - ブレークイーンモジュールは、価格が`BreakevenProfit`ステップ前進した後、ストップロスを`エントリー + Breakeven`（ロング）または`エントリー - Breakeven`（ショート）に移動します。
   - トレーリングストップは、ストップロスを最良価格から`TrailingStop`距離に維持し、価格が利益方向に少なくとも`TrailingStep`ステップ移動した場合にのみ更新します。

## パラメータ

| パラメータ | 説明 |
|-----------|-------------|
| `CandleType` | サブスクライブするローソク足データタイプ（デフォルト：1時間の時間軸）。 |
| `MaxSearch` | 高値/安値の極値を確認するために使用する最近のローソク足の数。 |
| `TakeProfitBuy` | ロングテイクプロフィット注文の価格ステップ単位の距離。`0`で目標を無効化。 |
| `TakeProfitSell` | ショートテイクプロフィット注文の価格ステップ単位の距離。`0`で目標を無効化。 |
| `TrailingStop` | 価格ステップ単位のトレーリングストップ距離。`0`でトレーリングを無効化。 |
| `TrailingStep` | トレーリングストップを進める前に必要な最小前進移動。トレーリングが有効の場合は正でなければなりません。 |
| `Indent` | ストップエントリーレベルを定義するために吸収バーの上/下に追加される価格ステップ単位のオフセット。 |
| `OrderExpirationHours` | 保留注文の有効期間。この期間後、トリガーされない場合に注文がキャンセルされます。 |
| `Breakeven` | ブレークイーンルールが発動したときにストップロスに適用されるオフセット。`0`でブレークイーンを無効化。 |
| `BreakevenProfit` | ストップロスをブレークイーンに移動する前に達成される必要がある利益閾値（価格ステップ）。 |

すべての距離ベースの入力は銘柄の価格ステップの倍数として表されます。デフォルトの戦略ボリュームは`0.1`に設定されています。

## リスク管理

戦略はエグジットに成行注文のみを使用します。ストップロス、テイクプロフィット、ブレークイーン、トレーリングルールは、バー内のレベルタッチを検出するためにローソク足の高値と安値を監視します。エグジット注文が送信されると、現在のポジションがフラットになるまで追加のエグジットリクエストは生成されません。
