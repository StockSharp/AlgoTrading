# DealersTradeZeroLag MACD戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、MetaTraderのエキスパートアドバイザー「Dealers Trade v 7.91 ZeroLag MACD」をStockSharpの高レベルAPIにポートしたものです。ゼロラグMACDの傾きを追跡して市場がロングまたはショートの蓄積段階にあるかどうかを判断し、適応的なスペースとリスク管理でポジションのグリッドを構築します。デフォルトの時間軸は元の作者が推奨する4時間足ですが、StockSharpがサポートする任意の足タイプを選択できます。

## トレードロジック
- **シグナル検出。** 2つのゼロラグ指数移動平均（ファストとスロー）がMACDラインを生成します。MACDが前のバーと比較して上昇すると、戦略は市場を強気として扱い、下落すると弱気として扱います。シグナルは`ReverseCondition`パラメーターで反転できます。
- **ポジショングリッド。** アルゴリズムは検出された方向にスケールします。エントリー間の距離はピップスで測定され、各フィル後に`IntervalCoefficient`で乗算されます。ロットサイズは各追加エントリーで`LotMultiplier`で乗算され、MQLバージョンのマーチンゲールスキームを模倣します。
- **ボリューム制御。** `BaseVolume`がゼロより大きい場合、初期注文量として使用されます。そうでない場合、エンジンは`RiskPercent`、ストップ距離、インストゥルメントのステップパラメーターからサイズを導出します。各計算されたボリュームはインストゥルメントの制限に対して確認され、`MaxVolume`でキャップされます。
- **注文管理。** 各エントリーにはストップロス、テイクプロフィット、トレーリングストップ（すべてピップス単位）を装備できます。テイクプロフィット距離は連続するエントリーに対して`TakeProfitCoefficient`で乗算されてターゲットを広げます。
- **口座保護。** 未決済ポジションの総数が`PositionsForProtection`を超え、合算利益が`SecureProfit`に達すると、戦略は利益を確保するために最大の利益を持つトレードをクローズします。ポジションの総数が`MaxPositions`を超えると、新しいエントリーを受け入れる前に最も成績の悪いトレードをクローズします。

## ポジション処理
- ストップ、トレーリングロジック、ターゲットは、クローズ、高値、安値の価格を使用して完成した足で評価されます。
- すべての未決済ポジションは、独自のボリューム、エントリー価格、トレーリング状態で追跡されます。最後のフィル価格は将来のエントリーの最小スペースを強制するために再利用されます。
- 口座残高が`MinimumBalance`を下回ると、小規模口座でのオーバートレードを避けるために戦略は自分自身を停止します。

## パラメーター
| パラメーター | 説明 |
|-----------|-------------|
| `BaseVolume` | 初期注文サイズ。`RiskPercent`によるリスクベースのサイジングを有効にするにはゼロに設定してください。 |
| `RiskPercent` | ポジションサイズがストップ距離から導出される場合にリスクにさらすポートフォリオ資本の割合。 |
| `MaxPositions` | 同時に開かれる最大エントリー数。 |
| `IntervalPips` | ピップスでのグリッドエントリー間の初期スペース。 |
| `IntervalCoefficient` | 各追加エントリー後にスペースに適用される乗数。 |
| `StopLossPips` | ストップロス距離（ピップス）。無効にするにはゼロに設定してください。 |
| `TakeProfitPips` | ピップスでのベーステイクプロフィット距離。エントリーごとに`TakeProfitCoefficient`で乗算されます。 |
| `TrailingStopPips` / `TrailingStepPips` | トレーリングストップ距離とトレーリングが調整される前に必要なアドバンス。 |
| `TakeProfitCoefficient` | 後のエントリーでテイクプロフィット距離を広げるための乗数。 |
| `SecureProfit` | 十分な数のポジションがオープンになったときに口座保護をトリガーする利益閾値。 |
| `AccountProtection` | 最良のトレードをクローズすることで自動的な利益ロックを有効にします。 |
| `PositionsForProtection` | 口座保護がアクティブになる前に必要な最小未決済ポジション数。 |
| `ReverseCondition` | MACDの傾き解釈を反転させます。 |
| `FastLength`, `SlowLength`, `SignalLength` | ゼロラグ指数移動平均の期間。 |
| `MaxVolume` | 単一エントリーのボリューム上限。 |
| `LotMultiplier` | 各グリッドエントリーでポジションサイズをスケールする乗法的因子。 |
| `MinimumBalance` | 取引を継続するために必要な最低口座残高。 |
| `CandleType` | 計算に使用する足データタイプ。 |

## 使用上のメモ
1. 起動前に戦略をポートフォリオと証券に接続してください。
2. ピップ変換が正確であることを確認するために、インストゥルメントのステップと価格設定を確認してください。
3. デフォルトパラメーターは元のエキスパートアドバイザーの動作を再現しますが、StockSharpのオプティマイザーで最適化できます。
4. この戦略にはPythonの翻訳は含まれていません。
