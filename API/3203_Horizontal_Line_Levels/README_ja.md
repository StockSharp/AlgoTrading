# Horizontal Line Levels戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

**Horizontal Line Levels**戦略は、同名のMetaTrader 5エキスパートアドバイザーを模倣します。現在の相場の周りに2つの価格レベルを継続的に再構築し、市場がそれらを突破すると、ユーザーに通知します。実装はLevel1（bid/ask）市場データに依存し、注文を送信せずに元のOnTick/OnTimerワークフローを模倣します。

## コアアイデア

1. Level1データを購読し、最新のベストbidとベストaskの価格をキャッシュします。
2. MetaTraderのポイント距離をStockSharpの価格スケールに変換します。
3. ベストaskを上にオフセットし、ベストbidを設定された距離だけ下にオフセットして、2つの仮想水平線を作成します。
4. 内部タイマーを通じて定期的にbidまたはaskがそれらの参照レベルを突破するかどうかを確認し、戦略ジャーナルにアラートを記録します。

## パラメーター

| 名前 | デフォルト | 説明 |
| --- | --- | --- |
| `TimerPeriodMinutes` | `1` | 2回の連続したタイマーチェック間の分数。正の値でなければなりません。 |
| `OffsetPoints` | `50` | 線を構築する際にaskの上とbidの下に適用するMetaTraderポイントでの距離。 |

## 動作の詳細

- **データ購読**：`GetWorkingSecurities`はLevel1ストリームを登録して、ローソク足なしでも戦略がbid/ask更新を受け取れるようにします。
- **初期化**：ベストbidとベストaskの両方が初めて利用可能になると、`RecalculateLevels`は現在の上部と下部の水平レベルを保存します。
- **タイマー**：各タイマーティックは欠落したレベルを再作成し（初期化が相場準備前に発生した場合）、市場がいずれかの境界を突破するとログメッセージを発行します。
- **MetaTraderポイント変換**：ヘルパー`EnsurePointSize`は`Security.PriceStep`を使用してMetaTraderの「ポイント」を絶対価格増分に変換します。同じ技術は他の変換された戦略でも数値的互換性を維持するために使用されています。
- **取引なし**：戦略は注文を送信しません。`AddInfoLog`を通じてアラートのみを生成します。これは価格がいずれかの線に触れるとポップアップアラートを表示した元のエキスパートと一致します。
- **停止/リセット**：戦略を停止するとタイマーがキャンセルされ、すべてのキャッシュされた値がクリアされるため、次の実行はクリーンな状態から始まります。

## 典型的な使用方法

1. 戦略を目的のインストゥルメントに接続し、DesignerのUIで`TimerPeriodMinutes`と`OffsetPoints`を設定します。
2. 戦略を開始します。完全な相場スナップショットが届くと、`Horizontal levels updated. Upper: 1.12345, Lower: 1.12245.`のようなログエントリーが計算された閾値を確認します。
3. ログウィンドウを監視します。askが上部レベルを上回る（またはbidが下部レベルを下回る）と、戦略は対応するアラートメッセージを印刷します。
4. オフセットを変更したり戦略を再起動したりすると、レベルは新しいパラメーターを使用して再計算されます。

## 分類

- **カテゴリ**：ユーティリティ / アラート
- **方向**：なし
- **実行スタイル**：イベント駆動型モニタリング
- **データ要件**：Level1 bid/ask
- **複雑さ**：基本
- **推奨タイムフレーム**：任意（純粋に相場主導）
- **リスク管理**：該当なし（ポジションなし）

この変換は、戦略タイマーやLevel1購読などのStockSharp抽象化を活用しながら、MetaTraderオリジナルのアラート中心の動作を維持します。
