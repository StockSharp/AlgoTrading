# JS Chaos戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
JS Chaos戦略は、StockSharpの高レベルAPIを使用してMetaTraderエキスパートアドバイザー「JS-Chaos」の動作を複製します。この戦略はビル・ウィリアムズのアリゲーター構造とフラクタルレベルを中心にブレイクアウトエントリーを構築し、Awesome OscillatorとAcceleration/Decelerationの確認を組み合わせ、トレーリングストップ、ブレイクイーブンロジック、豊富な時間フィルターでオープンエクスポージャーを管理します。

## コアロジック
1. **インジケータースタック**
   - ビル・ウィリアムズのアリゲーター（中央値価格でサンプリングされた13/8/5ピリオドと8/5/3バーシフトを持つ平滑化移動平均）。
   - Awesome OscillatorとそのAOの5ピリオドSMAでAcceleration/Decelerationオシレーターを導出。
   - トレーリングストップエンジン用の21ピリオド平滑化移動平均。
   - 追加のトレーリング条件として使用される10ピリオド標準偏差。
   - 最後の5本の高値/安値のフラクタル検出、10バーの間の最新の形成を保存。
2. **シグナル生成**
   - 強気コンテキストは`AO[0] > AO[1] > 0`と`Lips > Teeth > Jaw`を必要とします。
   - 弱気コンテキストは`AO[0] < AO[1] < 0`と`Lips < Teeth < Jaw`を必要とします。
3. **注文配置**
   - 条件が一致し、現在の時間が取引可能な場合、戦略は方向ごとに2つのストップスタイルエントリーをキューに入れます：プライマリ注文（2×基本ボリューム）とセカンダリ注文（1×基本ボリューム）。両方ともアリゲーターのリップスを超える最新の適格フラクタルでトリガーします。
   - プライマリテイクプロフィットは`Lips ± (Fractal − Lips) * Fibo1`を使用します。セカンダリテイクプロフィットは`Fibo2`乗数を使用します。
4. **取引管理**
   - リップスが前のキャンドルオープンを超えて上昇（ロングの場合）または下落（ショートの場合）したときのオプションの早期エグジット。
   - 標準偏差、AO、ACがすべて取引方向に進む際に、トレーリングストップが保護レベルを21ピリオドSMMAに引き寄せます。
   - ブレイクイーブンロジックは、プライマリ取引が完了し、価格が設定された追加pipsを移動した後、セカンダリ取引のストップをシフトします。
   - ストップロスとテイクプロフィットレベルの手動監視は、対応する価格境界が違反されたときに成行注文を通じて取引を閉じます。
5. **時間フィルター**
   - 開始/終了時間（折り返しサポート付き）と季節フィルターで定義された取引ウィンドウ：月曜日03:00前、金曜日18:00後、1月の最初の9日間、12月20日以降は無効。`Use Time`をfalseに設定するとフィルターが完全にオフになります。

## パラメーター
| 名前 | 説明 |
| ---- | ----------- |
| `UseTime` | 時間フィルターを有効にします。 |
| `OpenHour` / `CloseHour` | 取引の時間境界（0-23）。 |
| `BaseVolume` | ベース注文ボリューム、2つのステージエントリーのサイジングに使用（プライマリに2×、セカンダリに1×）。 |
| `IndentingPips` | ストップ注文を配置する前にフラクタルレベルから加算/減算されるオフセット（pipsで表現）。 |
| `Fibo1` / `Fibo2` | テイクプロフィット目標のためのリップスとフラクタル間の距離に適用されるフィボナッチスタイルの乗数。 |
| `UseClosePositions` | リップスが前のキャンドルオープンをクロスするときに反対のポジションを閉じます。 |
| `UseTrailing` | MA/オシレーターベースのトレーリングストップを有効にします。 |
| `UseBreakeven` | セカンダリポジションのブレイクイーブン管理を有効にします。 |
| `BreakevenPlusPips` | ストップをブレイクイーブンに移動する際にエントリー価格に追加する追加pips。 |
| `CandleType` | 戦略が処理するキャンドルの時間軸。 |

## 注意事項
- 変換はStockSharpのキャンドルサブスクリプションワークフローを活用しながら、元のMQL5ロボットのステージ注文構造と管理ロジックを維持しています。
- すべての計算は完成したキャンドルに依存します；元のEAのイントラバーティックロジックは、価格レンジがブレイクアウトを確認したら成行注文を通じて反映されます。
- Pip変換は、3桁または5桁の小数点以下で見積もりを持つインストゥルメント（フォレックスのようなシンボル）に自動的に適応します。
