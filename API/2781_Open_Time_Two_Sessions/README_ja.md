# 二つのセッション開始時間戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**二つのセッション開始時間戦略**は、取引日中に 2 つの独立したセッションを管理できる時間スケジュールによるトレードプランを自動化します。各セッションは独自の方向、リスクパラメーター、および任意の強制クローズウィンドウで構成できます。この変換は元の MetaTrader ロジックに従いますが、設定と最適化のために StockSharp の高レベル API、ローソク足、パラメーターオブジェクトを使用します。

## トレードロジック
1. **セッションクローズウィンドウ。** 各インターバルに任意のクローズウィンドウを定義できます。ローソク足の時間がウィンドウ内（開始時間にグローバル期間を加えたもの）に入ると、戦略は対応するインターバルを強制的にクローズし、状態をクリアします。
2. **トレーリングストップの維持。** トレーリングストップとステップが正の場合、トレーリングロジックは完成したローソク足を監視します。価格がポジションに有利な方向に少なくとも `(TrailingStop + TrailingStep)` 動いた後、ストップは `TrailingStop` だけ前進します。ノイズの多い再計算を避けるためにステップ距離が必要です。
3. **ストップロスとテイクプロフィットの確認。** 各インターバルには pips で測定された独立したストップロスとテイクプロフィット距離があります。完成した各ローソク足で、高値/安値がこれらのレベルと比較され、レベルが突破されると即座にインターバルがクローズされます。
4. **曜日フィルター。** 取引は有効になっている曜日のみ進行します。現在のローソク足が無効な日に属する場合、新しい取引は開きません。
5. **オープニングウィンドウ。** 各インターバルには開始時間と終了時間を持つオープニングウィンドウがあります。グローバル期間値はウィンドウを終了側に拡張します。ウィンドウがアクティブでインターバルにオープンポジションがない場合、戦略は設定された方向に成行注文を出します。
6. **ポジション同期。** アクティブなインターバルはターゲットのネットポジションに貢献します。戦略は `BuyMarket` または `SellMarket` を呼び出してネットポジションがインターバルのエクスポージャーの合計と一致するようにします。各インターバルは独自のエントリー価格、ストップ/テイクレベル、トレーリングストップ状態を保持します。

## パラメーター参照
- **Close Window #1 / Close Window #2** – 各インターバルの専用強制クローズウィンドウを有効化または無効化。
- **Close Start #1 / Close Start #2** – 各インターバルのクローズウィンドウが始まるローカル時刻。
- **Trailing Stop / Trailing Step** – トレーリングロジックで使用する pips での距離。トレーリングを有効にするには両方がゼロより大きくなければなりません。
- **Trade Monday … Trade Friday** – 曜日フィルター。取引を許可するには少なくとも 1 日有効のままにしておく必要があります。
- **Open Start #1 / Open End #1 / Open Start #2 / Open End #2** – 各インターバルのオープニングウィンドウの制限。グローバル期間は終了時間を超えてウィンドウを拡張します。
- **Window Duration** – オープニングウィンドウとクローズウィンドウの両方に追加される追加の時間スパン。
- **Direction #1 / Direction #2** – 各インターバルの取引方向フラグ（ロングは `true`、ショートは `false`）。
- **Trade Volume** – 各インターバルの成行注文のボリューム。戦略は元のエキスパートアドバイザーのように両インターバルに同一ボリュームを想定します。
- **Stop Loss #1 / Take Profit #1 / Stop Loss #2 / Take Profit #2** – インターバルごとのストップロスとテイクプロフィットレベルの pips 距離。値がゼロの場合、対応するレベルを無効化します。
- **Candle Type** – 戦略を駆動するために使用するローソク足シリーズ。タイムウィンドウとリスク確認を含むすべての計算は、これらのローソク足が終了したときに実行されます。

## リスク管理の詳細
- pip 距離は証券の価格ステップを使って価格単位に変換されます。銘柄が 3 または 5 桁の小数点を使用する場合、MetaTrader の pip 定義を複製するためにステップは 10 倍されます。
- トレーリングロジックは両方のインターバルで共有されますが、ストップロスとテイクプロフィットの値は独立したままです。
- ストップまたはトレーリングレベルが発動すると、インターバルはその状態をリセットして、時間が許せば同じウィンドウ内で再度オープンできます。

## 制限と注記
- StockSharp はネッティングポジションモデルで動作します。インターバル #1 と #2 が逆方向に設定されている場合、結果のネットポジションは 2 つのヘッジされた取引を同時に開いたままにするのではなく、平坦化されます。真のヘッジが必要な場合は、ヘッジ可能なポートフォリオを使用してください。
- 決定は選択したローソク足シリーズに基づいています。大きな時間軸を使用すると、MetaTrader のティックベースの実装と比較して反応が遅れる可能性があります。
- 戦略は取引日時の比較がローカル時間ベースであるため、取引所とターミナルのクロックが同期していることを期待します。

## 使用のヒント
- スケジュールに使用する時間粒度に合わせてローソク足タイプを構成します（例：きめ細かい制御のための 1 分）。
- 曜日フィルターとクローズウィンドウを組み合わせて、望ましくないセッションをまたいでポジションを持ち越すことを避けます。
- 組み込みの `StrategyParam` オブジェクトを通じてパラメーターを最適化します – 主要フィールドにはすでに `SetCanOptimize` が有効になっています。
