# 新しいFSCEA戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
新しい FSCEA 戦略は、元の MetaTrader 4 エキスパート アドバイザー `new_fscea.mq4` から移植された MACD ベースのトレンド追跡システムです。この戦略は、古典的な MACD クロスオーバー確認と EMA スロープ フィルター、静的なテイクプロフィット ターゲット、オープン ポジションを管理するためのトレーリング ストップを組み合わせたものです。一度に単一のシンボルを取引し、市場でポジションを 1 つだけオープンします。

## 取引ロジック
### ロングエントリー
- MACD メインラインはゼロを下回っていますが、現在閉じているローソク足のシグナルラインを上回っています。
- 前のローソク足にはまだシグナルラインの下に MACD のラインがありました (クロスオーバーを確認します)。
- MACD ラインの絶対値が `OpenLevelPoints` のしきい値 (価格ステップごとに調整) を超えています。
- シフトされた EMA の傾きは正です (`EMA_shifted_now > EMA_shifted_previous`)。
- 現在募集中のポジションはありません。

### ショートエントリー
- MACD メインラインはゼロの上にありますが、現在閉じているローソク足のシグナルラインの下で交差しています。
- 前のローソク足ではまだシグナルラインの上に MACD ラインがありました。
- MACD のメインラインが `OpenLevelPoints` のしきい値を超えています (価格ステップによって調整されます)。
- シフトされた EMA の傾きは負です (`EMA_shifted_now < EMA_shifted_previous`)。
- 現在募集中のポジションはありません。

### 長い出口
- MACD がゼロ以上の状態で信号線を下回り、MACD の値が `CloseLevelPoints` のしきい値を超えるとトリガーされます。
- または、ローソク足の高値が仮想テイクプロフィットレベル（`entry + TakeProfitPoints * priceStep`）に達したとき。
- または、ローソク足の安値がトレーリングストップレベルに達したとき（価格が有利に動くと動的に更新されます）。

### 短い出口
- MACD がゼロ未満のまま信号線の上を横切り、MACD の絶対値が `CloseLevelPoints` のしきい値を超えるとトリガーされます。
- または、ローソク足の安値が仮想テイクプロフィットレベル（`entry - TakeProfitPoints * priceStep`）に達したとき。
- または、ローソク足の高値がトレーリングストップレベルに達したとき（価格が有利に動くと動的に更新されます）。

## リスク管理
- テイクプロフィットは商品ポイントで表され、`Security.PriceStep`を乗じて価格に変換されます。
- トレーリングストップはポイント単位で機能し、変動利益がトレーリング距離よりも大きくなると引き締められます。
- MT4 エキスパートアドバイザーの動作を反映して、いつでもオープンできるポジションは 1 つだけです。
- 位置保護は、組み込みの `StartProtection()` ヘルパーを通じて有効になります。

## インジケーター
- **MACD (12, 26, 9)** – メインのクロスオーバー エンジン。ヒストグラムの大きさは、開始閾値と終了閾値を提供します。
- **EMA (TrendPeriod)** – 終値に適用されます。スロープ比較では、構成可能なシフト (`TrendShift`) を使用して MT4 `ma_shift` パラメーターをエミュレートします。

## パラメーター
| パラメータ | デフォルト | 説明 |
|-----------|---------|-------------|
| `TakeProfitPoints` | 300 | 利益目標までの距離 (ポイント単位)。シンボル価格ステップを使用して価格に変換されます。 |
| `TrailingStopPoints` | 20 | トレーリングストップのサイズ (ポイント単位)。取引がこの距離を超えて有利になった場合にのみ有効になります。 |
| `OpenLevelPoints` | 3 | 新しい取引が許可される前に、最小 MACD マグニチュード (ポイント) が必要です。 |
| `CloseLevelPoints` | 2 | MACD クロスオーバーを介して取引を終了するには、MACD マグニチュード (ポイント) が必要です。 |
| `TrendPeriod` | 10 | EMA トレンド フィルターの長さ。 |
| `TrendShift` | 2 | 傾きを評価するときに EMA に適用される水平方向のシフト (バー単位)。値が大きいほどトレンドの確認が遅れます。 |
| `TradeVolume` | 0.1 | 成行注文とともに送信されるデフォルトの注文量。 |
| `CandleType` | 1時間枠 | インジケーターの計算に使用されるローソク足のタイプ。ご希望の時間枠に合わせて変更可能です。 |

## 実装メモ
- この戦略は、ロジックを MT4 バージョンに近づけるために、完成したキャンドルのみを処理します。
- EMA シフトは、インジケーター出力をバッファリングし、`TrendShift` バー離れた値を比較することによってエミュレートされます。
- トレーリング ストップとテイク プロフィットは、高レベルの API 要件内に収まるように仮想的に実装されます (実際のストップ/指値注文はありません)。
- このコードは、リポジトリ ガイドラインに準拠するために、高レベルのキャンドル サブスクリプション API (`SubscribeCandles().BindEx(...)`) のみに依存しています。
