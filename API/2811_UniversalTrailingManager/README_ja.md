# ユニバーサル・トレーリング・マネージャー戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

**ユニバーサル・トレーリング・マネージャー戦略**は、MetaTraderのエキスパートアドバイザー「Universal 1.64 (barabashkakvn's edition)」をC#に変換したものです。
裁量取引またはセミオートマチック取引の取引管理タスクを自動化し、スケジュールされたエントリー、グリッド形式の未決注文、
成行注文と未決注文のダイナミックなトレーリング、素早い利益のスキャルピング、およびアカウント証拠金が定義されたパーセンテージ動いた際の
ポートフォリオレベルの通知を処理します。

この戦略はローソク足データを提供するあらゆる銘柄で動作するよう設計されています。インジケーターに依存せず、価格レベルと時間ウィンドウに
反応するため、手動シグナルの確認または大規模な取引管理ワークフローへの統合に適しています。

## 主な特徴

- **スケジュールされたアクション**: 特定のターミナル時刻（時/分）に自動的に成行ポジションを開くか未決注文を出します。
- **未決注文グリッド**: 各々独立したオフセット、オプションのトレーリング、および価格が未決注文の有利な方向に動いた際の自動再登録を持つ、
  バイリミット、セルリミット、バイストップ、セルストップ注文を最大1件ずつ維持します。
- **成行ポジション保護**: 現在の集計ポジションにストップロス、テイクプロフィット、トレーリングロジックを適用します。トレーリングが始まる前に
  未実現利益を待つオプションも含みます。
- **スキャルピング退出**: 平均エントリー価格から固定ポイント数だけ価格が進んだ際に既存ポジションをクローズします。
- **ポートフォリオアラート**: ポートフォリオ証拠金を監視し、アカウントが設定されたパーセンテージで増減した際にメッセージを記録します。
- **ポジションゲーティング**: 新しいエントリーや未決注文を受け付ける前に「ポジションがクローズされるまで待機」モードと、方向ごとのオープンポジション数の
  設定可能な制限をサポートします。

## パラメーター

| グループ | パラメーター | 説明 |
|---------|-------------|------|
| 全般 | `TradeVolume` | 成行および未決エントリーに使用するロット単位の注文量。 |
| 全般 | `WaitClose` | `true` の場合、その方向のオープンポジション数が `MaxMarketPositions` を下回る場合にのみ新規注文が許可されます。 |
| 成行 | `MaxMarketPositions` | `WaitClose` が有効な場合の方向ごとの最大アクティブポジション数。 |
| 成行 | `MarketTakeProfitPoints` | オープンポジションに適用されるテイクプロフィット距離（価格ポイント単位）。0で無効化。 |
| 成行 | `MarketStopLossPoints` | オープンポジションに適用されるストップロス距離（価格ポイント単位）。0で無効化。 |
| 成行 | `MarketTrailingStopPoints` | トレーリングストップ距離（価格ポイント単位）。0でトレーリングを無効化。 |
| 成行 | `MarketTrailingStepPoints` | トレーリングストップが動く前に必要な最小改善幅（ポイント単位）。 |
| 成行 | `WaitForProfit` | 有効な場合、利益が `MarketTrailingStopPoints` を超えた後にのみトレーリングが開始されます。 |
| 成行 | `ScalpProfitPoints` | 即時ポジションクローズをトリガーする利益しきい値（ポイント単位）。0でスキャルピングを無効化。 |
| 未決 | `AllowBuyLimit`, `AllowSellLimit`, `AllowBuyStop`, `AllowSellStop` | 各未決注文タイプのマスタースイッチ。 |
| 未決 | `LimitOrderOffsetPoints`, `StopOrderOffsetPoints` | 対応するリミット/ストップ注文を出す現在の終値からの距離。銘柄の最小ストップ距離を超える必要があります。 |
| 未決 | `LimitOrderTakeProfitPoints`, `StopOrderTakeProfitPoints` | 未決注文によって作成された新規オープンポジションに付けられる利益目標（ポイント）。 |
| 未決 | `LimitOrderStopLossPoints`, `StopOrderStopLossPoints` | 未決注文によって作成された新規オープンポジションに付けられる保護ストップ（ポイント）。 |
| 未決 | `LimitOrderTrailingStopPoints`, `StopOrderTrailingStopPoints` | アクティブな未決注文のトレーリング距離。ゼロでトレーリングロジックを無効化。 |
| 未決 | `LimitOrderTrailingStepPoints`, `StopOrderTrailingStepPoints` | トレーリング中に未決注文が動く前に必要な最小改善幅。 |
| 時間 | `UseTime` | スケジュールされたアクションブロックを有効化します。 |
| 時間 | `TimeHour`, `TimeMinute` | スケジュールされたブロックが評価されるターミナル時刻。 |
| 時間 | `TimeBuy`, `TimeSell` | スケジュールされた時刻に成行買い/売りポジションを開く。 |
| 時間 | `TimeBuyLimit`, `TimeSellLimit`, `TimeBuyStop`, `TimeSellStop` | メインの許可スイッチに関係なく、スケジュールされた時刻に対応する未決注文を出す。 |
| グローバル | `UseGlobalLevels` | ポートフォリオレベルの監視を有効化します。 |
| グローバル | `GlobalTakeProfitPercent`, `GlobalStopLossPercent` | 情報ログメッセージをトリガーする証拠金パーセンテージのしきい値。 |
| データ | `CandleType` | 定期処理に使用するローソク足タイプ（デフォルト: 1分）。 |

## 実行フロー

1. **ローソク足の到着**: 各完成したローソク足で戦略は注文参照を更新し、スケジュールされたシグナルを同期し、取引ロジックを評価します。
2. **時間ウィンドウ**: ローソク足のクローズが設定された時間ウィンドウと一致する場合、適切なブール値（`TimeBuy` 等）が設定され、
   成行/未決注文が即座に登録されます。
3. **未決注文**: 戦略はタイプごとに1件の未決注文を出します。価格の動きがトレーリングルールを満たすと、注文はキャンセルされ、
   オフセットを保持したままより市場に近い位置で再発行されます。
4. **成行保護**: オープンポジションに対して戦略は専用のストップロスとテイクプロフィット注文を維持し、トレーリング設定に基づいて調整し、
   ボリュームが集計ポジションと一致することを確認します。
5. **スキャルピングチェック**: `ScalpProfitPoints` が正の場合、現在のクローズ価格が平均ポジション価格からターゲットデルタに達すると
   ポジションはクローズされます。
6. **グローバルアラート**: ポートフォリオ証拠金は毎サイクルチェックされ、しきい値に達すると情報メッセージが記録されます。

## 使用上の注意

- ローソク足が継続的に配信される取引スキーム内に戦略を配置してください（例：1分足ローソク足）。ロジックはローソク足駆動のため、
  より細かい時間軸の方がより応答性の高いトレーリングが得られます。
- 戦略は集計された `Position` プロパティを使用します。ショートからロング（またはその逆）に反転する場合、新しいポジションを開く前に
  既存のポジションをフラットにするために実行された注文サイズが自動的に増加します。
- 未決注文のオフセットとトレーリングステップは *価格ポイント*（`Security.PriceStep` の倍数）で測定されます。銘柄のステップ値が
  正しく設定されていることを確認してください。設定されていない場合、戦略はステップサイズ1にフォールバックします。
- グローバルな損益監視は情報ログメッセージのみを提供します。ポジションを自動的にクローズしません。これはオリジナルのエキスパートアドバイザーの
  動作を反映しています。
- `WaitClose` が有効な場合、サイドごとのオープンポジション数は集計ポジションを `TradeVolume` で割った値から算出されます。
  正確なゲーティング動作を得るために一貫したボリュームサイズを使用してください。

## ログ

すべての重要なアクション（注文の配置、トレーリングの調整、グローバルレベルアラート）は `LogInfo` 経由で戦略ログに書き込まれます。
特にオフセットとトレーリングパラメーターを調整する際は、決定プロセスを追跡するためにログを監視してください。
