# Exp Cronex AO戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、MetaTraderエキスパートアドバイザー**Exp_CronexAO**をStockSharpの高レベルAPIに移植します。元のロボットはCronex Awesome Oscillator（AO）の2本のライン間のクロスを取引します。StockSharpバージョンは設定可能なローソク足シリーズにサブスクライブし、AOを計算し、Cronexラインを再現するために移動平均で2回平滑化し、ファストラインがスローラインをクロスしたときにポジションを開くまたは閉じます。

## トレードロジック

1. 選択したローソク足からAwesome Oscillatorを構築する。
2. 単純移動平均でオシレーターを2回平滑化する。最初の平滑化が「ファスト」Cronexラインを作成し、2回目の平滑化が「シグナル」ラインを生成する。
3. `SignalBar`完成ローソク足をルックバックし、そのバーと前のバーのCronexラインを比較する。
4. ファストラインがスローラインを上回り、ルックバックバーで上方クロスを実行したとき**買い**シグナルが現れる。戦略はオプションでショートポジションを閉じ、許可されていれば成行ロング注文を開く。
5. **売り**シグナルは前のルールを反映する：ファストラインがスローラインを下回り、ルックバックバーで下方クロスを実行した必要がある。戦略はオプションでロングポジションを閉じ、許可されていれば成行ショート注文を開く。
6. インストゥルメントポイントで表現されたストップロスとテイクプロフィットのレベルは、新しい取引が開かれるたびに結果のポジションに添付される。

1つのネットポジションのみが維持される。方向が変わると、戦略は反対ポジションを閉じるのに必要なボリュームと新しい取引ボリュームを組み合わせて、MetaTraderのネッティングモードをエミュレートする。

## パラメーター

| パラメーター | 説明 |
|-----------|-------------|
| `CandleType` | Cronex AO計算に使用するローソク足のデータタイプ。デフォルトは8時間足。 |
| `FastPeriod` | Awesome Oscillatorに適用する第1平滑化の長さ。 |
| `SlowPeriod` | ファストラインに適用する第2平滑化の長さ。 |
| `SignalBar` | クロスシグナルを含む必要がある完成バーの過去参照数。戦略は方向を確認するために次のバーも検査する。 |
| `BuyOpenEnabled` / `SellOpenEnabled` | ロングまたはショートポジションの開設を有効または無効にする。 |
| `BuyCloseEnabled` / `SellCloseEnabled` | 逆シグナルが現れたとき反対ポジションを閉じることができるかどうかを制御する。 |
| `TakeProfit` | ポイント単位の利益ターゲット。ゼロより大きい場合、すべての新規エントリー後に適用される。 |
| `StopLoss` | ポイント単位の保護ストップ。ゼロより大きい場合、すべての新規エントリー後にも適用される。 |

## リスク管理

ストップロスとテイクプロフィットの距離は、MetaTraderバージョンのポイントベース入力を模倣する。新しい取引が送信されるたびに再計算され、保護注文が常に現在のネットポジションサイズと一致するようにする。

## MetaTraderバージョンとの違い

- StockSharp実装は両方のCronex平滑化ステージに単純移動平均を使用する。元のXMA実装はいくつかの平滑化方法を許可しているが、デフォルト設定はここで再現される単純平均に対応する。
- `TradeAlgorithms`ライブラリからのスリッページと資金管理ルーティンは複製されない。ポジションサイジングは標準的な`Volume`プロパティで制御される。
- 取引実行はStockSharpのネッティング動作に依存する。方向が反転すると、ポジションを1ステップでフラット化して転換するのに十分なボリュームで単一の成行注文が発行され、MT5ネッティング口座ロジックを反映する。
