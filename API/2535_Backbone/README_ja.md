# Backbone戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略はStockSharpの高レベルAPIを使用して、元の**Backbone** MQL5エキスパートアドバイザーのコア動作を再現します。強気と弱気のトレードサイクルを交互に繰り返し、リスク割合に従ってポジションをスケールアップし、固定目標とトレーリングストップでオープントレードを保護します。

## 核心的なアイデア

1. **初期方向の検出** – 戦略は起動後に最高値と最安値を追跡します。どちらかの極値からトレーリングストップ距離より大きな動きが、最初にどちら側を取引するかを定義します。
2. **方向サイクル** – サイクルが始まったら、アルゴリズムはすべてのポジションが閉じられるまでその方向にのみ取引します。最後のポジションが決済されると、すぐに反転して反対のサイクルの準備をします。
3. **リスクベースのスケーリング** – 各追加エントリーは、アカウント資本、`MaxRisk`割合、設定された制限`MaxTrades`、ストップロス距離から導出された動的なボリュームを使用します。これは元のEAのロットサイジング関数を模倣します。
4. **保護的な決済** – 各エントリーは現在のサイクルの出来高加重平均価格周辺のストップロスとテイクプロフィットレベルを再計算します。未実現利益が設定されたトレーリング距離を超えると、トレーリングストップが保護ストップを締めます。

## パラメーター

| パラメーター | デフォルト値 | 説明 |
|-----------|---------|-------------|
| `MaxRisk` | 0.5 | 現在の方向のすべてのポジションに利用可能なアカウント資本の割合。 |
| `MaxTrades` | 10 | 方向サイクルごとの連続エントリーの最大数。 |
| `TakeProfitPips` | 170 | エントリー平均とテイクプロフィット目標間の距離（pip単位）。 |
| `StopLossPips` | 40 | エントリー平均と保護ストップ間の距離（pip単位）。 |
| `TrailingStopPips` | 300 | 初期方向の決定と利益のトレーリングの両方に使用される距離（pip単位）。 |
| `CandleType` | 5分の時間軸 | シグナル評価に使用するローソク足タイプ。 |

> **pipの定義** – 戦略はインストゥルメントの`PriceStep`に基づいてpipサイズを自動的に調整します。3桁または5桁の小数で見積もられているシンボルは10×乗数を使用し、元のMetaTraderのpip処理を再現します。

## トレードロジック

1. 完成したローソク足を待ちます。戦略がウォームアップ中またはトレーディングが無効な場合は処理をスキップします。
2. まだ方向が選択されていない間は極値価格を更新します。最高値が上方向に（`TrailingStopPips`より多く）突き抜けると最初のサイクルはショートになり、最安値が下方向に突き抜けると最初のサイクルはロングになります。
3. サイクルがロングの間：
   - （a）前のサイクルがショートでロングポジションが開いていない場合、または（b）前のサイクルもロングで開いているロング数が`MaxTrades`を下回っている場合に、新規ロングエントリーを追加します。
   - テイクプロフィットまたはストップロスに達したとき、またはトレーリングストップが保護レベルを現在のストップより上に引き上げたときにロングサイクル全体を決済します。
4. サイクルがショートの間は、条件を反転させて同じルールが適用されます。
5. サイクルが閉じた後、カウンターをリセットして反対のセットアップを待ちます。

## ポジションサイジング

各新規エントリーのポジションサイズは以下のように計算されます：

```
qty = equity * fraction / (pipSize * stopLoss)
ここで fraction = 1 / (MaxTrades / MaxRisk - openTrades)
```

数量はインストゥルメントのボリュームステップに合わせて調整され、最小/最大ボリューム境界内に制限されます。必要なサイズが許容最小値を下回った場合は最小値が使用されます。資本情報が利用できない場合、デフォルト戦略ボリュームがフォールバックとして機能します。

## 決済管理

- **ストップロス / テイクプロフィット** – 現在のサイクルのすべてのトレードが平均エントリー価格に基づく同じ合算レベルを共有するように、新しい注文が追加されるたびに再計算されます。
- **トレーリングストップ** – ロングサイクルでは、未実現利益がその閾値を超えると、ストップは`Close - TrailingStopPips * pipSize`に移動します。ショートサイドのトレーリングは対称的に処理されます。

## 注意事項と制限

- StockSharpはネッティング環境でトレードを実行するため、各方向サイクルは個別のチケットではなく合算ポジションを管理します。交互のロジックとリスク計算式は、APIモデルに適合させながら元の動作を再現します。
- 戦略は完了したローソク足に依存します。ローソク足の範囲より小さいバー内の動きは評価されません。
- トレードを期待する前に、選択されたローソク足タイプとインストゥルメントが初期極値を構築するのに十分なデータを生成することを確認してください。
