# 階段戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

**階段戦略**は、元の MetaTrader 専門家の行動を再現します。まず、現在の売り値を中心に対称的な逆指値注文を発注し、その後、最新の約定を中心にグリッドを継続的に再構築します。利益は、ソース スクリプトとまったく同様に、出来高による重み付けを行わずに、価格ステップ (pips) で累積されます。利益目標が達成されると、この戦略は成行注文に従ってすべてのポジションを清算し、保留中のストップをすべて削除し、グリッドをリセットします。

## 取引ロジック

1. オープンなポジションがない場合は、現在の売り値の上下 `ChannelSteps / 2` 価格ステップの距離に買いストップと売りストップを配置します。
2. 最初の逆指値注文が約定した後、約定価格付近でグリッドを再準備します。
   - アクティブなストップ注文が 2 つ未満の場合は、古いストップ注文をキャンセルします。
   - 現在の入札価格が最後のエントリーからチャネル距離の半分以内にある限り、最新の約定から離れた新しい買いストップと売りストップ `ChannelSteps` を配置します。
   - `AddLots` が有効な場合、各約定後に未決注文の数量が基本ロット分増加します。
3. MT4 バージョンで使用されるヘッジ バスケットを再現するために、すべてのロングおよびショート エントリを含む 2 つの実行リストを維持します。
4. ロングの最良の買い値とショートの最良の売り値を使用して、完成したすべてのローソク足のバスケットの未実現利益を計算します。距離は、元のポイント計算を反映して、商品価格ステップによって正規化されます。
5. いずれかのしきい値を超えた場合に完全清算をトリガーします。
   - `ProfitSteps` – 現在のシンボルのみによってもたらされる利益。
   - `CommonProfitSteps` – バスケット全体の利益。
6. 清算では、ロングエクスポージャーとショートエクスポージャーを個別に閉じるために成行注文を送信します。保留中のストップ注文は、バスケットがフラットになるとキャンセルされます。

> **注意**: 元のエキスパートは未決注文を登録するときにストップロスレベルを付けました。 StockSharp は、高レベルの API による注文ごとの保護レベルをサポートしていません。そのため、ポートは、上記の利益ベースのロジックのみを通じて取引をクローズします。

## パラメーター

| パラメータ | 説明 | デフォルト |
|-----------|-------------|---------|
| `ChannelSteps` | 対称ストップ注文間の距離 (最小価格ステップ)。 | `1000` |
| `ProfitSteps` | ローカル バスケットを閉じるために必要な利益しきい値 (段階単位)。 | `1500` |
| `CommonProfitSteps` | 完全な清算を強制するグローバル利益のしきい値 (段階的)。 | `1000` |
| `AddLots` | 有効にすると、各約定後に次の未決注文の数量が基本ロット分増加します。 | `true` |
| `BaseVolume` | ストップ注文の最初のペアに使用されるボリューム。 | `0.1m` |
| `CandleType` | キャンドルのサブスクリプションと取引管理に使用される時間枠。 | `1 minute` |

## 実装メモ

- StockSharp の高レベルの API を `SubscribeCandles()` および `Bind()` とともに使用して、完成したキャンドルのみを処理します。
- `OnOwnTradeReceived` 内の個々のエントリを追跡することで、利益計算で MQL バージョンのヘッジ ロジックを模倣できるようになります。
- 利益しきい値は、MT4 エキスパートがピップを合計する方法と一致し、実行量を乗算せず、純粋な価格ステップ距離に基づいて動作します。
- すべての逆指値注文は `BuyStop` と `SellStop` を通じて作成されますが、エグジットは成行注文で実行され、データ プロバイダー間でのロジックの移植性を維持します。
