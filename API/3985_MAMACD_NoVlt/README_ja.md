# MMACD ノーボラティリティ戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
MMACD ノー ボラティリティは、MetaTrader 4 エキスパート アドバイザー `MAMACD_novlt.mq4` の直接ポートです。この戦略は、ローソク足の安値で計算された 3 つの移動平均を組み合わせ、MACD モメンタム フィルターで終了します。高速の EMA が 2 つの低ベース LWMA フィルターを下回る (ロングの場合) か上に上昇する (ショートの場合) まで待機し、保留中のセットアップを準備し、MACD メインラインが勢いの変化を確認した後にのみエントリーをトリガーします。

## インジケーター
- **高速 EMA** (`FastEmaPeriod`) は終値に基づいて計算されます。
- **最初の LWMA** (`FirstLowWmaPeriod`) は低価格で計算されています。
- **2 番目の LWMA** (`SecondLowWmaPeriod`) は低価格に基づいて計算されます。
- **MACD のメインライン**。高速期間 `FastSignalEmaPeriod` と低速期間 `SlowEmaPeriod` があります。

すべてのインジケーターは、`CandleType` で定義された時間枠で動作します (デフォルト: 5 分足ローソク足)。

## パラメーター
| パラメータ | 説明 | デフォルト |
| --- | --- | --- |
| `FirstLowWmaPeriod` | ローソク足安値から構築された最初の LWMA の期間。 | 85 |
| `SecondLowWmaPeriod` | ローソク足の安値から構築された 2 番目の LWMA の期間。 | 75 |
| `FastEmaPeriod` | ローソクから構築された高速な EMA の期間が終了します。 | 5 |
| `SlowEmaPeriod` | MACD の計算にかかる EMA 期間が遅いです。 | 26 |
| `FastSignalEmaPeriod` | MACD の計算にかかる高速な EMA 期間。 | 15 |
| `StopLossPoints` | 価格ステップのストップロス距離 (0 はストップロスを無効にします)。 | 15 |
| `TakeProfitPoints` | 価格ステップでの利食い距離 (0 は利食いを無効にします)。 | 15 |
| `TradeVolume` | 市場エントリーに使用される注文量。 | 0.1 |
| `CandleType` | すべてのインジケーターに使用されているキャンドルシリーズ。 | 5分間の時間枠 |

## 取引ルール
1. **アームロングセットアップ**: 高速 EMA は両方の LWMA フィルターを下回ります。
2. **アームショートセットアップ**: 高速 EMA は両方の LWMA フィルターを上回ります。
3. **長めに入力してください**:
   - 高速の EMA が両方の LWMA を上回ってクロスバックします。
   - 長いセットアップは以前に装備されていましたが、
   - MACD のメインラインがプラスであるか、前の値と比較して増加しています。
   - 現在のネットポジションは長くありません。
4. **短く入力してください**:
   - 高速の EMA が両方の LWMA を下回ってクロスバックします。
   - 短いセットアップは以前に装備されていましたが、
   - MACD メインラインがマイナスであるか、前の値と比較して減少しています。
   - 現在のネットポジションはショートではありません。
5. **リスク管理**: オプションのテイクプロフィットとストップロスは、内蔵の保護サービスを通じて自動的に適用されます。

この戦略は専用の終了信号を実装しません。ポジションは、設定されたストップロス/テイクプロフィットレベルまたは手動介入によって管理されます。

## 注意事項
- MACD 確認は、MQL ロジックを複製します。メインラインは、ゼロより上か上昇（ロングの場合）、またはゼロより下か下降（ショートの場合）のいずれかである必要があります。
- LWMA の計算では、元のインジケーター構成を反映するためにローソク足の安値を使用します。
- ボリューム スケーリングは、注文ごとに `TradeVolume` パラメータを使用することで、元の EA を反映します。
