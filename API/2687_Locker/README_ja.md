# Locker 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

グリッドベースのヘッジ戦略で、ロングとショートの成行注文を交互に発注して浮動損失をロックし、口座残高の小さな割合の利益を獲得します。

## 取引ロジック
* 最初のローソク足が閉じると同時に、設定された開始ボリュームで最初のロングポジションを建てます。
* それ以降の各エントリーを追跡し、買いレグと売りレグの内部台帳を維持して、合算された未実現・実現損益を推定します。
* アクティブなレグ数が8に達した場合、そのローソク足で他の処理を行う前に、最も古い利用可能な買い/売りペアを決済してポジションをコントロール下に置きます。
* 合算損益がポートフォリオ価値の目標パーセンテージを上回った場合、残りのすべてのポジションを決済して内部状態をリセットします。
* 合算損益が負の目標値を下回った場合、最後のエントリー価格と現在の市場価格の距離を測定します。価格が設定されたステップだけ上昇した場合は新しいショートレグを追加し、同じ距離だけ下落した場合は新しいロングレグを追加します。
* 各決済は記録されたエントリーの反対方向への成行注文を使用して、ヘッジを即座に解除します。

## パラメーター
* **Profit %** – 帳簿を決済する前にロックインすべき現在のポートフォリオ価値のパーセンテージ。
* **Start Volume** – グリッドを開始する最初のロングエントリーに使用する数量。
* **Step Volume** – 損失閾値を超えた後の各ヘッジ注文に提出する数量。
* **Step Points** – グリッドレベル間の価格ステップ数。実際の価格距離を計算するために銘柄の価格ステップで乗算されます。
* **Enable Automation** – 無効化すると全取引ロジックを一時停止するマスタースイッチ。
* **Candle Type** – 各完成バーで決済ロジックをトリガーするために使用するローソク足シリーズ。

この変換は元のMetaTraderエキスパートロジックを複製しながら、StockSharpの高レベルAPIへの注文配置を適応し、損益計算がMQLバージョンと一致するように戦略内に詳細な取引状態を保存します。
