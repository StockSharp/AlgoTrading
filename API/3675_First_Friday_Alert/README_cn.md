# 首个星期五提醒策略

## 概述
**首个星期五提醒策略** 是 MetaTrader 4 专家顾问 `FirstFriday.mq4` 的直接移植版本。原始算法会持续监听日线蜡烛图，当检测到某个月的第一个星期五时，在平台日志中打印提示信息。StockSharp 版本通过订阅日线蜡烛并在满足条件时输出信息日志，从而忠实地复现了这一行为。

## 交易逻辑
1. 为所选证券订阅日线蜡烛数据。
2. 保存最近一次处理的蜡烛开盘时间，以避免在历史回放或重复推送时触发重复处理。
3. 对于每根已完成的蜡烛，判断其开盘时间是否为星期五，并且日历日期是否位于 1 至 7 号之间。
4. 一旦条件成立，使用 `AddInfoLog` 输出包含该日期的提醒信息。

该策略不会发送任何交易指令，只会在日志中报告事件，与原始 EA 仅在 MetaTrader 日志里打印消息的行为完全一致。

## 参数
| 名称 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `CandleType` | `DataType` | 日线时间框架（`TimeSpan.FromDays(1).TimeFrame()`） | 指定用于检测首个星期五的蜡烛类型。如需使用自定义日线或其他周期，可调整该参数。 |

## 实现说明
- 采用高级 StockSharp API（`SubscribeCandles().Bind(...)`）来接收蜡烛更新。
- 使用私有字段记录最近处理过的蜡烛时间，防止重复提醒。
- 按项目要求，代码中的注释全部使用英文。
- 重写 `GetWorkingSecurities()`，明确策略需要的证券与蜡烛数据类型。

## 使用方式
1. 将策略附加到任意提供日线数据的证券上。
2. 启动策略。当新的月份出现首个星期五的日线蜡烛时，日志中会出现类似 `Detected the first Friday daily candle for Friday, 3 March 2023.` 的提示信息。
3. 可以将日志输出作为人工流程的提醒，或作为其他自动化流程的输入信号。

由于策略不下单，因此在模拟或实盘环境中运行都不会产生任何交易风险。
