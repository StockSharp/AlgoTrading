# アプリ価格レベルのクロス戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- MetaTrader 4 エキスパートアドバイザー **BT_v4** (`MQL/8543/BT_v4.mq4`) の変換。
- 高レベルの StockSharp 戦略 API (キャンドル サブスクリプション、インジケーターなしの処理、組み込みの保護) を使用して再実装されました。
- ユーザー定義の水平レベル (`AppPrice`) を超える終値に反応することに重点を置きます。

## 取引ロジック
1. 完成した各ローソク足は内部バッファを最新の終値で更新します。
2. 前回の終値がそのレベル**以下**であったにもかかわらず、終値が**{PH000}}を**上回った場合、戦略は
   - `BuyOnly = true` の場合にのみ取引します (元の EA のデフォルトを反映します)。
   - 未決注文をキャンセルし、同じ成行注文量を介して既存のショートを相殺し、計算されたロットサイズのロングポジションを確立します。
3. 前回の終値がそのレベル**以上**であったにもかかわらず、終値が** `AppPrice`を下回った場合、戦略は
   - `BuyOnly = false`の場合のみ取引します（EAの売り専用モード）。
   - 未決注文をキャンセルし、既存のロングポジションを相殺し、計算されたロットサイズのショートポジションを確立します。
4. シグナルは完成したキャンドルに対して厳密に評価されます。部分的に形成されたキャンドルは、MQL スクリプトと同様に無視されます。

## ポジションサイズ
- `EnableMoneyManagement = false` → `FixedVolume` を使用します（MQL `Lots` 入力と同等）。
- `EnableMoneyManagement = true` → 元の式を使用してロットを計算します。

\[
\text{ロット} = \text{round}_{\text{LotPrecision}} \left( \frac{\text{LotBalancePercent}}{100} \times \frac{\text{Balance}}{\text{divisor}} \right)
\]

  - 10 進数 1 のロットの場合は `divisor = 1000`、10 進数 2 のロットの場合は `100` (MQL の `LotPrec` と同じルール)。
  - 結果は [`MinLot`、`MaxLot`] にクランプされ、セキュリティ制約 `VolumeStep`、`VolumeMin`、および `VolumeMax` に合わせて調整されます。
  - ポートフォリオ残高データが利用できない場合、戦略は `FixedVolume` に戻ります。

## リスク管理
- `StopLossPoints` と `TakeProfitPoints` は商品価格ポイント (ティック) で測定されます。
- いずれかの値が正の場合、`Security.PriceStep` を通じて変換されたオフセットを使用して `StartProtection` がアクティブになります。
- 距離を `0` に設定すると、元の EA の動作と一致して、その特定の保護脚が無効になります。

## パラメーター
| 名前 | 説明 | デフォルト |
| ---- | ----------- | ------- |
| `AppPrice` | 終値がそれを横切ったときに取引をトリガーするレベル。 | `0` |
| `BuyOnly` | `true` = ロングのみモード（元のデフォルト）、`false` = ショートのみ。 | `true` |
| `FixedVolume` | MM が無効な場合のロット サイズ。 | `0.1` |
| `EnableMoneyManagement` | バランスパーセンテージのサイジングを有効にします。 | `false` |
| `LotBalancePercent` | MM がオンのときに使用される残高のパーセント。 | `10` |
| `MinLot` / `MaxLot` | 計算されたロットサイズの境界。 | `0.1` / `5` |
| `LotPrecision` | 計算されたロットを四捨五入する小数点以下の桁数。 | `1` |
| `StopLossPoints` | 価格ポイントでのストップロス距離 (0 = 無効)。 | `140` |
| `TakeProfitPoints` | 価格ポイントでの利食い距離 (0 = 無効)。 | `180` |
| `CandleType` | クロス検出に使用されるローソク足のタイムフレーム。 | `1 Minute` |

## 実装メモ
- `SubscribeCandles(...).Bind(...)` を使用するため、インジケーターは不要です。終値はコールバックで直接到着します。
- 成行注文 (`BuyMarket`/`SellMarket`) は、新しい注文を開く前に反対のポジションをフラットにするようにサイズ設定されており、エントリーする前に反対の注文を閉じるという EA ロジックを反映しています。
- `CancelActiveOrders()` は、意図しない未決注文を避けるために、すべての成行注文の前に呼び出されます。
- `Magic`、`Slippage` などのパラメータ、および MQL ファイルの色設定は、StockSharp には直接同等のものがないため省略されます。
- 価格/数量調整がブローカーのルールと一致するように、`Security` メタデータ (`PriceStep`、`VolumeStep`、`VolumeMin`、`VolumeMax`) が入力されていることを確認します。

## 使用のヒント
- `AppPrice` を監視したい水平レベル (心理的価格、毎日のピボットなど) に設定します。
- 元の「販売専用」モードを複製するには、`BuyOnly` をオフにします。オンのままにすると、提供されているデフォルトのロングのみの動作が実行されます。
- 資金管理を有効にする場合は、ポートフォリオ接続が残高更新を提供していることを確認してください。それ以外の場合、戦略は固定ボリュームに戻ります。
- リクエストごとに Python ポートは提供されません。 C# ストラテジーのみが生成されます。
