# Clouds Trade 2 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略はVladimir Karputovの「cloud's trade 2」エキスパートアドバイザーのC#版である。2つの最近のビル・ウィリアムズのフラクタルとストキャスティクス・オシレーターの買われすぎ/売られすぎのクロスオーバーによって確認されたブレイクアウトを取引する。取引管理は設定可能なストップロス、テイクプロフィット、トレーリングストップ、最小利益ロックでオリジナルのインプットを反映する。

## トレーディングロジック

- **データ**: 単一時間軸のローソク足（デフォルト15分）。
- **インジケーター**:
  - 設定された %K ルックバック、スローイング、%D スムージングを使用するストキャスティクス・オシレーター。
  - 上部および下部フラクタルを再構築するための5本のローソク足高値/安値のローリングウィンドウ。
- **エントリー**:
  - **ロング**: 上部フラクタルよりも最近2つの連続した下部フラクタルが現れる、**または**ストキャスティクス %D が %K を下向きにクロスしながら20を下回る。ポジションが開いていてはならず、オプションの1日1取引フィルターが新規エントリーを許可しなければならない。
  - **ショート**: 2つの連続した上部フラクタルが先に現れる、**または**ストキャスティクス %D が %K を上向きにクロスしながら80を上回る。
- **エグジットと保護**:
  - エントリー価格からの静的なストップロスとテイクプロフィットのオフセット。
  - 現在の利益が設定されたトレーリング距離プラスステップを超えた場合にのみ移動するオプションのトレーリングストップ。
  - 金銭ベースまたは価格距離ベースの利益目標のいずれかが達成されると、ポジションを決済する。
  - ストップはローソク足の高値/安値を検査することでエミュレートされ、MQL バージョンのブローカー管理動作に合わせている。

## パラメーター

- **Order Volume**: エントリーのベース注文サイズ。
- **Stop/Take Offsets**: 絶対価格距離；オリジナルの pip ベースのインプットを再現するために銘柄のティック値に合わせて調整する。
- **Trailing Stop & Step**: ストップがいつ移動するかを制御する価格単位のオフセット。
- **Min Profit (Currency / Points)**: 未実現利益がこれらの閾値を超えると取引を決済する。
- **Use Fractals / Use Stochastic**: いずれかのシグナルコンポーネントを独立して有効にする。
- **One Trade Per Day**: 同じ取引日に複数のエントリーを防ぐ。
- **Stochastic Settings**: %K ルックバック、%K スローイング、%D スムージングの長さ。
- **Candle Type**: 戦略のローソク足サブスクリプションの時間軸。

## 注意事項

- ポジション利益チェックは、価格変動×ポジションサイズを使用してオリジナルの手数料/スワップ調整を近似する。
- トレーリングロジックは、ストップを移動する前に利益がトレーリング距離プラスステップを超えることを要求することで MQL 実装に従う。
- Forex ペアでデフォルトの MQL pip ベースのインプットを模倣するには、stop/take オフセットを希望する pip 値×銘柄のポイント値に設定する（例：5桁の相場では50pips ≈ 0.005）。
