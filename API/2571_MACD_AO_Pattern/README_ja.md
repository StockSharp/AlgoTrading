# MACD AO パターン戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、FORTRADERの`MACD.mq5`エキスパートアドバイザーを忠実にStockSharpへ移植したものです。MACDオシレーターがゼロラインから大きく乖離した後、中立方向へフックバックする「AOP」パターンを実装しています。フックが確認されると、予想される反転方向にエントリーし、pips単位の固定ストップロスとテイクプロフィットを即座に適用します。

## 戦略ロジック
### データ準備
- `CandleType`パラメーターで選択されたローソク足シリーズ（デフォルト：5分足）で動作します。
- 設定可能な速足、遅足、シグナル期間の標準MACDインジケーターを使用します（デフォルト：12/26/9）。
- MQLのインデックスベースアクセス（`iMACD(...,1..3)`）を再現するため、直近3本の完成したローソク足のMACD主線値を保存します。

### ショート設定（弱気フック）
1. **アーミング** – 最後に閉じたローソク足のMACD主線が`BearishExtremeLevel`（デフォルト−0.0015）を下回ると、戦略は反転の監視を開始します。
2. **中立プルバック** – MACDが`BearishNeutralLevel`（デフォルト−0.0005）を上回ると、フック検証ステージが有効になります。
3. **フック確認** – 直前の3つのMACD値がローカル最大値（`macd₁ < macd₂ > macd₃`）を形成し、最新値が中立レベルを下回り、古い値が上回っている必要があります。これにより、モメンタムが衰退していることを確認する元のパターンが再現されます。
4. **エントリー** – ロングポジションが開いていない場合（`Position <= 0`）、`OrderVolume`の成行売り注文が送信されます。保護レベルが即座に計算されます：エントリーの`StopLossPips`上にストップロス、`TakeProfitPips`下にテイクプロフィット（`GetPipSize`で価格に変換）。
5. MACDの正の読みはセットアップをキャンセルし、新しい深い負の動きが現れるまで内部の弱気ステートマシンをリセットします。

### ロング設定（強気フック）
1. **アーミング** – MACDが`BullishExtremeLevel`（デフォルト+0.0015）を上回ると、強気監視モードが有効になります。
2. **即時キャンセル** – MACDがゼロを下回ると、MQLのロジックを反映して強気シナリオが放棄されます。
3. **中立プルバック** – `BullishNeutralLevel`（デフォルト+0.0005）を下回る下落でフック確認の準備が整います。
4. **フック確認** – 3つの保存されたMACD値が中立しきい値を尊重しながらローカル最小値（`macd₁ > macd₂ < macd₃`）を作る必要があります。
5. **エントリー** – ショートのエクスポージャーがない場合（`Position >= 0`）、戦略は`OrderVolume`で成行買いし、ショートルールと対称的にエントリー周辺にストップロスとテイクプロフィットを設定します。

### リスク管理
- ストップロスとテイクプロフィットは常に`_stopPrice`と`_takePrice`で有効です。元のEAのブローカー側の執行をエミュレートするため、記録された高値/安値を使用して完成したローソク足ごとに評価されます。
- Pipsは`Security.PriceStep`を使用して絶対価格に変換されます。3桁と5桁のFXシンボルの場合、分数pipsのMQL調整に合わせてステップに10が乗算されます。
- 保護レベルによりポジションを終了すると、即座にクリアして次のローソク足での新しいセットアップを待ちます。

## パラメーター
| パラメーター | 説明 | デフォルト値 |
|-------------|------|------------|
| `CandleType` | 戦略が処理するローソク足データシリーズ。 | 5分時間軸 |
| `OrderVolume` | 各成行注文で送信する出来高。 | 0.1 |
| `TakeProfitPips` | pipsでの利益目標までの距離。最適化対象。 | 60 |
| `StopLossPips` | pipsでのストップロスまでの距離。最適化対象。 | 70 |
| `MacdFastPeriod` | MACDの速EMA長。 | 12 |
| `MacdSlowPeriod` | MACDの遅EMA長。 | 26 |
| `MacdSignalPeriod` | MACDのシグナルEMA長。 | 9 |
| `BearishExtremeLevel` | ショート機会をアームする負のMACDしきい値。 | −0.0015 |
| `BearishNeutralLevel` | 弱気フックの検証に使用する負のMACDしきい値。 | −0.0005 |
| `BullishExtremeLevel` | ロング機会をアームする正のMACDしきい値。 | +0.0015 |
| `BullishNeutralLevel` | 強気フックの検証に使用する正のMACDしきい値。 | +0.0005 |

## 追加メモ
- 戦略はMQLの元の`PrevBars`ガードを模倣して、完成したローソク足ごとに1回のみ反応します。
- ストップロス/テイクプロフィット管理は純粋に価格ベースです。ステートマシンの完全なサイクルが再び完了するまで、トレーリング調整や再エントリーはありません。
- ソースEAはヘッジアカウント向けに設計されていますが、この移植版は新規注文送信前に`Position`をチェックして単一ネットポジションを強制します。
- 要望通りPythonバージョンは提供されていません。
