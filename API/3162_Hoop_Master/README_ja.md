# Hoop Master戦略

## 概要

Hoop Master戦略は、現在の価格の周りに2つのストップ注文を継続的に保持するペンディングブレイクアウトシステムです。元のMetaTrader 5エキスパートアドバイザーは市場の上にバイストップを、市場の下にセルストップを配置します。一方のサイドが発動すると、反対側の注文がキャンセルされ、両サイドはより大きなボリュームで再作成されます。StockSharpポートは単一の戦略クラス内でストップ注文とオプションのマーチンゲールサイジングを管理することで同じアイデアを踏襲しています。

戦略はまた、開いているポジションに保護的なストップロスとテイクプロフィット注文を付加することもできます。トレーリングストップモジュールは、市場がトレード方向に進むにつれて保護的なストップを徐々に移動させます。

## トレードロジック

1. 各完成したローソク足で、戦略はブレイクアウトストップの配置レベルを再計算します。
2. ポジションが開かれていない場合、バイストップとセルストップの両方が現在のbid/askから設定可能なpips距離に登録されます。
3. ペンディングストップのどちらかが約定すると、反対側のストップが削除されます。新しいブレイクアウトストップはベースボリュームの2倍を使用して即座に送信されます。
4. トレードが開かれた後、戦略は独立したストップロスとテイクプロフィット注文を作成します。移動が十分に大きくなると、トレーリングエンジンがストップを価格に向けて移動できます。
5. ポジションが決済されると、すべての保護注文がキャンセルされ、ブレイクアウト注文は次のシグナルでベースボリュームで再初期化されます。

## パラメーター

| パラメーター | 説明 |
| --- | --- |
| **Candle Type** | バーごとのロジックに使用されるローソク足データタイプ。 |
| **Order Volume** | 各ブレイクアウト注文のベースボリューム。マーチンゲールステップはこの量の2倍を使用します。 |
| **Stop Loss (pips)** | エントリー価格と保護的なストップ注文の間のpips距離。無効にするには0に設定。 |
| **Take Profit (pips)** | エントリー価格と保護的な目標注文の間のpips距離。無効にするには0に設定。 |
| **Trailing Stop (pips)** | トレーリングストップを移動するときに使用される距離。トレーリングを無効にするには0に設定。 |
| **Trailing Step (pips)** | トレーリングストップが更新される前に必要な最小価格改善（pips単位）。 |
| **Indent (pips)** | ブレイクアウトストップを配置するときにaskの上とbidの下に追加されるpips単位のオフセット。 |

## 注文管理の詳細

- 戦略は最良のbid/ask見積もりを継続的に追跡します。見積もりが利用できない場合は、最後の取引価格またはローソク足クローズにフォールバックします。
- すべての注文は無効な価格を避けるためにインストゥルメントの価格ステップに揃えられます。
- 保護的なストップとテイクプロフィット注文は新しいポジションが現れるたびに置き換えられます。
- トレーリングはトレーリング距離とステップパラメーターの両方がゼロより大きい場合にのみ機能します。望ましい改善が十分に大きいとき、ストップはトレード方向に移動します。

## 注意事項

- 接続されたブローカーまたはシミュレーターが選択したインストゥルメントのストップおよびリミット注文をサポートしていることを確認してください。
- マーチンゲールステップはエクスポージャーを急速に増加させることがあります。許容可能なリスク制限内に留まるようにベースボリュームを調整してください。
- 戦略はブレイクアウト価格を正確に計算できるように、ローソク足データとともにレベル1データ（bid/ask）を受信することを期待しています。
