# MAMy Expert 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- Victor ChebotariovによるMetaTrader 5の"MAMy Expert"アドバイザーをStockSharpのハイレベル戦略APIにポーティングしたもの。
- 異なる価格ソース（始値、終値、加重価格）の3つの移動平均を比較する元のカスタムインジケーターを再現します。
- 完了したローソク足でのみ動作し、一度に最大1つのネットポジションを管理し、MQLエキスパートの動作を反映します。

## インジケーターの基礎
- 戦略は同じ長さと平滑化アルゴリズムを使用して3つの移動平均を構築します：
  - `MA(close)` – ローソク足の終値で計算されます。
  - `MA(open)` – ローソク足の始値で計算されます。
  - `MA(weighted)` – 加重価格 `(High + Low + 2 × Close) / 4` で計算されます。
- `MaType` パラメーターは3つのシリーズすべての平均化アルゴリズム（Simple、Exponential、Smoothed、またはWeighted LWMA）を選択し、MetaTraderの `MODE_*` オプションに対応します。
- "クローズバッファー"は差分 `MA(close) − MA(weighted)` として計算されます。
- "オープンバッファー"は移動平均がトレンド構成で整列した場合にのみ生成されます：
  - **下降トレンドセットアップ**：`MA(close)` と `MA(weighted)` の両方が下落し、終値MAが加重MAを下回り、両方が始値MAを下回り、クローズバッファーが減少します。
  - **上昇トレンドセットアップ**：`MA(close)` と `MA(weighted)` の両方が上昇し、終値MAが加重MAを上回り、両方が始値MAを上回り、クローズバッファーが増加します。
  - どちらかのセットアップが真の場合、オープンバッファーは `(MA(weighted) − MA(open)) + (MA(close) − MA(weighted))` になります；そうでなければゼロにリセットされます。
- 新しい正のオープンバッファーがクローズバッファーのネガティブクロスを伴う場合、元のインジケーターコードと同様にクローズバッファーはゼロに強制されます。

## シグナルロジック
- **エントリー条件**
  - **買い**：オープンバッファーがゼロを上向きにクロスした場合（`前回 ≤ 0`、`現在 > 0`）。
  - **売り**：オープンバッファーがゼロを下向きにクロスした場合（`前回 ≥ 0`、`現在 < 0`）。
  - エントリーは既存のポジションがない場合にのみ考慮されます。
- **エグジット条件**
  - **ロング決済**：クローズバッファーがゼロを下向きにクロスした場合（`前回 ≥ 0`、`現在 < 0`）。
  - **ショート決済**：クローズバッファーがゼロを上向きにクロスした場合（`前回 ≤ 0`、`現在 > 0`）。
  - エグジットは新しいエントリーの前に評価されるため、戦略はロングとショートのエクスポージャーを同時に保持することはありません。
- 注文は設定された `TradeVolume` を使用して市場で発行されます。`StartProtection()` による保護自動化はStockSharpのサンプルの安全呼び出しを反映します。

## チャートとデータフロー
- `CandleType` で定義された時間軸にサブスクライブし、完了したローソク足のみを処理します。
- 3つの移動平均とともに価格ローソク足を描画し、実行された注文を注釈して、MetaTraderで元のインジケーターが提供していたのと同じビジュアルキューを提供します。

## パラメーター
| 名前 | タイプ | デフォルト | 説明 |
| --- | --- | --- | --- |
| `CandleType` | `DataType` | `TimeSpan.FromHours(1).TimeFrame()` | インジケーターとシグナルにローソク足を提供するプライマリ時間軸。 |
| `MaPeriod` | `int` | `3` | 3つの移動平均すべてに適用される長さ。 |
| `MaType` | `MaCalculationType` | `Weighted` | 平均化アルゴリズム（Simple、Exponential、Smoothed、Weighted）。 |
| `TradeVolume` | `decimal` | `1` | 各市場注文エントリーに使用されるボリューム。 |

## 実装上の注意
- StockSharpのハイレベル `SubscribeCandles().Bind(...)` ワークフローと組み込み移動平均インジケーターを使用します；シグナル検出に必要な最新の値を超えたカスタムバッファーは保存されません。
- すべてのインジケーターが完全に形成され、戦略がライブトレーディングの準備ができた後にのみシグナルが評価されます（`IsFormedAndOnlineAndAllowTrading()`）。
- 戦略はポジションが開いている間の同時エントリーを意図的に無視し、ソースエキスパートアドバイザーのロジックを忠実に一致させます。
