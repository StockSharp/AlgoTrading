# Exp XBullsBearsEyes Vol 直接戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は、MetaTraderエキスパート **Exp_XBullsBearsEyes_Vol_Direct** をC#に変換したものです。Bulls PowerとBears Powerから
構築されたカスタムオシレーターを再現し、設定可能な出来高ソースで乗算し、元のインジケーターと同一の適応的平滑化を適用します。
取引の意思決定はインジケーターの方向バッファのみによって行われます：アルゴリズムはレベル交差ではなくモメンタムの変動に反応し、
平滑化されたヒストグラムの傾きが変化したときにポジションを開いたり閉じたりします。

多くの変換とは異なり、StockSharpバージョンはMQLコードで使用されていた出来高加重ステージと四レベルのガンマフィルターを
維持しています。オシレーターは同じ移動平均法で2回平滑化されます — ヒストグラム自体に1回、出来高ストリームに1回 — そのため
シグナルは両コンポーネントが完全に形成されたときのみ現れます。この戦略は完成したローソク足のみを処理し、ティック出来高または
実際に取引された出来高をサポートするため、異なる市場間で移植可能です。

## インジケーターロジック
1. `Period`本のローソク足にわたる終値の指数移動平均を使用してBulls PowerとBears Powerを計算する。
2. 元の四段階ガンマフィルター（パラメーター `Gamma`、`L0`–`L3`）を適用して、2つの力を-50から+50の正規化されたヒストグラムに結合する。
3. ヒストグラムを選択した出来高ソース（ティック数または取引出来高）で乗算する。
4. 同じ移動平均ファミリー（`Method`、`SmoothingLength`、`SmoothingPhase`）でヒストグラムと生の出来高を平滑化する。
5. 方向バッファを導出する：平滑化されたヒストグラムが上昇するとき色 `0`、下降するとき色 `1`。これはMetaTrader実装の
   `ColorDirectBuffer` を模倣している。

インジケーターの上位/下位閾値バッファは内部で計算されますが、方向転換のみに依存していた元のエキスパートアドバイザーの
動作に合わせて、取引フィルターには使用されません。

## 取引ルール
- **ショートをクローズ**：前のバーの方向が強気だったとき（`olderColor = 0`）。
- **ロングを開く**：ロングエントリーが許可されており、強気のバーに弱気のバーが続き（`currentColor = 1`）、戦略がすでにロングでない場合。
- **ロングをクローズ**：前のバーの方向が弱気だったとき（`olderColor = 1`）。
- **ショートを開く**：ショートエントリーが許可されており、弱気のバーに強気のバーが続き（`currentColor = 0`）、ロングポジションがアクティブでない場合。
- ポジションの反転は、まず反対側をクローズし、次に設定された `OrderVolume` で成行注文を送信します。

シグナルは設定可能なバーシフト（`SignalBar`）で評価されます。デフォルト値の `1` は、方向変化に反応する前に完全に閉じたローソク足を
待っていたMQLエキスパートをエミュレートします。

## パラメーター
| 名前 | 説明 |
|------|------|
| `CandleType` | 戦略がサブスクライブするローソク足タイプ/時間軸（デフォルト：2時間ローソク足）。 |
| `Period` | Bulls/Bears Powerに使用するルックバック期間。 |
| `Gamma` | 適応ガンマフィルターの平滑化係数（0…1）。 |
| `VolumeMode` | 出来高ソース：ティック数または取引出来高。 |
| `Method` | ヒストグラムと出来高を平滑化するために使用する移動平均ファミリー（SMA、EMA、SMMA、LWMA、Jurik；サポートされていないレガシータイプはSMAにフォールバック）。 |
| `SmoothingLength` | 両方の平滑化ステージの長さ。 |
| `SmoothingPhase` | Jurikフェーズパラメーター（互換性のために保持）。 |
| `SignalBar` | 方向バッファを評価する際に読み取るバー数。 |
| `AllowBuyOpen` / `AllowSellOpen` | ロング/ショートポジションのオープンを有効または無効にする。 |
| `AllowBuyClose` / `AllowSellClose` | 反対シグナルでの強制エグジットを有効または無効にする。 |
| `OrderVolume` | 新規エントリーの成行注文サイズ。 |
| `StopLossPoints` | 価格ステップでのオプションの保護ストップ（0はストップを無効にする）。 |
| `TakeProfitPoints` | 価格ステップでのオプションの保護ターゲット（0はターゲットを無効にする）。 |

## 使用上の注意
- この戦略は `GetWorkingSecurities()` によって返された単一の銘柄で動作し、安定した出来高ストリームを提供するシンボルで最も
  機能します。
- 実際の取引出来高が利用できないスポットFXシンボルにはティック出来高を推奨します。実行された出来高を公開する取引所には
  `VolumeMode` を `Real` に設定してください。
- ストップロスとテイクプロフィットの距離は価格ステップで表現され、銘柄の `PriceStep` を使用して絶対価格単位に変換されます。
- ロジックが方向転換に依存しているため、連続して等しいヒストグラム値は新しい傾きが現れるまで前の方向を維持します — MetaTrader
  バージョンと全く同じです。
- チャート出力はデフォルトで価格ローソク足のみをプロットします。視覚的な確認が必要な場合は、ヒストグラムのカスタムプロットを
  追加できます。
