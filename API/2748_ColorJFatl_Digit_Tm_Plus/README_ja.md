# ColorJFatl Digit TM Plus ストラテジー
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

ColorJFatl Digit TM Plus ストラテジーは、MetaTrader 5のエキスパートアドバイザー*Exp_ColorJFatl_Digit_Tm_Plus*の直接ポートです。ジュリク移動平均（JMA）で平滑化されたFast Adaptive Trend Line（FATL）の傾き反転を取引します。元のインジケーターは3つの色（上昇、フラット、下降）を出力します。ストラテジーは最新の完成バーで色が変化したときに反応し、新しい傾きに合わせてポジションを調整します。

StockSharpの実装はMQLバージョンの高レベル動作を維持しています：注文はクローズしたローソク足で生成され、時間ベースのエグジットはオプションで利用でき、ロットサイジングの入力は`TradeVolume`パラメーターで表されます。

## シグナルロジック

1. **インジケーター計算**
   - 価格は元のインジケーターに付属する39タップのFATLデジタルフィルターを通じて供給されます。
   - フィルタリングされた系列はジュリク移動平均で平滑化されます。長さ、適用価格、丸め精度はパラメーターでカスタマイズできます。
   - 色の状態は現在と前の平滑化された値の差の符号によって決まります：`2`は強気の傾き、`0`は弱気の傾き、`1`はニュートラル/変化なし。

2. **エントリー条件**
   - **ロングエントリー** – `EnableBuyEntries`で有効化。現在のバーの色が`2`になり、前のバーの色が`2`未満だった場合にトリガーされます。`EnableSellExits`がtrueの場合、既存のショートポジションが先に閉じられます。
   - **ショートエントリー** – `EnableSellEntries`で有効化。現在のバーの色が`0`になり、前の色が`0`より大きかった場合にトリガーされます。`EnableBuyExits`がtrueの場合、既存のロングポジションが先に閉じられます。
   - 同時に開けられるポジションは1つのみです。注文は確認ローソク足のクローズ時に送信されます。

3. **エグジット条件**
   - **傾き反転エグジット** – 傾きが逆方向に転換すると、対応する`EnableBuyExits`または`EnableSellExits`フラグがオープンポジションを閉じます。
   - **時間ベースのエグジット** – `UseTimeExit`が有効な場合、`HoldingMinutes`分間保有した後にポジションが閉じられます。
   - **保護レベル** – `StopLossPoints`と`TakeProfitPoints`は価格ステップで表されます。セッションの高値/安値とエントリー価格を比較することで、完成したローソク足ごとに評価されます。

## パラメーター

| パラメーター | 説明 |
|------------|------|
| `TradeVolume` | マーケットエントリーに使用する数量。 |
| `StopLossPoints` | 価格ステップでの保護ストップ距離。`0`に設定すると無効化。 |
| `TakeProfitPoints` | 価格ステップでの利益目標距離。`0`に設定すると無効化。 |
| `EnableBuyEntries` / `EnableSellEntries` | ロング/ショートエントリーを有効または無効化。 |
| `EnableBuyExits` / `EnableSellExits` | 傾きベースのエグジットを有効または無効化。 |
| `UseTimeExit` | 時間制限付きエグジットロジックを有効化。 |
| `HoldingMinutes` | 時間制限付きエグジットがアクティブな場合の保有期間（分単位）。 |
| `CandleType` | 計算に使用する時間軸（デフォルト4時間）。 |
| `JmaLength` | FATL出力に適用するジュリク移動平均の平滑化長。 |
| `AppliedPrices` | デジタルフィルターの価格ソース（終値、始値、中間値、Demark等）。 |
| `RoundingDigits` | 平滑化ラインを丸める際に使用する桁数。 |
| `SignalBar` | インジケーター状態を評価するために使用する完成バーのオフセット。 |

## 注意事項

- ストラテジーは完全に完了したローソク足のみを処理するため、歴史的なバックテストに適しています。
- `AppliedPrices.Demark`は元のMQLインジケーターと同じ計算を再現します。
- StockSharpは注文の実行を非同期で処理するため、エントリー価格の内部追跡は新しいポジションが開かれるたびに更新され、エグジット注文が送信されるたびにクリアされます。
