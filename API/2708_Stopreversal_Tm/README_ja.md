# Stopreversal Tm戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Stopreversal Tm戦略は、元のMetaTrader 5エキスパートアドバイザー`Exp_Stopreversal_Tm.mq5`の直接翻訳です。取引のアイデアはStoprevarsalカスタムインジケーターに従っており、価格の周囲に動的なトレーリングストップを維持し、価格がそのトレーリング境界をクロスするたびにリバーサルアラートを生成します。戦略は単一のインストゥルメントと単一のキャンドルフィードで動作し、ユーザー定義のセッションフィルターを使用したトレンドリバーサル取引向けに設計されています。

## シグナル生成
Stoprevarsalインジケーターは選択した適用価格モードからの参照価格を計算し、次に`Sensitivity`（`nPips`パラメーター）でトレーリングストップレベルを調整します。前のバーがその下にあった間に新しい適用価格がトレーリングストップを上回ってクロスするたびに、強気のシグナルが生成されます。逆に、新しい価格がその上にあった後にトレーリングストップを下回ると弱気のシグナルが現れます。各強気のシグナルは既存のショートポジションのクローズと新しいロングのオープンを同時に要求し、各弱気のシグナルはロングを閉じてショートを開きます。

元のMetaTrader実装の動作を再現するため、戦略は複数の完成したバーによるシグナルの実行を遅延させることができます（`Signal Bar Delay`）。これはエキスパートアドバイザーの`SignalBar`入力を複製し、まだ形成中のキャンドルでの取引を防ぎます。

## セッションフィルターとポジション処理
エキスパートアドバイザーは指定された時間ウィンドウ内でのみ取引を許可していました。変換された戦略は同じロジックを維持します：`Use Time Filter`フラグが有効な場合、注文は`Start Hour/Minute`と`End Hour/Minute`で設定されたセッション内でのみ許可されます。現在の時間が許可ウィンドウを離れると、オープンポジションは即座にフラット化されます。シグナル駆動のエグジットはセッションが無効な場合でもアクティブのままです。

戦略はネットポジションで動作します。クローズアクションは常に反対のエントリーの前に実行され、重複するエクスポージャーなしに方向が変わることを保証します。

## パラメーター
- **Allow Buy Entries / Allow Sell Entries** – 対応するシグナルが受信されたときに新しいロングまたはショートポジションの開設を有効または無効にします。
- **Allow Long Exits / Allow Short Exits** – 反対のシグナルが既存のポジションを閉じることを許可するかどうかを制御します。
- **Use Time Filter** – 取引セッションウィンドウを切り替えます。
- **Start Hour / Start Minute / End Hour / End Minute** – 取引ウィンドウの包括的な開始と排他的な終了を定義します。時間フィルターは終了時間が開始時間より早いオーバーナイトセッションをサポートします。
- **Sensitivity (`nPips`)** – トレーリングストップを価格に近づけたり遠ざけたりするために使用される相対的な距離（乗数として表現、例：`0.004 = 0.4%`）。
- **Signal Bar Delay (`SignalBar`)** – シグナルに基づいて行動する前に待つ完成したキャンドルの数。`0`はクローズキャンドルで即座に実行し、`1`は前のバーで行動するデフォルト動作を再現します。
- **Candle Type** – インジケーター計算に使用するキャンドルサブスクリプションの時間軸。
- **Applied Price** – トレーリングストップ計算を供給する価格シリーズの選択（クローズ、オープン、中央価格、トレンドフォローモード、Demark価格など）。

## 実装上の注意
- インジケーターは外部バッファに依存せず直接戦略内に実装されており、`nPips`トレーリングストップロジックが元のMQL5コードと一致することを保証します。
- セッション管理とシグナルシーケンシングは元のエキスパートに従います。これには既存のエクスポージャーを閉じてから新しい取引を開く優先度が含まれます。
- 変換はStockSharpの高レベルAPI、つまりキャンドルサブスクリプション、遅延シグナルキュー、成行注文（`BuyMarket` / `SellMarket`）に焦点を当てています。StockSharp戦略はすでに明示的なポジションサイズで動作するため、MetaTrader口座メトリクスに関連したマネーマネジメント機能は省略されました。
