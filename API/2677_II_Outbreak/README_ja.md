# II ブレイクアウト戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
II ブレイクアウト戦略は、元々 MetaTrader 4 向けに書かれた高頻度ブレイクアウトシステムです。独自のタイミングオシレーターとボラティリティ圧力ゲージを組み合わせて強い方向性の動きにエントリーし、適応型トレーリングストップとピラミッド取引を使って取引を管理します。このコンバージョンは StockSharp のハイレベル API 上で元のロジックを再現し、スプレッド、ボラティリティ、カレンダーフィルターのための同じ保護機能を維持します。

## 変換されたトレーディングロジック
### タイミングオシレーター
* 新しい M1 ローソク足ごとに「典型価格」（high、low、close の平均に 100 を掛けたもの）が生成され、レガシーの平滑化カスケードに供給されます。
* カスケードは元のネストされた移動平均/差分パイプライン（dtemp/atemp バッファ）を再構築し、0 から 100 のタイミング値を生成します。
* 買いシグナル：タイミング値が前回の読み値を上回る方向に交差する（buffer[0] > buffer[1] かつ buffer[1] ≤ buffer[2]）。
* 売りシグナル：タイミング値が下方向に交差する（buffer[0] < buffer[1] かつ buffer[1] ≥ buffer[2]）。

### ボラティリティフィルター
* 終値の 10 期間標準偏差が `StdDevLimit` を下回る必要があります。制限を超えると新規ポジションは許可されず、オプションで警告がログに記録されます。
* カスタムボラティリティスコアが元の振幅 × ティック密度の式を再現します：現在と前の分足ローソク足の重なりと1秒あたりの平均ティック数を使用します。スコアは設定可能な `VolatilityThreshold` を超える必要があります。

### エントリールール
* 戦略は `CandleType` パラメーターで提供される単一のシンボル/時間軸ペアで動作します（デフォルトは1分足）。
* ポジションが開いておらずカレンダーフィルターが取引を許可している場合、エンジンは `CalculateOrderVolume()` でロットサイズを更新し、レベル1の bid/ask データを使用して現在のスプレッドを `SpreadThreshold` と照合します。
* タイミングオシレーターが買いシグナルを出し、ボラティリティスコアが有効な場合にロングポジションが開かれます。ショートポジションは逆の条件に従います。エントリー時に、執行価格の下/上に `TrailStopPoints` の 2 倍の距離に静的ストップが置かれます。

### ピラミッド取引とトレーリング
* トレーリングモジュールは、集計ポジションが少なくとも `TrailStopPoints + int(Commission) + SpreadThreshold` ポイントの未実現利益を獲得したら起動します。
* ストップは最新の終値の後ろ `TrailStopPoints` に引き締められます（ロングとショートで別々に追跡）。1ポイントを超える改善があればトレーリング価格が更新されます。
* ボラティリティ、タイミング、スプレッドの条件が有効である限り、戦略は追加利益の `max(10, SpreadThreshold + 1)` ポイントごとに新しい注文をピラミッドできます。新しい注文は静的ストップを無効にし、純粋にトレーリングロジックに依存します。

### リスクと資金管理
* ポジションサイズは各注文の前に再計算されます：`残高 × MaximumRisk ÷ (500000 / AccountLeverage)` を銘柄のボリュームステップに丸めたもの。残高情報が利用できない場合は `Volume` または最小ロットにフォールバックします。
* 簡略化されたマージンチェックが元の MetaTrader の保護を近似します（`volume × price / leverage × (1 + MaximumRisk × 190)`）。口座価値がその金額を賄えない場合、注文は無視されます。
* ピラミッド取引が有効になった後、戦略は浮動損失を監視します。未実現ドローダウンが口座価値の `TotalEquityRisk` パーセントを超えると、すべてのポジションが清算されます。

### カレンダーとスプレッドの保護
* 取引は金曜日のサーバー時間 23:00 以降、および年末の最後の取引日（年の日数 358、359、365 または 366）の 16:00 以降に停止します。
* 各エントリーと追加注文は現在の bid/ask スプレッドを確認し、設定されたしきい値を超えた場合は執行をスキップします。

## パラメーター
| パラメーター | デフォルト | 説明 |
|------------|-----------|------|
| `Commission` | 4 | トレーリング起動オフセットの計算に使用するラウンドロット手数料（ポイント）。 |
| `SpreadThreshold` | 6 | 新規エントリーまたはピラミッド取引に許可される最大スプレッド（ポイント）。 |
| `TrailStopPoints` | 20 | トレーリングストップの距離（ポイント）；初期ストップはこの値の2倍。 |
| `TotalEquityRisk` | 0.5 | ピラミッド取引後に強制決済をトリガーする口座資産の損失パーセント。 |
| `MaximumRisk` | 0.1 | ボリュームをサイジングする際に各注文に充当する口座残高の割合。 |
| `StdDevLimit` | 0.002 | 新規取引を受け入れるための最大 10 期間標準偏差。 |
| `VolatilityThreshold` | 800 | 取引に必要な最小ボラティリティスコア（振幅 × ティック密度）。 |
| `AccountLeverage` | 100 | マージン近似とポジションサイジングに使用する口座レバレッジ。 |
| `WarningAlerts` | true | 標準偏差フィルターがエントリーをブロックする際のログ記録を有効にします。 |
| `CandleType` | 1分 | すべての計算に使用するローソク足タイプ。 |

## インジケーター
* 終値に対する `StandardDeviation(Length = 10)` — ボラティリティフィルター用。
* 元の EA から再現したカスタムタイミングオシレーター（StockSharp インジケーターオブジェクトを使わずインラインで実装）。

## 実装上の注意
* スプレッドフィルタリングにはライブレベル1データ（`Security.BestBid`/`BestAsk`）が必要です。フィードがない場合、戦略はスプレッドゼロと仮定します。
* 元の EA が MetaTrader 固有の口座プロパティとコントラクトサイズに依存していたため、マージンと資産チェックは近似値です。ブローカーモデルに合わせて `AccountLeverage`、`MaximumRisk`、または `Volume` を調整してください。
* コンバージョンは StockSharp のハイレベル API（`Bind` を使ったローソク足サブスクリプション）を使用し、すべてのコメントを英語で維持します。この戦略には Python ポートは生成されません。
