# SAR Trading v2.0戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

**SAR Trading v2.0戦略**は、StockSharpの高レベルAPI内でクラシックなCronexエキスパートアドバイザーを再現します。単純移動平均（SMA）とParabolic SARを組み合わせてエントリーのタイミングを計り、固定の保護注文とピップベースのトレーリングストップでポジションを管理します。

- インジケーター: 単純移動平均、Parabolic SAR。
- デフォルトの時間軸: 15分足（`CandleType`で設定可能）。
- 市場: 意味のある`PriceStep`（pip）値を提供する任意のインストゥルメント。

## トレードロジック
- 戦略はポジションが開いていないときのみエントリーを評価します。
- **ロング設定:** Parabolic SARの値がSMAを下回るか、`MaShift`本前のクローズ価格がSMAを下回る場合。これはMQLルール`SAR < MA || Close[shift] < MA`を反映しています。
- **ショート設定:** Parabolic SARの値がSMAを上回るか、`MaShift`本前のクローズがSMAを上回る場合。
- エグジット注文を送信した後、アルゴリズムはポジションがフラットになるまで待ってから新しいシグナルを検討し、元のEAの単一ポジション動作に一致します。

## リスク管理
- `StopLossPips`と`TakeProfitPips`は`Security.PriceStep`を使用してピップスを絶対価格距離に変換します。
- `TrailingStopPips`はトレードが利益になると、保護ストップを価格の後ろの固定のピップ距離に維持します。
- `TrailingStepPips`はトレーリングストップを再び移動させる前に追加のピップバッファを要求し、MQLコードの「トレーリングステップ」ロジックをエミュレートします。
- 市場がストップロスまたはテイクプロフィットレベルに達した場合、ポジションはマーケットでクローズされます。

## パラメーター
- `MaPeriod`（デフォルト**18**）: SMAで使用される足の数。
- `MaShift`（デフォルト**2**）: SMAと比較する際に何本前のクローズ価格を読み取るか。
- `SarStep`（デフォルト**0.02**）: Parabolic SARの加速係数。
- `SarMaxStep`（デフォルト**0.2**）: Parabolic SARの最大加速係数。
- `StopLossPips`（デフォルト**50**）: ピップスでの固定ストップロス距離。
- `TakeProfitPips`（デフォルト**50**）: ピップスでの固定テイクプロフィット距離。
- `TrailingStopPips`（デフォルト**15**）: ピップスでのトレーリングストップ距離。
- `TrailingStepPips`（デフォルト**5**）: トレーリングストップが再び動く前に必要な追加のピップ利益。
- `CandleType`: 計算に使用する足のサブスクリプション。

## 補足説明
- 戦略はMQLバージョンで使用される`iClose(shift)`呼び出しを再現するために、クローズの内部履歴を維持します。
- 決定には完成した足のみを使用し、元のエキスパートアドバイザーとの一貫性を確保します。
- ボリュームは戦略の`Volume`プロパティから取得されます。デフォルトでは各シグナルは1ロットのマーケット注文を送信します。
