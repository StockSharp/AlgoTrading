# トリプルMAチャネルクロスオーバー戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**トリプルMAチャネルクロスオーバー戦略**は、高速移動平均が中速と低速の移動平均を両方とも突き抜けるときに方向性のある
ブレイクアウトを取引します。ドンチャン形式の価格チャネルを使用して出口を管理し、オプションの自動ストップロスと
テイクプロフィットレベルを提供します。この変換はオリジナルのMetaTrader「3MACross EA」に基づいており、その設定可能な
移動平均構造、リスク管理、およびトレーリングロジックを維持しています。

戦略は設定可能な数のポジションまでスケールし、手動のpipベースのリスクターゲットをサポートし、適応的な出口のために
チャネルに従うことができます。有効化されると、ブレイクイーブントリガーはストップロスをエントリー価格プラス安全
バッファに押し上げます。

## 取引ロジック
- **エントリー条件**
  - *ロング:* 高速移動平均が中速と低速の両方の平均を上回るように交差する。`Trade On Close`が有効な場合、交差は完全に
    閉じたローソク足で発生する必要があります。無効な場合、高速平均が両方の低速平均を上回っている間はロングシグナルが
    許可されます。
  - *ショート:* 高速移動平均が同じ確認ロジックで中速と低速の平均を下回るように交差する。
  - 反対側の既存ポジションは即座にクローズされ反転されます。同じ方向へのスケーリングは`Max Positions`に達するまで
    許可されます。
- **エグジット条件**
  - 設定されたテイクプロフィットまたはチャネルベースのターゲットに到達する価格。
  - 動的ストップレベルに触れる価格（手動距離、トレーリングストップ、ブレイクイーブン移動、またはチャネルベースのストップ）。
  - オプションのトレーリングストップは、価格が少なくともトレーリングステップ距離だけ有利に動いた後に調整されます。

## リスク管理
- ストップとターゲットは手動でpipsで定義するか、`Auto SL/TP`が有効な場合は価格チャネルから導出できます。
- トレーリングストップとブレイクイーブンロジックはオリジナルのエキスパートアドバイザーを反映します。ストップは
  有利な方向にのみ動き、決して緩められません。
- ドンチャンチャネルは自動ストップロスとテイクプロフィット配置に使用できる自然なサポート/レジスタンス境界を提供します。
- `Max Positions`はスケーリングステップの数を制限し、制御されていないピラミッディングを防ぎます。

## 主要パラメーター
| パラメーター | 説明 |
|-------------|------|
| `Volume` | 各スケーリングステップの注文サイズ。 |
| `Stop Loss (pips)` | 保護ストップの固定距離。`0`に設定して無効にする。 |
| `Take Profit (pips)` | 利益目標の固定距離。`0`に設定して無効にする。 |
| `Trailing Stop (pips)` | トレーリングストップが使用する距離。`0`はトレーリングを無効にする。 |
| `Trailing Step (pips)` | トレーリングストップを更新する前に必要な最小前進。 |
| `Break Even (pips)` | ブレイクイーブンストップを固定する前に必要な利益。 |
| `Auto SL/TP` | 固定距離の代わりにドンチャンチャネルをストップロスとテイクプロフィット配置に使用する。 |
| `Trade On Close` | 交差が閉じたローソク足で確認されることを要求する。無効にすると、各バーで平均のアライメントが確認される。 |
| `Max Positions` | 方向ごとの最大スケーリングステップ数。 |
| `Fast/Middle/Slow MA Period` | 移動平均の長さ。 |
| `Fast/Middle/Slow MA Shift` | 各移動平均に適用されるオプションのシフト（バー単位）。 |
| `Fast/Middle/Slow MA Type` | 移動平均計算モード（単純、指数、平滑化、加重）。 |
| `Channel Period` | ドンチャンチャネルの高値/安値のルックバック。 |
| `Candle Type` | 戦略が処理するローソク足の時間軸。 |

## 実装ノート
- pip距離は`Security.PriceStep`を使用して変換されます。有効なティックサイズのないインストゥルメントの場合、戦略は
  pip当たり`1`価格単位の距離にフォールバックします。
- 自動チャネル管理はストップロスとテイクプロフィットレベルを現在価格により近い方向にのみ移動させ続けます；決して
  拡大されません。
- ブレイクイーブン有効化はトレーリングステップを追加バッファとして再利用し、オリジナルEAの動作に合わせます。
- 戦略はStockSharpの高レベルAPIとの使用のために設計されており、視覚的分析のためのチャートレンダリング（MAとドンチャン
  チャネル）を処理します。
- クロスオーバーシグナルが有効であるように、低速移動平均とチャネル期間に対して十分な履歴データの深さを確保してください。

## 使用方法
1. 戦略をインストゥルメントに適用し、希望するローソク足の時間軸を設定する。
2. オリジナルEAまたはお使いの適応に合わせて移動平均の期間/方法を設定する。
3. 手動pipsベースのリスク設定と自動チャネル出口の有効化のどちらかを選択する。
4. 戦略を開始する；設定されたローソク足を購読し、インジケーターを計算し、クロスオーバー条件が満たされたときに取引する。
5. ログとチャートオーバーレイを通じてトレーリングストップとブレイクイーブン調整を監視する。

> **免責事項:** 自動取引には重大なリスクが伴います。ライブ市場に展開する前に、履歴データとシミュレーション環境で
> 戦略を徹底的にテストしてください。
