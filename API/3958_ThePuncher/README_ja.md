# パンチャー戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
パンチャー戦略は、オリジナルの MetaTrader 4 エキスパート アドバイザー「The Puncher by L. Bigger」から変換された勢い反転システムです。低速の Stochastic オシレーターと古典的な RSI フィルターを組み合わせて、極端な買われすぎと売られすぎの状態を取引します。両方のオシレーターが市場が延長されることに同意すると、戦略はローソク足の終値での反転を探し、反対方向の成行注文を入力します。

## 取引ロジック
- **買いセットアップ:** Stochastic シグナルラインと RSI が売られ過ぎレベルを同時に下回るとトリガーされます。既存のショートポジションがあれば、それが最初にクローズされてから、新しいロングポジションがオープンされます。
- **売りセットアップ:** 両方のオシレーターが買われ過ぎレベルを超えたときにトリガーされます。オープンしているロングは、新たなショートが設定される前に清算されます。
- **決済ルール:** ポジションは反対のシグナルまたは保護ルール (ストップロス、テイクプロフィット、ブレークイーブン、トレーリングストップ) によってクローズされます。

この戦略は、バー内ノイズを回避するために、選択したタイムフレームの終了したローソク足のみを処理し、ソース EA の「バー終了時の取引」動作を再現します。

## リスク管理
- **ストップロス/テイクプロフィット:** ピップ単位で測定されるオプションの固定距離。無効 (ゼロ) にすると、対応する保護は無視されます。
- **損益分岐点:** 取引で要求された利益バッファーが蓄積された後、ストップをエントリー価格に移動します。
- **トレーリングストップ:** 設定可能な距離と最小ステップで価格に追従し、価格が十分に上昇した後でのみストップがきつくなります。
- **出来高:** 注文では固定の出来高パラメータが使用され、MT4 バージョンのロットサイズ入力が反映されます。

## パラメーター
| 名前 | 説明 | デフォルト |
| --- | --- | --- |
| `OrderVolume` | 新規参入の取引高。 | `1` |
| `StochasticLength` | Stochastic オシレーターのルックバック長 (%K)。 | `100` |
| `StochasticSignalPeriod` | 信号線を適用する前の %K の平滑化期間。 | `3` |
| `StochasticSmoothingPeriod` | %D 信号線の平滑化期間。 | `3` |
| `RsiPeriod` | RSI フィルターの計算期間。 | `14` |
| `OversoldLevel` | 売られ過ぎの状態を検出するためにオシレーターによって共有されるしきい値。 | `30` |
| `OverboughtLevel` | 買われ過ぎの状態を検出するためにオシレーターによって共有されるしきい値。 | `70` |
| `StopLossPips` | 保護停止の距離 (0 を指定すると無効になります)。 | `2000` |
| `TakeProfitPips` | 利益ターゲットの距離 (0 を指定すると無効になります)。 | `0` |
| `TrailingStopPips` | トレーリングストップの距離 (0 は無効にします)。 | `0` |
| `TrailingStepPips` | トレーリングストップを締める前の最小限の有利な動き。 | `1` |
| `BreakEvenPips` | 損益分岐点にストップを移動する前に利益が必要です。 | `0` |
| `CandleType` | キャンドルの作成に使用されるデータ型。 | `M15` |

## 注意事項
- ピップサイズは証券の価格ステップまたは小数点から導出され、ストップ距離とトレーリング距離が金融商品の精度を確実に考慮します。
- この戦略は、元の EA が使用されていた任意のバックテストに適しており、StockSharp をさらに強化するためのベースとして機能します。
- MT4 バージョンの音声アラート、電子メール、チャート上のラベルはプラットフォーム固有の機能であるため、意図的に省略されています。
