# Amstell グリッド・マネージャー戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

双方向の平均化グリッドを実行するMetaTraderエキスパート「exp_Amstell-SL」の高レベルポートです。戦略は各サイドの最新の約定価格を追跡し、価格が十分に離れたときに追加の成行注文を発し、固定のテイクプロフィットまたはストップロス距離に達したときにオープンバッチを清算します。実装はStockSharpのローソク足サブスクリプションと高レベル注文ヘルパーを使用しているため、単一の銘柄のローソク足データを提供する任意の環境に接続できます。

翻訳されたロジックはStockSharpのネットポートフォリオモデルに若干適応されています。ロングとショートのグリッドは依然として別々に管理されますが、同時には保持されません。ロンググリッドはネットポジションが非負の間アクティブであり、すべてのロングエクスポージャーが平坦化された後にのみショートグリッドが引き継ぎます。

## 動作方法

### マーケットデータと実行フロー
- 設定された `CandleType`（デフォルト: 1分足ローソク足）をサブスクライブし、完成したローソク足のみを処理します。
- 銘柄の `PriceStep` からpipsベースのオフセットを計算します。ステップが3桁または5桁の場合、MetaTraderの3/5桁pip調整を模倣するために10倍されます。
- すべての取引は `BuyMarket`/`SellMarket` ヘルパーを通じて配置されます。ペンディング注文は使用しません。

### ロングサイドの管理
- 既存のロングエクスポージャーがなく、戦略がショートを閉じる最中でない場合に最初のロングポジション（`OrderVolume`）を開きます。
- 最新のロング約定価格とアクティブなロングバッチの出来高加重平均エントリー価格を追跡します。
- 終値が最後のロング約定価格より少なくとも `BuyDistancePips`（価格単位に変換）下落した場合、常に `OrderVolume` サイズの追加ロング注文を発します。

### ショートサイドの管理
- ロングバッチが完全に閉じられ、ネットポジションが非正になったら、戦略はショートエントリーを許可します。
- ショートエクスポージャーがない場合に最初のショート注文を発します。価格が前のショート約定価格より `BuyDistancePips * SellDistanceMultiplier` 上昇した後に追加のショートが開かれます。
- アクティブなショートバッチの最新のショート約定価格と出来高加重平均エントリー価格を維持します。

### エグジットルール
- 各方向について、平均約定に対する未実現利益を計算します。
- 利益が `TakeProfitPips` pipsに達するか、ドローダウンが `StopLossPips` pipsに達したときに、成行売りでロングバッチ全体を閉じます。
- 利益が `TakeProfitPips` pipsに達するか、不利な動きが `StopLossPips` pipsに達したときに、成行買いでショートバッチ全体を閉じます。
- 清算後、キャッシュされたすべての価格とボリュームがリセットされ、次のローソク足で新しいグリッドを開始できます。

### オリジナルMQLエキスパートとの違い
- StockSharpバージョンは個別のティックではなくローソク足のクローズで動作します。
- ロングとショートのグリッドはStockSharpのデフォルトネッティングモードに合わせて、同時ではなく順次実行されます。
- すべての保護距離は各チケットを個別にではなく平均エントリー価格に対して確認され、集計されたネットポジションの動作を反映します。

## パラメーター

| パラメーター | デフォルト | 最適化範囲 | 説明 |
|-----------|---------|--------------------|-------------|
| `OrderVolume` | `0.01` | `0.01` – `0.10` (ステップ `0.01`) | 各グリッド注文で送信される数量。正である必要があります。 |
| `TakeProfitPips` | `30` | `10` – `150` (ステップ `10`) | アクティブバッチの利益目標（pips単位）。 |
| `StopLossPips` | `30` | `10` – `150` (ステップ `10`) | バッチを諦める前の最大不利な動き。 |
| `BuyDistancePips` | `10` | `5` – `60` (ステップ `5`) | 別の買いを追加するための最後のロング約定からの最小下落。TPとSLの両方より小さい必要があります。 |
| `SellDistanceMultiplier` | `10` | `2` – `15` (ステップ `1`) | ショートエントリーを間隔を置く際のロング距離に適用される乗数。 |
| `CandleType` | 1分足時間軸 | — | シグナル生成に使用するローソク足シリーズ。 |

## 実装に関する注意事項
- `BuyDistancePips` は `TakeProfitPips` と `StopLossPips` の両方より厳密に小さくなければなりません。そうでない場合、戦略は起動時に例外をスローし、MetaTraderの検証を再現します。
- pip サイズは銘柄の `PriceStep` から導出されます。楽器が非標準のティックサイズを使用している場合はパラメーターを調整してください。
- すべての内部状態は `OnReseted` でクリアされ、残りのグリッドデータなしに戦略を再起動できます。
- カラーカスタマイズや手動のインジケーター登録は使用されず、このリポジトリの高レベルAPIガイドラインに準拠しています。
