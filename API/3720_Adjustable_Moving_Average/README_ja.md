# 調整可能な移動平均戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、StockSharp の高レベルの API を使用して、MetaTrader の「調整可能な移動平均」エキスパート アドバイザーを再作成します。同じタイプで長さが異なる 2 つの移動平均が、それらの距離を監視します。速いカーブが少なくとも設定可能なギャップだけ遅いカーブを横切ると、戦略は反対のポジションを閉じ、オプションで新しい方向に取引を開始します。追加のセッション フィルター、保護出口、およびオプションのトレーリング ストップにより、元のロボットと同じ操作上の柔軟性が提供されます。

## 取引ロジック

- 2 つの移動平均 (高速と低速) は同じ計算方法を共有します。入力が小さいほど周期が速くなり、入力が大きいほど周期が遅くなるように自動的に設定されます。
- シグナルは、両方の移動平均が完全に形成され、その絶対距離が価格単位に変換された `MinGapPoints` しきい値を超えた後にのみ生成されます。
- 高速 MA が低速 MA よりも必要なギャップだけ上にある場合、内部信号状態は強気になります。弱気状態は、低速 MA が高速 MA を上回っているときに記録されます。
- `CloseOutsideSession` が有効になっている場合、または現在時刻がセッション ウィンドウ内にある場合、状態反転により既存のポジションがクローズされます。新しい注文は、選択した `Mode` (買いのみ、売りのみ、または両方) に従い、固定ロットまたは自動ロット サイズ設定ルールのいずれかを使用します。
- 完成したキャンドルごとに保護ロジックがチェックされます。
  - ストップロスとテイクプロフィットの距離は商品ポイントで測定され、ローソク足の範囲に対して評価されます。
  - 価格がポジションに有利に少なくとも `TrailStopPoints` ポイント動くと、トレーリングストップが有効になります。ストップは、セッション フィルターで末尾が許可されている場合、または `TrailOutsideSession` が有効になっている場合にのみ強化されます。ストップが設定されると、取引時間外でもアクティブなままになります。

## ポジションサイジング

- `EnableAutoLot = false` を使用すると、ストラテジーは `FixedLot` のボリュームを送信します（機器ステップ、最小値と最大値の制限を適用した後）。
- `EnableAutoLot = true` の場合、ボリュームは利用可能なポートフォリオ値 `(PortfolioValue / 10,000) * LotPer10kFreeMargin` から近似され、小数第 1 位のロットに四捨五入されます。計算されたボリュームも交換制約に合わせて調整されます。

## パラメーター

| 名前 | タイプ/デフォルト | 説明 |
| --- | --- | --- |
| `CandleType` | `TimeFrame` = 5 分キャンドル | 移動平均の計算に使用される時間枠。 |
| `FastPeriod` | `int` = 3 | 移動平均の長さが短い。 `SlowPeriod` とは異なる必要があります。 |
| `SlowPeriod` | `int` = 9 | 長い移動平均の長さ。 `FastPeriod` とは異なる必要があります。 |
| `MaMethod` | `MovingAverageMethod` = 指数関数 | 移動平均アルゴリズム (単純、指数、平滑化、加重)。 |
| `MinGapPoints` | `decimal` = 3 | 機器ポイントの高速平均と低速平均の間の最小距離。商品価格ステップを使用して換算されます。 |
| `StopLossPoints` | `decimal` = 0 | 計器点での保護停止距離。無効にするには、ゼロに設定します。 |
| `TakeProfitPoints` | `decimal` = 0 | 器具ポイント単位での利益目標距離。無効にするには、ゼロに設定します。 |
| `TrailStopPoints` | `decimal` = 0 | 計器ポイントでのトレーリングストップ距離。無効にするには、ゼロに設定します。 |
| `Mode` | `EntryMode` = 両方 | 新しい取引の方向を許可 (両方、買いのみ、売りのみ)。 |
| `SessionStart` | `TimeSpan` = 00:00 | セッションの開始時刻 (プラットフォーム クロック)。 |
| `SessionEnd` | `TimeSpan` = 23:59 | セッション終了時刻 (プラットフォーム クロック)。 `SessionEnd < SessionStart` の場合、夜間セッションをサポートします。 |
| `CloseOutsideSession` | `bool` = true | true の場合、セッションウィンドウの外側でも反対のポジションがクローズされます。 |
| `TrailOutsideSession` | `bool` = true | true の場合、セッションが終了した後もトレーリングストップは更新され続けます。 |
| `FixedLot` | `decimal` = 0.1 | 自動サイズ調整が無効な場合に使用されるボリューム。 |
| `EnableAutoLot` | `bool` = 偽 | ポートフォリオ値からのボリューム推定を有効にします。 |
| `LotPer10kFreeMargin` | `decimal` = 1 | 自動ロットモードでポートフォリオ値の 10,000 単位ごとに割り当てられるロット。 |
| `MaxSlippage` | `int` = 3 | 完全を期すために保持されます。 StockSharp 成行注文は直接のスリッページ パラメーターを公開しません。 |
| `TradeComment` | `string` = "AdjustableMovingAverageEA" | 取引実行時にログメッセージに含まれるテキスト。 |

## 注意事項

- オリジナルの MetaTrader バージョンでは、注文の変更によりストップロス、テイクプロフィット、トレーリングストップが適用されていました。 StockSharp ポートは、ローソク足の範囲を評価し、反対の成行注文を送信することで動作をエミュレートします。
- MetaTrader の `AccountFreeMargin()` は StockSharp では利用できないため、ポートフォリオの値は自由証拠金の概算として使用されます。
- 機器に有効な `PriceStep` がない場合、ポイントベースの計算 (ギャップ、ストップ、トレーリング) は非アクティブなままになります。
