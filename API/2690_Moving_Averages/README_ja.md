# 移動平均戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
移動平均戦略は、シフトされた単純移動平均（SMA）に対する価格のクロスを取引するクラシックなMetaTraderエキスパートを再現したものです。アルゴリズムは完成したローソク足のみを処理し、すべての取引判断が完全に形成されたバーに基づくことを保証します。ポジションサイジングは口座の資産に連動したダイナミックなリスクモデルに従い、連続損失に適応し、オリジナルのMQL実装を模倣しています。

## 取引ロジック
- 設定可能な期間と完成したバーで測定した追加の前方シフトで単純移動平均が計算されます。
- 各完成ローソク足で、戦略はバーがシフトされたSMAを上回って始まり下回って終わったか（弱気クロス）、または下回って始まり上回って終わったか（強気クロス）を確認します。
- システムは一度に1つのポジションのみを管理します。アクティブなポジションに対してクロスが発生した場合、まずポジションが閉じられ、同じバーでは反転注文が送られません。
- ポジションがオープンしていない場合：
  - 強気クロスがロングポジションを開きます。
  - 弱気クロスがショートポジションを開きます。

## ポジション管理
- ロングポジションは弱気クロスが発生した時に閉じられます。
- ショートポジションは強気クロスが発生した時に閉じられます。
- 取引の実行は戦略銘柄での成行注文を使用します。
- ポジション決済時に損益を測定できるように、有効なエントリー価格を計算するために取引履歴が追跡されます。

## リスク管理とポジションサイジング
- 基本注文量は、ポートフォリオの資産に**Maximum Risk**パラメーターを掛けて現在の終値で割ることで導出されます。資産が利用できない場合、戦略はデフォルトの戦略ボリュームにフォールバックします。
- **Decrease Factor**パラメーターは、連続する負けトレードが検出された場合に計算された注文量を減らします。削減はMQLバージョンの適応型サイジングロジックを再現して、連続損失ストリークに比例します。
- 注文量は決してマイナスにならず、削減がベース金額を完全に相殺した場合はトレードがスキップされます。

## パラメーター
| 名前 | 説明 | デフォルト |
| --- | --- | --- |
| `MaximumRisk` | 各取引でリスクにさらす口座資産の割合。 | `0.02` |
| `DecreaseFactor` | 連続損失後にボリュームを縮小するために使用する除数。 | `3` |
| `MovingPeriod` | シグナルに使用するSMAの期間。 | `12` |
| `MovingShift` | SMAを前方にシフトするために使用する完成バー数。 | `6` |
| `CandleType` | 計算に使用するローソク足シリーズ（時間軸）。 | `5m`ローソク足 |

## 注記
- 移動平均シフトは内部の循環バッファを通じて実装されており、戦略はMetaTraderのインジケーターシフトパラメーターと同様に数バー前のSMA値を使用します。
- SMAとシフトバッファの両方が完全に形成された場合にのみ注文が生成され、ウォームアップ中の早まった取引を防ぎます。
- ログメッセージはエントリー、エグジット、取引結果を記録し、デバッグとパフォーマンス分析を支援します。
