# Blau Ergodic MDI Time戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

**Blau Ergodic MDI Time戦略**は、MetaTraderエキスパート`Exp_BlauErgodicMDI_Tm.mq5`をStockSharpに直接変換したものです。上位時間軸のキャンドルで動作し、元のアルゴリズムの3つのシグナルモード（**Breakdown**、**Twist**、**CloudTwist**）を再現します。この戦略は、選択されたキャンドル価格に適用される多段階の指数移動平均（EMA）スムージングプロセスに依存します。すべての計算は追加インジケーターなしで戦略内部で行われ、MetaTraderエキスパートのロジックと一致しながらStockSharpの高レベルAPIと互換性を保ちます。

スムージングパイプラインはBlau Ergodic MDIオシレーターのロジックに従います：

1. 選択した価格をEMAでスムージング（長さ`BaseLength`）。
2. スムージングされた値を生価格から引いて差分系列を取得。
3. 差分に3つの連続したEMAを適用（長さ`FirstSmoothingLength`、`SecondSmoothingLength`、`ThirdSmoothingLength`）。
4. 中間（`histogram`）および最終（`signal`）出力をインストゥルメントの価格ステップでスケーリング。これらの値がトレードシグナルを駆動します。

## シグナルモード

### Breakdown

* `SignalBar`で制御される2バー前のヒストグラムを使用。
* 前のヒストグラム値が正で選択したバーが非正領域に移動すると、戦略はロングエントリーを準備し、オプションでショートポジションを決済。
* 前のヒストグラム値が負で選択したバーが非負領域に上昇すると、戦略はショートエントリーを準備し、オプションでロングポジションを決済。

### Twist

* 2つの履歴バーにわたるヒストグラムの傾きを比較。
* ヒストグラムが上方向に加速（バー`SignalBar + 1` < バー`SignalBar + 2`）し、最新の選択バーが前のバーより上にある場合、ロングエントリーシグナルが生成。ショートポジションは同じブロックで決済可能。
* ヒストグラムが下方向に加速（バー`SignalBar + 1` > バー`SignalBar + 2`）し、最新の選択バーが前のバーより下にある場合、戦略はショートエントリーを準備し、ロングポジションを決済可能。

### CloudTwist

* ヒストグラムと追加のスムージングラインの両方を使用。
* 前のヒストグラムがシグナルラインより上に留まるが選択バーがその下に移動すると、ロングエントリーが準備されショートポジションを決済可能。
* 前のヒストグラムがシグナルラインより下にあるが選択バーがその上にクロスすると、戦略はショートエントリーを準備しロングポジションから離脱可能。

## 時間ウィンドウフィルター

元のエキスパートは設定可能なセッションにトレードを制限します。StockSharp版はパラメーター`UseTimeFilter`、`StartHour`、`StartMinute`、`EndHour`、`EndMinute`を通じて同じルールを複製します。セッションロジックはMetaTrader実装と同一の深夜をまたぐウィンドウをサポートします：

* 開始時刻が終了時刻より早い場合、セッションは1日以内に収まります。
* 開始時刻と終了時刻が等しい場合、分数がその時間内の短い間隔を定義します。
* 開始時刻が終了時刻より遅い場合、セッションは深夜をまたぎます。

セッションフィルターによりトレードが無効化されている場合、戦略は開いているポジションをフラット決済し、セッションが再開するまで新規エントリーをブロックします。

## リスク管理

パラメーター`StopLossPoints`と`TakeProfitPoints`はエキスパートのストップロスとテイクプロフィットの距離を反映します。距離は価格ステップで表現されます。新しいポジションが開かれるたびに保護価格を再計算します。各完成したキャンドルはバーのレンジが保護レベルに触れたかどうかを確認し、トリガーされた場合はすぐにポジションを決済します。

## 価格入力

`PriceMode`パラメーターは元のインジケーターと同じ価格ソースのリストを公開します：

| モード | 説明 |
| ------ | ---- |
| Close | 終値。 |
| Open | 始値。 |
| High | 高値。 |
| Low | 安値。 |
| Median | (High + Low) / 2。 |
| Typical | (High + Low + Close) / 3。 |
| Weighted | (High + Low + 2 × Close) / 4。 |
| Simple | (Open + Close) / 2。 |
| Quarter | (Open + High + Low + Close) / 4。 |
| TrendFollow0 | 強気キャンドルでHigh、弱気でLow、中立でClose。 |
| TrendFollow1 | トレンド方向のキャンドル極値とCloseの平均。 |
| Demark | Demarkの価格（キャンドル方向で重み付け）。 |

## パラメーター

| パラメーター | デフォルト | 説明 |
| ------------ | ---------- | ---- |
| `Mode` | Twist | Breakdown、Twist、またはCloudTwistシグナル評価を選択。 |
| `PriceMode` | Close | オシレーターに使用される価格ソース。 |
| `BaseLength` | 20 | 生価格に適用されるEMA長。 |
| `FirstSmoothingLength` | 5 | 最初の差分スムージングのEMA長。 |
| `SecondSmoothingLength` | 3 | 2番目の差分スムージングのEMA長。 |
| `ThirdSmoothingLength` | 8 | 3番目の差分スムージングのEMA長。 |
| `SignalBar` | 1 | シグナルチェックに使用される完成バー数（1はMetaTraderのデフォルトに一致）。 |
| `AllowLongEntry` / `AllowShortEntry` | true | ロング/ショートエントリーを有効/無効化。 |
| `AllowLongExit` / `AllowShortExit` | true | 対応するサイドのエグジットを有効/無効化。 |
| `UseTimeFilter` | true | トレーディングセッションフィルターを有効化。 |
| `StartHour`, `StartMinute`, `EndHour`, `EndMinute` | 0/0/23/59 | セッション境界。 |
| `StopLossPoints` | 1000 | 価格ステップ単位のストップロス距離（0で無効化）。 |
| `TakeProfitPoints` | 2000 | 価格ステップ単位のテイクプロフィット距離（0で無効化）。 |
| `CandleType` | 4h時間軸 | 計算に使用されるキャンドルサブスクリプション。 |
| `Volume` | 0.1 | 注文量。エキスパートの`MM`入力に対応。 |

## トレーディングルールの概要

1. 設定された時間軸のキャンドルにサブスクライブ。
2. 各完成キャンドルで4段階EMAパイプラインを更新し、ヒストグラムとシグナル値をローリングバッファーに保存。
3. 最小履歴深度に達するまで待機（元の`min_rates_total`計算に対応）。
4. バー`SignalBar`と古い値を使って選択モードを評価し、開く/閉じるフラグを設定。
5. 対応するエグジットフラグが立つか時間フィルターがトレードをブロックしている場合は、まずポジションを決済。
6. 各フラグが設定され、時間フィルターがトレードを許可し、現在のポジションが同じ方向を向いていない場合にのみ新規ロングまたはショートトレードを開く。反転時は戦略が既存のエクスポージャーと設定量をカバーする注文サイズを自動計算。
7. キャンドルの極値を使用して保護ストップとターゲットを維持し、トリガーを検出。

## 使用上の注記

* 戦略はプロジェクトガイドラインに従い、インデントにタブを使用します。
* 起動時に一度`StartProtection()`を呼び出し、StockSharpのセーフティ機能をポジション変更に対応させます。
* インジケーター値はシグナルに必要な最小バー数のみ保存されます。リポジトリの指示に従い、大きなコレクションは作成されません。
* MetaTraderバージョンの他のスムージング方法を試す場合は、EMA長を調整してください。EMAベースのパイプラインはStockSharpの高レベルAPIでサポートされる最も近い近似を提供します。

## 戦略の実行

1. StockSharpソリューションに戦略クラスを追加してプロジェクトをコンパイル。
2. パラメーター（インストゥルメント、キャンドル時間軸、モード、セッション、リスク設定）を設定。
3. 必要なマーケットデータを提供するコネクターに戦略を接続。
4. 戦略を開始すると、設定されたキャンドルに自動サブスクライブし、上記ルールに従って注文を管理します。
