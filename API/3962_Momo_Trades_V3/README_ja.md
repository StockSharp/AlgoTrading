# Momo Trades V3 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Momo Trades V3 は、オリジナルの MetaTrader エキスパート アドバイザーから変換されたモメンタム戦略です。これは、複数条件の MACD パターン検出器と変位指数移動平均 (EMA) フィルターを組み合わせたものです。 StockSharp ポートは、EA の任意要素を保持し、オプションの損益分岐点処理を追加し、スクリプトの自動ロット ロジックを反映するリスクベースのポジション サイジング モードを提供します。

## 取引ロジック
1. **MACD の運動量パターン** – この戦略は、従来の `(12, 26, 9)` パラメーターと追加の変位 (`MacdShift`) を使用して、メインの MACD ラインを監視します。次の 2 つの強気パターンが受け入れられます。
   - 3 番目の値が 0 に等しく、その後の 2 つのサンプルが上昇し続ける厳密な上昇シーケンス。
   - MACD がゼロより上で交差するシーケンス。次のサンプルは正のままですが、前の値は負です。
弱気のエントリーには、値が減少し、ラインがゼロ未満で交差するミラー条件が必要です。
2. **EMA 距離フィルター** – シフトバー (`MaShift`) の終値は、ロング取引の場合は少なくとも EMA よりも `PriceShiftPoints` MetaTrader ポイント上、ショートの場合は EMA より下である必要があります。これにより、価格が平均に達した場合のエントリーが回避されます。
3. **単一ポジション体制** – この戦略は、フラットな場合にのみ新しいポジションをオープンします。取引がアクティブな間、反対のシグナルは無視されます。
4. **セッション終了終了** – `CloseEndDay` が有効な場合、戦略は夜間の露出を避けるためにプラットフォーム時間の 23:00 (金曜日は 21:00) にポジションを清算します。
5. **オプションの損益分岐点** – `UseBreakeven` がオンの場合、価格がエントリー価格に `BreakevenOffsetPoints` を加えた値にストップを置くほど大きく移動すると、戦略は損益分岐点レベルを確保します。その後、価格がそのレベルまたはそれを超えて戻った場合、ポジションは市場でクローズされます。

## リスク管理
- **初期プロテクション** – `StopLossPoints` と `TakeProfitPoints` は商品価格ステップを通じて絶対価格距離に変換され、`StartProtection` に渡されるため、プロテクション注文が自動的に付加されます。
- **自動出来高** – `UseAutoVolume` が true の場合、注文サイズは現在のポートフォリオの資本から計算されます。この戦略は、株式の `RiskFraction` を取引に割り当て、契約額 (`price × lot size`) で除算し、結果を取引高ステップに正規化し、`VolumeMin`/`VolumeMax` の境界を尊重します。自動サイズ調整が無効になっている場合は、`TradeVolume` が直接使用されます。

## インジケーター
- **移動平均収束発散 (MACD)** – 主要な運動量信号を提供し、`MacdShift` を使用して履歴サンプルで評価されます。
- **指数移動平均 (EMA)** – 置き換えられたトレンド フィルターとして使用されます。

## パラメーター
| 名前 | 種類 | デフォルト | 説明 |
|------|------|---------|-------------|
| `CandleType` | `DataType` | `TimeFrame(15m)` | シグナル生成に使用されるプライマリ タイムフレーム。 |
| `MaPeriod` | `int` | `22` | ディスプレイスメント フィルターの EMA 期間。 |
| `MaShift` | `int` | `1` | 終値と EMA をサンプリングするときに使用される完了したバーの数。 |
| `FastPeriod` | `int` | `12` | MACD の高速な EMA の長さ。 |
| `SlowPeriod` | `int` | `26` | MACD の EMA の長さが遅いです。 |
| `SignalPeriod` | `int` | `9` | MACD の信号 EMA の長さ。 |
| `MacdShift` | `int` | `1` | MACD パターンを評価するときに適用される追加のディスプレイスメント。 |
| `PriceShiftPoints` | `decimal` | `10` | ポジションをオープンするために必要な、シフトされた終値と EMA の間の最小距離 (MetaTrader ポイント単位)。 |
| `TradeVolume` | `decimal` | `0.1` | 自動サイジングが無効になっている場合のデフォルトの取引量。 |
| `RiskFraction` | `decimal` | `0.1` | `UseAutoVolume` が true の場合、注文のサイジングに使用されるポートフォリオ資本の割合。 |
| `UseAutoVolume` | `bool` | `false` | リスクベースのボリュームサイジングを可能にします。 |
| `StopLossPoints` | `decimal` | `100` | 最初のストップロス距離は MetaTrader ポイントで表されます。 `0` は保護停止を無効にします。 |
| `TakeProfitPoints` | `decimal` | `0` | 最初のテイクプロフィット距離（MetaTrader ポイント）。 `0` はターゲットを無効にします。 |
| `CloseEndDay` | `bool` | `true` | オープンポジションは取引日の終わり近く（23:00、金曜日は21:00）に決済されます。 |
| `UseBreakeven` | `bool` | `false` | 損益分岐点管理ロジックをアクティブ化します。 |
| `BreakevenOffsetPoints` | `decimal` | `0` | 損益分岐点のエグジットを準備するときに、オフセットがエントリー価格に追加されます。 |

## 使用上の注意
- 機器に有効な `PriceStep` があることを確認してください。それ以外の場合、戦略は、MetaTrader ポイントを価格距離に変換するときに、`0.0001` ポイント値に戻ります。
- MACD フィルターは完成したキャンドルに依存します。この戦略は、未完成のバーが元の EA の動作と一致するように早期に終了します。
- 一度に許可されるポジションは 1 つだけであるため、取引ごとのリスクは単一の `TradeVolume` (または自動サイズ調整された同等のもの) によって制御されたままになります。
