# Anubis 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Anubis 戦略は、強い逆張りスパイクに対するリバーサルを捉えるために、マルチタイムフレームのボラティリティとモメンタムフィルターを組み合わせています。MetaTrader 5 のオリジナルエキスパートアドバイザーは H4 のインジケーターでエントリーをフィルタリングし、M15 のシグナルでタイミングを計っていました。このコンバージョンは同じ構造を維持しながら、StockSharp のハイレベル API にロジックを適応させ、豊富なランタイムテレメトリを提供します。

## 戦略ロジック
- **時間軸**
  - メインシグナルの時間軸：設定可能なローソク足タイプ（デフォルトは15分足）。
  - 上位時間軸の確認：CCI と標準偏差に使用する固定の4時間足。
- **インジケーター**
  - 上位時間軸の *Commodity Channel Index (CCI)* が買われすぎ/売られすぎの極値を検出します。
  - 上位時間軸の *2つの標準偏差* がテイクプロフィットのサイジングのためのボラティリティ測定を提供します。
  - シグナル時間軸の *MACD* がモメンタムクロスオーバーの確認を提供します。
  - シグナル時間軸の *Average True Range (ATR)* が異常なローソク足レンジによる決済を定義します。
- **エントリー条件**
  - **ロング：** CCI が `-CciThreshold` を下回り、MACD メインラインがシグナルラインを上抜け、前の MACD ヒストグラムが負だった場合。
  - **ショート：** CCI が `+CciThreshold` を上回り、MACD メインラインがシグナルラインを下抜け、前の MACD ヒストグラムが正だった場合。
  - 戦略は必要に応じて新しいポジションを積み上げる前に反対方向のポジションを閉じ、連続エントリー間の最小価格間隔を強制します。
- **ポジション管理**
  - 最大 `MaxLongPositions` または `MaxShortPositions` の積み上げエントリーが許可され、それぞれ `TradeVolume` 枚で開かれます。
  - ストップロスとテイクプロフィットの距離は pips ベースの設定と上位時間軸のボラティリティから導出されます。
  - 価格が `BreakevenPips` 動いたら、保護ストップが平均エントリー価格まで引き上げられます。
- **エグジット条件**
  - ハードストップ：確定済みローソク足ごとにストップロスとテイクプロフィットのレベルを監視します。
  - レンジ決済：前のローソク足レンジが `CloseAtrMultiplier × ATR` を超えた場合、ポジションを閉じます。
  - モメンタム決済：MACD モメンタムがトレードに逆らって転換し、利益が `ThresholdPips` を超えた場合、十分な利益のあるポジションを閉じます。

## パラメーター
| 名前 | デフォルト | 説明 |
| --- | --- | --- |
| `TradeVolume` | 1 | 各エントリーの注文サイズ。 |
| `CciThreshold` | 80 | 極値検出に使用する4時間チャートの絶対 CCI レベル。 |
| `CciPeriod` | 11 | 上位時間軸の CCI のルックバック長さ。 |
| `StopLossPips` | 100 | pips で表したストップロス距離。0 に設定すると初期ストップが無効になります。 |
| `BreakevenPips` | 65 | ストップをブレークイーブンに移動する前の利益距離（pips）。 |
| `ThresholdPips` | 28 | MACD ベースの決済が発動する前に必要な追加利益クッション。 |
| `TakeStdMultiplier` | 2.9 | テイクプロフィット距離を計算する際に遅い標準偏差に適用する乗数。 |
| `CloseAtrMultiplier` | 2 | レンジベースの決済に使用するシグナル時間軸 ATR の乗数。 |
| `SpacingPips` | 20 | 同方向の連続エントリー間の最小価格距離。 |
| `MaxLongPositions` | 2 | 同時ロングエントリーの最大数。 |
| `MaxShortPositions` | 2 | 同時ショートエントリーの最大数。 |
| `MacdFastLength` | 20 | シグナル時間軸の MACD の速い EMA 長さ。 |
| `MacdSlowLength` | 50 | シグナル時間軸の MACD の遅い EMA 長さ。 |
| `MacdSignalLength` | 2 | MACD のシグナル平滑化の長さ。 |
| `AtrLength` | 12 | シグナル時間軸の ATR のルックバック期間。 |
| `StdFastLength` | 20 | 速い標準偏差の期間（診断用）。 |
| `StdSlowLength` | 30 | テイクプロフィット距離を駆動する遅い標準偏差の期間。 |
| `CandleType` | 15m 足 | MACD と ATR 計算に使用するメイン時間軸。 |

## トレーディングの注意事項
- 上位時間軸は4時間に固定されています；異なる市場とメインシグナルの時間軸を同期させたい場合は `CandleType` を調整してください。
- StockSharp はデフォルトでネットポジションを集計するため、ロングとショートのエクスポージャーは同時に保有されません；反対方向のシグナルは新しい注文を発注する前にオープンポジションを解消します。
- 標準偏差の計算は StockSharp の実装に従います。遅い長さはオリジナル MQL バージョンの EMA ベースの偏差を近似します。
- 選択した銘柄が有効な `PriceStep` を公開していることを確認し、pips ベースのパラメーターが正確に価格距離に変換されるようにしてください。
