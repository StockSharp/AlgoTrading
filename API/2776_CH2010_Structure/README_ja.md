# CH2010 Structure マルチ時間軸ブレイクアウト戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、複数の外国為替ペアを 2 つの時間軸で追跡することで、元の **ch2010structure.mq5** エキスパートの動作を再現します。各インストゥルメントは日足ローソク足を監視して方向バイアスを決定し、次に 30 分足ローソク足を観察して前日の値幅を超えるブレイクアウトを探します。ブレイクアウトが日次トレンドと一致する場合にマーケットポジションが開かれ、保護的なストップロスとテイクプロフィットレベルを使用してクローズされます。

## 主要なロジック

1. **日次バイアス検出**  
   * 戦略は USDCHF、GBPUSD、AUDUSD、USDJPY、EURGBP の日足ローソク足をサブスクライブします。  
   * 日足ローソク足が完了すると、クローズ/オープンの関係がバイアスを定義します：強気、弱気、または中立。  
   * 日次の高値、安値、クローズはセッションの日付とともに保存され、イントラデイロジックが同じセッションを取引していることを確認できます。

2. **イントラデイ・ブレイクアウト実行**  
   * 30 分足ローソク足はクローズしたら評価されます。  
   * クローズが前日の高値と設定可能なバッファーを超え、バイアスが弱気でない場合、ロングトレードが引き起こされます。  
   * クローズが前日の安値からバッファーを引いた値を下回り、バイアスが強気でない場合、ショートトレードが引き起こされます。  
   * 過剰取引を避けるために、1 日に各インストゥルメントでロングとショートのブレイクアウトは各 1 回のみ活性化できます。

3. **元のヘルパー関数に着想を得たリスク管理**  
   * ボリュームは `MinTradeVolume` と `MaxTradeVolume` の間に制限され、すべてのインストゥルメントにわたる集計ポジションは `MaxAggregateVolume` によって制限されます。  
   * 各フィルドポジションは、エントリー価格からのパーセントオフセットを使用して、絶対ストップロスとテイクプロフィットレベルをすぐに計算します。  
   * ストップまたはターゲットに達するとすぐに成行注文でポジションがクローズされ、`ExitInProgress` フラグで繰り返しの出口注文が防がれます。

4. **状態追跡**  
   * 各インストゥルメントについて、戦略はその固有の日次レベル、最後に知られたポジション、エントリーサイド、出口注文、ブレイクアウトフラグを `InstrumentContext` で追跡します。  
   * これにより、コンテキストクラスの外にカスタムコレクションを維持せずにマルチシンボルワークフローが可能になります。

## 戦略パラメーター

| パラメーター | 説明 |
| --- | --- |
| `TradeVolume` | 新規エントリーに使用される基本ボリューム（ボリューム制限の対象）。 |
| `MinTradeVolume` と `MaxTradeVolume` | 元の MQL リスクフィルターを反映する境界。 |
| `MaxAggregateVolume` | 取引されるすべてのペアにわたる絶対ポジションの最大合計。 |
| `StopLossPercent` | 検出されたエントリー価格からのパーセントで表した保護ストップオフセット。 |
| `TakeProfitPercent` | 検出されたエントリー価格からのパーセントで表したテイクプロフィットオフセット。 |
| `BreakoutBufferPercent` | ブレイクアウトトリガーに追加される前日の値幅のパーセント。 |
| `DailyCandleType` | 上位時間軸のローソク足をリクエストするために使用される DataType。 |
| `IntradayCandleType` | 実行時間軸のローソク足をリクエストするために使用される DataType。 |
| `UsdChfSecurity` .. `EurGbpSecurity` | デフォルトで監視される 5 つの外国為替シンボルのセキュリティオブジェクト。 |

## 必要なデータ

* 設定された各シンボルの日足ローソク足（デフォルト：1 日の時間軸）。  
* 同じシンボルのイントラデイローソク足（デフォルト：30 分）。  
* 各証券の成行注文を送信するリアルタイム注文ルーティング。

## 使用上の注意

1. 戦略を開始する前に 5 つのセキュリティパラメーターを設定します。必要に応じて他のインストゥルメントに置き換えることができます。  
2. 他の StockSharp 戦略と同様にポートフォリオとコネクターを設定します。  
3. オプションでブレイクアウトバッファーまたはリスクパラメーターを調整して、目標ブローカーの契約仕様を反映させます。  
4. 戦略を開始します。各インストゥルメントの両方のローソク足ストリームを自動的にサブスクライブし、日次構造をログに記録し、有効なイントラデイブレイクアウトを待ちます。  
5. `Daily candle captured` や `Enter Buy` などのエントリーのログを監視して、意思決定フローを確認します。

## 元の MQL エキスパートとの違い

* ブレイクアウト条件が観察されたら即座の成行注文で保留注文が置き換えられます。これにより、ロジックが StockSharp 高レベル API と互換性を保ちながら、露出を制限し毎日 1 方向に 1 回だけ反応するというアイデアを保持します。  
* `DebugOrderSend` ヘルパーのボリューム制限は、個別のトレードサイズと総露出を制限するパラメーターに適応されました。  
* StockSharp でのデバッグを容易にするために、日次レベル、エントリー理由、出口トリガーを英語のコメントで示す広範なログが追加されました。
