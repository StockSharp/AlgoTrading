# 移動平均トレードシステム戦略 (2518)
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

この戦略は、MetaTrader「Moving Average Trade System」エキスパートアドバイザーのStockSharpポートです。ローソク足の中央値価格で計算された4本の単純移動平均（SMA）を使用してトレンドを分析します。システムは、より速い平均がトレンドの一致を確認する一方で、中期と長期の平均間の確認されたクロスオーバーを待ちます。確認が届くと、戦略は新しいトレンドの方向にポジションを反転させ、価格ステップで定義された固定のテイクプロフィット、ストップロス、トレーリングストップオフセットでリスクを管理します。

## トレードロジック

1. **インジケーター**
   - `SMA(5)`（高速）中央値価格に基づく。
   - `SMA(20)`（中速）中央値価格に基づく。
   - `SMA(40)`（シグナル）中央値価格に基づく。
   - `SMA(60)`（低速）中央値価格に基づく。

2. **ロングエントリー**
   - `SMA(5) > SMA(20) > SMA(40)`。
   - `SMA(40)`が`SMA(60)`より少なくとも`SlopeThresholdSteps`価格ステップ上にある。
   - `SMA(40)`が現在のバーで`SMA(60)`を上抜けした（前の`SMA(40)`は低速SMA以下だった）。
   - ショートポジションが開いている場合、戦略はそれを閉じて希望のロングサイズを確立するのに十分なボリュームを買います。

3. **ショートエントリー**
   - `SMA(5) < SMA(20) < SMA(40)`。
   - `SMA(40)`が`SMA(60)`より少なくとも`SlopeThresholdSteps`価格ステップ下にある。
   - `SMA(40)`が現在のバーで`SMA(60)`を下抜けした（前の`SMA(40)`は低速SMA以上だった）。
   - ロングポジションが開いている場合、戦略はそれを閉じて希望のショートサイズを確立するのに十分なボリュームを売ります。

4. **リスク管理**（バーに新しいエントリーがトリガーされない場合にのみ評価）：
   - **トレンドエグジット：** `SMA(40) <= SMA(60)`のときにロングをクローズし、`SMA(40) >= SMA(60)`のときにショートをクローズします。
   - **テイクプロフィット：** 価格がエントリー価格から設定したテイクプロフィット距離に達したら退場します。
   - **ストップロス：** 価格がポジションに対して設定したストップロス距離だけ動いたら退場します。
   - **トレーリングストップ：** 価格がエントリーを超えて進むと、エントリー以来の最高値（ロングの場合）または最安値（ショートの場合）を使用して、保護ストップを`TrailingStopSteps`価格ステップでトレーリングします。

すべてのストップと利益のオフセットは**価格ステップ**（インストゥルメントの`PriceStep`）で測定されます。有価証券が価格ステップを報告しない場合、`1`がフォールバックとして使用されます。

## パラメーター

| 名前 | 説明 | デフォルト値 | 最適化可能 |
| --- | --- | --- | --- |
| `Volume` | 新しいポジションを開く際に使用する注文ボリューム。 | `1` | いいえ |
| `TakeProfitSteps` | 価格ステップで測定したテイクプロフィット目標までの距離。 | `50` | はい |
| `StopLossSteps` | 価格ステップで測定した保護ストップまでの距離。 | `50` | はい |
| `TrailingStopSteps` | 価格ステップでのトレーリングストップオフセット（`0`でトレーリングを無効化）。 | `11` | はい |
| `SlopeThresholdSteps` | ブレイクアウトを検証するための`SMA(40)`と`SMA(60)`間の最小分離（価格ステップ単位）。 | `1` | はい |
| `FastPeriod` | 高速SMAの長さ。 | `5` | はい |
| `MediumPeriod` | 中速SMAの長さ。 | `20` | はい |
| `SignalPeriod` | シグナルSMAの長さ（低速SMAと比較）。 | `40` | はい |
| `SlowPeriod` | バックグラウンドトレンドを定義する低速SMAの長さ。 | `60` | はい |
| `CandleType` | インジケーター計算に使用されるローソク足シリーズ。 | `1h時間軸` | いいえ |

## 実装上の注意

- インジケーターは高レベルの`Bind` APIを通じてローソク足サブスクリプションにバインドされており、計算がイベント駆動であり、手動バッファアクセスに依存しないことを保証します。
- 中央値価格はすべてのSMA計算に使用され、オリジナルのMetaTrader EAの動作を再現しています。
- ポジション管理は`OnNewMyTrade`を使用して実際の約定価格を保存し、各約定後にストップロス、テイクプロフィット、トレーリングストップレベルを再計算します。
- ポジションを反転させる際、戦略は既存のエクスポージャーを閉じ、新しいポジションを開く単一のマーケット注文を送信し、オリジナルアルゴリズムのヘッジ対応動作を模倣します。
- C#ソースファイル内のすべてのコメントはリポジトリガイドラインに従って英語で書かれています。

## 使用のヒント

- インストゥルメントのロットサイズまたは契約乗数に応じて`Volume`パラメーターを設定してください。
- インストゥルメントのボラティリティに合わせてストップと利益距離を調整してください（デフォルト値はFXペアで50 pipsのストップ/テイクプロフィットと11 pipsのトレーリングストップというMetaTrader設定を反映しています）。
- `SlopeThresholdSteps`パラメーターを`0`に設定すると、追加のスペーシングフィルターを削除し、`SMA(40)`/`SMA(60)`のクロスオーバーに反応できます。
- バックテストまたはライブ取引では、有価証券が有効な`PriceStep`を提供していることを確認してください。そうでなければ、戦略は1価格単位を1ステップとして扱います。
