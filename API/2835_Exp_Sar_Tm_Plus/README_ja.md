# Exp Sar Tm Plus 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

エキスパートアドバイザー**Exp_Sar_Tm_Plus**のStockSharp高レベルポートです。この戦略は設定可能な時間軸でParabolic SARの反転を監視し、元のマネー管理とタイムアウト機能を再現しながら、StockSharpの高レベルAPIと互換性のあるロジックを維持します。

## 取引ロジック

- キャンドルは`CandleType`パラメーターからサブスクライブされます（デフォルト：4時間足）。Parabolic SARインジケーターはユーザー定義の`SarStep`と`SarMaximum`係数で計算されます。
- 完成した各キャンドルについて、アルゴリズムは終値とSAR値をバッファリングします。`SignalBar`パラメーターはどの完了したキャンドルを評価するかを選択し（デフォルト：最後に閉じたバー）、前のキャンドルと比較してSARの方向変化を検出します。
- 価格がSARを**上方**にクロスし（前のキャンドルがSARの下、選択されたキャンドルがSARの上）、ロング取引が有効な場合、**ロング**ポジションが開かれます。方向を切り替える前に既存のショート建玉が自動的に閉じられます。
- 価格がSARを**下方**にクロスし（前のキャンドルがSARの上、選択されたキャンドルがSARの下）、ショート取引が有効な場合、**ショート**ポジションが開かれます。既存のロング建玉が最初に清算されます。
- ポジションはSARが逆行したとき（`AllowLongExit` / `AllowShortExit`）、オプションのストップロス/テイクプロフィットレベルが違反されたとき、または最大保持時間（`UseTimeExit` + `HoldingMinutes`）が切れたときに閉じられます。
- ストップロスとテイクプロフィットのレベルは、インストゥルメントの`PriceStep`を使用してエントリーごとに再計算されます。両方のレベルはオプションで、対応する値がゼロの場合は無視されます。

## パラメーター

- `MoneyManagement` – 各エントリーで取引されるベース`Volume`の割合。値 ≤ 0 は単純な`Volume`値にフォールバックします。インストゥルメントの`VolumeStep`に正規化されます。
- `ManagementMode` – 元のエキスパートから保持された列挙型。このポートでは現在すべてのモードが`Lot`（固定ボリューム）として動作します。
- `StopLossPoints` / `TakeProfitPoints` – エントリー価格周辺の保護レベルを設定するために使用される価格ステップ単位の距離。無効にするにはゼロに設定。
- `DeviationPoints` – 元のスリッページ設定。完全性のために保持されていますが、高レベルAPIはこの値を使用せずに成行注文を実行します。
- `AllowLongEntry`, `AllowShortEntry` – ロング/ショートポジションを開くためのトグル。
- `AllowLongExit`, `AllowShortExit` – 価格がSARを反対方向にクロスした際にポジションを閉じるためのトグル。
- `UseTimeExit` – 市場での`HoldingMinutes`分後のポジション清算を有効にします。
- `HoldingMinutes` – 時間ベースのエグジットウィンドウの期間。
- `CandleType` – SAR分析用のキャンドルデータタイプ。
- `SarStep`, `SarMaximum` – Parabolic SAR設定。
- `SignalBar` – シグナル評価をシフトするための閉じたキャンドルの数（0 = 現在の完成したキャンドル、1 = 前のキャンドルなど）。

## リスク管理と注意事項

- 戦略は開始時に`StartProtection()`を呼び出し、StockSharpの組み込み保護サービスを有効にします。
- 時間ベースのエグジットはキャンドルの`CloseTime`（利用できない場合は`OpenTime`にフォールバック）に依存して、保持期間を正確に測定します。
- 常に1つのネットポジションのみが維持されます。ポジションの反転は新しい取引に入る前に自動的に反対側を閉じます。
- 実装は元のMQL5エキスパートのパラメーターセットを維持しています。一部のオプション（非`Lot`マネー管理モードや注文`DeviationPoints`など）は、高レベルAPIがブローカー側の仕組みを抽象化するためのプレースホルダーです。
