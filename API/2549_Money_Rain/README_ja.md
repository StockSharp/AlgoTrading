# Money Rain 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- MQL5の元のエキスパートアドバイザー **MoneyRain (barabashkakvn版)** をStockSharp高レベルAPIに変換したものです。
- DeMarkerオシレーターを使って方向を選択します：0.5を超える値はロングエントリーをトリガーし、0.5以下の値はショートエントリーをトリガーします。
- 同時に一つのポジションのみを取引し、ポイントで表現された固定のストップロス/テイクプロフィットオフセットに依存します。

## マーケットデータとインジケーター
- 設定可能な `CandleType`（デフォルト：30分時間軸）を購読します。
- 調整可能な `DeMarkerPeriod`（デフォルト：31）を持つ単一の `DeMarker` インジケーターを計算します。
- アダプティブなポジションサイジングロジックに必要な現在のスプレッドを近似するため、レベル1クォートを購読します。

## トレードロジック
1. MQLの元の「新しいバー」ロジック（`iTime(0)` チェック）と整合するため、完成したローソク足のみを処理します。
2. ポジションが存在する間、事前計算されたストップロスおよびテイクプロフィットレベルに対してローソク足の高値/安値を監視します。どちらかに触れた場合、成行注文でポジションを決済し、結果を損失または利益としてマークします。
3. 未決済ポジションがなく、損失制限に達していない場合、取引数量を計算します。
4. `DeMarker > 0.5` でロングエントリー；それ以外はショートエントリー。戦略は成行注文を送る前に保留中の注文をキャンセルします。

## 資金管理
- 以下を追跡することでMQLバージョンの `getLots()` ロジックを再現します：
  - `_lossesVolume`：基本ロットサイズでスケーリングされた最近の負けトレードの累積数量。
  - `_consecutiveLosses` と `_consecutiveProfits`：損失アキュムレーターをリセットするタイミングを決めるストリークカウンター。
- 負けストリーク後に最初の利益取引が現れた場合（`_consecutiveProfits == 0`）、次の注文サイズは元の式に従って増加します：
  \[
  \text{volume} = \text{BaseVolume} \times \frac{_lossesVolume \times (\text{StopLossPoints} + \text{spread})}{\text{TakeProfitPoints} - \text{spread}}
  \]
- スプレッドは最良の買値/売値クォート（ポイント単位）から推定され、レベル1データがまだ利用できない場合は無視されます。
- `FastOptimize = true` を設定すると、アダプティブサイジングが無効になり、常に基本ロットを使用します。

## リスクコントロール
- `StopLossPoints` と `TakeProfitPoints` は、3桁または5桁のシンボル向けに追加の10倍乗数を持つ証券の価格ステップを使用して絶対価格に変換されます（MQLの `digits_adjust` ロジックを反映）。
- `LossLimit` は、連続損失数がユーザー定義の閾値を超えると、さらなる取引をブロックします（デフォルト：1,000,000で実質的に無効化）。

## パラメーター
| パラメーター | 説明 | デフォルト値 |
| --- | --- | --- |
| `DeMarkerPeriod` | DeMarkerインジケーターの平均期間。 | 31 |
| `TakeProfitPoints` | DeMarkerスタイルのポイントでのテイクプロフィットオフセット。 | 5 |
| `StopLossPoints` | DeMarkerスタイルのポイントでのストップロスオフセット。 | 20 |
| `BaseVolume` | デフォルトの注文数量（ロットサイズ）。 | 0.01 |
| `LossLimit` | 一時停止前に許可される最大連続損失数。 | 1,000,000 |
| `FastOptimize` | `true` の場合、アダプティブポジションサイジングを無効化。 | `false` |
| `CandleType` | 計算に使用するローソク足データタイプ。 | 30分足ローソク足 |

## 実装メモ
- ストップとターゲットはローソク足の極値をチェックすることでエミュレートされます。イントラバーの約定順序は復元できないため、同時タッチはストップロス分岐を優先します（保守的な仮定）。
- `OnOwnTradeReceived` は保護的な決済注文が完了したことを検出するために使用され、戦略がストリークカウンターと損失数量アキュムレーターを更新できるようにします。
- コードはタブによるインデントを維持し、リポジトリのガイドラインに従って英語のコメントを使用します。

## MQLバージョンとの違い
- ブローカー側の保護注文は、ローソク足の範囲に基づく成行決済に置き換えられています。
- スプレッドはシンボルメタデータから直接ではなく、レベル1クォートから近似されます。
- StockSharp環境が接続を別途管理するため、メール機能と明示的な `IsTradeAllowed` チェックは省略されています。
