# Blau Ergodic MDI戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Blau Ergodic Market Directional Indicator (MDI) 戦略は、MetaTraderのエキスパートアドバイザー `Exp_BlauErgodicMDI` の動作を再現します。アルゴリズムは高い時間軸のローソク足ストリーム (デフォルト4H) で動作し、選択された価格入力に三重スムージングパイプラインを適用してモメンタムヒストグラムとシグナルラインを構築します。トレードの決定は、3つの設定可能なエントリーモードの1つを使用してそのヒストグラムから導き出されます：

1. **Breakdown** – ヒストグラムがゼロラインを越えるときに取引します。
2. **Twist** – ヒストグラムの傾きの逆転に反応します (モメンタムが方向を変える)。
3. **CloudTwist** – ヒストグラム/シグナルラインのクロスオーバーで行動します。

各シグナルは、ユーザーが提供する許可フラグに応じて、オプションで反対ポジションを閉じたり、新しいトレードを開いたりできます。

## インジケーターロジック
1. 設定された移動平均タイプと `PrimaryLength` で選択した適用価格をスムージングし、ベースライン価格を得る。
2. モメンタム差分 `(price - baseline) / point_value` を計算する。
3. そのモメンタムを `FirstSmoothingLength` と `SecondSmoothingLength` でスムージングしてヒストグラムを構築する。
4. ヒストグラムを `SignalLength` でもう一度スムージングしてシグナルラインを得る。
5. シグナルが終値済みローソク足で確認できるよう、`SignalBarShift` に従って過去の値をバッファリングする。

サポートされているスムージングファミリーは **EMA**、**SMA**、**SMMA/RMA**、**WMA** です。適用価格の選択はMetaTrader実装を反映します (終値、始値、高値、安値、中間値、標準値、加重値、単純値、四分の一値、トレンドフォロービリアント)。

## パラメーター
| 名前 | 説明 |
| ---- | ----------- |
| `Volume` | ポジションを開く際に使用する注文サイズ。 |
| `StopLossPoints` | インストゥルメントポイントでのストップロス距離 (0は無効)。 |
| `TakeProfitPoints` | インストゥルメントポイントでのテイクプロフィット距離 (0は無効)。 |
| `SlippagePoints` | 成行注文に適用される最大価格スリッページ (ポイント単位)。 |
| `AllowLongEntries` / `AllowShortEntries` | 各方向へのポジション開始を許可する。 |
| `AllowLongExits` / `AllowShortExits` | 反対シグナルで既存ポジションを閉じることを許可する。 |
| `Mode` | エントリーモード (Breakdown / Twist / CloudTwist)。 |
| `CandleType` | 計算に使用するローソク足の時間軸 (デフォルト4H)。 |
| `SmoothingMethods` | すべてのスムージングステップで使用する移動平均ファミリー。 |
| `PrimaryLength` | 適用価格のベーススムージング長。 |
| `FirstSmoothingLength` | モメンタムに適用する第1スムージング長。 |
| `SecondSmoothingLength` | ヒストグラムを形成する第2スムージング長。 |
| `SignalLength` | シグナルラインを作成するためのヒストグラムのスムージング長。 |
| `AppliedPrices` | インジケーター計算で使用する価格ソース。 |
| `SignalBarShift` | シグナル評価時に振り返る終値済みバーの数。 |
| `Phase` | 互換性のために予約されたパラメーター (現在の実装では使用されていない)。 |

## シグナル条件
* **Breakdown**
  * ロング: `SignalBarShift` のヒストグラムが正で、前のバーはそうでない。
  * ショート: `SignalBarShift` のヒストグラムが負で、前のバーはそうでない。
* **Twist**
  * ロング: `SignalBarShift` のヒストグラムが下落期間後に上昇している (前 < 最新 かつ 2バー前 > 前)。
  * ショート: `SignalBarShift` のヒストグラムが上昇期間後に下落している (前 > 最新 かつ 2バー前 < 前)。
* **CloudTwist**
  * ロング: ヒストグラムがシグナルラインを上回るクロス (最新ヒストグラム > 最新シグナル、前のヒストグラム <= 前のシグナル)。
  * ショート: ヒストグラムがシグナルラインを下回るクロス。

各シグナルは反対側のエクスポージャーをフラット化する (退場が許可されている場合) とともに、設定されたボリュームで新しいトレードを開くことができます。

## リスク管理
`StartProtection` は指定されたストップロスとテイクプロフィット距離 (インストゥルメントのティックサイズを使ってポイントから価格単位に変換) で初期化されます。いずれかの距離がゼロの場合、それぞれの保護は省略されます。スリッページも同じティックサイズを使って価格単位に変換されます。

## メモ
* 元のMetaTraderの動作を反映するため、シグナルは完了したローソク足のみで処理されます。
* `SignalBarShift` により、最新のバーでの行動を避けるためにトレード確認を遅らせることができます。
* `Phase` パラメーターは完全性のために保持されていますが、サポートされているスムージング方法使用時は効果がありません。
* すべてのコードコメントは将来のメンテナンスを簡略化するために英語で提供されています。
