# Hans Indicator クラウドシステム戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略は MQL5 エキスパートアドバイザー `Exp_Hans_Indicator_Cloud_System` を StockSharp 高レベル API に移植します。各取引日を2つの参照セッションに分割する Hans インジケーターの「クラウド」レンジを再現し、インジケーターがそれらの動的レンジを上回るまたは下回るブレイクアウトを報告したときに取引します。実装は設定可能なローソク足シリーズ (デフォルト: M30) を消費し、完了したローソク足のみを処理し、色の変化後の次のバーで行動することにより元のスクリプトの遅延実行ロジックを反映します。

## Hans インジケーターの再現
元のインジケーターはすべてのタイムスタンプをブローカーのタイムゾーン (`LocalTimeZone`) からターゲットのタイムゾーン (`DestinationTimeZone`) にシフトします。StockSharp ポートは各日を2つのセッションに分割する前に同じオフセットを適用します:

1. **セッション 1 (ターゲット時間 04:00–08:00)** – 戦略はこのウィンドウ内に入るすべてのローソク足の最高値の高値と最低値の安値を記録します。ウィンドウが終了すると、ゾーンは完成したとみなされます。
2. **セッション 2 (ターゲット時間 08:00–12:00)** – 2番目のウィンドウでプロセスが繰り返されます。このセッションが終了すると、その高値/安値がその日の残りの時間で最初のゾーンに取って代わります。

価格ステップで表現された設定可能なバッファ (`PipsForEntry`) がアクティブゾーンの高値の上と安値の下に追加されます。インジケーターのカラーマップは次のように再現されます:

- `0` – クローズが上部ゾーンより上にあり、ローソク足のボディが強気。
- `1` – クローズが上部ゾーンより上にあり、ローソク足のボディが弱気。
- `3` – クローズが下部ゾーンより下にあり、ローソク足のボディが強気。
- `4` – クローズが下部ゾーンより下にあり、ローソク足のボディが弱気。
- `2` – ブレイクアウトなし (ニュートラル状態)。

これらの値は MQL5 エキスパートが実行する `CopyBuffer` の参照をエミュレートするために保存されます。

## 取引ロジック
- 戦略はカラーコードのローリング履歴を維持し、ソースの `CopyBuffer(..., SignalBar, 2, ...)` 呼び出しと一致して `SignalBar` バー (デフォルト 1) プラス1つの追加バーを遡ります。
- **ロングを開く**: 古いバー (`SignalBar + 1`) が色 `0` または `1` を報告し、より新しいバー (`SignalBar`) が `0`/`1` で色付けされていない。`TradeVolume` 単位の新しいロングを開く前に、既存のショートエクスポージャーがクローズされます。
- **ショートを開く**: 古いバーが色 `3` または `4` を報告し、より新しいバーが `3`/`4` で色付けされていない。既存のロングエクスポージャーが最初にフラット化され、その後新しいショートが開かれます。
- **ロングをクローズ**: 古いバーが `3` または `4` で色付けされており、ロング決済が有効な場合はいつでも。
- **ショートをクローズ**: 古いバーが `0` または `1` で色付けされており、ショート決済が有効な場合はいつでも。

決済は `TradeAlgorithms.mqh` 内のヘルパー関数と全く同様にエントリーの前に処理され、新しい注文を出す前に反対のポジションがクローズされることを確認します。

## パラメーター
- **ローソク足タイプ** (`CandleType`): 処理されるローソク足の時間軸。
- **シグナルバー** (`SignalBar`): 色の変化を検査するために遡る完了したローソク足の数。
- **ローカルタイムゾーン** (`LocalTimeZone`): ブローカー/サーバーのタイムゾーン (時間単位)。
- **宛先タイムゾーン** (`DestinationTimeZone`): セッションウィンドウを定義するターゲットタイムゾーン。
- **ブレイクアウトバッファ** (`PipsForEntry`): 検出されたセッション範囲の上下に追加する価格ステップ数。
- **ロングエントリー/決済を有効化** (`BuyPosOpen`, `BuyPosClose`): ロングポジション管理のトグル。
- **ショートエントリー/決済を有効化** (`SellPosOpen`, `SellPosClose`): ショートポジション管理のトグル。
- **取引ボリューム** (`TradeVolume`): 各新しいポジションに使用する注文サイズ; 開始時に `Strategy.Volume` とも同期されます。

## 注意事項
- Python の翻訳はリクエストに応じて意図的に省略されています。
- `TradeAlgorithms.mqh` の資金管理ヘルパー (マージンモード、動的なポジションサイジング、ストップロス/テイクプロフィットの配置) は固定の取引ボリュームと明示的な決済ルールに簡略化されています。
- セキュリティが `PriceStep` を公開していない場合、ブレイクアウトバッファは絶対価格単位として解釈され、ティックサイズ情報なしで利用可能な最善の近似に対応します。
