# EA Moving Average戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- MetaTraderのエキスパートアドバイザー**「EA Moving Average」**（barabashkakvn版）から変換されました。
- ロングとショートのエントリーとエグジットを制御するために4つの独立した移動平均線を使用します。
- ネッティングモードの単一銘柄用に設計されています。デフォルトのキャンドルタイプは15分足の時間軸ですが、任意の通常のキャンドルタイプを選択できます。
- 戦略は一度に最大1つのポジションを開きます。ポジションがアクティブの間は、エグジットルールのみが評価されます。

## 取引ロジック
### ロングエントリー
1. 現在のキャンドルは*Buy Open*移動平均線の下で始まり、その上で終値を形成する必要があります（単一バー内での真のクロスオーバー）。
2. `UseBuy`が有効になっている必要があります。
3. `ConsiderPriceLastOut`が有効な場合、現在の価格は最後のクローズド取引の価格以下でなければなりません。これにより最新のエグジットより上での買いが防止されます。
4. 条件が満たされると、戦略はリスクモデルによってサイズが決まる成行買い注文を送信します。

### ロングエグジット
1. ネットポジションがロングの間のみアクティブです。
2. キャンドルは*Buy Close*移動平均線の上で始まり、その下に戻って終値を形成する必要があり、弱気のクロスオーバーを示します。
3. トリガーされると、全ポジションが成行注文で閉じられます。

### ショートエントリー
1. キャンドルは*Sell Open*移動平均線の上で始まった後、その下で終値を形成する必要があります。
2. `UseSell`が有効になっている必要があります。
3. `ConsiderPriceLastOut`が有効な場合、現在の価格は最後のエグジット価格以上でなければなりません。これにより以前のカバーより低いところでのショートが回避されます。
4. リスクベースのボリュームを使用して成行売り注文が送信されます。

### ショートエグジット
1. ポジションがショートの間のみアクティブです。
2. キャンドルは*Sell Close*移動平均線の下で始まり、その上で終値を形成する必要があります。
3. ショートポジションは市場で完全にカバーされます。

## リスクとポジションサイジング
- `MaximumRisk`は取引ごとのリスク資本をポートフォリオ資産の割合として表します。戦略はこのリスク額を現在の価格で割って生のボリューム推定値を取得します。
- `DecreaseFactor`はオリジナルのMetaTraderのロット削減をエミュレートします。2回以上の連続した負け取引の後、ボリュームは損失連敗を`DecreaseFactor`で割った値に比例して削減されます。
- ボリュームは銘柄のボリュームステップに合わせられ、1ステップを下回ることはありません。リスク計算が失敗した場合のフォールバックは戦略の`Volume`プロパティ（デフォルトで1契約/ロット）です。

## パラメーター
| パラメーター | デフォルト | 説明 |
|------------|-----------|------|
| `MaximumRisk` | `0.02` | 取引ごとにリスクにさらされる資産の割合。 |
| `DecreaseFactor` | `3` | 連続した損失後のロット削減係数。`0`で無効化。 |
| `BuyOpenPeriod` | `30` | ロングエントリーに使用する移動平均線の期間。 |
| `BuyOpenShift` | `3` | ロングエントリー移動平均線に適用される前方シフト（バー）。 |
| `BuyOpenMethod` | `Exponential` | ロングエントリーの移動平均メソッド（`Simple`、`Exponential`、`Smoothed`、`LinearWeighted`）。 |
| `BuyOpenPrice` | `Close` | ロングエントリー移動平均線の価格入力。 |
| `BuyClosePeriod` | `14` | ロングエグジット移動平均線の期間。 |
| `BuyCloseShift` | `3` | ロングエグジット移動平均線に適用されるシフト（バー）。 |
| `BuyCloseMethod` | `Exponential` | ロングエグジット移動平均線のメソッド。 |
| `BuyClosePrice` | `Close` | ロングエグジット移動平均線の価格入力。 |
| `SellOpenPeriod` | `30` | ショートエントリー移動平均線の期間。 |
| `SellOpenShift` | `0` | ショートエントリー移動平均線に適用されるシフト（バー）。 |
| `SellOpenMethod` | `Exponential` | ショートエントリー移動平均線のメソッド。 |
| `SellOpenPrice` | `Close` | ショートエントリー移動平均線の価格入力。 |
| `SellClosePeriod` | `20` | ショートエグジット移動平均線の期間。 |
| `SellCloseShift` | `2` | ショートエグジット移動平均線に適用されるシフト（バー）。 |
| `SellCloseMethod` | `Exponential` | ショートエグジット移動平均線のメソッド。 |
| `SellClosePrice` | `Close` | ショートエグジット移動平均線の価格入力。 |
| `UseBuy` | `true` | ロング取引を有効または無効にする。 |
| `UseSell` | `true` | ショート取引を有効または無効にする。 |
| `ConsiderPriceLastOut` | `true` | 再エントリーの前に最後のエグジットに対して価格改善を要求する。 |
| `CandleType` | 15m足 | 計算に使用するキャンドルシリーズ。 |

## 追加の注意事項
- 最後のエグジット価格と連続損失カウンターは取引執行から追跡され、MetaTraderの動作を反映します。
- StockSharpは完了したキャンドルで実行するため、エントリー価格フィルターはキャンドルの終値と比較し、元のティックベースのask/bid比較を近似します。
- 戦略はネッティング口座を前提としており、複数ポジションの同時ヘッジはサポートされていません。
- 実際の資本で取引する前に、必ず過去データでの検証を行ってください。
