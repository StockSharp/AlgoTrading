# AOCCI 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- MetaTrader 5 のエキスパートアドバイザー `AOCCI` を StockSharp 高レベル API に変換したもの。
- Awesome Oscillator と Commodity Channel Index をシンプルなピボットレベルフィルターと組み合わせる。
- 不安定な価格行動をスキップするための「ビッグジャンプ」と「ダブルジャンプ」フィルターによるスプレッド保護を含む。
- ショートセットアップがロングセットアップと同じ条件を使用するオリジナルの MQL5 ロジックを再現する。

## データとインジケーター
- シグナル生成に `CandleType` で定義された主要時間軸を使用する。
- トレンドフィルターとして前の終値を読み取るために追加の上位時間軸（`HigherCandleType`、デフォルト1時間）にサブスクライブする。
- インジケーター：
  - モメンタム方向を検出するための `AwesomeOscillator`。
  - 設定可能な期間とオプションのシグナルシフトを持つ `CommodityChannelIndex`。
- 作業時間軸の `SignalCandleShift + 1` にある足からピボットレベルを計算する：`(High + Low + Close) / 3`。

## エントリーロジック
1. 両インジケーターが完全に形成され、少なくとも6本の完成したローソク足が利用可能になるまで待機する。
2. 設定されたシフトで CCI 値を収集する（現在の比較は `SignalCandleShift`、前のバーは `SignalCandleShift + 1`）。
3. いずれかのジャンプフィルターが発動した場合はバーを拒否する：
   - `BigJumpPips` は最後の5つのインターバルの連続する始値を比較する。
   - `DoubleJumpPips` は1本挟んだバーの始値を比較する。
4. 以下のすべての条件が満たされ、アクティブなポジションがない場合にロングエントリー：
   - 現在のバーで Awesome Oscillator が正である。
   - シフトされた CCI 値がゼロ以上である。
   - 現在の終値がピボットレベルより上にある。
   - 前のデータで少なくとも1つの確認が弱気である：前の AO 値がゼロ未満、前のシフトされた CCI ≤ 0、または最後の上位時間軸終値がピボット未満。
5. ショートエントリーはロングエントリーとまったく同じルールセットを使用する（オリジナルエキスパートは両方向に同一の条件を含む）。

## エグジットロジックとリスク管理
- 取引が開かれると、銘柄の検出されたピップサイズで乗算した設定されたピップ距離を使用してオプションのストップロスとテイクプロフィットレベルが割り当てられる。
- 完成した各ローソク足で、戦略はローソク足の極値を使用してテイクプロフィットまたはストップロスのヒットを確認し、ポジションを成行で決済する。
- `TrailingStopPips` と `TrailingStepPips` の両方が正の場合にトレーリングストップが有効化される：
  - ロング取引は、価格がエントリーから少なくとも `TrailingStopPips + TrailingStepPips` 進むと、ストップを `Close - TrailingStopPips` に移動する。
  - ショート取引は、価格が同じ合計距離を下落すると、ストップを `Close + TrailingStopPips` に移動する。
- ポジションが決済された場合（ストップ、ターゲット、またはトレーリングによって）、戦略は次のローソク足まで待ってから新しいエントリーを評価する。

## パラメーター
| パラメーター | デフォルト | 説明 |
|------------|-----------|------|
| `TradeVolume` | 1 | 成行エントリーに使用するベース注文量。 |
| `StopLossPips` | 50 | 保護ストップの pip 距離。0に設定すると無効化。 |
| `TakeProfitPips` | 50 | テイクプロフィットの pip 距離。0に設定すると無効化。 |
| `TrailingStopPips` | 5 | トレーリングストップ距離（pips）。`TrailingStepPips` > 0 が必要。 |
| `TrailingStepPips` | 5 | トレーリングストップが更新される前の追加バッファー。 |
| `CciPeriod` | 55 | Commodity Channel Index の期間。 |
| `SignalCandleShift` | 0 | CCI バッファーとピボット足を読み取る際に適用されるシフト。 |
| `BigJumpPips` | 100 | 最後の足の連続する始値間の最大許容差（pips）。 |
| `DoubleJumpPips` | 100 | 1本おきの足の始値間の最大許容差（pips）。 |
| `CandleType` | 15分足 | 主要シグナルの作業時間軸。 |
| `HigherCandleType` | 1時間足 | 確認のための前の終値を取得するために使用する上位時間軸。 |

## 注意事項
- pip サイズは `Security.PriceStep` から導出され、3桁または5桁の小数で表示される銘柄に対して調整される。
- オリジナルの EA が両方向に同一のフィルターを使用したため、ショート取引はロング条件も通過し、戦略が売ることを許可されている場合にのみ発生する。不要な場合は外部でショート取引を無効にする。
- ジャンプフィルターは最初の取引が評価される前に少なくとも6本の完成したローソク足が必要である。
