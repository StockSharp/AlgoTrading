# Exp XBullsBearsEyes Vol 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略はMetaTraderのエキスパート **Exp_XBullsBearsEyes_Vol** のC#変換版です。元のアドバイザーはBulls PowerとBears Powerの
読み値を組み合わせ、結果をローソク足のボリュームで乗算し、その結果のモメンタムに応じてヒストグラムを着色します。ロングと
ショットの両側で2つの独立したポジションスロットが維持され、色の強度が増すにつれてシステムがスケールできます。StockSharpへの
ポートは、高レベルAPIコールを使った注文とリスク制御を行いながら、マルチステージフィルター、色ロジック、トレード管理を再現します。

アルゴリズムは設定可能な時間軸をサブスクライブし、カスタムXBullsBearsEyesインジケーターを再構築し、完成したローソク足にのみ
反応します。色の遷移がエントリーとイグジットの両方を決定します：強気の色はショートトレードを閉じ、1つまたは2つのロングスロット
を開く可能性があります；弱気の色は鏡のアクションを実行します。ストップロスとテイクプロフィットの距離は `StartProtection` パラ
メーターに変換され、プラットフォームのリスクマネージャーが保護注文を処理できます。

## インジケーターロジック
1. Bulls PowerとBears Powerの値は、ローソク足の高値/安値を平滑化された終値と比較し、周期 `IndicatorPeriod` のEMAで再構築されます。
2. 4段階の適応フィルターがコエフィシェント `Gamma` で強気（`CU`）と弱気（`CD`）の圧力を蓄積します。インジケーター値は `CU / (CU + CD) * 100 - 50` です。
3. フィルタリングされた値は、`VolumeType` に応じてティックボリュームまたはリアルボリュームで乗算されます。
4. 乗算された系列と生のボリュームの両方が、`SmoothingMethod`、`SmoothingLength`、`SmoothingPhase` で選択された移動平均で平滑化されます（基底クラスが公開している場合はJurikフェーズが適用されます）。
5. 色レベルは `HighLevel1`、`HighLevel2`、`LowLevel1`、`LowLevel2` から導出されます。上バンドより上の値は色 `0` または `1` を生成し、下バンドより下の値は色 `3` または `4` を生成します。色 `2` はニュートラル状態を示します。
6. 色の履歴は保存され、シグナルをバー `SignalBar` で評価できます（デフォルト：1本の閉じたローソク足前）。現在のシグナルバーの色が前の色と比較され、遷移を検出します。

## トレードルール
- 色 `1` と `0` は強気の圧力を示します。色がこれらの値のいずれかに変化し、前の色が弱かった場合、スロット1（`PrimaryVolume`）またはスロット2（`SecondaryVolume`）がそれぞれロングポジションを開きます。`AllowShortExit` が有効な場合、両イベントは既存のショートエクスポージャーを閉じます。
- 色 `3` と `4` は弱気の圧力を示します。色がこれらの値に移動し、前の色が強かった場合、スロット1またはスロット2がそれぞれショートポジションを開きます。`AllowLongExit` が有効な場合、両イベントは既存のロングエクスポージャーを閉じます。
- 各スロットはすでにオープンポジションがあるかどうかを記憶し、対応する方向が閉じられるまで繰り返しシグナルを無視します。
- `SignalBar` は色を評価する前にスキップする完了ローソク足の数を定義します（0 = 最後に完成したローソク足）。コードは比較するために少なくとも2つの履歴色を必要とします。
- ポイントで表現されたストップロスとテイクプロフィット（`StopLossPoints`、`TakeProfitPoints`）は `Security.PriceStep` で絶対価格距離に変換され、マーケットイグジットによるプラットフォーム保護の開始に使用されます。

## パラメーター
| 名前 | 説明 |
|------|------|
| `PrimaryVolume` | 最初のスロットのボリューム（色1 / 3でトリガー）。 |
| `SecondaryVolume` | 2番目のスロットのボリューム（色0 / 4でトリガー）。 |
| `StopLossPoints` / `TakeProfitPoints` | 価格ステップでの保護距離。ゼロに設定して無効化。 |
| `AllowLongEntry` / `AllowShortEntry` | 対応する方向へのスケーリングを有効化。 |
| `AllowLongExit` / `AllowShortExit` | 反対の色が現れたときの自動イグジットを有効化。 |
| `CandleType` | ローソク足とインジケーター計算のためにサブスクライブする時間軸（デフォルト：8時間）。 |
| `IndicatorPeriod` | Bulls/Bears Powerを再構築するために使用するEMA周期。 |
| `Gamma` | 4段階フィルターの適応平滑化係数（0.0 – 0.999）。 |
| `VolumeType` | 重み付けにティックボリュームまたはリアルボリュームを選択。 |
| `HighLevel1`、`HighLevel2`、`LowLevel1`、`LowLevel2` | 色の閾値を定義するレベル乗数。 |
| `SmoothingMethod` | インジケーターとボリュームを平滑化するために使用する移動平均タイプ（SMA、EMA、SMMA、LWMA、Jurik、JurX、ParMA→EMA、T3、VIDYA→EMA、AMA）。 |
| `SmoothingLength` | 平滑化移動平均の長さ。 |
| `SmoothingPhase` | Jurikフェーズパラメーター（[-100, 100]にクランプ）。 |
| `SignalBar` | 色の遷移を評価する前にステップバックする閉じたローソク足の数。 |

## 使用上の注意
- 戦略は `GetWorkingSecurities()` が返す単一の銘柄で動作し、エントリーとイグジットに成行注文を使用します。
- スロット管理はネットです：追加エントリーはネットポジションに追加され、イグジットは影響を受けた側のエクスポージャー全体をフラットにします。
- プラットフォームがティックボリュームのみを提供する場合、`VolumeType = Real` を選択すると利用可能なティックカウントにフォールバックします。
- VIDYAとパラボリック平滑化はExponential Moving Averageにフォールバックします。これはStockSharpがこれらの実装を直接公開しているためです。
- `StopLossPoints` と `TakeProfitPoints` が意図した絶対距離に変換されるよう、銘柄の価格ステップを正しく設定してください。
