# 移動平均ポジションシステム戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

移動平均ポジション システムは、MetaTrader 4 エキスパート アドバイザ「MovingAveragePositionSystem.mq4」の直接ポートです。この戦略は長期ルックバック移動平均を監視し、完了したローソク足で発生する価格交差に反応します。手動のロット選択と、MetaTrader ポイントで表現される累積損益に反応するオプションのマーチンゲールのような数量増加ルーチンの両方をサポートします。

## 取引ロジック

1. **信号検出**
   - システムは、構成可能な移動平均 (単純、指数、平滑化、または線形加重) を計算します。
   - 最も最近終了したローソク足の終値が前の終値とは反対の方向に移動平均を横切ると、戦略は新しいポジションをオープンします。
   - 各方向に 1 つの位置のみが許可されます。戦略がすでにロングの場合、現在の戦略がクローズされるまでポジションは追加されません。同様のことがショートトレードにも当てはまります。
2. **ポジション管理**
   - ロングポジションがオープンしている間にクローズしたローソク足が移動平均を下回って終了した場合、そのポジションは市場で直ちにクローズされます。
   - ショートポジションがオープンしている間にローソク足が移動平均を上回って引けた場合、ショートポジションはクローズされます。
   - 価格ステップ (ポイント) で表現される MetaTrader スタイルのテイクプロフィットは、戦略パラメータを通じて有効化できます。それ以外の場合、ストップは移動平均クロスによって管理されます。
3. **お金の管理**
   - マーチンゲール ブロックが有効になっている場合、この戦略は現在のサイクルで実現および変動損益を MetaTrader ポイントに蓄積します。
   - 累積損失が設定された損失しきい値を超えた場合、次の取引量は 2 倍になり (最大ロット サイズを超えることはありません)、すべてのオープン ポジションはフラット化されます。
   - 累積利益が設定された利益目標を超えると、ボリュームは開始ロットサイズにリセットされ、オープンポジションはすべてクローズされて利益を固定します。

## パラメーター

| パラメータ | 説明 |
|-----------|-------------|
| **MaType** | 移動平均の計算方法: 単純、指数関数、平滑化、または線形重み付け。元の専門家の `TypeMA` の入力を反映しています。 |
| **Ma期間** | 移動平均のルックバック期間 (デフォルトは 240)。 |
| **マシフト** | 信号を生成する前に移動平均値に適用される前方シフト。 `SdvigMA` 入力と同等。 |
| **キャンドルの種類** | 信号の計算に使用されるキャンドルのデータ型。デフォルトは1時間足のローソク足です。 |
| **初期ボリューム** | マーチンゲール ルーチンが変更する前に使用されていたボリューム。 `Lots` 入力に対応します。 |
| **開始ボリューム** | 収益サイクルの後にマーチンゲールがリセットされる基本ロット サイズ (`StarLots`)。 |
| **最大音量** | 取引量の上限 (`MaxLots`)。この制限を超えると、この戦略では作業量が半分になります。 |
| **損失閾値ピップス** | ボリューム倍増イベント (`LossPips`) をトリガーする損失しきい値 (MetaTrader ポイント)。 |
| **ProfitThresholdPips** | ボリュームを開始値 (`ProfitPips`) にリセットするポイント単位の利益目標。 |
| **TakeProfitPips** | 組み込みの保護ヘルパー (`TakeProfit`) を通じて適用される、オプションのポイント単位の固定テイク プロフィット ディスタンス。 |
| **お金の管理を使う** | マーチンゲールのような位置サイジング ルーチン (`MM`) を有効または無効にします。 |

## 使用上の注意

- MetaTrader で使用されたものと同じシンボルと時間枠を使用して戦略を構成します。デフォルトの期間 240 は H1 キャンドルでうまく機能し、元の設定を複製します。
- ポイントしきい値は、機器が有効な `PriceStep` と `StepPrice` を提供することを前提としています。このメタデータが欠落しているシンボルの場合は、しきい値を手動で調整する必要がある場合があります。
- 元のコードは各エントリの前にマージンを再計算するため、ポートは取引サイズが `MaxVolume` を超えるたびに、取引サイズを半分にする簡略化された出来高正規化ステップを実行します。必要に応じて、標準の StockSharp リスク プロバイダーを介して追加のリスク管理を追加できます。
- 完了したローソク足のみがエントリーとエグジットをトリガーし、新しい各足の `Close[1]` と `Close[2]` の値をチェックした MQL 実装をミラーリングします。
