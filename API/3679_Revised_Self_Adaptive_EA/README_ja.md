# 改訂された自己適応型 EA
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

MetaTrader 5 エキスパート アドバイザー `revised_self_adaptive_ea.mq5` を StockSharp の高レベル戦略フレームワークに移植します。

## 戦略の概要

このアルゴリズムは、設定可能なローソク足シリーズをスキャンし、勢いとトレンド フィルターによって確認される巻き込み反転セットアップを探します。

* **パターン検出** – 最後に閉じたローソク足を前のローソク足と比較して評価します。強気のセットアップには、前のローソク足が弱気である間に、前の終値より下で開く緑色の実体が必要です。ミラーロジックは弱気セットアップに適用されます。ローソク足の実体はローリング平均と比較され、弱いシグナルが除去されます。
* **モメンタム フィルター** – 古典的な RSI により、強気の取引は売られ過ぎの領域からのみトリガーされ、弱気の取引は買われ過ぎの状況からのみトリガーされるようになります。
* **トレンド フィルター** – 短い単純移動平均は取引の方向と一致する必要があります。これにより、確認なしに強いトレンドが薄れるのを防ぎます。
* **リスク管理** – ATR によるストップロスとテイクプロフィットのレベルは、新しいポジションごとに計算されます。オプションのトレーリングストップは、保護を決して低下させずに、有益な動きを追跡し続けます。価格が保護レベルに達すると、ポジションは強制的にクローズされます。
* **スプレッドとリスクガード** – 現在のスプレッドが設定されたしきい値を超える場合、または ATR ベースのストップのリスクが価格の許容パーセンテージを超える場合、取引はスキップされます。

## パラメーター

| 名前 | 説明 |
| --- | --- |
| `CandleType` | 分析にはローソク足アグリゲーションが使用されます。デフォルトは 1 時間足です。 |
| `AverageBodyPeriod` | 平均身体サイズフィルターの計算に使用されるキャンドルの数。 |
| `MovingAveragePeriod` | 方向性フィルターとして機能する単純移動平均の長さ。 |
| `RsiPeriod` | RSI の長さは売られすぎ/買われすぎの確認に使用されます。 |
| `OversoldLevel` | RSI のしきい値は、強気の反転を受け入れる前に満たす必要があります。 |
| `OverboughtLevel` | 弱気反転を受け入れる前に満たす必要がある RSI のしきい値。 |
| `AtrPeriod` | ATR の長さは、変動性に基づく保護距離に使用されます。 |
| `StopLossAtrMultiplier` | ストップロス距離の乗算係数が ATR に適用されます。 |
| `TakeProfitAtrMultiplier` | テイクプロフィット距離の乗算係数が ATR に適用されます。 |
| `TrailingStopAtrMultiplier` | ATR 距離はトレーリング ストップ ロジックによって維持されます。 |
| `UseTrailingStop` | トレーリングストップスーパーバイザーを有効にします。 |
| `MaxSpreadPoints` | 最大許容スプレッド (価格ステップ/ピップスで表されます)。市場がより広い場合、シグナルは無視されます。 |
| `MaxRiskPercent` | エントリー価格に対する ATR ストップに基づく最大許容リスク率。 |
| `TradeVolume` | 成行注文に使用される基本ロットサイズ。 |

## 行動メモ

* MetaTrader の実装をミラーリングするために方向を反転する前に位置が平坦化されます。
* 保護ストップ/テイク レベルは、充填のたびに、最新の ATR 読み取り値を使用して再計算されます。
* トレーリングストップは取引方向にのみ移動し、ATR データがまだ利用できない場合は無効になります。
* 信頼できる買値/売値のない商品で戦略が実行されている場合、スプレッド フィルターは自動的に非アクティブなままになります。

## オリジナルの MQL との違い

元のスクリプトでは、信号検出ルーチンの概要のみが説明されていました。この移植では、欠落している要素が、指定されたパラメーターを使用して再構築されました。

* MQL ソースで宣言された MA ハンドルを使用するための移動平均確認を追加しました。
* 元のエキスパートで定義されたボラティリティ ハンドルを使用して、ATR ベースのストップロス、テイクプロフィット、トレーリング ストップ ロジックを実装しました。
* リスクパーセンテージガードを追加し、特大の ATR ストップが盲目的に実行されるのではなくスキップされるようにしました。
* StockSharp 戦略はデフォルトではチャート上にオブジェクトを描画しないため、視覚化要素 (チャートの矢印) は省略されました。

## 使用法

1. Hydra またはカスタム StockSharp ホスト内のポートフォリオとセキュリティに戦略をアタッチします。
2. キャンドルのサブスクリプションが意図した時間枠 (デフォルト: 1 時間) と一致していることを確認してください。
3. 商品のボラティリティを反映するようにリスクパラメータを調整します。
4. 戦略を開始します。条件が満たされた場合、ローソク足を自動的に購読し、インジケーターを計算し、成行注文を出します。
