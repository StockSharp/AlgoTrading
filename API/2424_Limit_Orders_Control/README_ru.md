# Стратегия контроля лимитных заявок
[English](README.md) | [中文](README_cn.md)

Утилитарная стратегия контролирует количество активных отложенных заявок (лимитных и стоп) по текущему инструменту. Предназначена для мониторинга внешней сетки заявок и остановки торговли при её разрушении.

Стратегия подписывается на поток сделок и после каждой сделки пересчитывает число отложенных заявок. Если остаётся менее двух заявок, все они отменяются. Когда заявок не остаётся, стратегия прекращает работу и выводит сообщение в лог.

## Параметры

- **Magic Number** – идентификатор заявок. Сохранён для совместимости с исходной версией MQL, в StockSharp не используется.
- **Write Comments** – при значении `true` стратегия пишет в лог количество активных заявок и своё состояние.

## Детали

- **Критерии входа**: отсутствуют; предполагается, что заявки выставляются внешним образом.
- **Лонг/Шорт**: оба варианта в зависимости от имеющихся заявок.
- **Критерии выхода**: стратегия останавливается при отсутствии отложенных заявок.
- **Стопы**: не используются.
- **Значения по умолчанию**:
  - `Magic Number` = 0
  - `Write Comments` = true
- **Фильтры**:
  - Категория: Утилита
  - Направление: Оба
  - Индикаторы: Нет
  - Стопы: Нет
  - Сложность: Простая
  - Таймфрейм: Любой
  - Сезонность: Нет
  - Нейросети: Нет
  - Дивергенция: Нет
  - Уровень риска: Низкий
