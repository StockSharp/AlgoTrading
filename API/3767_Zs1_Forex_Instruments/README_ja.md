# Zs1 外国為替商品戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、MetaTrader の専門家 **Zs1_www_forex-instruments_info** のヘッジ グリッド ロジックを再現しています。このアルゴリズムは同時売買ペアをオープンし、価格が開始点からどれだけ離れているかを監視し、5 つの個別の取引ゾーンに反応します。ヘッジの生き残ったレッグはマーチンゲール乗数で平均化され、バスケットは株式ベースのエグジットによって保護されます。

## 核となる行動

- 構成された基本ボリュームを使用して、初期市場ヘッジ (1 つの買いと 1 つの売り) を開きます。
- どちらかのレッグが利益が出たら、それをクローズし、負けている側をアンカー注文として保持します。
- `Orders Space (pips)` パラメータを使用して価格変動を追跡します。新しいゾーンに到達したら、元のエキスパートと同じ分岐ロジックを実行します。
  - ゾーン −2: 利益が出たらバスケットを閉じます。それ以外の場合は、動きに対して平均します。
  - ゾーン -1: 最初のアンカーの反対側の位置を追加します。
  - ゾーン 0: アンカーの方向に位置を追加します。
  - ゾーン +1: 利益が出たらバスケットを閉じます。そうでない場合は反対側を開きます。
- 3 つ以上の取引がアクティブなときは常に、変動利益がマイナスでない場合はすぐに終了します。
- すべてのポジションがクローズされると、サイクルが自動的に再開されます。

## パラメーター

| 名前 | 説明 |
| --- | --- |
| `Orders Space (pips)` | 隣接するグリッド レベル間の距離 (ピップ単位)。 |
| `Zone Offset (pips)` | 新しいゾーンが確認される前に突破する必要がある追加のバッファ。 |
| `Initial Volume` | 開始ヘッジとマーチンゲール スケーリングに使用されるベース ボリューム。 |

## 注意事項

- マーチンゲール乗数は、元のトンネル シーケンス (1、3、6、12、...) に従います。
- 数量検証では、注文を送信する前に証券の最小値、最大値、およびステップの制約が尊重されます。
- すべての決定は、MQL バージョンのティックベースのロジックと一致する、レベル 1 データからの最良の買値/売値の更新によって決定されます。
