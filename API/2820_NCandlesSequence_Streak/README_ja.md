# N本連続ローソク足シーケンス戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
N本連続ローソク足シーケンス戦略は、StockSharpの高レベルAPIを使用してオリジナルのMetaTraderエキスパート「N-_Candles_v7」の動作を再現します。完成したローソク足を監視し、設定可能な数の連続した強気または弱気のボディを探します。適格な連続が存在する場合、戦略は同じ方向にポジションを開き、設定可能なテイクプロフィット、ストップロス、トレーリングストップ、取引時間フィルター、浮動利益ロックで管理します。

## 取引ロジック
- 完成した各ローソク足を評価し、強気、弱気、またはニュートラル（ドージ）に分類します。ニュートラルなローソク足は連続カウンターをリセットし、「ブラックシープ」動作を引き起こす可能性があります。
- 同じボディ方向を持つ連続したローソク足の実行カウントを維持します。カウントが設定されたしきい値に達すると、現在の方向がアクティブなパターンになります。
- 強気の連続がアクティブな場合、戦略はロングポジションを開こうとします。弱気の連続がアクティブな場合は、ショートポジションを開こうとします。一度に1つのネットポジションのみが保持されます。
- ローソク足が統一した方向を破ると（「ブラックシープ」）、戦略は選択されたクローズモードに従って反応します：全てをクローズ、反対のポジションのみクローズ、または前の連続と整合したポジションのみクローズ。
- オプションで、開始時刻と終了時刻（両端含む）で定義される取引ウィンドウにエントリーを制限します。
- テイクプロフィット、ストップロス、トレーリングストップの更新、浮動利益しきい値のためにオープンポジションを継続的に監視します。

## ポジションとリスク管理
- 最初の保護ストップと目標は、銘柄の価格ステップで変換されたpip距離から計算されます。これらのレベルは新しいポジションが開かれるたびに再計算されます。
- トレーリングストップロジックはオリジナルのエキスパートを模倣します：価格がトレーリング距離にステップを加えた分だけ進むと、トレーリング距離を維持するためにストップが移動されます。
- 浮動利益ガード（`MinProfit`）は、オープン利益が設定値を超えるとポジション全体をクローズします。
- `MaxPositionVolume` パラメーターは、要求されたボリュームが許可された制限を超える場合にエントリーを防ぎます。`MaxPositions` はポジションがすでにアクティブな場合の再エントリーに対するガードとして機能します。
- 全ての退出は成行注文を呼び出してネットポジションをフラットにします。StockSharp戦略はネッティング環境で動作するためです。

## パラメーター
| 名前 | 説明 |
| --- | --- |
| `ConsecutiveCandles` | シグナルをトリガーするために必要な同じ方向を持つローソク足の数。 |
| `OrderVolume` | エントリーと退出に使用される成行注文量。 |
| `TakeProfitPips` | pipsで表されるテイクプロフィット距離。無効にするにはゼロに設定。 |
| `StopLossPips` | pipsで表されるストップロス距離。無効にするにはゼロに設定。 |
| `TrailingStopPips` | トレーリングストップ距離。トレーリングを無効にするにはゼロに設定。 |
| `TrailingStepPips` | トレーリングストップが動く前に必要な追加距離。 |
| `MaxPositions` | パターンあたりの最大同時エントリー数（戦略は単一のネットポジションを維持する）。 |
| `MaxPositionVolume` | 許可されたネットボリュームの上限。 |
| `UseTradeHours` / `StartHour` / `EndHour` | 取引時間ウィンドウを有効化して設定する（両端含む）。 |
| `MinProfit` | 完全退出をトリガーする浮動利益しきい値。 |
| `ClosingBehavior` | 「ブラックシープ」ローソク足が現れた際の反応を定義する。 |
| `CandleType` | 計算に使用されるローソク足シリーズ。 |

## 注意事項と前提条件
- 戦略はネットポジションで動作します。ヘッジスタイルの複数チケットは作成されません。これは、複数のヘッジされたポジションが共存できたオリジナルのエキスパートとは異なります。
- 浮動利益はロングポジションの場合 `(現在価格 - エントリー価格) * ボリューム`、ショートポジションの場合はその逆として近似されます。
- pip変換は銘柄の `PriceStep` に依存します。最小ステップが提供されていないシンボルの場合、デフォルトのpip 0.0001が仮定されます。
- 要求に応じてPythonポートは提供されません。
