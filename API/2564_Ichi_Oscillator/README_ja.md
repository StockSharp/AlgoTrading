# Ichiオシレーター戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
- MetaTrader 5エキスパート**Exp_ICHI_OSC**をStockSharp高レベルAPIに変換。
- 設定可能なローソク足シリーズで動作し、一目均衡表のラインで構築されたオシレーターからシグナルを導出します。
- 生のオシレーター値は`((Close - SenkouA) - (Tenkan - Kijun)) / Step`で、選択可能な移動平均で平滑化されます。
- 注文は戦略ボリュームで実行されます。元のコードの複雑な資金管理ブロックはStockSharpのポジション処理に置き換えられました。

## パラメーター
| 名前 | 説明 |
| --- | --- |
| `CandleType` | すべてのインジケーター計算に使用されるローソク足時間軸。 |
| `IchimokuBase` | Tenkan（`base * 0.5`）、Kijun（`base * 1.5`）、Senkou B（`base * 3`）の長さを定義する基本期間。 |
| `Smoothing Method` | オシレーターを平滑化するために使用される移動平均。オプション：`Simple`、`Exponential`、`Smoothed`、`Weighted`、`Jurik`、`Kaufman`。 |
| `Smoothing Length` | 選択した平滑化方法の期間。 |
| `Smoothing Phase` | 予約済み互換性パラメーター（MQLバージョンから保持、現在組み込み平滑化実装では使用されていない）。 |
| `Signal Bar` | オシレーターの色を読み取るために使用される最新の完成ローソク足からの何バー前か（デフォルト`1`）。 |
| `Enable Buy Entries / Enable Sell Entries` | それぞれロングまたはショートポジションの開設を許可する。 |
| `Enable Buy Exits / Enable Sell Exits` | 既存のロングまたはショートポジションのクローズを許可する。 |
| `Stop Loss (points)` | 価格ステップで表現された保護ストップ距離。 |
| `Take Profit (points)` | 価格ステップで表現されたテイクプロフィット距離。 |
| `Order Volume` | 成行注文で使用されるベース注文ボリューム。 |

## トレードロジック
1. 要求されたローソク足シリーズをサブスクライブし、導出された一目均衡表期間を使用してTenkan、Kijun、Senkou Aの値を計算します。
2. 価格、Senkou A、Tenkan、Kijun間の差からオシレーターを構築し、選択したスムーザーに通します。
3. 各平滑化された値にカラーコードを割り当てます：
   - `0` — オシレーターがゼロより上で上昇中。
   - `1` — オシレーターがゼロより上で下降中。
   - `2` — ニュートラル（ゼロレベルまたは変化なし）。
   - `3` — オシレーターがゼロより下で減少中。
   - `4` — オシレーターがゼロより下で上昇中。
4. 2つの色を読みます：`SignalBar + 1`のバー（前の色）と`SignalBar`のバー（現在の色）。
   - 前の色が`0`または`3`の場合、許可されている場合はショートを閉じて、現在の色が`2`、`1`、または`4`の場合はロングを開きます。
   - 前の色が`4`または`1`の場合、許可されている場合はロングを閉じて、現在の色が`0`、`1`、または`3`の場合はショートを開きます。
5. 注文は設定されたボリュームで配置されます。ロングとショートは積み重ねられません：オープンシグナルは同じバーでエグジットロジックが実行された後にのみ評価されます。

## リスク管理
- 保護注文は`StartProtection`を通じて管理され、価格ステップでのストップロスとテイクプロフィット距離を使用します。
- デフォルトではトレーリングや部分的なエグジットは有効になっていません。

## 注意事項
- 元の資金管理モジュール（ロット計算、偏差処理、取引タイマー）はStockSharpのポジションおよびボリューム制御に置き換えられます。
- StockSharpに存在しない平滑化方法（例：JurX、ParMA、VIDYA、T3）は利用できません。提供されたリストから最も近い代替を選択してください。
- ログのシグナルタイムスタンプには、MQLの`TimeShiftSec`の使用を反映して、ローソク足クローズ時刻に完全な1ローソク足期間を加えたものが含まれます。
