# ADXストップ注文テンプレート戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略はAverage Directional Index（ADX）とその方向性指標コンポーネントを使用してペンディングストップ注文を出す方法を示します。古典的なMQLテンプレートのコアロジックを再現しています：市場が強いトレンドを示し+DIと-DIラインがクロスすると、システムは固定距離に買いストップまたは売りストップを出します。保護的なストップロスとテイクプロフィットレベルは自動的に管理されます。

例は意図的にシンプルに設定されており、注文処理に焦点を当てています。トレーダーは追加フィルターやマネーマネジメントルールで拡張して、より高度なシステムを構築できます。

## 詳細

- **エントリー条件**:
  - ADX値が`ADX Threshold`パラメーターを上回っている。
  - **ロング**: `+DI`が`-DI`より大きく、2本前のローソク足で`+DI`が`-DI`より低かった。
  - **ショート**: `+DI`が`-DI`より小さく、2本前のローソク足で`+DI`が`-DI`より高かった。
  - 現在のスプレッドが`Max Spread`パラメーターを下回っている。
- **注文配置**:
  - ペンディングストップ注文は現在のビッドまたはアスクから`Pips`価格ステップ離れた位置に配置されます。
  - 一度に有効なペンディング注文は1つのみです。新しいシグナルが現れると古い注文はキャンセルされます。
- **エグジット条件**:
  - `-DI`が`+DI`を上抜けするとロングポジションがクローズされます。
  - `+DI`が`-DI`を上抜けするとショートポジションがクローズされます。
- **ストップ**:
  - ストップロスとテイクプロフィットは`Stop Loss`と`Take Profit`パラメーターを使用して`StartProtection`経由で適用されます。
- **デフォルト値**:
  - `ADX Period` = 14
  - `ADX Threshold` = 5
  - `Pips` = 10価格ステップ
  - `Take Profit` = 1000価格ステップ
  - `Stop Loss` = 500価格ステップ
  - `Max Spread` = 20価格ステップ
  - `Candle Type` = 15分足
- **フィルター**:
  - カテゴリ: トレンドフォロー
  - 方向: 両方
  - インジケーター: ADX, DMI
  - ストップ: はい
  - 複雑さ: 中程度
  - 時間軸: イントラデイ
  - スプレッドフィルター: はい
