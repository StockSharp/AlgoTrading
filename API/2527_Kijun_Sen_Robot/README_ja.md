# Kijun-Sen Robotストラテジー
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**Kijun-Sen Robotストラテジー**は、MetaTrader 5のエキスパートアドバイザー「Kijun Sen Robot」をStockSharpの高レベル戦略APIに直接変換したものです。デフォルトで30分足ローソク足で動作し、20期間の線形加重移動平均（LWMA）によって確認されたIchimokuのKijun-senによる価格クロスに焦点を当てています。この戦略は、最も活発な時間帯にのみ取引を行い、ダイナミックなストップ、ブレークイーブン、トレーリングロジックによるポジション保護を強制するという元のエキスパートの考えを維持しています。

## インジケーターとデータ
- **Ichimoku**：Tenkan、Kijun、Senkou Span Bを6/12/24期間に設定。
- **線形加重移動平均（LWMA）** 20バー：傾き確認と距離フィルタリング用。
- **30分足ローソク足**（デフォルト）：シグナル生成用。`CandleType`パラメーターで他の時間軸を選択可能。

## トレードロジック
### ロングエントリー
1. ローソク足がKijunラインを下から突き抜ける。ローソク足はラインの下で開いてその上で閉じるか、または前の終値もラインの下にある状態でバー中にラインに触れる必要があります。
2. 現在のKijunが2バー前と比べて横ばいまたは上昇している。
3. LWMAがKijunレベルの少なくとも`MaFilterPips`（価格単位に変換）下にあり、基準線を移動平均の上に保つ。
4. LWMAの傾きが正（現在のLWMAが前の値より大きい）。
5. 取引時間が`[TradingStartHour, TradingEndHour)`内、デフォルトは取引所時刻07:00–19:00。

すべての条件が満たされ、戦略がすでにネットロングでない場合、成行買い注文が送信されます（既存のショートがある場合はまず決済）。エントリー価格はローソク足の終値です。

### ショートエントリー
1. ローソク足がKijunラインを上から突き抜ける（ロングロジックの鏡）。
2. Kijunが2バー前と比べて横ばいまたは下落している。
3. LWMAがKijunレベルの少なくとも`MaFilterPips`上にある。
4. LWMAの傾きが負（現在のLWMAが前の値より小さい）。
5. エントリーは許可された取引ウィンドウ内でのみ発生。

成行売り注文が置かれます（既存のロングエクスポージャーはショートを開く前に閉じられます）。

### ポジション管理と決済
- **初期ストップロス** – エントリー価格から`StopLossPips`下/上に置かれます（インストゥルメントの価格ステップを使って価格単位に変換）。これはMQLバージョンの保護ストップを再現します。
- **ブレークイーブン移動** – 未実現利益が`BreakEvenPips`を超えたら、ストップをエントリー価格プラス1pip（ロング）またはマイナス1pip（ショート）に移動します。閾値は同じpip変換ロジックで測定されます。
- **トレーリングストップ** – 価格が`TrailingStopPips`進んだ後、ストップはその距離で価格に追従し、有利な方向にのみ動きます。
- **固定テイクプロフィット** – `TakeProfitPips`で定義されるオプションの目標。無効化するにはゼロに設定。
- **Kijun傾きによる決済** – ストップがブレークイーブンを超えて移動する前にLWMAがトレードに逆らって転換した場合、ポジションは即座に閉じられ、元のエキスパートの緊急脱出と一致します。
- **時間フィルター** – 新規トレードは設定されたウィンドウ外では無視されますが、オープンなトレードは上記のルールで閉じられるまで引き続き管理されます。
- **注文処理** – StockSharp戦略は成行注文のみを使用。ティックデータの代わりにローソク足データが使用されるため、元のEAの複雑なリミット対成行エントリーロジックは簡素化されています。

同じバー内でストップロスとテイクプロフィットの両方のレベルが違反される場合、バー内情報なしで保守的を維持するためにストップロスが優先されます。

## パラメーター
| パラメーター | デフォルト値 | 説明 |
|-----------|---------|-------------|
| `TenkanPeriod` | 6 | Ichimoku Tenkan-senの長さ。 |
| `KijunPeriod` | 12 | Ichimoku Kijun-senの長さ。 |
| `SenkouSpanBPeriod` | 24 | Ichimoku Senkou Span Bの長さ。 |
| `LwmaPeriod` | 20 | LWMAの確認フィルターの長さ。 |
| `MaFilterPips` | 6 | pipでの最小LWMA対Kijun距離。 |
| `StopLossPips` | 50 | 初期保護ストップ距離。 |
| `BreakEvenPips` | 9 | ストップをブレークイーブンに移動するために必要な利益。 |
| `TrailingStopPips` | 10 | トレーリングストップ移動の距離。 |
| `TakeProfitPips` | 120 | オプションの固定テイクプロフィット距離。 |
| `TradingStartHour` | 7 | 最初の許可取引時間（含む）。 |
| `TradingEndHour` | 19 | 最後の許可取引時間（除く）。 |
| `CandleType` | 30分の時間軸 | シグナル評価に使用するデータタイプ。 |

すべてのpipベースのパラメーターはインストゥルメントの`PriceStep`を使って価格単位に変換されます。3桁または5桁の小数を持つインストゥルメントは、古典的なFX pipサイズを再現するために自動的に10の係数を受け取ります。

## 実装メモ
- 変換は`_pendingLongLevel`と`_pendingShortLevel`を通じてストラテジーのステートフル変数（`longcross`、`shortcross`の動作）を維持し、新規ポジションが新鮮なKijunクロスを必要とすることを保証します。
- MT5バージョンの「最後のbid/ask」のようなバー内チェックは、ローソク足レベルの条件（`Open`、`Close`、`High`、`Low`）で近似されます。これにより、StockSharpのバックテストでのロジックが決定論的になります。
- ポジション保護は、MT5注文変更の代わりに`ClosePosition()`と手動ストップ追跡を使用します。ブレークイーブンとトレーリングの調整は完了したローソク足ごとに一度実行されます。
- ヘルパーメソッド`ConvertPips`は、`Security.PriceStep`または`Security.MinPriceStep`を使用してpipから価格への変換を行い、MT5の`digits_adjust`ルールをエミュレートするために3桁または5桁のティックサイズに10×乗数を適用します。
- 戦略は高レベルAPIに結びついているため、インジケーターは`SubscribeCandles().BindEx(...)`を介してバインドされ、チャートの描画（ローソク足、Ichimoku、LWMA、自己トレード）が自動的に設定されます。

## 使用ガイドライン
1. 30分足ローソク足をサポートするインストゥルメントに戦略を適用します（または別の`CandleType`を設定）。
2. 開始前に戦略インスタンスの`Volume`を希望の注文サイズに設定します。
3. オプションで、インストゥルメントのボラティリティを反映するか、特定の通貨ペアの最適化された設定を再現するためにpipベースのパラメーターを調整します。
4. 高レベルバックテスターまたはライブ環境で実行。戦略は元のエキスパートと同じ取引ウィンドウ、ストップ、トレーリングルールを適用します。
5. ブレークイーブンとトレーリングの更新を確認するためにログまたはチャートを監視します。コード内のすべてのコメントは、要求に応じて明確さのために英語で記述されています。

Pythonバージョンは意図的に省略されています。このフォルダーにはC#実装のみが提供されています。
