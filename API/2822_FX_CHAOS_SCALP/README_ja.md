# FX-CHAOSスキャルピング戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
FX-CHAOSスキャルピング戦略は、複数の時間軸でAwesome OscillatorとフラクタルベースのZigZagレベルを組み合わせたMT5エキスパートアドバイザーを再現します。StockSharpポートは取引実行のために時間足ローソク足を、上位時間軸フィルターのために日足ローソク足をサブスクライブします。内部トラッカーは5本のローソク足のフラクタルパターンを検出し、それらを交互のスイングポイントに結び付けることで「ZigZag on Fractals」ロジックを再構築します。

## トレーディングワークフロー
1. **データ収集**
   - 時間足ローソク足がエントリーとリスク管理を駆動します。
   - 日足ローソク足が上位時間軸ZigZagフィルターにデータを提供します。
   - Awesome Oscillator（5、34）が時間足フィードで計算されます。
2. **フラクタルZigZagトラッキング**
   - 各完成したローソク足は5要素のスライドウィンドウに入力されます。
   - 中央のバーが上昇/下降フラクタルを形成すると、最新のスイング値が更新されます。同方向の連続するスイングはより極端な値によってのみ置き換えられます。
3. **時間足終値でのシグナル検出**
   - ロングシグナルは、ローソク足が前の高値を下回って始まり、その高値を上回って終わり、最新の時間足ZigZagスイングを下回ったまま、最新の日足ZigZagレベルを上回り、Awesome Oscillatorが負の場合に現れます。
   - ショートシグナルは前の安値と反対のオシレーター極性を使用してロジックを反映します。
4. **注文実行**
   - 反対のポジションが存在する場合は、設定されたボリュームで新しいエントリーが配置される前にそれらが決済されます。
   - エントリー価格はその後のストップロスとテイクプロフィット管理のために保存されます。

## パラメータ
| 名前 | 説明 |
| --- | --- |
| `Volume` | ロット単位の取引ボリューム。各成行注文に適用されます。 |
| `Stop Loss (pts)` | 保護ストップのポイント単位の距離。値は証券の価格ステップで乗算されます。無効にするには`0`を設定してください。 |
| `Take Profit (pts)` | 利益目標のポイント単位の距離。同じ方法で価格ステップで変換されます。無効にするには`0`を設定してください。 |
| `Trading Candle` | エントリーに使用される主要な時間軸（デフォルトは1時間）。 |
| `Daily Candle` | ZigZagフィルターに使用される上位時間軸（デフォルトは1日）。 |

## リスク管理
- 各完成した時間足ローソク足で、戦略は保存されたエントリー価格から導出されたストップロスまたはテイクプロフィットレベルに価格が触れたかどうかを確認します。
- 保護注文が執行されると、ポジションはすぐに決済され、エントリー価格フラグがリセットされ、同じローソク足サイクルでの再エントリーが防止されます。
- 反対方向に新しいシグナルが現れた場合もポジションは決済されます。

## 実装ノート
- カスタムZigZagロジックは直接のインジケーターバッファを避け、最小限のローカル状態でローソク足サブスクリプションで動作することでリポジトリガイドラインに従います。
- ZigZag値は、十分なローソク足が処理されるまで（潜在的なフラクタルの各側に2本のバー）`null`のままです。時間足と日足の両方のトラッカーが有効なスイングを生成するまでトレードは停止されます。
- Awesome Oscillatorは`BindEx`経由でリクエストされ、すべての入力が準備できた際に戦略が最終的なインジケーター値のみを使用することを保証します。
- 価格距離は`Security.PriceStep`でスケーリングされます。インジケーターにステップがない場合、戦略は1ポイント乗数にフォールバックします。
