# ProtoType IX 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
ProtoType IX は、オリジナルの MetaTrader 4 エキスパート アドバイザーから変換されたマルチフィルター トレンドフォロー戦略です。このアルゴリズムは、Williams %R スイングを観察して新たな衝動的な動きを検出し、平均トゥルー レンジ (ATR) 拡張で検証します。予測される報酬対リスクの比率が十分に魅力的で、ブレイクアウトが確認された場合にのみ、取引が開始されます。

## インジケーターとシグナル
- **Williams %R (期間設定可能)** – 買われすぎ/売られすぎのローテーションを監視します。この戦略は、インジケーターが極端なゾーンを離れるときに表示される 2 つの最新のスイング高値とスイング安値を記録します。
- **平均トゥルーレンジ (ATR)** – 現在のボラティリティを測定します。最新のスイングと前のスイングの間の距離が `ATR × multiplier` を超えた場合、ブレイクアウトは有効であるとみなされます。

## エントリールール
1. 最近のスイングの高値と安値の両方が記録されるまで待ちます。
2. Williams %R 方向を決定します。インジケーターが上限しきい値を超えている場合、強気バイアスが保存されます。下限閾値を下回っている場合、弱気バイアスが保存されます。
3. ATR でスイング構造を確認します。
   - 強気トレンド - 最新のスイング高値は前のスイング高値を少なくとも `ATR × multiplier` 上回っており、最新のスイング高値は前回のスイング高値よりも高い必要があります。
   - 弱気トレンド – 最新のスイング安値は前のスイング安値を少なくとも `ATR × multiplier` 下回る必要があり、最新のスイング高値は以前のスイング高値よりも低い必要があります。
4. 現在の終値を使用して報酬/リスク比を評価します。
   - **ロング**: ターゲット = 最大(最後のスイング高、前のスイング高); stop = max(最後のスイングが低い、前のスイングが低い)。
   - **ショート**: ターゲット = min(最後のスイングが低い、前のスイングが低い); stop = min(最後のスイング高、前のスイング高)。
5. `take profit distance / stop loss distance ≥ TP/SL criteria` でターゲット距離が最小スプレッド要件より大きい場合にのみポジションをオープンします。

## 終了ルール
- 最初の保護命令は入国直後に出されます。ストップロスとテイクプロフィットのレベルは、StockSharp の保護注文を使用するための価格ステップに変換されます。
- 設定された `Zero Bar` 遅延が期限切れになると、ATR ベースのトレーリング モデルを使用してストップロスが強化されます。
  - ロングポジションはストップから `max(previous stop, close − 2 × ATR)` まで続きます。
  - ショートポジションはストップから `min(previous stop, close + 2 × ATR)` まで続きます。

## ポジションサイズ
ロットサイズは、ポートフォリオ値と `Risk %` パラメーターから推定されます。価格ステップのストップロス距離は、許容される金銭的リスクを出来高に換算するために使用されます。音量は楽器の音量ステップに正規化され、`Max Order Size` によって上限が設定されます。

## パラメーター
| 名前 | 説明 |
| --- | --- |
| Williams %R 期間 | Williams %R インジケーターの長さ。 |
| 基準 WPR | 買われ過ぎ/売られ過ぎゾーンを定義する絶対しきい値。 |
| ATR 期間 | 平均トゥルーレンジインジケーターの長さ。 |
| ATR乗数 | ブレークアウト検証のために ATR に適用される乗数。 |
| ゼロバー | ATR の末尾を有効にするまでの小節の数。 |
| 最小ターゲットスプレッド | 最小許容ターゲット距離は拡散倍数で表されます。 |
| TP/SL基準 | 取引に参加するために必要な最小限のテイクプロフィット/ストップロス比率。 |
| 最大注文数 | 同時にオープンされる注文の最大数。 |
| 最大注文サイズ | サイジング後の注文量の上限。 |
| リスク% | ポジションサイジングに使用されるリスクパーセンテージ。 |
| キャンドルの種類 | 計算用のローソク足データ型。 |

## 注意事項
- この戦略は単一の証券に焦点を当てていますが、元の EA のマルチフィルター ロジックは維持しています。
- 保護注文は商品の価格ステップに依存します。戦略を実行する前に、機器のメタデータが設定されていることを確認してください。
- ボリューム ステップまたはステップ価格のゼロ値は、サイジング ルーチンを安定に保つために適切なデフォルト値に置き換えられます。
