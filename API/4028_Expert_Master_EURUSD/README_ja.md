# エキスパートマスター EURUSD 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

Expert Master EURUSD 戦略は、MetaTrader 4 Expert Advisor *Expert Master* を複製します。
MACD のメイン ラインとシグナル ライン上の 4 つのキャンドル パターンを評価します (高速 EMA = 5、低速 EMA = 15、シグナル EMA = 3)。
アルゴリズムは、反対方向のブレイクアウトエントリーをトリガーする前に、インジケータが一方向に勢いを増すことを期待します。

## 取引ロジック

### 長いセットアップ
1. MACD シグナル ラインは、前の 3 つのローソク足で下降シーケンスを形成し、現在のローソク足で上昇に転じます。
2. MACD のメインラインは「V」字型を形成し、現在の値が前の 3 つの読み取り値を上回ります。
3. 以前のメインライン値は、設定可能な下限しきい値 (デフォルトは -0.00020) を下回っています。
4. 最も古い主線の値はゼロ未満ですが、現在の値は上限しきい値 (デフォルトは 0.00020) を超えています。

### 簡単なセットアップ
1. MACD シグナル ラインは、前の 3 つのローソク足で上昇シーケンスを形成し、現在のローソク足で下降に転じます。
2. MACD のメインラインは逆「V」を形成し、現在の値が前の 3 つの読み取り値を下回ります。
3. 以前の主線の値が上限しきい値 (デフォルトは 0.00020) を超えています。
4. 最も古いメインライン値はゼロより上ですが、現在の値は短いしきい値 (デフォルト -0.00035) を下回ります。

## ポジション管理

- **モメンタム損失による決済:** 現在の MACD のメイン値が前の値を下回ると、ロングポジションがクローズされます。
現在の MACD のメイン値が前の値を上回った場合、ショート ポジションはクローズされます。
- **トレーリング ストップ:** 取引に有利に設定されたポイント数だけ価格が変動した後、トレーリング ストップがアクティブになります。
ストップは、ローソクの終値からトレーリング距離を引いた値またはプラスした値を使用して、終了したローソクごとに更新されます。
価格がトレーリングストップに戻った場合、戦略は成行注文を通じて終了します。

## リスク管理

- 取引量のデフォルトは固定ロット サイズですが、**リスク パーセント** パラメーターを通じて動的に調整できます。
リスクサイジングが有効な場合、戦略は元の EA の動作を模倣し、すべてのエントリでポートフォリオ値の一部をリスクにさらします。

## パラメーター

| 名前 | 説明 | デフォルト |
| --- | --- | --- |
| `TrailingPoints` | 価格帯でのトレーリングストップ距離。 | 25 |
| `FixedVolume` | リスクサイジングが利用できない場合のフォールバック取引量。 | 1 |
| `RiskPercent` | ポジションのサイズ設定に使用されるポートフォリオ値の割合。 | 0.01 |
| `MacdFastPeriod` | MACD 幹線の高速 EMA 長さ。 | 5 |
| `MacdSlowPeriod` | MACD 本線の EMA の長さが遅いです。 | 15 |
| `MacdSignalPeriod` | MACD インジケーターの信号 EMA の長さ。 | 3 |
| `UpperMacdThreshold` | エントリには正の MACD しきい値が必要です。 | 0.00020 |
| `LowerMacdThreshold` | 長い信号で使用される負の MACD しきい値。 | −0.00020 |
| `ShortCurrentThreshold` | 負の MACD しきい値がショートの現在の値に適用されます。 | −0.00035 |
| `CandleType` | インジケーターの計算に使用されるローソク足のタイプ。 | 1分の時間枠 |

## 注意事項

- 高レベルの StockSharp API と足並みを揃えるために、完成したローソク足でのみ取引してください。
- この変換では、リスクベースのロットサイジングやトレーリングストップの動作など、元の EA ロジックが維持されますが、最適化を容易にするために広範なパラメーター化が追加されます。
