# MACD EA 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、`MQL/20010`フォルダーにある MetaTrader 5 エキスパートアドバイザー `MACD EA (barabashkakvn's edition).mq5` の StockSharp ポートです。StockSharp の高レベル API を使用しながら、同じ MACD クロスオーバーロジック、部分的な利益確定、および資金管理機能を再現しています。

## トレードロジック

* **シグナルソース** – 設定可能な高速、低速、シグナル期間を使ってクラシックな MACD インジケーターを計算します。戦略は、2 本および 4 本前の完成したローソク足における MACD ラインとシグナルラインの差を調べます。強気のクロスオーバー（差がマイナスからプラスに転換）でロングトレードを開き、逆の条件でショートトレードを開きます。
* **ポジション管理** – 各注文は、pips 単位で測定された設定可能なストップロスとテイクプロフィットのオフセットで保護されます。オフセットは、銘柄の価格ステップを使って価格に変換され、銘柄が 3 または 5 桁の小数点を持つ場合は 10 倍にして、元の EA のポイント調整を模倣します。
* **部分利確** – 有効にすると、価格がトレード方向に `PartialProfitPips` 進んだ時点で、オープンポジションの半分が決済されます。残りの部分は継続します。
* **ブレイクイーブン** – 価格が `BreakevenPips` 有利方向に進んだ後、戦略はブレイクイーブンガードを有効にします。価格が元のエントリーレベルに戻ると、EA がストップをブレイクイーブンに移動させるのと同様に、エントリー価格でポジションが決済されます。
* **逆 MACD シグナル** – 逆の MACD クロスオーバーが残りのエクスポージャーを即座に決済し、戦略がインジケーターのトレンドに逆らったポジションを保持しないことを保証します。

## 資金管理

`UseMoneyManagement` が有効な場合、連続する負けトレードの後にポジションサイズが増加します。次のトレードは連続損失数に基づくマルチプライヤーを使用します（1 回の損失後は x2、2 回の損失後は x3、6 回以上の損失では x7 まで）。マルチプライヤーは `RiskMultiplier` パラメーターと組み合わされ、元のコードのマーチンゲールスタイルのサイジングを再現します。勝ちトレードは損失カウンターをゼロにリセットします。

## パラメーター

| パラメーター | 説明 |
|------------|------|
| `FastPeriod` / `SlowPeriod` / `SignalPeriod` | MACD 計算の長さ。
| `StopLossPips` | pips 単位の保護ストップまでの距離（0 で無効化）。
| `TakeProfitPips` | pips 単位の利益目標までの距離（0 で無効化）。
| `PartialProfitPips` | ポジションの半分を決済するために必要な pips（0 で部分決済を無効化）。
| `BreakevenPips` | ブレイクイーブンモードが有効になる前に必要な pips（0 でブレイクイーブンを無効化）。
| `UseMoneyManagement` | 損失連続に基づく動的なポジションサイジングを有効化。
| `RiskMultiplier` | 資金管理がアクティブな場合に適用される追加マルチプライヤー。
| `BaseVolume` | スケーリング前の基本トレードボリューム。
| `CandleType` | インジケーター計算に使用するローソク足シリーズ。

## 注記

* 戦略は `SubscribeCandles` とインジケーターバインディングを使用し、推奨される高レベル API パターンに従っています。
* Python バージョンはまだ利用できません。`CS` フォルダーの C# 実装のみが提供されています。
* リクエストに従い、テストは追加または変更されていません。
