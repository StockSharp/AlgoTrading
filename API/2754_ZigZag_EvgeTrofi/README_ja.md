# ZigZag EvgeTrofi ストラテジー
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

ZigZag EvgeTrofi ストラテジーは、クラシックなMetaTraderのエキスパートアドバイザーをStockSharpの高レベルAPIにポートしたものです。ZigZagスタイルのプロセスで検出された最新のスウィングを監視し、ピボットがまだ新鮮な間に素早く反応します。

## コンセプト

* 元のアドバイザーはZigZagバッファの最初の非ゼロポイントを分析し、最後に確認されたスウィングが高値だったか安値だったかを判断します。
* スウィング高値はデフォルトでロングエントリーを生成します。**SignalReverse**を有効にするとロジックが反転します。
* ポジションは新しいピボットが最近のものと見なされる間のみ開かれます。**Urgency**パラメーターはピボット後にトレードを開始できるバーの数を制限します。
* 既存の逆方向のポジションは新しい注文を出す前にすぐに平坦化されます。緊急性ウィンドウが開いている間、ストラテジーは連続したバーで同じ方向にスケールインできます。

このポートは対立的な動作を維持します：新しい高値がロングトレードを引き起こし、新鮮な安値がショートを引き起こし、元のセットアップを模倣します。

## 動作の仕組み

1. 2つのローリングインジケーター（`Highest`と`Lowest`）がMetaTrader ZigZagの深度ロジックを近似します。
2. 価格がそれらのバンドの上/下に新しい極値を印刷し、その動きが**Deviation**（価格ステップ）を超えるたびに、ピボットが記録されます。
3. アルゴリズムはピボット以降に経過したバーの数を追跡します。カウンターが**Urgency**を超えると、シグナルは期限切れになります。
4. アクティブウィンドウ中のすべての閉じたローソク足で、ストラテジーは`VolumePerTrade`を使用して参入します。逆のエクスポージャーが最初に閉じられるため、フリップトレードはきれいに発生します。

## パラメーター

| パラメーター | デフォルト | 説明 |
|------------|-----------|------|
| `Depth` | 17 | スウィング高値/安値を探すための遡るバーのウィンドウ。ZigZagの深度入力を反映。 |
| `Deviation` | 7 | 新しいピボットを受け入れるために必要な最小価格変位（シンボルの価格ステップで乗算）。 |
| `Backstep` | 5 | インジケーターが逆のピボット方向に切り替わる前に経過する必要があるバー数。 |
| `Urgency` | 2 | エントリーが許可されるピボット後の最大バー数。 |
| `SignalReverse` | `false` | 高値/安値のロング/ショートシグナルへのマッピングを反転。 |
| `CandleType` | 5分ローソク足 | 分析に使用する時間軸。ミラーしたいチャートに合わせて調整。 |
| `VolumePerTrade` | 0.10 | 各エントリーで送信される注文サイズ。元のロット入力と一致。 |

## トレードに関する注意事項

* ロジックにはストップやターゲットが**含まれていません**。リスク管理が必要な場合は、オーバーレイまたはポートフォリオ設定を通じて追加する必要があります。
* システムは緊急性ウィンドウ内のすべてのバーでポジションに追加できるため、強いトレンドではポジションサイズが急速に成長する可能性があります。
* ボラティリティの高いシンボルでは、過度なピボットを避けるために高い深度を使用してください。低い深度はストラテジーをより反応的にしますが、よりノイジーになります。
* **SignalReverse**がtrueの場合、動作はブレイクアウトフォローになります：スウィング高値がショートを引き起こし、スウィング安値がロングを引き起こします。
