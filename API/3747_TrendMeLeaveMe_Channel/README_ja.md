# TrendMeLeaveMe 保留中のチャネル戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この StockSharp 実装は、元の MetaTrader「TrendMeLeaveMe」エキスパート アドバイザーを再作成します。そのアイデアは、動的なトレンド チャネルを手動で追跡し、価格がトレンド ラインに達するたびに保留中のストップ注文を使用してブレイクアウトをキャッチすることです。 StockSharp はユーザーが描画したチャート オブジェクトでは機能しないため、この戦略は線形回帰指標を使用してチャネルの中心を自動的に再構築し、MQL バージョンが上部と下部のガイドラインに適用したものと同じオフセット ロジックを再現します。

このアプローチはロングエントリーとショートエントリーの両方に向けて設計されています。逆指値注文がトリガーされると、ポジションは、EA で設定された距離を反映する静的なストップロス注文と利食い注文によって直ちに保護されます。未決注文は常に更新されるため、アクティブ化レベルは最新の回帰直線値を追跡します。

## 戦略の仕組み

1. ローソク足サブスクリプションは、中央のトレンドラインとして機能する `LinearRegression` インジケーターを駆動します。
2. ユーザーは商品価格ステップで 4 つのオフセット (売買シナリオの上限/下限) を定義します。この戦略は、それらを回帰直線の上または下の価格に変換します。
3. 最後のローソク足がトレンドラインと設定された下側オフセットの間で閉じると、買いストップが上側オフセットに配置されます。対称的に、価格がラインと上部オフセットの間で閉じると、売りストップが下部境界に配置されます。
4. 市場がこれらのアクティブ化ゾーンの外に移動した場合、対応する未決注文はキャンセルされるため、戦略が帳簿を乱雑にすることはありません。
5. ストップ注文が実行されると、取引は元のエキスパートアドバイザーと同じポイント距離を使用する静的なストップロスとテイクプロフィットでラップされます。

## 信号

- **買いのセットアップ**: ローソク足の終値は回帰線以下ですが、それでも買いの下限オフセットを上回っています。買いストップ注文は上限オフセットで発注され、条件が有効な間はラインに従います。
- **売り設定**: ローソク足終値は回帰直線以上ですが、まだ売り上限オフセットを下回っています。売りストップ注文は、より低いオフセットで配置され、トレンドラインをたどります。
- **設定なし**: 価格がアクティベーションコリドーの外にある場合、既存の未決注文は削除されます。

## リスク管理

- 買い取引では、`BuyStopLossSteps` と `BuyTakeProfitSteps` を使用して、エントリー価格から固定のストップロスとテイクプロフィットのレベルを計算します。
- 売り取引では、同じ目的で `SellStopLossSteps` と `SellTakeProfitSteps` を使用します。
- 保護注文は、ネットポジションが反転した場合にのみ再計算され、MetaTrader が各未決注文に直接ストップレベルを付ける方法を模倣します。

## パラメーター

- `CandleType` – トレンドラインの計算に使用されるローソク足の集計。
- `TrendLength` – 線形回帰ウィンドウ内のローソク足の数。
- `BuyStepUpper` / `BuyStepLower` – 長いセットアップの上限トリガーと下限アクティベーションしきい値を定義するオフセット (価格ステップ単位)。
- `SellStepUpper` / `SellStepLower` – 短期間のセットアップのアクティベーションコリドーを定義するオフセット（価格ステップ単位）。
- `BuyTakeProfitSteps` / `BuyStopLossSteps` – 価格ステップで表される、ロングポジションのエグジットの距離。
- `SellTakeProfitSteps` / `SellStopLossSteps` – ショートポジションの出口の距離。
- `BuyVolume` / `SellVolume` – 両側の未決注文に使用されるボリューム。

## 注意事項

- 手動の傾向線がないため、回帰インジケーターは MQL 戦略のチャート オブジェクトを置き換えます。ユーザーは回帰長を試して、手動の傾向分析に近似することができます。
- この戦略は、取引所接続がライブ (`IsFormedAndOnlineAndAllowTrading`) である場合にのみ取引されます。
- 同じ方向のポジションがすでに存在する場合、保留中の注文は自動的にキャンセルされ、元の EA の単一注文の動作が再現されます。
