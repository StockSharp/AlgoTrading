# 矢印と曲線ストラテジー
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
このストラテジーは、MetaTrader 5の「Arrows and Curves」エキスパートアドバイザー（EA）のC#移植版です。StockSharpの高レベルAPIを使用して、元のインジケーターベースのロジックを再現しています。システムは単一のシンボルを取引し、Arrows and Curvesインジケーターが生成するカスタムチャネルシグナルに反応します。同時にアクティブにできるポジションは1つのみで、新しいシグナルが来るたびに新しいトレードを開くか、既存のトレードを閉じます。

## ストラテジーロジック
- 設定可能な時間軸のローソク足は`SubscribeCandles`を通じてストリーミングされます。処理ルーティンは、新しいバーの開始時のEA動作を再現するために、完成したローソク足のみを処理します。
- Arrows and Curvesチャネルはストラテジー内で再構築されます：アルゴリズムはMT5インジケーターと同様に、`Relay`オフセットでシフトされた`SSP`ルックバックウィンドウ内の最高値と最安値をスキャンします。それらの値から2つのエンベロープが導出されます（外側バンドには`Channel %`、内側バンドには`Channel Stop %`）。
- インジケーターの状態変数（`uptrend`と`uptrend2`）は、元のMQLコードと全く同じ順序で更新されます。前のローソク足がSell矢印を生成した場合、ストラテジーはロングエントリーを準備し、Buy矢印を生成した場合はショートエントリーを準備します。これは、次のバーでインデックス1でシグナルを読み取るEAの動作を再現しています。
- ポジションが開いていない場合、前のバーから保存されたシグナルを使用して、矢印とは逆方向の成行注文を出します（Sell矢印→買いトレード、Buy矢印→売りトレード）。
- すでにポジションが存在し、逆のシグナルが現れた場合、現在のポジションは閉じられますが、逆ポジションはすぐには開かれません。これはMT5のソースコードに一致しており、まず閉じてから次のバーで再びエントリーを評価します。

## リスク管理
- ストップロスとテイクプロフィットの距離はpipsで定義され、インスツルメントの`PriceStep`を使用して絶対価格オフセットに変換されます。小数点以下3桁または5桁で引用されるインスツルメントの場合、変換はステップを10倍にし、EAのpip調整を再現します。
- トレーリングストップ機能はEAを反映しています：浮動利益が`Trailing Stop + Trailing Step`を超えると、最小ステップを守りながら設定された距離だけ保護ストップが引き寄せられます。
- 保護レベルは、ローソク足の高値/安値を使用してイントラバートリガーを近似するために、完成したローソク足ごとに確認されます。
- ポジションサイジングは`Volume`パラメーターで固定できます。`Volume`がゼロの場合、ストラテジーは設定されたストップロス距離に対してポートフォリオ価値の`Risk %`をリスクにさらすことで動的な数量を算出します。

## パラメーター
- `Volume`: 固定注文サイズ。ゼロに設定するとリスクベースのサイジングが有効になります。
- `Risk %`: ボリュームがゼロの場合にリスクにさらすポートフォリオ価値の割合。
- `Stop Loss (pips)`: 保護ストップの距離（pips単位）。
- `Take Profit (pips)`: 利益目標の距離（pips単位）。
- `Trailing Stop (pips)`: トレーリングストップの距離（pips単位）；ゼロに設定すると無効になります。
- `Trailing Step (pips)`: トレーリングストップが再び移動する前に必要な最小追加移動量。
- `SSP`: チャネル範囲を計算するために使用するローソク足の数。
- `Channel %`: 外側エンベロープの割合、MT5設定と同一。
- `Channel Stop %`: 2次状態を切り替えるために使用する内側エンベロープの割合。
- `Relay`: チャネル計算に適用されるシフト。
- `Candle Type`: インジケーターに供給する時間軸またはローソク足タイプ。

## 実装メモ
- ストラテジーはインジケーターに必要な最小限の過去の高値、安値、終値のみを保存します（`SSP + Relay + 5`バー）。
- すべてのコメントとヘルパーメソッドは、リポジトリのガイドラインに合わせて英語で書かれています。
- MT5とは異なり、ストップロスとテイクプロフィットの注文はローソク足データでシミュレートされるため、イントラバーの約定は元のEAと異なる場合があります。それ以外はすべて同じ決定ルールに従っており、移植版はソーススクリプトに忠実です。
