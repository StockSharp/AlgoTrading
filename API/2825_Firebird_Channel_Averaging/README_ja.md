# Firebirdチャネル平均化戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Firebirdチャネル平均化戦略は、StockSharpの高レベルAPIを使用してMetaTrader 5のエキスパート「Firebird v0.60」を再現します。設定可能な移動平均チャネルで取引し、価格がチャネルから離れた場合にポジションを段階的に平均化します。グリッドスタイルのエントリーとpipベースのリスクコントロールが必要な平均回帰フォレックス取引向けに設計されています。

## インジケーター設定
- 選択したローソク足シリーズで移動平均（単純、指数、平滑化、または加重）が計算されます。価格ソース（終値、高値、安値、中央値など）は設定可能です。
- 上側および下側チャネルバンドは、移動平均をユーザー定義のパーセンテージでオフセットすることによって導出されます。

## エントリーロジック
1. **買い条件**
   - 選択したローソク足ソースの価格が下側バンドを下回って終了します。
   - ポジションが存在しないか、`Step Exponent`成長を考慮した場合に新しいエントリーが最新のフィルから少なくとも`Step (pips)`離れています。
   - 戦略はエントリー間に2つのローソク足間隔のクールダウンを強制します。
2. **売り条件**
   - 価格が上側バンドを上回って終了します。
   - ロングロジックと同一の距離およびクールダウンチェックが満たされる必要があります。

有効なシグナルが発生すると、戦略は設定されたロットボリュームで成行注文を送信します。一度に1つの方向のみが維持されます — 反対のシグナルは現在の在庫がリスクルールによって閉じられるまで待機します。

## ポジション管理
- 各エントリーは、戦略がオープングリッドの平均価格を計算できるように保存されます。
- ストップロスとテイクプロフィットレベルはpipで定義されます。単一ポジションの場合、ストップロスはエントリー価格から`Stop Loss (pips)`を引いた/加えた値に等しく、テイクプロフィットはエントリー価格に`Take Profit (pips)`を加えた/引いた値に等しくなります。
- 複数のポジションが存在する場合、ストップロス距離はエントリー数で割られ、オリジナルエキスパートの平均化動作をエミュレートします。
- 利益目標は平均価格に対して固定のままですが、ストップロスエグジットは各ローソク足で再計算されます。
- 取引は金曜日にオプションで無効にすることができます。

## パラメータ
| パラメータ | 説明 |
| --- | --- |
| `Volume` | 各平均化エントリーのロット単位の注文サイズ（デフォルト0.1）。 |
| `Stop Loss (pips)` | pip単位の保護ストップ距離（デフォルト50）。 |
| `Take Profit (pips)` | pip単位のテイクプロフィット距離（デフォルト150）。 |
| `MA Period` | 移動平均のルックバック長（デフォルト10）。 |
| `MA Shift` | 移動平均出力に適用される前向きシフト（ローソク足単位）。 |
| `MA Type` | 移動平均計算メソッド：Simple、Exponential、Smoothed、またはWeighted。 |
| `Price Source` | インジケーター計算に使用されるローソク足価格（デフォルトは終値）。 |
| `Channel %` | バンドを形成するために使用される移動平均からのパーセンテージオフセット（デフォルト0.3%）。 |
| `Trade Friday` | 金曜日の取引を有効または無効にします。 |
| `Step (pips)` | 平均化注文間の最小pip距離（デフォルト30）。 |
| `Step Exponent` | オープンエントリー数に基づいてステップをスケーリングする指数（0でステップを一定に保ちます）。 |
| `Candle Type` | 作業ローソク足の時間軸。 |

## 注意事項
- 戦略は銘柄の`PriceStep`が1 pipを表すと想定しています。利用できない場合は0.0001にフォールバックします。
- 保護エグジットは、高レベルAPIとの一貫性を維持するために、ネイティブのストップ/リミット注文ではなく成行注文で実行されます。
- 平均化グリッドは、クールダウンロジックと、ゼロより大きいステップ指数が使用される場合の増加する距離によって制限されます。
