# サイベリア トレーダー AI 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、**CyberiaTrader.mq4 (ビルド 8553)** エキスパート アドバイザーの StockSharp 変換です。オリジナルの MQL プログラムは、
オプションのトレンド フィルターのコレクションを備えた確率エンジン。 C# ポートは同じ構造を維持します: 確率モデル検索
最も信頼性の高いサンプリング期間を設定し、オプションの MACD、EMA、および反転フィルターで取引を拒否できます。

## インジケーターと内部モデル

- **確率エンジン** – 候補サンプリング期間 (`MaxPeriod`) を反復し、`SamplesPerPeriod` の履歴セグメントを評価します。
各期間について、エンジンは以下を計算します。
  - サンプリング期間ごとに連続した強気/弱気の 1 分足ローソク足に基づいた意思決定の方向 (買い/売り/フラット)。
  - 買い、売り、未定義の結果の平均「可能性」の振幅と、上記の成功した結果の割合
`SpreadThreshold`。
  - 最もパフォーマンスの高い期間を選択する成功率。
- **EMA トレンド フィルター** – 現在の傾きに逆らった取引をブロックするオプションの指数移動平均 (`EnableMa`)。
- **MACD フィルター** – 勢いに逆らった取引を禁止するオプションの移動平均収束/発散 (`EnableMacd`)。
- **逆転検出器** – 確率が上回った場合に権限を反転するオプションのスパイク検出器 (`EnableReversalDetector`)
平均の `ReversalFactor` 倍です。

## パラメーター

| 名前 | 説明 |
| --- | --- |
| `MaxPeriod` | 確率エンジンによって検査された最大のサンプリング ストライド。 |
| `SamplesPerPeriod` | 期間候補ごとに処理されるセグメントの数 (MQL `ValuesPeriodCount` を反映)。 |
| `SpreadThreshold` | 成功確率の結果としてカウントされる最小振幅。 |
| `EnableCyberiaLogic` | 売買を無効にできるサイベリア確率スイッチを有効にします。 |
| `EnableMacd` | MACD 運動量フィルターを有効にします。 |
| `EnableMa` | EMA スロープ フィルターを有効にします。 |
| `EnableReversalDetector` | 極端なスパイクで許可を切り替える逆転検出機能を有効にします。 |
| `MaPeriod` | トレンド フィルターで使用される EMA の長さ。 |
| `MacdFast` / `MacdSlow` / `MacdSignal` | MACD 高速 EMA、低速 EMA、信号期間。 |
| `ReversalFactor` | 反転検出器をトリガーする乗算器。 |
| `CandleType` | モデルによって処理されるローソク足のデータ型 (デフォルトは 1 分)。 |
| `TakeProfitPercent` | パーセントで表されるオプションの保護テイクプロフィット。 |
| `StopLossPercent` | パーセントで表されるオプションの保護ストップロス。 |

## 取引ロジック

1. 完了した各ローソク足はローカル履歴キューを更新し、1 から 2 までのすべての期間の確率統計を再計算します。
`MaxPeriod`。成功率が最も高い期間がアクティブな構成になります。
2. Cyberia ロジックは、MQL コードと同じ比較を使用して、`DisableBuy`/`DisableSell` フラグを設定します。
   - 期間が増加または減少したときに、買い/売りの平均可能性とその成功加重バリアントを比較します。
   - 新しい可能性が成功した平均の 2 倍を超える場合、エントリを無効にします。
3. オプションのフィルターは、MACD、EMA スロープ、反転検出器の順に適用されます。
4. オープンなポジションがない場合、現在の決定が買い (または売り) であり、対応する可能性が超過した場合にストラテジーがエントリーされます。
逆方向が無効になっている間の成功した平均。
5. ポジションが存在する間、コードは確率エンジンが反転したとき、またはフィルターがポジションを禁止したときと同じ条件をチェックしてクローズします。
現在の方向。
6. `StartProtection` は、ゼロ以外のリスク パラメーターが指定された場合に、元の資金管理ブロックを再現します。

## 変換時の注意点

- ポートは統計計算を保持しますが、ティックベースのスプレッドチェックを構成可能な `SpreadThreshold` に置き換えます。
- MQL スクリプトからの自動ロットサイジングとバランス診断は実装されていません。 StockSharp の音量は `Volume` を介して制御されます。
- MoneyTrain モジュールと Pipsator モジュールは、高レベルの API の使用法に合わせて、上記の統合されたエントリ/エグジット ロジックに凝縮されています。
- この戦略では、デザイナーでの検証を容易にするために、ローソク足 EMA および MACD のチャート描画を追加します。
