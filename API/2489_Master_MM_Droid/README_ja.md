# Master MM Droid戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Master MM Droid戦略は、元のMetaTrader 5エキスパートアドバイザーのマルチモジュールポートです。StockSharpの実装は、ローソク足サブスクリプション、インジケーターバインディング、注文管理に高レベルAPIを使用しながら、レガシーロボットのコアアイデアを保持します。4つの独立したマネーマネジメントブロックのオン/オフが切り替え可能で、戦略はモメンタムエントリー、スケジュールされたブレイクアウト注文、週次ギャップ取引を組み合わせることができます。

## モジュール
1. **RSIブロック**
   - 設定されたローソク足タイプで14期間の相対力指数を使用します。
   - RSIが売られすぎしきい値から上にクロスしたときにロング、買われすぎレベルから下にクロスしたときにショートに入ります。
   - 固定価格ステップで区切られた設定可能な数の追加エントリーによるピラミッドを可能にします。
   - ポイント距離に基づく固定初期ストップを適用し、ポジションが開いたらトレーリングストップを有効にします。
2. **ボックスブレイクアウトブロック**
   - 1日3回ブレイクアウトボックスを再構築します（デフォルトはセッションシフト時間6、12、20）。
   - 設定可能なバッファでセッション高値の上とセッション安値の下に括弧付きストップ注文を置きます。
   - セッションリセット時（時間0、10、16）に保留中の注文とポジションをクリアし、元のエキスパートの動作を模倣します。
3. **週次ブレイクアウトブロック**
   - 月曜の価格アクションを追跡し、セッションの最初の部分の最高値/最安値の推移を保存します。
   - 週がOCOブレイクアウトで始まるよう、限られた有効化ウィンドウ（`StartHour` – `WeeklySetupEndHour`）内に対称的なストップ注文を置きます。
   - 週末エクスポージャーを避けるため、金曜日の夜にフラット状態を強制します。
4. **ギャップブロック**
   - 新しい日次始値を前日の高値/安値と比較します（シフトされたカレンダーを使用）。
   - 強いギャップダウン始値を買い、強いギャップアップ始値を売ります。
   - 設定可能な距離で保護ストップを設定し、さらなる管理をトレーリングエンジンに引き渡します。

## パラメーター
| 名前 | 説明 |
| ---- | ---- |
| `CandleType` | インジケーター計算と時間ウィンドウチェックに使用する時間軸。 |
| `TimeShiftHours` | 時間スケジュールが元のEAと一致するようにローソク足タイムスタンプに適用するセッションシフト。 |
| `StartHour` | 週次モジュールの月曜ベース開始時間（シフト適用前）。 |
| `EnableRsiModule`、`EnableBoxModule`、`EnableWeeklyModule`、`EnableGapModule` | 4つの独立したブロックのトグル。 |
| `RsiPeriod`、`RsiLowerLevel`、`RsiUpperLevel` | RSI計算とトリガーレベル。 |
| `RsiMaxEntries`、`RsiPyramidPoints` | RSIブロックのピラミッド制御。 |
| `RsiStopLossPoints`、`RsiTrailingPoints` | RSI主導トレードの初期ストップとトレーリングストップサイズ（ポイント）。 |
| `BoxEntryPoints`、`BoxTrailingPoints` | ボックス注文のブレイクアウトバッファとトレーリング距離。 |
| `WeeklyEntryPoints`、`WeeklySetupEndHour`、`WeeklyTrailingPoints` | 週次ブレイクアウト設定。 |
| `GapStopLossPoints`、`GapTrailingPoints` | ギャップモジュールの保護ストップとトレーリング距離。 |

すべてのポイントベースのパラメーターは、価格オフセットを得るために銘柄の `TickSize` で乗算されます。これにより戦略が異なるシンボルに適応します。

## 取引ロジック
- **インジケーターバインディング**: 単一のRSIインジケーターがローソク足サブスクリプションにバインドされています。完了した各ローソク足が `ProcessCandle` をトリガーし、4つのモジュールハンドラーに値を送信します。
- **日次状態追跡**: 戦略はギャップロジックをサポートし、週次モジュールの歴史的参照を保持するために、シフトされた各日の始値/高値/安値を集約します。
- **注文配置**: 注文は高レベルAPIのベストプラクティスに沿って `BuyMarket`、`SellMarket`、`BuyStop`、`SellStop` を通じて送信されます。スケジュールされたモジュールは重複を避けるために再武装する前に常にアクティブな注文をキャンセルします。
- **トレーリング管理**: ポジションがアクティブになると、`_activeTrailingPoints` がモジュール固有の距離を保存します。`UpdateTrailing` メソッドは有利な方向にのみストップ注文を移動します。

## リスク管理
- RSIとギャップモジュールによって作成された成行注文のみが、ポイントで計算された即時ストップによって保護されます。
- ブレイクアウトモジュールは有効化後にトレーリングエンジンに依存します。必要に応じて外部ポートフォリオ保護と組み合わせることができます。
- `ClosePosition()` の呼び出しがフラット化の正規の方法であり、StockSharpリスクツールとの互換性を保持します。

## 使用上の注意
- 戦略は単一の証券で動作し、サイジングにグローバルな `Volume` 値を使用します。ポジションごとのリスク制限が必要な場合は、ポートフォリオ保護を別途調整してください。
- セッション時間は `TimeShiftHours` を適用した後に評価されます。たとえば、デフォルト値 `2` では、時間 `0` でのボックスリセットはサーバー時間02:00に対応します。
- StockSharp戦略はネットポジションを管理するため、同時ロング/ショートバスケット（MetaTraderのヘッジングアカウントで可能）は統合されます。これが元のEAとの主な動作上の違いであり、検証時に考慮する必要があります。

## ログと監視
- 各モジュールはポジションがフラットに戻ると内部フラグをリセットし、オペレーターがどのブロックがトレードを生成したかを診断するのに役立ちます。
- 詳細な分析が必要な場合は、StockSharpの機能を通じてオプションのチャートやログを追加してください。
