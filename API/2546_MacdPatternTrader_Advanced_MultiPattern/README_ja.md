# MacdPatternTrader 上級マルチパターン戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
MacdPatternTrader戦略は、オリジナルのMQLエキスパートアドバイザー *MacdPatternTraderAll* のStockSharp高レベルAPI変換です。システムは完成したローソク足を聴き、6つの独立したMACDベースのエントリーパターンを評価します。各パターンは、MACD主線でのリバーサルと継続構造を認識するために、独自の速い・遅い指数移動平均と専用の閾値レベルを使用します。シグナルは同時に到着する場合があり、それぞれが現在のマーチンゲールボリュームでサイズ調整された成行注文を送信します。

戦略はエントリーロジックをアダプティブリスク管理で補完します。ストップロス価格は最近の高値または安値にオフセットを加えて計算され、テイクプロフィットターゲットはMQL実装と同様に連続した履歴ブロックを通じて延長されます。オープンポジションは、EMA/SMAフィルターと未実現利益の閾値に基づく部分エグジットによって積極的に管理されます。各フラットクローズの後、マーチンゲール乗数は実現結果に応じてリセットするかロットサイズを倍増させます。

## 取引ルール
1. **パターン1 – 閾値リバーサル**
   * MACDの主線が上限閾値を超えて上昇し、その後プラスのまま下回ったときを追跡します。
   * MACDがマイナス領域から回復するときの下限閾値の動作を反映します。
2. **パターン2 – ゼロレベルバウンス**
   * 売りの前に、まずMACDのプラス段階を要求し、次にゼロライン下への弱気なフックが必要です。
   * 買いのためにゼロ上の強気なフックに対して対称的なロジックを使用します。
3. **パターン3 – マルチステージシーケンス**
   * ネストしたフラグと閾値ペアを使用してMQLソースからの3段階のクレストとトラフ認識を再現します。
   * 各実行取引後に補助カウンター（`bars_bup`）をリセットします。
4. **パターン4 – ローカルピーク/バレー**
   * 前の2つのバーに対してMACDのローカル高値または安値を待って、それぞれショートとロングシグナルを設定します。
5. **パターン5 – ニュートラルバンドブレイクアウト**
   * ニュートラルバンドを下回った後、即座に弱気の制限以下に戻ったときのショートエントリーを探します。
   * ニュートラルバンドを上回った後、強気の制限を飛び越えたときのロングエントリーを探します。
6. **パターン6 – 連続バーカウンター**
   * 設定された閾値を上回るまたは下回るバーの数をカウントし、カウンターが`TriggerBars`値を超えながら`MaxBars`制限を下回る場合にのみトリガーします。

## リスク管理と取引管理
* **ストップロス** – ショート取引については最後の`StopLossBars`ローソク足の最高価格（ロング取引については最低価格）に設定されたオフセットを価格ステップ単位に変換して加えたもの。
* **テイクプロフィット** – MQLバージョンのネストされた`iLowest`/`iHighest`ループと全く同様に、`TakeProfitBars`ローソク足の連続した履歴セグメントを検索します。次のセグメントがより極端な値を生成する間、ターゲットは延長されます。
* **部分エグジット** – 未実現利益が5通貨単位を超え（価格差×ポジションボリュームで近似）、EMA/SMAフィルターが合意すると、戦略はオープンボリュームの3分の1をクローズし、次に残りの半分をクローズします。
* **マーチンゲールロット管理** – フラットエグジット後、クローズした取引が利益を得た場合、戦略はロットを`InitialVolume`にリセットします；そうでなければ、ボリュームが倍増します（`UseMartingale`が有効な場合）。
* **時間フィルター** – `UseTimeFilter`が有効な場合、戦略は`(StartTime, StopTime)`ウィンドウ内でのみエントリーを評価します。ストップは引き続きすべての完成したローソク足でチェックされます。

## パラメーター
| グループ | 名前 | 説明 |
| --- | --- | --- |
| パターン1 | `Pattern1Enabled` | 最初のMACDパターンを有効にします。 |
| パターン1 | `Pattern1StopLossBars`, `Pattern1TakeProfitBars`, `Pattern1Offset` | ストップロス/テイクプロフィットのルックバックとオフセット設定。 |
| パターン1 | `Pattern1Slow`, `Pattern1Fast` | MACD計算の遅いと速いEMAの長さ。 |
| パターン1 | `Pattern1MaxThreshold`, `Pattern1MinThreshold` | 上限と下限のMACD閾値。 |
| パターン2 | パターン1と同じ構造で独自の値を持ちます。 |
| パターン3 | 階層化されたクレスト/トラフ認識を再現するために追加の閾値`Pattern3MaxLowThreshold`と`Pattern3MinHighThreshold`を追加します。 |
| パターン4 | `Pattern4AdditionalBars`を含みます（元のコードとの互換性のために保持）。 |
| パターン5 | バンドブレイクアウト検出のためのニュートラル閾値を使用します。 |
| パターン6 | バーカウンターロジックを管理するために`Pattern6MaxBars`、`Pattern6MinBars`、`Pattern6TriggerBars`を追加します。 |
| 管理 | `EmaPeriod1`, `EmaPeriod2`, `SmaPeriod3`, `EmaPeriod4` | 部分エグジットフィルターの移動平均。 |
| 全般 | `InitialVolume`, `UseTimeFilter`, `StartTime`, `StopTime`, `UseMartingale`, `CandleType` | グローバルな動作制御。 |

## 注意事項
* 変換は、セグメント化されたテイクプロフィット検索とマーチンゲールリセットルールを含む元のロジック構造を維持します。
* StockSharpの高レベルAPIはポジションごとの生のターミナル利益値を公開しないため、利益ベースの部分エグジットは近似を使用します；代わりに価格差×ボリュームが使用されます。
* `Pattern4AdditionalBars`は、元のMQLコードが直接参照しなかったにもかかわらず、互換性のために保存されます。
* StockSharpは高レベルAPIで保護注文を自動的に添付しないため、ストップとテイクプロフィットは閉じたローソク足で評価されます。
