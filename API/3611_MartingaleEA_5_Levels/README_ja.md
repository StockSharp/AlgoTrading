# MartingaleEA-5 レベル戦略 (StockSharp)
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

**MartingaleEA-5 レベル戦略**は、MetaTrader 5 エキスパート アドバイザー「MartingaleEA-5 レベル」を StockSharp の高レベル API に直接移植したものです。このシステムは既存のポジションを監視し、市場がそれに反する動きをするたびに 5 段階の平均グリッドを構築します。すべてのロジックは完成したローソク足で実行され、履歴テストとライブ取引の両方で動作の再現性が保たれます。

## 取引ロジック

1. **既存のエクスポージャーの監視** – この戦略では、最初のロングまたはショート ポジションが存在することが予想されます。最初の取引は手動で行うことも、他の戦略を通じて行うこともできます。
2. **逆動き検出** – 完了したローソク足ごとに、戦略は現在の価格がアクティブ グループの最悪価格のエントリー (最高値ロングまたは最低値ショート) からどれだけ離れているかを測定します。
3. **Martingale の追加** – グループの変動損失がマイナスで、逆の動きが設定された累積距離を超える場合、ストラテジーは追加の成行注文を送信します。追加の注文ごとに、前の注文に `VolumeMultiplier` が乗算されます。最大 5 つのレベルを設定できます。 `MaxAdditions` パラメータは、実際に使用される数を制限します。
4. **損益目標** – グループがオープンしている間、戦略はその方向の未実現損益を継続的に合計します。合計が `TakeProfitCurrency` に達するか、`StopLossCurrency` を下回ると、その側のすべての注文が成行注文でクローズされ、マーチンゲール カウンターがリセットされます。
5. **数量の正規化** – すべての注文数量は商品の `VolumeStep`、`MinVolume`、および `MaxVolume` を経由して、約定不可能な数量の送信を回避します。

## パラメーター

| 名前 | 説明 | デフォルト |
| --- | --- | --- |
| `EnableMartingale` | 平均化および清算ロジックをオンまたはオフにします。 | `true` |
| `VolumeMultiplier` | 新しいレベルを追加するときに前の注文量に適用される係数。 | `2.0` |
| `MaxAdditions` | 方向ごとのマーチンゲール ステップの最大数 (最大 5)。 | `4` |
| `Level1DistancePips` | 2 番目のオーダーをオープンする前の最初の不利な距離 (pips 単位)。 | `300` |
| `Level2DistancePips` | 3次オーダーには追加の距離が必要です。 | `400` |
| `Level3DistancePips` | 4 次の場合は追加の距離が必要です。 | `500` |
| `Level4DistancePips` | 5 次の場合は追加の距離が必要です。 | `600` |
| `Level5DistancePips` | 6 次の場合は追加の距離が必要です (許可されている場合)。 | `700` |
| `TakeProfitCurrency` | グループ全体を締めくくる未実現利益（口座通貨）。 | `200` |
| `StopLossCurrency` | 緊急退場を余儀なくされる含み損（口座通貨）。 | `-500` |
| `CandleType` | 評価に使用される時間枠 (デフォルトの 1 分ローソク足)。 | `TimeFrame(1m)` |

> **ピップ換算** – すべての距離に商品価格ステップ (`PriceStep` または `MinPriceStep`) が掛けられます。小数ピップで引用されたシンボルの場合は、それに応じて値を調整します。

## 注意事項と推奨事項

- この実装は、一度に 1 つの方向バスケットのみがアクティブであるという前提を含め、元の EA を反映しています。両方向で同時にポジションをオープンすると、各側が独立して管理されます。
- この戦略はローソク足の終値にのみ反応するため、必要な反応性に一致する時間枠を選択してください。タイムフレームが低いほど、ティックレベルの動作がより厳密にエミュレートされます。
- Martingale の手法はリスクを増幅します。ライブ市場で戦略を有効にする前に、常に現実的なスリッページと手数料モデルでバックテストを行い、保守的なストップレベルを定義してください。
- この戦略ではまだ Python ポートを作成しません。要求に応じて、C# の高レベル実装のみが含まれます。
