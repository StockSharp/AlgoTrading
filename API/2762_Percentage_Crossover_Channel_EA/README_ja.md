# パーセンテージクロスオーバーチャネル戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
パーセンテージクロスオーバーチャネル戦略は、MetaTrader 5のエキスパートアドバイザー *Percentage_Crossover_Channel_EA* に由来します。高速移動平均を中心に構築されたカスタムチャネルに依存し、バンドタッチまたは中央線クロスオーバーに反応します。このStockSharp実装は、ハイレベルAPIを使用して完了済みローソク足を処理しながら同じロジックに従います。

## チャネルの構築
基礎となるインジケーターは、選択された価格 (デフォルトでは終値) を中心に動的チャネルを構築します：

1. 設定された **Applied Price** モードを使用してベース価格を計算する。
2. 1期間の単純移動平均を適用して短期参照価格を得る。
3. **Percent** パラメーターを使用して2つの境界を計算する (例：50 → ±0.5%)。
4. 新しい境界内に前の中央線を制限して現在の中央値を得る。
5. 上下のバンドは制限された中央値に ±パーセント係数を掛けたものです。

この再帰により、価格が強いトレンド中にチャネルが遅れる一方、価格が統合するときには密接なエンベロープを維持できます。

## トレーディングロジック
2つの異なるシグナルモードが利用可能です：

- **バンドタッチモード (デフォルト):**
  - 前のローソク足の安値が下限バンドを上回っており、最後に完了したローソク足がそれに触れるか貫通する場合にロングエントリー。
  - 前のローソク足の高値が上限バンドを下回っており、最後に完了したローソク足がそれに触れるか貫通する場合にショートエントリー。
- **中央線クロスオーバーモード (TradeOnMiddleCross = true):**
  - 価格が中央線を上から下へクロスするときにロングエントリー。
  - 価格が中央線を下から上へクロスするときにショートエントリー。

**ReverseSignals** フラグはロングとショートのルールを入れ替えます。戦略は常に既存のポジションを閉じて反転させ、設定された **OrderVolume** と現在のポジションの絶対値の合計のボリュームで単一の成行注文を送信します。

## リスク管理
オプションの保護レベルは元のMT5のストップロスとテイクプロフィット設定をエミュレートします：

- **StopLossPoints** – 推定エントリー価格から差し引かれる (ロング) またはそれに追加される (ショート) 価格ステップでの距離。
- **TakeProfitPoints** – エントリー価格に追加される (ロング) またはそこから差し引かれる (ショート) 価格ステップでの距離。

いずれかのパラメーターがゼロの場合、対応する保護は無効になります。ストップは、ローソク足の高値と安値を保存されたレベルと比較することで、完了した各ローソク足で評価されます。トレーリングロジックは適用されません。

## パラメーター
| パラメーター | 説明 |
|-----------|-------------|
| `CandleType` | サブスクライブするローソク足データタイプ (デフォルトは15分の時間軸)。 |
| `Percent` | 価格のパーセントでのチャネル幅 (±パーセント/100係数に変換)。 |
| `PriceMode` | チャネルに使用する適用価格。オプション：Close、Open、High、Low、Median (H+L)/2、Typical (H+L+C)/3、Weighted (H+L+2C)/4、Average (O+H+L+C)/4。 |
| `TradeOnMiddleCross` | バンドタッチロジックと中央線クロスオーバーロジックを切り替える。 |
| `ReverseSignals` | ロングとショートの条件を反転させる。 |
| `StopLossPoints` | 証券の価格ステップで表現された保護ストップ距離。 |
| `TakeProfitPoints` | 証券の価格ステップで表現された利益目標距離。 |
| `OrderVolume` | 成行エントリーのベースボリューム。戦略は1つのトランザクションで反転するために絶対オープンポジションを追加します。 |

## 実装メモ
- 注文はローソク足が終了した後のみ発行されます。これは前のバーのデータを使って次のバーの開始時に行動したMT5エキスパートを反映しています。
- チャネルインジケーターは履歴コレクションを保存せずに戦略内で再作成され、スカラー状態変数に依存しています。
- 保護ストップとターゲットは手動で確認され、MT5からのプラットフォーム固有の注文処理を再現します。
- 選択したインストゥルメントが有効な `PriceStep` を公開していることを確認してください。そうでない場合、ストップロスとテイクプロフィットの距離は無視されます。
