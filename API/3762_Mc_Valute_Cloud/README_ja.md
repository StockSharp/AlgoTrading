# マックバリューのクラウド戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

このフォルダには、MetaTrader エキスパート アドバイザー「Mc_valute」の StockSharp ポートが含まれています。ショート丈を組み合わせたオリジナルロボット
3 つの平滑化移動平均、Ichimoku クラウド フィルター、および複数の MACD インスタンスを含む指数移動平均 (EMA)
トレンドにスケーリングします。 StockSharp の実装は、コアのトレンド確認スタックを維持しながら、ポジション管理を簡素化します
ロジックが高レベルの API に自然に適合するように、各方向で単一の露出に対応します。

## 取引ロジック

1. **価格フィルター EMA** – `FilterMaLength` EMA は、2 つの滑らかな動きの上（ロングの場合）または下（ショートの場合）に配置する必要があります
平均 (`BlueMaLength` と `LimeMaLength`)。平滑化された平均は、MT4 テンプレートの「青」と「ライム」の線をエミュレートします。
2. **Ichimoku クラウドの確認** – EMA もクラウドの外にある必要があります。ロングトレードには両方の上にフィルター EMA が必要です
センコウはスパンですが、ショートトレードでは雲底を下回ることが要求されます。
3. **MACD の勢いチェック** – メインの MACD ラインは、ロングエントリーの場合はシグナルラインより上にあり、ショートエントリーの場合はシグナルラインより下にある必要があります。
残りのコピーは最終的な MQL バージョンで無効になっているため、元の EA の最初の MACD セットのみが保持されます。
4. **単一ポジション管理** – 新しいシグナルが現れるたびに、戦略は既存の反対ポジションを相殺し、ポジションをオープンします。
設定された `Volume` との新たな取引。保護注文は成行注文が送信された直後に更新されます。
5. **ローソクごとの評価** – すべてのインジケーターは、`CandleType` で定義された時間枠で動作します。取引に関する決定が下される
閉じたバーを処理した MT4 `start()` ハンドラーをミラーリングするために、終了したローソク足でのみ使用します。

## リスク管理

- `TakeProfit` と `StopLoss` は価格ポイントで測定されます。各エントリの後に、ヘルパー `SetTakeProfit` と `SetStopLoss`
関数は、予想される結果のポジションサイズを使用して呼び出されます。これは、ストップが適用されたMT4の動作を反映しています。
チケット。
- 元のエキスパートアドバイザーは、`Step` の距離を使用して、最大 3 つの追加注文をピラミッド化しました。 StockSharp ポートは
上位レベルのオーダーヘルパー内に留まる単一のポジション。スケーリングが必要なユーザーは、`Volume` を増やすか、クローンを作成できます。
複数のポートフォリオにわたる戦略。

## パラメーター

| パラメータ | 説明 |
| --- | --- |
| `Volume` | 高レベルの `BuyMarket`/`SellMarket` 呼び出しで使用される基本取引サイズ。 |
| `CandleType` | インジケーターとトレードロジックを動かす主要なローソク足シリーズ。 |
| `FilterMaLength` | EMA トレンド フィルターの長さ。 |
| `BlueMaLength`, `LimeMaLength` | 方向性バンドとして機能する 2 つの平滑化移動平均の長さ。 |
| `MacdFastLength`, `MacdSlowLength`, `MacdSignalLength` | MACD 確認の EMA の長さ。 |
| `TenkanLength`, `KijunLength`, `SenkouLength` | Ichimoku クラウドフィルターのKinko Hyo設定。 |
| `TakeProfit`, `StopLoss` | 保護距離は価格で表されます。 |

## 使用上の注意

1. **インジケーター シフト** – MetaTrader では、平滑化移動平均を作成するときにゼロ以外の「シフト」パラメーターが許可されました。 StockSharpさん
インジケーターは現在のバーで動作するため、ポートは元の期間を維持しながらこれらのシフトを無視します。
2. **MACD のバリアント** – ソース コードでは 3 つの MACD ブロックが宣言されていますが、ライブ信号に参加するのは最初のブロックのみです。港
その行動に従います。追加の MACD フィルターは、インジケーター バインディングを複製することで再度有効にすることができます。
3. **取引のスケーリング** – MT4 ロボットは、`Step` ポイントで区切られた最大 3 つの平均注文を送信しました。この動作は文書化されています
ただし、高レベルの戦略は単一の集約されたポジションで機能するため、意図的に省略されています。
4. **保護ブロック** – `StartProtection()` は起動時に 1 回呼び出され、組み込みインフラストラクチャが停止を監視します。
再接続後も注文をターゲットにします。
