# SilverTrend V3戦略 (C#)
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

SilverTrend V3戦略は、MetaTrader 5のエキスパートアドバイザー「SilverTrend v3」を起源とするモメンタムフォローシステムです。StockSharpへのポートは、高レベル戦略APIに適合させながら元のロジックを再現しています。中心的なアイデアは、SilverTrendチャネル計算を使って強気または弱気のモメンタムを検出し、J_TPOマーケットプロファイルオシレーターで確認し、保護ストップ、トレーリングロジック、および金曜日セッションフィルターで結果ポジションを管理することです。

## シグナルエンジン

1. **SilverTrendの方向**
   - 動的サポート（`smin`）とレジスタンス（`smax`）を計算するために、9バーのスムージングパラメーターを持つ350バーのローリングウィンドウを使用します。
   - 現在の終値が`smin`を下回ると、システムは弱気レジームにフラグを立て、`smax`を上回る終値は強気レジームに切り替わります。
   - 計算は最古のバーから最新のバーへ反復し、元のMQLコードの再帰的な性質を再現します。

2. **J_TPOの確認**
   - 価格が短期分布内にどのように集中するかを測定する、元の14期間J_TPOオシレーターを実装しています。
   - オシレーターが正の場合のみロングエントリーを許可し、負の場合のみショートエントリーを許可することで、弱いモメンタムシフトをフィルタリングします。

3. **シグナル変化の検出**
   - 新たに計算されたSilverTrendの方向が前回の値と異なる場合にのみトレードが開始され、戦略がノイズではなく真のレジームシフトに反応することを保証します。

## トレード管理

- **成行エントリー** – 戦略は設定された`Volume`でトレードします。反対のポジションが開いている場合、1つの成行注文で決済して反転します。
- **初期ストップロス** – オプション。エントリー価格に対する価格ステップで定義されます（インストゥルメントの`PriceStep`で変換）。
- **テイクプロフィット** – オプション。価格ステップで定義され、元の注文変更動作を模倣するためにローソク足の高安値に対して評価されます。
- **トレーリングストップ** – 設定されたトレーリング距離だけ価格が有利方向に動いた後にアクティブになります。ロングポジションではストップが上方に引き上げられ、ショートでは下方に引き下げられ、MetaTraderのロジックと一致します。
- **反対シグナルでの決済** – 前のレジームが反対方向を指している場合、既存のポジションは次のローソク足の終値で清算されます。
- **金曜日トレードブロック** – 週末ギャップを避けるために、元のEAと同様に、金曜日の指定時刻以降は新規ポジションをスキップします。

## パラメーター

| 名前 | デフォルト値 | 説明 |
| --- | --- | --- |
| `TrailingStopPoints` | 50 | 価格ステップで測定したトレーリングストップ距離。トレーリングを無効にするにはゼロに設定。 |
| `TakeProfitPoints` | 50 | 価格ステップでのテイクプロフィット距離。ゼロはターゲットを無効にします。 |
| `InitialStopLossPoints` | 0 | 価格ステップでの初期保護ストップ。ゼロは初期ストップなしでポジションを残します。 |
| `FridayCutoffHour` | 16 | 金曜日に新規エントリーが許可されない取引所時刻。終日取引を許可するには`0`を使用。 |
| `CandleType` | 1時間足 | インジケーターに供給するデータシリーズ。サポートされている任意の時間軸を使用できます。 |
| `Volume` | 1ロット | 各ポジションのトレードサイズ（StockSharpの`Volume`プロパティ）。 |

すべての距離は実行時に`PriceStep`で乗算され、銘柄のティックサイズ（3/5桁のフォレックスシンボルを含む）に自動的に適合します。

## データと環境要件

- SilverTrendとJ_TPOの両方のバッファが完全に形成されるよう、ライブシグナルが生成される前に少なくとも360本の完成したローソク足が必要です。
- `SubscribeCandles`による単一インストゥルメントのトレーディング向けに設計されています。`GetWorkingSecurities`オーバーライドにより、戦略は設定された銘柄と時間軸のみを購読することが保証されます。
- 起動時に一度`StartProtection()`を使用して、標準のStockSharpポジション保護サービスを有効にします。

## 使用上の注意

- アルゴリズムは主要なフォレックスペアや流動性の高い先物などのトレンドインストゥルメントを想定しています。市場のボラティリティに合わせて時間軸を調整してください。
- SilverTrendの計算は再帰的なため、履歴ローソク足が不十分な状態で戦略を再起動すると、十分なデータが収集されるまでシグナル形成が遅延します。
- 高レベルAPI実装はローソク足の高安値を使用して注文管理（ストップロス、テイクプロフィット、トレーリング）をシミュレートします。ライブトレーディングでは、インフラが必要とする場合、実際のストップ/リミット注文とロジックを組み合わせることを検討してください。
- ポートは内部状態（`_previousSignal`、`_entryPrice`、トレーリングストップ）を完了したローソク足ごとに正確に1回格納し、元のEAの「1バーあたり1トレード」の動作と一致します。

## 変換の詳細

- ネストされた配列J_TPOアルゴリズムを含む`SilverTrend v3.mq5`の数学的ルーティンを忠実に再現しています。
- C#のベストプラクティスを適用：パラメーターは`StrategyParam<T>`を通じて公開され、すべてのコメントは英語で、インデントはリポジトリのガイドラインに従いタブを使用しています。
- タスク要件に従い、このリリースにはPythonバージョンは含まれていません。
