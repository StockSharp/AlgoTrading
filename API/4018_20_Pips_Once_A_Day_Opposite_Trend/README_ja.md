# TwentyPipsOnceADay戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

MetaTrader エキスパート **20pipsOnceADayOppositeLastNHourTrend** のポートで、StockSharp の高レベル API を実装しました。この戦略は、設定された時間ごとに 1 回取引し、最後の `N` 時間足ローソク足のドリフトに対して逆張りポジションをオープンします。ポジションサイズはマーチンゲールラダーに従い、最近の取引が損失で終了した場合にのみロットが増加します。この実装では、毎日の取引スケジュール、オプションのトレーリング保護、最大保有期間も強制されます。

## 取引ロジック

1. この戦略は時間足ローソク足をサブスクライブします (`CandleType` を通じて設定可能)。
2. ローソク足が終了し、次の時間が `TradingHour` と一致すると、戦略は方向を評価します。
   - 完了した最後の 1 時間の終値と `HoursToCheckTrend` 時間前の終値を比較します。
   - 市場がその期間内に下落した場合は、ロングポジションをオープンします（弱気ドリフトを弱めます）。
   - 市場が上昇した場合は、ショートポジションをオープンします。
3. 一度にアクティブにできるポジションは 1 つだけです (`MaxOrders` によって制御されます)。
4. 各取引は、固定のテイクプロフィットとオプションのストップロス/トレーリングストップを継承し、どちらも商品のピップサイズに対する相対的なピップで表されます。
5. ポジションが `OrderMaxAgeSeconds` より長くオープンのままである場合、または次の 1 時間が `TradingDayHours` で定義された許可されたセッションの範囲外である場合、ストラテジーは取引を強制的に終了します。

## 資金管理

- `FixedVolume` は基本ロットを定義します。 `RiskPercent` を使用してポートフォリオ値からロットを導出するには、これを `0` に設定します。リスクベースのサイジングは、元の EA ロジック: `(portfolio value * RiskPercent) / 1000` を反映しています。
- 基本ロットが計算されると、商品の `VolumeMin/VolumeMax/VolumeStep` とユーザー定義の `MinVolume` / `MaxVolume` の両方の境界によってクランプされます。
- マーチンゲール ラダーは、それぞれの過去の取引が損失で終了した場合にのみ、次のロットを増やします。
  - `FirstMultiplier` は、最新の取引が失われた場合に適用されます。
  - `SecondMultiplier` は、最新の取引が勝ったが、前の取引が負けた場合に適用されます。
  - チェーンは `FifthMultiplier` まで続き、元の 5 段階のエスカレーションと一致します。

## パラメーター

| パラメータ | 説明 |
|-----------|-------------|
| `FixedVolume` | 固定取引量。リスクベースのサイジングを有効にするには、`0` を使用します。 |
| `MinVolume` / `MaxVolume` | サイズ変更後に下限と上限が適用されます。 |
| `RiskPercent` | `FixedVolume` がゼロに等しい場合、ポートフォリオのパーセンテージが出来高に変換されます。 |
| `MaxOrders` | 同時にオープンするポジションの最大数 (デフォルトは `1`)。 |
| `TradingHour` | 新しい取引を開始できる時刻 (0 ～ 23)。 |
| `TradingDayHours` | オープンポジションの資格を維持できるカンマ区切りの時間または範囲（例: `0-7,13-22`）。次の 1 時間がこのセットの範囲外になると、戦略は終了します。 |
| `HoursToCheckTrend` | 逆張り比較に使用される時間足ローソク足でのルックバック。 |
| `OrderMaxAgeSeconds` | 強制終了するまでの最大保持時間 (秒単位)。 |
| `FirstMultiplier` … `FifthMultiplier` | 過去 5 つの終了した取引で見つかった損失に割り当てられた乗数は Martingale です。 |
| `StopLossPips` | 初期ストップロス距離 (pips)。無効にするには、`0` に設定します。 |
| `TrailingStopPips` | トレーリングストップの距離 (pips)。無効にするには、`0` に設定します。 |
| `TakeProfitPips` | 利食い距離をピップス単位で指定します。 |
| `CandleType` | シグナル生成に使用されるローソク足タイプ (デフォルトは 1 時間枠)。 |

## リスク管理と出口

- **テイクプロフィット / ストップロス**: `TakeProfitPips` および `StopLossPips` を通じて設定され、商品価格単位への自動変換が行われます。
- **トレーリングストップ**: 有効にすると、設定されたピップ数を超えて取引が増加すると、ストップがトレーリングされます。
- **タイムアウト終了**: `OrderMaxAgeSeconds` より古いポジションは、現在のローソク足の終値でクローズされます。
- **セッション フィルター**: 次の時間が `TradingDayHours` に含まれていない場合、ポジションはクローズされます。

## 使用上の注意

- この戦略は、時間足ローソク足と有効な `PriceStep` を提供するあらゆる商品で機能します。機器が小数ピップ (小数点以下 3 桁または 5 桁) を使用する場合、ピップ サイズは自動的に調整されます。
- MetaTrader の動作を再現するには、`CandleType` を時間枠に設定して単一の商品で戦略を実行し、デフォルトの `TradingDayHours` (0-23) を維持して、1 日を通して取引できるようにします。
- マーチンゲール ラダーは、最大 5 つの関連する過去の取引を想定しています。戦略をリセットすると、この履歴が消去されます。
- この戦略はクローズドローソク足データを使用して設定された時間の開始時に取引されるため、新しい時間の開始時に利用可能な価格で約定が発生します。
