# トレーリングストップとテイク戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**トレーリングストップとテイク戦略**は、`MQL/19963` の MetaTrader エキスパートアドバイザーの直接 StockSharp への適応です。アクティブなトレード管理に焦点を当てています：ポジションが開かれると、戦略は初期ストップロスとテイクプロフィットレベルを設定し、価格が動くにつれて両方のレベルを追従させます。トレーリング調整は設定可能な最小ステップサイズ、ブレークイーブン保護、およびトレードがまだ損失状態にある間はトレーリングを避けるオプションを遵守します。

戦略は完成したローソク足を使用して単一の証券で動作します。戦略がフラットの場合、最新のローソク足のボディの方向にポジションを開きます（強気のクローズはロングにつながり、弱気のクローズはショートにつながります）。これは MQL スクリプトが使用した元のテスト動作を反映し、トレーリングエンジンが管理するための継続的なポジションのフローを提供します。

## 仕組み
1. 設定されたローソク足タイプをサブスクライブし、完成したローソク足のみを処理します。
2. ポジションが開いていない場合、強気のローソク足でロングまたは弱気のローソク足でショートに入ります（ポジションタイプフィルターを遵守）。
3. 新しいポジションで、`InitialStopLossPoints`/`InitialTakeProfitPoints` を使用してストップロスとテイクプロフィットの距離を初期化します。これらがゼロの場合、代わりにトレーリング距離が使用されます。
4. 各ローソク足のクローズで、更新されたトレーリングターゲットを計算します：
   - ストップは市場がトレーリングステップ分進んだ後にのみ価格に近づきます。
   - テイクプロフィットは価格が少なくともトレーリングステップ分引き戻された時に近づきます。
   - `AllowTrailingLoss` が無効な場合、ブレークイーブン保護によってレベルが損失ゾーンに移動するのを防ぎます。
5. 価格がトレーリングストップまたはテイクプロフィットレベルを越えた時、成行注文で出口し、すべての保存されたレベルをリセットします。

## トレーリングロジック
### ロングポジション
- 初期ストップはエントリーから少なくとも `SpreadMultiplier * PriceStep` の距離に制限されます。
- 初期テイクプロフィットはエントリーより少なくとも同じ最小距離上に位置します。
- トレーリングストップはトレーリングステップとオプションのブレークイーブンフィルターを遵守しながら、`TrailingStopLossPoints` だけクローズ価格を下方に追従します。
- トレーリングテイクプロフィットは価格が引き戻された時に締まり、損失のトレーリングが許可されていない場合はブレークイーブンレベルを下回りません。

### ショートポジション
- 初期ストップはエントリーの上に設定され、スプレッド乗数距離より近くはなりません。
- 初期テイクプロフィットはエントリーの下から同じ最小距離ルールで開始します。
- トレーリングストップは価格が下落するにつれて下落しますが、損失トレーリングが許可されていない限りブレークイーブンより高く移動しません。
- トレーリングテイクプロフィットは引き戻しで価格に向かって上昇し、必要に応じてブレークイーブンに制限されます。

## パラメーター
| パラメーター | 説明 |
|------------|------|
| `CandleType` | 価格評価に使用されるローソク足の集約。 |
| `Volume` | エントリーとエグジットのデフォルト注文数量。 |
| `PositionType` | エンジンをロングポジション、ショートポジション、または両方の管理に制限します。 |
| `InitialStopLossPoints` | 価格ポイント単位の初期ストップロスサイズ（ゼロの場合はトレーリング距離を使用）。 |
| `InitialTakeProfitPoints` | 価格ポイント単位の初期テイクプロフィットサイズ（ゼロの場合はトレーリング距離を使用）。 |
| `TrailingStopLossPoints` | 価格とトレーリングストップ間の距離。 |
| `TrailingTakeProfitPoints` | 価格とトレーリングテイクプロフィット間の距離。 |
| `TrailingStepPoints` | ストップまたはターゲットを調整する前に必要な最小ポイント移動。 |
| `AllowTrailingLoss` | トレードがまだブレークイーブン以下にある間もトレーリングを有効にします。 |
| `BreakevenPoints` | ブレークイーブンバリアを形成するためにエントリー価格に追加するポイントオフセット。 |
| `SpreadMultiplier` | 最小ストップ距離近似のための乗数（MQL の `StopLevel` をシミュレート）。 |

## 注意事項
- ストップとターゲットはトリガーされた時に成行注文で実行され、実装をシンプルに保ち、元のストップ変更を反映します。
- `SpreadMultiplier` は、ストップレベルが現在のスプレッドより近くに配置できないという MQL の動作を近似します。実行場所に合わせてこの値を調整してください。
- 戦略は意図的に Python バージョンを避け、要求通り C# 実装のみに焦点を当てています。
- 必要に応じて、組み込みエントリーを無効にして外部注文を注入することで、トレーリングエンジンを独自のエントリーフィルターと組み合わせることを検討してください。
