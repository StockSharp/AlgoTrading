# バタフライパターン戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要

**バタフライ パターン戦略**は、元の MetaTrader「Cypher EA」高調波パターン ロジックを StockSharp の高レベルの API に変換します。この戦略は、設定可能なローソク足シリーズをスキャンして強気および弱気のバタフライ形成を調べ、調和比率を検証し、3 段階のテイクプロフィットターゲットで市場ポジションをオープンします。オプションのリスク管理機能は、MetaTrader の専門家を反映しています。損益分岐点ロックとトレーリング ストップの更新は、部分的な終了後に利用できます。

## 仕組み

1. ローソク足は、`PivotLeft`/`PivotRight` ウィンドウを使用してピボット ポイントを確認できるまでバッファリングされます。
2. 5 つの交互ピボットが利用可能な場合、ストラテジーはバタフライ パターンに必要な Fibonacci 比率をチェックします。
3. 認定されたセットアップは再検証され (オプション)、高調波品質スコア (`MinPatternQuality`) によって評価されます。
4. 閉じたキャンドルでパターンが確認されたら、次のようにします。
   - 成行注文は、固定数量またはリスクベースのサイジングを使用して発注されます。
   - ポジション量は 3 つのテイクプロフィットレベル (`TP1/TP2/TP3`) に分割されます。
   - 幾何学的なストップロスはパターン構造から導き出されます。
5. ポジションの存続期間中、戦略はローソク足を監視して、設定されたしきい値に従って部分的な出口、損益分岐点ロック、およびトレーリング調整をトリガーします。

> **ヒント:** MetaTrader バージョンは複数のタイムフレームで同時に動作します。この動作を StockSharp で再現するには、異なる `CandleType` 値を使用して戦略のインスタンスをいくつか起動します。

## 主要パラメータ

| パラメータ | 説明 |
| --- | --- |
| `CandleType` | ピボットとパターンの検出に使用されるタイムフレーム。 |
| `PivotLeft` / `PivotRight` | ピボットの高値/安値を確認するために必要な左/右のローソク足の数。 |
| `Tolerance` | バタフライ パターンの検証時に許容される最大高調波比偏差。 |
| `AllowTrading` | パターン確認後の注文生成を有効または無効にします。 |
| `UseFixedVolume` / `FixedVolume` | 一定の取引量を強制します。無効にすると、戦略は `RiskPercent` を介してポジションのサイズを決定します。 |
| `RiskPercent` | 取引ごとにリスクがかかるポートフォリオ値の割合 (`UseFixedVolume` が false の場合にのみ使用されます)。 |
| `AdjustLotsForTakeProfits` | 部分ボリュームを正規化して、合計がエントリ サイズと一致するようにします。 |
| `Tp1Percent` / `Tp2Percent` / `Tp3Percent` | 3 つのテイクプロフィットレベル間の総量の分布。 |
| `MinPatternQuality` | 検出されたパターンを受け入れるために必要な最小高調波スコア (0 ～ 1)。 |
| `UseSessionFilter`, `SessionStartHour`, `SessionEndHour` | 取引を特定の取引所セッションウィンドウに制限します。 |
| `RevalidatePattern` | ポジションをオープンする前に二次価格チェックを強制します。 |
| `UseBreakEven`, `BreakEvenAfterTp`, `BreakEvenTrigger`, `BreakEvenProfit` | 指定されたテイクプロフィットレベルと追加の利益バッファー後の損益分岐点のアクティブ化を制御します。 |
| `UseTrailingStop`, `TrailAfterTp`, `TrailStart`, `TrailStep` | テイクプロフィットレベルに達し、最小の有利なエクスカーションが達成されたら、トレーリングストップを有効にします。 |

## リスク管理

- ストップロス、損益分岐点、およびトレーリングレベルは、追加の注文を作成することなく内部で管理されます。部分的な決済とストップクローズは、MetaTrader ロジックをエミュレートする成行注文でトリガーされます。
- `UseFixedVolume` が無効な場合、ポジション サイズは停止距離、商品ティック値、および `RiskPercent` 設定から計算されます。

## 使用上の注意

- 接続された機器が設定された `CandleType` と価格ステップをサポートしていることを確認してください。サポートされていない場合、最小距離チェックにより検証ロジックが信号を拒否する可能性があります。
- 損益分岐点フィーチャとトレーリングフィーチャでは、それぞれのテイクプロフィットレベルを満たす必要があります (`BreakEvenAfterTp` および `TrailAfterTp`)。
- 複数の戦略インスタンスを異なる証券またはタイムフレームで同時に実行して、元の EA のマルチタイムフレーム スキャンを再現できます。
