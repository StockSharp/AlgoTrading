# iCCI iMA 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は MetaTrader のエキスパートアドバイザー「iCCI iMA」を複製します。Commodity Channel Index（CCI）のクロスオーバーを、CCI ストリームに直接適用された指数移動平均（EMA）に対してトレードします。独自の期間で計算されるセカンダリ CCI が、±100 バンド周辺の買われ過ぎ/売られ過ぎの反転を監視します。注文はロットでサイズが設定され、オプションで口座残高によってスケーリングされ、各トレードは pip で表現された設定可能なストップロスとテイクプロフィットレベルで保護されます。

## 仕組み
* **データソース** – 設定可能なローソク足シリーズ（デフォルトで 1 分足ローソク足）が、ローソク足の典型的な価格 `(high + low + close) / 3` を使用してすべてのインジケーター計算を供給します。
* **コアインジケーター** – プライマリ CCI は `CciPeriod` の長さでモメンタムを測定します。その CCI の EMA（長さ `MaPeriod`）がオシレーターを平滑化し、シグナルラインとして機能します。セカンダリ `CciClosePeriod` CCI が閾値クロスオーバーを監視します。
* **エントリーロジック** – 現在の CCI がその EMA を上回り、2 本の完成したローソク足前の値が EMA を下回っていた場合にロングポジションがオープンし、上向きクロスオーバーを示します。ショートポジションは CCI が下向きにクロスするときにこのロジックを反映します。アルゴリズムは、すべてのインジケーターが完全に形成され、2 つの履歴バーが利用可能になってから、MQL 実装の元のルックバックを再現するためにのみトレードします。
* **エグジットロジック** – 既存のロングは、セカンダリ CCI が +100 を下回って戻るとき、または 2 バー前に EMA の上にあった後にプライマリ CCI が EMA を下回って下落するときにクローズします。ショートは、セカンダリ CCI が −100 を超えて上昇するとき、または同じ 2 バー確認の下で CCI が EMA の上に戻って上昇するときに出ます。保護ストップは完成した各ローソク足を監視します：ロングポジションは価格が `entry − stopLossPips * pipSize` まで下がるとクローズし、`entry + takeProfitPips * pipSize` で利益を取ります；ショートは `entry + stopLoss` と `entry − takeProfit` で対称レベルを使用します。pip サイズは有価証券の価格ステップから導出され、ティックサイズに 10 を掛けることで 3 桁または 5 桁の見積もりに適応し、MetaTrader の変換に一致します。
* **ポジションサイジング** – ベースロットサイズ（`LotSize`）は、注文が取引所の制約を尊重するように商品の `VolumeStep`、`MinVolume`、`MaxVolume` の値に対して検証されます。マネー管理が有効な場合、戦略は口座残高を `DepositPerLot` で割った整数係数（20 に制限）をロットサイズに掛け、各バーで更新し、元のエキスパートからの整数ステップスケーリングを再現します。

## パラメーター
- **ローソク足タイプ** – インジケーター計算に使用されるデータシリーズ。
- **CCI 期間** – クロスオーバーシグナルを駆動するプライマリ CCI の長さ。
- **CCI クローズ期間** – ±100 反転を監視するために使用されるセカンダリ CCI の長さ。
- **CCI EMA 期間** – プライマリ CCI 値を平滑化する EMA の期間。
- **ロットサイズ** – スケーリング前のロット単位のベース取引ボリューム。
- **マネー管理を有効化** – 残高ベースのロットサイズスケーリングを切り替えます。
- **ロットあたり預金** – ロット乗数を 1 増やすために必要な残高増分（マネー管理が有効な場合のみアクティブ）。
- **ストップロス (pips)** – pip 単位の保護ストップ距離；無効化するにはゼロに設定。
- **テイクプロフィット (pips)** – pip 単位の利益目標距離；無効化するにはゼロに設定。

アルゴリズムは、2 バークロスオーバー比較がソース MQL ロジックと一致するように、取引を開始する前に 2 本の完全に完成したローソク足を必要とします。ストップロスとテイクプロフィットのチェックは、高値/安値の極値を使用して閉じたローソク足で評価され、StockSharp の高レベル API 内で MetaTrader のサーバー側保護注文を近似します。
