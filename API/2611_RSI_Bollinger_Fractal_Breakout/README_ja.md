# RSIボリンジャーフラクタルブレイクアウト戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
この戦略はMetaTraderのエキスパートアドバイザー「RSI and Bollinger Bands」をStockSharpで再現します。RSIオシレーターにボリンジャーバンドを適用し、最近のフラクタルブレイクアウトレベルを待って設定可能なオフセットでそのレベルを超えたところにストップ注文を配置します。Parabolic SARトレーリングフィルターがポジションがオープンになったらストップを動的に引き締めます。

## インジケーターとシグナル
- **RSI**（デフォルト8期間）– メインオシレーター。買われすぎと売られすぎの閾値が保留注文をキャンセルするために使用されます。
- **RSI上のボリンジャーバンド**（デフォルト14期間、偏差1.0）– RSIが上限または下限バンドの外でクローズしたときのみエントリーがトリガーされます。ボリンジャーがRSI値で供給される元のスクリプト動作に対応しています。
- **ビル・ウィリアムズフラクタル**– 戦略は最後の確認された上昇・下降フラクタル（5バーパターン）をスキャンし、そのプライスをベースブレイクアウトレベルとして使用します。
- **Parabolic SAR**（ステップ0.003、最大0.2）– ポジションがアクティブになったらトレーリングストップ参照を提供します。

## エントリーロジック
1. 選択した時間軸（デフォルト4時間）の完成したロウソク足で処理が行われます。
2. **上昇フラクタル**が現れ、RSIが**ボリンジャー上限バンド**より上でクローズし、前のクローズがフラクタルより下に留まっている場合、**バイストップ**が配置されます：
   - エントリー価格 = フラクタル高値 + indent（デフォルト15pips）。
   - オプションのストップロス = エントリー − StopLossPips。
   - オプションのテイクプロフィット = エントリー + TakeProfitPips。
3. 対称的に、**下降フラクタル**が形成され、RSIが**ボリンジャー下限バンド**より下でクローズし、前のクローズがフラクタルより上に留まっている場合、フラクタルの下に**セルストップ**が配置されます。
4. RSIがチャネル内に戻ると保留注文がキャンセルされます：
   - RSI < 下限閾値でバイストップがキャンセル。
   - RSI > 上限閾値でセルストップがキャンセル。

## エグジットとリスク管理
- 固定ストップロスとテイクプロフィット距離（pips単位）がMQLの入力を再現します。任意の距離を`0`に設定するとその保護が無効になります。
- Parabolic SARトレーリングロジックは、SARが現在価格から少なくとも`SarTrailingPips`離れている必要があり、有利な方向にのみストップを動かします。
- トレーリングストップが価格を横切るか、価格が固定テイクプロフィットに達したとき、ポジションは成行注文でクローズされます。
- ポジションをオープンすると反対の保留注文が自動的にクリアされ、意図した保護レベルが保存されます。

## パラメーター
| パラメーター | 説明 | デフォルト |
| --- | --- | --- |
| `RsiPeriod` | RSI平滑化の長さ。 | 8 |
| `BandsPeriod` | RSI上のボリンジャー期間。 | 14 |
| `BandsDeviation` | RSI上のボリンジャーの標準偏差乗数。 | 1.0 |
| `SarStep` | Parabolic SAR加速ステップ。 | 0.003 |
| `SarMax` | Parabolic SAR最大加速。 | 0.2 |
| `TakeProfitPips` | pips単位のテイクプロフィット距離。 | 50 |
| `StopLossPips` | pips単位のストップロス距離。 | 135 |
| `IndentPips` | ストップ注文を配置する前のフラクタルを超えたオフセット。 | 15 |
| `RsiUpper` | セルストップをキャンセルするRSI閾値。 | 70 |
| `RsiLower` | バイストップをキャンセルするRSI閾値。 | 30 |
| `SarTrailingPips` | トレーリング前の価格とSAR間の最小ギャップ（pips単位）。 | 10 |
| `CandleType` | 処理用のデータタイプ/時間軸。 | 4時間ロウソク足 |

## 注記
- Pythonバージョンは要請に従い意図的に省略されています。
- ベースクラスで`Volume`を使用してロットサイズを設定してください（指定がない場合はデフォルト1）。
- 戦略は元のEA設定と同じ時間軸で実行する必要があります（提供された`.set`ファイルによればEURUSD H4）。
