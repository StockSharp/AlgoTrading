# Polish Layer戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**Polish Layer戦略**は、`MQL/17484`のMetaTraderエキスパートアドバイザーをStockSharpの高レベルAPIに変換したものです。5分または15分足ローソク足を使用してフォレックスペアの短期トレンド継続を狙います。トレンドの方向は高速と低速の指数移動平均の関係と相対力指数（RSI）の最近のモメンタムによって定義されます。エントリー確認にはストキャスティクスオシレーター、DeMarker、Williams %Rからの同期シグナルが必要です。

## インジケーター
- **指数移動平均（EMA）** – 高速（`ShortEmaPeriod`）と低速（`LongEmaPeriod`）のトレンドフィルター。
- **相対力指数（RSI）** – 前のローソク足の値から導出されたモメンタム傾きフィルター。
- **ストキャスティクスオシレーター** – %Kの閾値クロスによって買われすぎ/売られすぎのリバーサルを検出。
- **DeMarker** – 蓄積/分配フェーズを確認。
- **Williams %R** – 極値レベルでのモメンタムリバーサルを検証。

## パラメーター
| パラメーター | デフォルト値 | 説明 |
|-----------|---------|-------------|
| `ShortEmaPeriod` | 9 | 高速EMAトレンドフィルターの長さ。 |
| `LongEmaPeriod` | 45 | 低速EMAトレンドフィルターの長さ。 |
| `RsiPeriod` | 14 | モメンタム傾き比較に使用するRSIルックバック。 |
| `StochasticKPeriod` | 5 | %Kラインのルックバック。 |
| `StochasticDPeriod` | 3 | %Dの平滑化期間。 |
| `StochasticSlowing` | 3 | %Kに適用される最終的な減速係数。 |
| `WilliamsRPeriod` | 14 | Williams %Rのルックバックウィンドウ。 |
| `DeMarkerPeriod` | 14 | DeMarkerのルックバックウィンドウ。 |
| `TakeProfitPoints` | 17 | 価格ポイントでの利益目標への距離（`Security.PriceStep`を使用）。 |
| `StopLossPoints` | 77 | 価格ポイントでの保護ストップへの距離。 |
| `CandleType` | 5分 | 戦略が処理するローソク足データタイプ。 |
| `Volume` | 1 | 成行エントリーに使用するトレードサイズ。 |

## トレードロジック
1. **トレンドフィルター** – ロングシナリオでは、前のローソク足が高速EMAが低速EMAの上にあり、RSIが上昇している（前のRSI > 2バー前のRSI）必要があります。逆の設定がショートシナリオを定義します。
2. **オシレーター確認** – 戦略がフラットで以下のすべての条件が満たされた場合にのみエントリーが考慮されます：
   - **ストキャスティクス %K** がロングでは19を上方クロス、ショートでは81を下方クロス。
   - **DeMarker** がロングでは0.35を上方クロス、ショートでは0.63を下方クロス。
   - **Williams %R** がロングでは-81を上方クロス、ショートでは-19を下方クロス。
3. **注文実行** – 戦略は`BuyMarket(Volume)`または`SellMarket(Volume)`を使用して成行注文を送信し、`StartProtection`に依存してストップロスとテイクプロフィットのオフセットを自動的に添付します。

## リスク管理
- 保護注文は`StartProtection`を通じて作成され、`TakeProfitPoints`と`StopLossPoints`をインストゥルメントの`PriceStep`に基づいて絶対的な価格距離に変換します。
- アルゴリズムは既存のポジションが保護注文によって閉じられるまで市場外に留まり、元のエキスパートアドバイザーの動作を反映します。

## 使用上の注意
- 5分または15分足ローソク足を持つ流動性の高いフォレックスペアで最も効果的に機能します。
- インストゥルメントのメタデータに有効な`PriceStep`が含まれていることを確認してください。そうでない場合は、インストゥルメントのティックサイズに合わせて`TakeProfitPoints`と`StopLossPoints`を調整してください。
- 確認シーケンスはインジケーターの平滑化とブローカーの価格刻みに敏感なため、ライブデプロイ前にフォワードテストを検討してください。
