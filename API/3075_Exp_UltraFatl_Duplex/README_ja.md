# Exp UltraFATL Duplex 戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
**Exp UltraFATL Duplex 戦略**は、MetaTrader 5のエキスパートアドバイザー `Exp_UltraFatl_Duplex` をC#に変換したものです。システムは2つの独立したUltraFATLインジケーターパイプラインを実行します：1つはロングの機会専用、もう1つはショートのセットアップ用に調整されています。各パイプラインは平滑化されたFATL値の階段を評価し、何段階が上昇または下降しているかをカウントします。強気と弱気のカウンターのバランスが次のトレードの方向を定義します。

## トレードロジック
1. 各方向ブロックの設定されたローソク足の時間軸にサブスクライブします。
2. FATLカーネル（39タップデジタルフィルター）で適用価格をフィルタリングします。
3. フィルタリングされたシリーズを、設定されたステップで長さが増加する移動平均の階段に通します。階段はユーザーが指定した平滑化方法を使用します。
4. 階段内の連続した値を比較して強気と弱気の票数をカウントします。両方のカウンターを2番目の移動平均で平滑化します。
5. 選択したシグナルシフト（デフォルト：完全に閉じた1本のローソク足）でカウンターを評価します：
   - **ロングブロック**は前のローソク足が強気の優位を示したが、現在のローソク足でカウンターが下向きにクロス（強気 ≤ 弱気）した場合にポジションを開きます。前のローソク足で弱気が強気を上回ったときにロングポジションを閉じます。
   - **ショートブロック**は逆方向で動作します：前のローソク足が弱気に支配され、現在のローソク足が上向きにクロス（強気 ≥ 弱気）した場合にショートを開きます。前のローソク足で強気がリードしたときにショートを閉じます。
6. オプションのストップロスとテイクプロフィットレベルは、銘柄の価格ステップを使用してローソク足データで評価されます。

戦略はネットポジションを強制します：ショートシグナルは開く前に既存のロングを閉じ、その逆も同様です。エントリーとエグジットには市場注文が使用されます。

## パラメーター
### ロングブロック
- **Long Volume** – ロングトレードを開くときの注文サイズ。
- **Allow Long Entries** – 新しいロングポジションを有効または無効にします。
- **Allow Long Exits** – 反対のシグナルでロングを閉じることを許可します。
- **Long Candle Type** – ロングUltraFATLパイプラインに使用する時間軸。
- **Long Applied Price** – FATLカーネルに供給する価格ソース（終値、典型値、DeMark など）。
- **Long Trend Method / Start Length / Phase / Step / Steps** – 階段の平滑化設定。
- **Long Counter Method / Counter Length / Counter Phase** – 強気/弱気カウンターの平滑化設定。
- **Long Signal Bar** – シグナルオフセットとして使用する完了ローソク足の数（1未満の値は1として扱われます）。
- **Long Stop (pts)** – 価格ステップ単位のオプションのストップロス距離。
- **Long Target (pts)** – 価格ステップ単位のオプションのテイクプロフィット距離。

### ショートブロック
ショートパイプラインの対称的な設定：**Short Volume**、**Allow Short Entries**、**Allow Short Exits**、**Short Candle Type**、**Short Applied Price**、**Short Trend Method / Start Length / Phase / Step / Steps**、**Short Counter Method / Counter Length / Counter Phase**、**Short Signal Bar**、**Short Stop (pts)**、**Short Target (pts)**。

## 実装上の注意
- 平滑化方法はStockSharpインジケーターにマッピングされます。Jurikベースのオプションは `JurikMovingAverage` を再利用します；`Parabolic` と `T3` などのメソッドは、元のカスタムカーネルが利用できないため、指数移動平均またはJurik移動平均で近似されます。
- ストップロスとテイクプロフィットレベルはローソク足の高値/安値で評価されます；サーバーサイドの保護注文ではありません。
- StockSharpポートは完了したローソク足にのみ反応するため、1バー未満のシグナルオフセットは再現できません。したがって、シグナルバーをゼロに設定すると、シフト1と同じように動作します。
- 両方のインジケーターパイプラインは、視覚的な検査のために専用のチャートエリアに平滑化されたカウンターを描画します。

## 使用方法
戦略をStockSharpソリューションに追加し、トレーディングプランに従って方向ブロックを設定し、Designer、Shell、またはRunnerで実行します。銘柄が必要なローソク足シリーズを提供し、`LongVolume`/`ShortVolume` パラメーターが目的の注文サイズに設定されていることを確認してください。
