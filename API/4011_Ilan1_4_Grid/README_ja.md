# Ilan 1.4 バスケットグリッド戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
Ilan 1.4 は、古典的な平均グリッド システムです。変換された戦略は、単一のローソク足シリーズをサブスクライブし、最後の 2 つの完了したローソク足の方向に基づいて初期市場ポジションをオープンします。新しい終値が古い終値を下回っている場合、バスケットは売りから始まり、そうでない場合は買いを開始します。構成された **ピップ ステップ**によって価格がアクティブ バスケットに対して移動すると、オプションでストラテジーは同じ方向に新しいポジションを追加し、加重平均エントリー価格を再計算します。

バスケット内のすべての取引は成行注文で実行されます。終値が平均エントリー価格に **利益確定距離** を加えた値に達すると、バスケット全体が閉じられます。トレーリング ストップ、固定ストップロス、株式ベースの緊急ストップ、最大ライフタイム セーフガードは、元の MetaTrader 専門家による保護ブロックを再現します。

## 取引ルール
1. 次に終了するローソク足を待ち、最後の 2 つの終値を評価します。
2. エクスポージャーがない場合は、最新の終値が前の終値よりも高い場合はロングバスケットをオープンし、それ以外の場合はショートバスケットをオープンします。
3. 最新の約定価格とアクティブなバスケットの加重平均エントリー価格を追跡します。
4. **追加の使用** が有効で、価格がポジションに対して **ピップ ステップ** ポイント移動した場合、次のロット サイズを計算し、追加の市場取引を開始します。 **追加する前に閉じる** が有効な場合、既存のバスケットが最初に閉じられ、スケールされたボリュームで再度開かれます。
5. 約定ごとに平均エントリー価格を再計算します。価格が平均テイクプロフィットレベルに達するか、リスクルールのいずれかが発動すると、バスケットは清算されます。
6. バスケットが閉じると、ロジックはすぐに最後の 2 つのローソク足の終値を使用して新しいシグナルを準備します。

## 資金管理モード
**Money Management** パラメータは、元の `MMType` スイッチを再現します。
- **修正** – すべての新しい注文は、設定された**初期ボリューム**を使用します。
- **ジオメトリック** – 後続の注文では、基本取引高に `LotExponent^n` を掛けます。ここで、`n` は現在のオープン取引数に等しくなります。
- **RecoverLastLoss** – バスケットに負けた後、次のポジションでは、最後に終了した取引のボリュームに**ロット指数**を乗じたものが使用されます。収益性の高いバスケットは、ボリュームを基準値にリセットします。

取引量は **出来高桁** と証券量ステップに従って四捨五入されます。丸めによりゼロが生成される場合、代わりに丸められていない入力ボリュームが使用されます。

## リスク管理
- **利益確定** – 価格が平均エントリー価格±設定ポイントに達するとバスケット全体を閉じます。
- **ストップロス** – 価格が平均エントリー価格に対して指定されたポイント数だけ変動したときにバスケットを閉じます。
- **トレーリング ストップ**を **トレイル スタート** および **トレイル ストップ** とともに使用する – バスケットが十分なポイントを獲得すると、トレーリング レベルがアクティブになります。利益を保護するために、トレーリングオフセットは価格に従います。
- **株式リスク %** で **株式ストップを使用** – ポートフォリオの価値を監視し、変動損失が記録された株式ピークの選択された割合を超えたときにバスケットを閉じます。
- **最大オープン時間** でタイムアウト**を使用する – バスケットが許容時間数を超えて開いたままになっている場合、バスケットを強制的に閉じます。

## パラメーター
- **ローソク足タイプ** – 取引シグナルの生成に使用される時間枠。
- **初期ボリューム** – 新しいバスケットの開始ロットサイズ。
- **体積桁数** – 計算された体積を四捨五入するときに使用される精度。
- **資金管理** – ボリューム計算モード (`Fixed`、`Geometric`、`RecoverLastLoss`)。
- **ロット指数** – 幾何学スキームおよび回復スキームによって適用される乗数。
- **追加する前に閉じる** – 次の平均化注文を行う前に、開いている取引をすべて閉じます。
- **追加を使用** – 平均注文を完全に有効または無効にします。
- **ピップステップ** – 新しい取引を追加する前の最小の逆方向の動き（価格ステップで）。
- **テイクプロフィット** – 平均エントリー価格からの利益目標。
- **ストップロス** – 平均エントリー価格から許容される最大の逆方向逸脱。
- **トレーリング ストップ / トレイル スタート / トレイル ストップを使用する** - トレーリング ストップ構成。
- **最大取引数** – バスケット内で許可される平均取引の最大数。
- **株式ストップ / 株式リスク %** を使用します – 浮動損失保護のパラメーター。
- **タイムアウト/最大営業時間を使用** – 各バスケットの寿命を制御します。

## 変換メモ
- 元のコードでは平均化ロジックが常に即時に実行されるため、MetaTrader 未決注文ヘルパーは直接成行注文に置き換えられました。
- 後続ブロックは、各注文を個別に変更するのではなく、集約されたバスケットで機能するようになりました。トリガー距離は元のデフォルトと一致します。
- ポートフォリオの資産は、エキスパートの資産停止ルーチンをエミュレートするために、StockSharp ポートフォリオ オブジェクトを通じて監視されます。
- ポジション平均とバスケット統計は、高レベルの API ガイドラインに従って、トレードごとのコレクションを保存せずにストラテジー内で計算されます。
