# Improve MA & RSI ヘッジ戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、高レベル API を使用してオリジナルの MetaTrader エキスパート「Improve」を StockSharp に移植したものです。戦略に選択されたメイン銘柄とヘッジ銘柄の2つの銘柄を同時に取引します。取引方向は、メイン銘柄における2つの平滑化移動平均と相対力指数（RSI）の関係によって定義されます。ヘッジの脚はメインの脚の方向を反映し、同期したモメンタムの動きから利益を得ながら単一銘柄のリスクを制限しようとするペアのエクスポージャーを作成します。

## 戦略ロジック

- 設定可能な高速および低速の期間でプライマリ銘柄に2つの平滑化移動平均（SMMA）を計算します。
- 同じロウソク足で RSI を計算し、売られすぎ/買われすぎのしきい値を監視します。
- 低速 SMMA が高速 SMMA を上回り、RSI が売られすぎのしきい値以下のとき、両方の銘柄で**ロング**に入ります。
- 低速 SMMA が高速 SMMA を下回り、RSI が買われすぎのしきい値以上のとき、両方の銘柄で**ショート**に入ります。
- 両方の脚の合計オープン利益が設定された金額目標を超えるまでポジションは開いたまま保持され、その時点で戦略は両側を清算します。

アルゴリズムは各銘柄の最新の終値を追跡します。合計利益は、現在の終値と各脚の保存されたエントリー価格の差から推定されます。ストップロスが適用されないため、価格が利益目標に達しない場合、ポジションは長期間開いたままになる可能性があります。

## パラメーター

| パラメーター | 説明 |
| --- | --- |
| **Volume** | プライマリとヘッジの両方の銘柄に対する注文数量。 |
| **Profit Target** | 両方の脚で共有される金額目標；達成されると戦略はすべてのオープンポジションを閉じます。 |
| **Hedge Security** | プライマリ銘柄と並行して取引されるセカンダリ銘柄。 |
| **Fast MA** | 高速平滑化移動平均の期間（デフォルト 8）。 |
| **Slow MA** | 低速平滑化移動平均の期間（デフォルト 21）。高速 MA 期間より大きくなければなりません。 |
| **RSI Period** | RSI の計算に使用される長さ（デフォルト 21）。 |
| **Oversold** | MA 条件と共にロングエントリーをトリガーする RSI レベル（デフォルト 30）。 |
| **Overbought** | MA 条件と共にショートエントリーをトリガーする RSI レベル（デフォルト 70）。 |
| **Candle Type** | 計算の時間軸；デフォルトで1時間ロウソク足ですが調整可能です。 |

## インジケーター

- **平滑化移動平均（SMMA）** – 高速と低速のトレンドコンポーネントを定義するために2回使用されます。
- **相対力指数（RSI）** – 確認のために売られすぎ/買われすぎ状態を決定します。

## エントリーと決済ルール

1. **ロングエントリー**
   - プライマリ銘柄で低速 SMMA &gt; 高速 SMMA。
   - RSI ≤ 売られすぎ。
   - 両方の脚が同じ方向の成行注文で開かれます（買い/買い）。
2. **ショートエントリー**
   - プライマリ銘柄で低速 SMMA &lt; 高速 SMMA。
   - RSI ≥ 買われすぎ。
   - 両方の脚が同じ方向の成行注文で開かれます（売り/売り）。
3. **決済**
   - `(プライマリ利益 + ヘッジ利益) ≥ Profit Target` のとき、戦略は成行注文を使用して両方のポジションを閉じます。
   - 追加のストップロスまたはトレーリングロジックは適用されません；必要な場合はリスク管理を外部で追加する必要があります。

## 使用上の注意

- 戦略を開始する前に、プライマリ銘柄とヘッジ銘柄の両方が割り当てられていることを確認してください；そうでなければ例外がスローされます。
- 合計利益の推定はロウソク足の終値に依存します。2つの脚間のスリッページと実行の違いが実際の実現利益に影響を与える可能性があります。
- 戦略は両方の脚を同時に開くため、同期した動きが期待される相関した銘柄（例えば通貨ペアや関連先物）に適しています。
- ライブ取引時にはポートフォリオレベルのリスク管理を追加することを検討してください。オリジナルのアルゴリズムは決済に仮想的な利益目標のみを使用しているためです。
