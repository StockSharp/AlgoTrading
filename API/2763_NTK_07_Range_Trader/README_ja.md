# NTK 07レンジトレーダー戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

NTK 07レンジトレーダー戦略は、MetaTraderのエキスパートアドバイザー「NTK 07」の移植版です。アルゴリズムは現在の市場価格の周囲に対称なストップ注文を維持し、設定可能なトレーリングとテイクプロフィットのロジックでオープンポジションを管理します。目標は、厳格なリスク管理を守りながら、短期価格レンジの境界または中心付近で発生するブレイクアウトを捉えることです。

## 核心的なアイデア

- **エントリートリガー** – 戦略がフラットな状態のとき、設定可能なルックバックレンジを評価します。価格がレンジの端またはその中間点付近にある場合 (選択したトレードモードによる)、価格ステップで定義されたオフセットでバイストップ注文とセルストップ注文の両方を置きます。
- **レンジ認識** – 直近 *N* 本の完了済みローソク足の最高値と最低値がトレーディングレンジを定義します。長さゼロはフィルターを無効にし、すぐに注文を置くことを許可します。
- **適応型リスク** – 各エントリーはベースボリュームを使用し、オプションのロット乗数はポジションが開かれた後、追加のストップ注文をピラミッドにできます。ポートフォリオ全体のボリューム制限は、エクスポージャーが上限を超えるときに新しい注文をブロックします。
- **退場管理** – ポジションが成立すると、反対のストップ注文がキャンセルされます。戦略は設定されたオフセットを使用してプロテクティブストップとオプションのテイクプロフィット注文を登録します。トレーリングは、前のローソク足の高値/安値、移動平均、または固定距離バッファを追跡できます。
- **セッションフィルター** – 取引は選択された開始時刻と終了時刻の間のみ許可され、週末は自動的に無効になります。

## パラメーター

| パラメーター | 説明 |
|-----------|-------------|
| **Entry Volume** | 各エントリー注文のベースサイズ。 |
| **Total Volume Limit** | 最大累積ポジションサイズ。`0` の値は上限を無効にします。 |
| **Net Step** | 市場とエントリーストップ注文間の価格ステップ距離。 |
| **Stop Loss** | エントリー価格に対して相対的な価格ステップでの初期ストップロスオフセット。 |
| **Take Profit** | 価格ステップでのテイクプロフィット距離。利益目標を無効にするには `0` に設定。 |
| **Trailing Stop** | トレーリングロジックに使用する価格ステップ距離。 |
| **Lot Multiplier** | 既存のポジションにピラミッド化するときに適用される乗数。 |
| **Trail High/Low** | 有効にすると、プロテクティブストップが前のローソク足の極値を追跡します。 |
| **Trail Moving Average** | 移動平均値を使用したトレーリングを有効にします。1つのトレーリングモードのみがアクティブにできます。 |
| **Trading Start/End Hour** | 取引のための包括的なプラットフォーム時間ウィンドウ。 |
| **Range Bars** | トレーディングレンジの計算に使用する完了済みローソク足の数。`0` はフィルターを無効にします。 |
| **Trade Mode** | `EdgesOfRange` は価格がレンジ境界に触れることを要求し、`CenterOfRange` は価格がレンジ中間点付近になるまで待ちます。 |
| **MA Period** | トレーリングに使用する移動平均の長さ。 |
| **Candle Type** | すべての計算に使用するローソク足の集計。 |

## ワークフロー

1. **データサブスクリプション** – 戦略は設定されたローソク足シリーズをサブスクライブし、移動平均と選択したレンジ長にわたる最高値・最低値を計算します。
2. **フラット状態** – ポジションが開いていない間、戦略はレンジ条件を評価します。満たされると、グローバルボリューム制限を尊重しながら、指定されたオフセットでバイストップとセルストップの注文ペアを置きます。
3. **ポジション処理** – エントリーが成立すると、反対のストップがキャンセルされます。戦略はすぐにプロテクティブストップロスとオプションのテイクプロフィット注文を置きます。トレーリングロジックは完了した各新しいローソク足でプロテクティブストップを更新します。
4. **ピラミッド化** – ロット乗数が `1` より大きい場合、合計ボリューム制限が許す限り、現在のポジションの方向に追加のストップ注文が置かれます。
5. **退場** – ストップまたはテイクプロフィットがポジションをフラット化し、残りの保護注文をキャンセルします。システムは次のレンジ相互作用の監視に戻ります。

## 注意事項

- 戦略はすべて価格ステップで動作するため、異なるティックサイズを持つインストゥルメントに適しています。
- 元のMQL実装の動作を反映するため、取引は土曜日と日曜日に自動的に無効になります。
- 一度に1つのトレーリングモードのみ有効にできます。両方を有効にすると起動時に設定エラーが発生します。
