# N Candles v5戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

## 概要
N Candles v5戦略は同一ローソク足の連続を探し、必要な連続が現れるとすぐに同じ方向
でトレードを開きます。Vladimir KarputovによるオリジナルのMQL実装はStockSharpの
高レベルAPIに移植されています。この戦略は完了したローソク足のみで動作し、あらゆる
時間軸で実行できますが、StockSharpバージョンのデフォルトは1時間ローソク足です。

## 取引ロジック
1. ローソク足がクローズすると、戦略はそれを陽線（終値が始値を上回る）、陰線
   （終値が始値を下回る）、または中立（終値が始値と等しい）として分類します。
2. 連続する陽線は陽線ストリークカウンターを増加させ、陰線カウンターをリセットします。
   陰線の場合はその逆です。中立のローソク足は両方のカウンターをリセットします。
3. 陽線ストリークカウンターが設定された`CandlesCount`値に達し、現在のネットポジション
   がフラットまたはショートの場合、戦略は成行買いを送信します。ショートのエクスポージャー
   が最初にカバーされ、次に設定された`TradeVolume`が追加されてロングポジションが確立されます。
4. 陰線ストリークカウンターが`CandlesCount`に達し、ポジションがフラットまたはロングの場合、
   戦略は成行で売り、ショートに入る前にロングのエクスポージャーを最初にカバーします。
5. トレードは`StartHour`と`EndHour`によって定義されるオプションの取引セッションウィンドウ内
   でのみ開かれます。保護アクション（テイクプロフィット、ストップロス、トレーリング）はポジション
   が安全に処理されるよう、セッション外でも引き続き動作します。
6. 戦略はMQLバージョンのボリューム保護を反映して、`MaxNetVolume`を超えてエクスポージャー
   を増加させることを拒否します。

## リスク管理
- **テイクプロフィット / ストップロス** – pipsで表現され、証券の価格ステップを使用して
  絶対価格距離に変換されます。両方のレベルはオプションであり、対応する値をゼロに設定することで
  無効にできます。
- **トレーリングストップ** – エントリー価格から`TrailingStopPips`だけ価格が進んだ後に
  有効になります。有効になると、取引方向にさらに`TrailingStepPips`動くたびにストップが
  引き締められます。
- **セッションフィルター** – `UseTradingHours`により開始・終了時間フィルターが有効になり、
  選択したウィンドウ外での新規エントリーを防ぎながら、リスク管理はポジションをクローズし続けます。
- **最大ネットボリューム** – 絶対ポジション（ロングまたはショート）は`MaxNetVolume`を
  超えることは決してできません。

## パラメーター
| パラメーター | 説明 | デフォルト |
|-------------|------|-----------|
| `TradeVolume` | 新規エントリーに使用される注文サイズ。 | `1` |
| `CandlesCount` | シグナルに必要な連続同一ローソク足の数。 | `3` |
| `TakeProfitPips` | pipsでのテイクプロフィット距離（0で無効）。 | `50` |
| `StopLossPips` | pipsでのストップロス距離（0で無効）。 | `50` |
| `TrailingStopPips` | トレーリングストップを有効にする距離（0で無効）。 | `10` |
| `TrailingStepPips` | トレーリングストップを引き締める前に必要な追加の進行。 | `4` |
| `UseTradingHours` | 取引時間フィルターを有効にする。 | `true` |
| `StartHour` | 新しいポジションが許可される最初の時間（0–23）。 | `11` |
| `EndHour` | 新しいポジションが許可される最後の時間（0–23）。 | `18` |
| `MaxNetVolume` | 許可される最大絶対ポジションサイズ。 | `2` |
| `CandleType` | 分析するローソク足データタイプ。デフォルトは1時間ローソク足。 | `TimeSpan.FromHours(1)` |

## 使用ノート
- 戦略は高レベル`SubscribeCandles` APIを通じてローソク足データを購読し、ローソク足
  シリーズを提供するあらゆるインストゥルメントで動作します。
- ロジックは完了したバーに依存するため、クローズ間のマーケットノイズが少ない
  イントラデイまたはより上位の時間軸に最も適しています。
- インストゥルメントのティックサイズに応じてpipベースのリスク設定を調整してください。
- スプレッド差が大きいインストゥルメントに展開する場合、通常のスプレッド拡大に
  よってストップが引き起こされないように、トレーリングストップのパラメーターを確認してください。
