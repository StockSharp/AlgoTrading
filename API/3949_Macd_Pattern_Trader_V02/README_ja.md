# Macd パターン トレーダー v02 (StockSharp ポート)
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、MetaTrader エキスパート **MacdPatternTraderv02.mq4** (ディレクトリ `MQL/8194`) の StockSharp の高レベルの API 変換です。オリジナルの MACD パターン検出とアクティブな位置管理ルールを再現し、さらに最適化するための便利なパラメーターを公開します。

## コアアイデア

1. 信号の長さが 1 つのローソク足 (MQL バージョンと一致) の高速および低速の EMA 期間 (`FastEmaPeriod`、`SlowEmaPeriod`) を使用して、MACD メイン ラインを計算します。
2. 完了したキャンドルのみを監視します。 MACD 値がゼロ線の周囲に特定の 3 小節シーケンスを描画する場合、短いパターンまたは長いパターンのいずれかを準備します。
   - **短いパターン**: 正の MACD フェーズに続いて、`MinThreshold` を超える負のプルバック、そして下向きの変曲点が必要です。
   - **ロング パターン**: 負の MACD フェーズに続いて、`MaxThreshold` を下回る正の引き戻し、その後上向きの変曲点が必要です。
3. パターンが確認されたら、`TradeVolume` を使用して成行注文を実行します。
4. 最近のスイングの極値 (`StopLossBars` ローソク足以上) を超えてストップロスを設定し、ポイント単位の追加オフセット (`OffsetPoints`) を加えて各ポジションを保護します。
5. 連続する `TakeProfitBars` セグメントをスキャンし、シーケンスが新しい極値を出力し続ける間に到達した最も極値の高値/最安値を選択することにより、テイクプロフィットレベルを定義します。
6. 元のエキスパートのアクティブなポジション マネージャーでオープン ポジションを管理します。最低 5 ポイントの利益が達成された後、戦略は前のローソク足がトレンドを確認したときに取引高の 3 分の 1 を決済し (`Ema2Period` フィルター)、価格が `SmaPeriod` と `Ema3Period` の正中線と相互作用したときに残りの半分を決済します。

## パラメーター

| パラメータ | 説明 |
|-----------|-------------|
| `StopLossBars` | ストップロススイングの極値を計算する際に検査される完了したローソク足の数。 |
| `TakeProfitBars` | テイクプロフィットターゲットを構築する逐次極値検索のウィンドウサイズ（ローソク単位）。 |
| `OffsetPoints` | インストゥルメントポイントで表される追加のオフセットがストップロスに追加されます。 |
| `FastEmaPeriod` | MACD 幹線の高速 EMA 長さ。 |
| `SlowEmaPeriod` | MACD 本線の EMA の長さが遅いです。 |
| `MaxThreshold` | 短いパターンの準備を終了する正の MACD しきい値。 |
| `MinThreshold` | 長いパターンの準備を終了する負の MACD しきい値。 |
| `Ema1Period` | 元の資金管理ブロックで使用される最初の EMA 期間 (完全を期すために保持)。 |
| `Ema2Period` | ロング/ショート ポジションの部分利益を検証するために使用される 2 番目の EMA 期間。 |
| `SmaPeriod` | 2 番目の部分クローズトリガーで使用される SMA 期間。 |
| `Ema3Period` | 低速の EMA 期間を SMA と組み合わせて、平均回帰出口を検出します。 |
| `TradeVolume` | 成行注文量 (ロット)。 |
| `CandleType` | すべてのインジケーターにフィードするために使用されるローソク足のデータ型。 |

## 取引ロジック

- **短いエントリ**: MACD シーケンス `(prev3, prev2, prev1, current)` が元の条件 (`macdPrev1 < macdPrev3`、`macdPrev1 > macdPrev2`、`current < prev1`、`current < 0`、およびマグニチュード チェック) と一致する場合にトリガーされます。既存のロングエクスポージャーは、新しいショートポジションを開く前にフラット化されます。
- **長いエントリ**: `current > 0`、MACD の値が鏡像パターンを形成し、マグニチュード チェックが満たされる対称ルール。既存のショートエクスポージャは、新しいロングポジションをオープンする前にフラット化されます。
- **ストップとターゲット**: 各エントリーの直後に計算され、新たな取引が実行された場合にのみ更新されます。
- **部分クローズ**: 利益が 5 ポイント (商品ポイント サイズと比較して) に達すると、前のローソク足が `EMA2` を超えて終了した場合、ストラテジーは残りのボリュームの 3 分の 1 をクローズします。次のステージでは、前のローソク足が `SMA` と `EMA3` の平均を突き抜けたときに残りのボリュームの半分を閉じます。
- **フルエグジット**: ストップロスまたはテイクプロフィットレベルの価格タッチにより、ポジション全体がクローズされます。強制終了するたびに、内部状態が自動的にリセットされます。

## 注意事項

- ポイント サイズは、`Security.PriceStep` から導出されます。使用できない場合は、セキュリティ小数点から導出されます。デフォルト値の `0.0001` は安全なフォールバックとして使用されます。
- MQL ヘルパー関数 `iHighest`、`iLowest`、および `TakeProfit()` からの連続極値スキャンを複製するために、ローソク足の履歴 (最大 1024 エントリ) が保存されます。
- リポジトリ ガイドラインの要求に従って、戦略内のすべてのコメントは英語のままです。
- このタスクでは、Python ポートは意図的に省略されています。
