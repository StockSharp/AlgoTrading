# Macd パターン トレーダーすべて v0.01
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は、「MacdPatternTraderAll v0.01」MetaTrader エキスパート アドバイザーを複製します。同じローソク足ストリームで 6 つの独立した MACD ベースのエントリー パターンを実行し、適応的なストップロスとテイクプロフィット レベルでリスクを管理し、段階的な利益確定を実行し、オプションで損失サイクル後にスロー マーチンゲール サイズ ルールを適用します。

## コア機能

- **6 つの MACD セットアップ** – すべてのパターンは、独自の高速/低速 EMA 期間としきい値レベル (`Pattern1` … `Pattern6`) を使用します。パターンは個別にオンまたはオフに切り替えることができます。
- **動的なリスク レベル** – ストップロス レベルは構成可能なオフセットを使用して最近の高値/安値から導出されますが、テイクプロフィット レベルは元の MQL 実装を反映するために連続したバー ブロックにわたって反復されます。
- **セッション フィルター** – `UseTimeFilter` が有効な場合、戦略は構成可能な `StartTime` / `StopTime` ウィンドウ内でのみ取引されます。
- **部分的なエグジット** – 元の `ActivePosManager` ロジックに従い、EMA/SMA フィルターで勢いが確認されると、収益性の高いポジションは 2 つのステップでスケールアウトされます。
- **遅いマーチンゲール** – `UseMartingale` が true の場合、損失サイクル後に次の取引サイズが 2 倍になり、利益サイクル後にリセットされます。

## パターン別エントリーロジック

1. **パターン 1 (タグ `Pattern1`)**
   - MACD メインラインが `Pattern1MaxThreshold` を上に押し上げた後、ショートアームし、その後、低い高いシーケンスでロールオーバーします。
   - `Pattern1MinThreshold` より下に伸びた後、アームを長く伸ばし、より高い低音シーケンスを生成します。
2. **パターン 2 (タグ `Pattern2`)**
   - ゼロライン付近の振動をカウントします。ショートは、ポジティブスイングが `Pattern2MinThreshold` 付近で失敗したときにトリガーされます。ロングは、マイナスのスイングが `Pattern2MaxThreshold` 付近でフェードアウトするときに表示されます。このアルゴリズムは、絶対値 MACD (`valueMin2` / `valueCurr2`) を比較することによって、元の距離チェックを再現します。
3. **パターン 3 (タグ `Pattern3`)**
   - 最大 3 つの下降 (または上昇) MACD トップを追跡して、「トリプル フック」を検出します。すべての中間しきい値 (`Pattern3MaxThreshold`、`Pattern3MaxLowThreshold`、`Pattern3MinThreshold`、`Pattern3MinHighThreshold`) が一致する場合にのみ、新しいポジションが許可されます。
4. **パターン 4 (タグ `Pattern4`)**
   - `Pattern4MaxThreshold` / `Pattern4MinThreshold` の外側で MACD のスパイクが発生し、その後新たな極端値を設定しようとして失敗するかどうかを監視します。追加のカウンター (`Pattern4AdditionalBars`) は互換性のために保存されています。
5. **パターン 5 (タグ `Pattern5`)**
   - エキスパートアドバイザーで使用されるニュートラルゾーンブレイクアウトを実装します。ショートでは、`Pattern5MinThreshold` より下からニュートラル ゾーン内にリバウンドして、もう一度失敗する必要があります。ロングは、`Pattern5MaxThreshold` の周りのミラーリングされたシーケンスに従います。
6. **パターン 6 (タグ `Pattern6`)**
   - しきい値レベルを上回る/下回る連続したバーの数をカウントします。買われ過ぎ/売られ過ぎエリア内で `Pattern6TriggerBars` を超えて支出し、しきい値を下回ったり上回ったりした後、`Pattern6MaxBars` がシグナルをブロックしない限り、この戦略は取引を開始します。

各パターンはヘルパー メソッド `TryOpenLong` / `TryOpenShort` を使用し、注文が発行される前にストップとターゲットが計算されることを保証します。

## リスクと貿易の管理

- **ストップロス**: `CalculateStopPrice` は、最新の `stopBars` の終了したローソク足 (アクティブなローソク足を除く) をスキャンし、設定されたポイント `offset` を適用します。価格は、MQL バージョンと同様に、10 進数の 3/5 の商品に合わせて調整されます。
- **テイクプロフィット**: `CalculateTakeProfit` は、新しい極値が見つからなくなるまで、`takeBars` ローソク足の連続ブロックを探索し、元のコードのネストされた `iLowest` / `iHighest` ループを模倣します。
- **部分エグジット**: 価格が `ema2` で確認されたとき、`ManageActivePositions` は `ProfitThreshold` の利益でポジションの 3 分の 1 をクローズします。価格が結合された `(sma3 + ema4) / 2` フィルターに達すると、2 番目の半分のサイズの exit が起動されます。
- **ハードエグジット**: `CheckRiskManagement` は、保存されているストップロスまたはテイクプロフィットのレベルに達すると、完全な市場エグジットを発行します。
- **Martingale コントロール**: `OnOwnTradeReceived` は、現在の横ばいから横ばいまでのサイクルで実現した損益を累積します。ポジションがフラットに戻ると、`AdjustVolumeOnFlat` は、利益後に取引高を `InitialVolume` にリセットするか、`UseMartingale` が有効になっている場合は損失後に取引量を 2 倍にします。

## パラメーター

すべての構成ノブは、StockSharp デザイナーでの最適化のために、`StrategyParam<T>` プロパティを通じて公開されます。

- **一般**: `CandleType`、`InitialVolume`、`UseTimeFilter`、`StartTime`、`StopTime`、`UseMartingale`。
- **パターン 1 ～ 6**: ストップロス/テイクプロフィットバーのカウント、オフセット、MACD の速い/遅い期間、および MQL スクリプトからの外部入力に一致するしきい値レベル。
- **位置マネージャー**: 部分終了フィルターで使用される EMA/SMA の長さ (`EmaPeriod1`、`EmaPeriod2`、`SmaPeriod3`、`EmaPeriod4`)。

すべてのデフォルト値は、`MacdPatternTraderAll v0.01` の `extern` 変数を反映します。

## 使用上の注意

- この戦略では、オフセットを正しく計算するために、有効な `PriceStep` および `Decimals` を持つシンボルが必要です。
- `CandleType` 経由でローソク足シリーズを指定します (例: `TimeSpan.FromMinutes(5).TimeFrame()`)。
- 複数のパターンが同時にトリガーされる場合、すべてのエントリーコールが希望の組み合わせのボリュームを再計算し、反対側のストップをクリアするため、ストラテジーは 1 つのポジションのみをオープンします。
- 段階的終了ロジックは集約されたポジションで機能するため、複数のパターンが同じ取引方向を共有している場合でも、部分的な決済が発生します。
