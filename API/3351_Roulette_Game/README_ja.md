# ルーレットゲーム
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

ルーレット ゲーム戦略は、MetaTrader のカジノのようなエキスパート アドバイザーを StockSharp 内に再現します。完成したすべてのローソク足をホイールの新しいスピンとして扱い、ランダムな方向を選択し、損失後に Martingale スタイルの進行を使用して注文サイズを調整します。この実装では、仮想バンクロールを追跡し、構成可能な上限を通じてエクスポージャーを制限します。

すべてのラウンドは、既存のポジションをフラット化し、仮想コインを投げて赤または黒を表し、選択した方向に成行注文を送信することから始まります。次のローソク足が閉じると、戦略はその終値が賭けに有利に動いたかどうかをチェックします。勝利すると賭け金が基本量にリセットされ、負けると賭け金が規定の上限まで増加します。最大連敗ガードにより、危険が極端になる前に強制的にリセットされます。オプションのクールダウン キャンドルをラウンド間に挿入して、賭けのペースを遅くすることができます。

この変換は、インジケーターシグナルではなく、元の専門家によって紹介されたギャンブルにインスピレーションを受けた資金管理に焦点を当てています。時間ベースのラウンドを調整し、内部状態を維持し、キャンドル サブスクリプションを通じて StockSharp の高レベルの API と対話する方法を示します。

## 詳細

- **エントリー基準**: 技術的なフィルターはありません。方向は完成したキャンドルの終わりにランダムに選択されます。
- **ロング/ショート**: 両方向、ラウンドごとにランダムに選択されます。
- **終了基準**: ポジションは次の終了ローソク足でクローズされ、価格がエントリーを上回ったか下回ったかを評価します。
- **停留所**: 従来の停留所はありません。リスクはステークキャップとストリーク制限で管理されます。
- **デフォルト値**:
  - `BaseVolume` = 1m
  - `LossMultiplier` = 2m
  - `MaxMultiplier` = 16m
  - `RoundCooldown` = 1
  - `MaxLosingStreak` = 5
  - `CandleType` = TimeSpan.FromMinutes(1)
- **フィルター**:
  - カテゴリー: お金の管理
  - 方向: 両方
  - インジケーター: なし
  - 停車駅: いいえ
  - 複雑さ: 初心者
  - 期間: 短期
  - 季節性: いいえ
  - ニューラルネットワーク: いいえ
  - 発散: いいえ
  - リスクレベル: 高

## 注意事項

- 成行注文は、乗数で調整された賭け金に応じてサイズ設定され、商品の出来高ステップに四捨五入されます。
- 勝利すると賭け金が基本量にリセットされます。損失は​​、最大乗数または連敗制限に達するまで、乗数によってスケールされます。
- クールダウン バーにより、即時再入力が防止され、より遅いデータ フィードとの同期が可能になります。
