# スターター戦略 (Starter)
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

**スターター戦略 (Starter)** はMetaTrader 5エキスパートアドバイザー「Starter (barabashkakvn's edition)」の変換です。このシステムは
商品チャンネル指数（CCI）が極端な売られ過ぎまたは買われ過ぎの領域からリバウンドするのを待ち、長期移動平均の傾きで動きを確認します。
モメンタムがトレンドフィルターと一致すると、戦略はポートフォリオの設定可能なリスク割合によって決定されるサイズの単一市場ポジションを
開きます。保護ストップとオプションのトレーリングメカニズムは、元のエキスパートの資金管理ルールを再現します。

## 取引ロジック

- **トレンドフィルター** — 設定可能な移動平均（MA）は、ロング取引を許可するために `MaDelta` より速く上昇し、ショート取引を許可するために
  `MaDelta` より速く下降する必要があります。この戦略はMQLバージョンと同じ平滑化方法（シンプル、指数、スムーズ、線形加重）をサポートします。
- **CCI確認** — 商品チャンネル指数は、ロングエントリーを発動するために下から `-CciLevel` を上回って再びクロスし、ショートを発動するために
  上から `CciLevel` を下回って再びクロスする必要があります。インジケーターは完成したローソク足のみで評価され、元のバーバイバー処理を
  反映します。
- **単一ポジションモデル** — アルゴリズムは最大1つのオープンポジションを維持します。現在の取引が閉じられるまで新しいシグナルは無視され、
  マジックナンバーとシンボルでフィルタリングするMetaTraderロジックに一致します。

### エントリールール

1. ローソク足のクローズを待つ。
2. 設定されたシフトで移動平均の最新と前の値を計算する。
3. CCIの現在と前の読み取り値を計算する。
4. 以下の場合に**ロングに入る**:
   - 移動平均の傾きが `MaDelta` を超える（現在のMAマイナス前のMA）。
   - 現在のCCI値が前の値より大きい。
   - CCIが `-CciLevel` を上方にクロスする（前が閾値を下回り、現在が上回る）。
5. 以下の場合に**ショートに入る**:
   - 移動平均の傾きが `-MaDelta` を下回る。
   - 現在のCCI値が前の値より小さい。
   - CCIが `CciLevel` を下方にクロスする（前が閾値を上回り、現在が下回る）。

### エグジットルール

- **初期ストップロス** — `StopLossPips` がゼロより大きい場合、執行されたエントリー価格が `StopLossPips * PriceStep` だけオフセットされて
  初期保護ストップが計算されます。
- **トレーリングストップ** — `TrailingStopPips` と `TrailingStepPips` の両方が正の場合、価格が少なくとも設定されたステップ分改善されると
  常にストップが前進します。ロング取引はストップを `Close - TrailingStop` に移動し、ショートは `Close + TrailingStop` に移動します。
- **手動エグジット** — ローソク足の範囲内でストップレベルに価格が触れると、戦略は成行注文でポジションをクローズし、保護状態をリセットします。

## リスク管理

- **ポジションサイジング** — ベースボリュームは `Portfolio.CurrentValue * MaximumRisk / price` です。ブローカーまたはバックエンドが
  無効な資本価値を報告すると、戦略は手動の `Volume` プロパティ（デフォルト1）にフォールバックします。
- **損失連続後の削減** — 2つ以上の連続する損失取引の後、ボリュームは `volume * losses / DecreaseFactor` だけ削減され、元の
  `DecreaseFactor` ルールを模倣します。どんな利益ある取引も損失カウンターをリセットします。

## パラメーター

| パラメーター | デフォルト | 説明 |
|-------------|-----------|------|
| `MaximumRisk` | `0.02` | ポジションをサイジングする際に取引ごとにリスクにさらされる資本の割合。 |
| `DecreaseFactor` | `3` | 2つ以上の連続する損失取引の後に適用されるロット削減除数。 |
| `CciPeriod` | `14` | 商品チャンネル指数が使用するバー数。 |
| `CciLevel` | `100` | CCIクロスの売られ過ぎ/買われ過ぎ閾値。 |
| `CciCurrentBar` | `0` | 現在のCCI値のシフト（0 = 最新ローソク足）。 |
| `CciPreviousBar` | `1` | 前のCCI値のシフト。 |
| `MaPeriod` | `120` | 移動平均トレンドフィルターの期間。 |
| `MaMethod` | `Simple` | 移動平均平滑化方法（Simple, Exponential, Smoothed, LinearWeighted）。 |
| `MaCurrentBar` | `0` | 移動平均値に適用されるシフト。 |
| `MaDelta` | `0.001` | 現在と前のMA読み取り値の間の最小傾き差。 |
| `StopLossPips` | `0` | ピップでの初期ストップロス距離（0はストップを無効にする）。 |
| `TrailingStopPips` | `5` | ピップでのベーストレーリングストップ距離（0はトレーリングを無効にする）。 |
| `TrailingStepPips` | `5` | トレーリングストップが前進する前の最小ピップ改善。 |
| `CandleType` | `30m` 時間軸 | 戦略によって処理される主要なローソク足サブスクリプション。 |

## 実装上の注意

- インジケーターバッファは内部でキャッシュされるため、戦略は任意のシフトで履歴値にアクセスでき、インジケーター配列をインデックス化する
  MQLアプローチを複製します。
- ピップサイズは `Security.PriceStep` から派生します。インジケーターが有効な価格ステップを報告しない場合、ストップとトレーリング距離は
  ゼロとして扱われます。
- コード内のすべてのコメントはリポジトリのガイドラインに従って英語で書かれています。
- Pythonバージョンは要求どおり意図的に省略されています。
