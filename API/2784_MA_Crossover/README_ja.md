# MA クロスオーバー マルチ時間軸戦略
[English](README.md) | [Русский](README_ru.md) | [中文](README_zh.md) | [Español](README_es.md) | [Deutsch](README_de.md) | [Português](README_pt.md)

この戦略は MetaTrader 4 の元の **MA Crossover** エキスパートアドバイザーのアイデアを再現しています。異なる時間軸から来る可能性のある 2 つの移動平均を比較します。強気のクロスオーバー（速い MA が遅い MA より上）でロングポジションを開き、弱気のクロスオーバーでショートポジションを開きます。オプションのフィルターは、許可された取引方向、アクティブな取引スケジュール、および資産ガードを制御します。内部のストップロス、テイクプロフィット、トレーリングロジックは MQL バージョンの「隠れた」決済を模倣します。

## トレードロジック

1. 2 つのローソク足ストリーム（現在および前の時間軸）を購読し、選択されたタイプの移動平均を計算します。
2. 比較前に移動平均値に設定されたバーシフトを適用します。
3. 未完成のローソク足を無視し、両方の移動平均が形成されるまで待ちます。
4. 設定された日付/時間ウィンドウ外や資産ガードが発動した場合は取引をスキップします。
5. 強気のクロスオーバー時：
   - `ClosePositionsOnCross = true` の場合、オプションでショートポジションをクローズ。
   - ロング取引が許可されている場合はロングポジションを開く。
6. 弱気のクロスオーバー時：
   - `ClosePositionsOnCross = true` の場合、オプションでロングポジションをクローズ。
   - ショート取引が許可されている場合はショートポジションを開く。
7. エントリー価格のパーセンテージとして表現されたストップロス、テイクプロフィット、トレーリングルールでオープンポジションを管理します。

## パラメーター

| パラメーター | 説明 |
|------------|------|
| `AllowedDirection` | 取引方向フィルター（`LongOnly`、`ShortOnly`、`LongAndShort`）。 |
| `ClosePositionsOnCross` | 新しい取引を開く前にクロスオーバー時に逆方向のポジションをクローズ。 |
| `MaType` | 移動平均計算タイプ（`Simple`、`Exponential`、`Smoothed`、`Weighted`）。 |
| `CurrentMaPeriod` | 速い移動平均の期間。 |
| `PreviousPeriodAddition` | 遅い移動平均に追加される余分な長さ（`PreviousMaPeriod = CurrentMaPeriod + addition`）。 |
| `CurrentShift` / `PreviousShift` | 移動平均値を後ろにシフトするために使用する完成バーの数。 |
| `CurrentCandleType` / `PreviousCandleType` | 速い移動平均と遅い移動平均のためのローソク足データ。 |
| `StopLossPercent` | エントリー価格のパーセンテージでのストップロス距離（隠れた決済）。 |
| `TrailingStopPercent` | 達成された最良価格に基づくパーセンテージでのトレーリングストップ距離。 |
| `TakeProfitPercent` | エントリー価格のパーセンテージでのテイクプロフィット距離（隠れた決済）。 |
| `StartDay` / `EndDay` | 取引活動の曜日フィルター。 |
| `StartTime` / `EndTime` | 新しい取引を開くためのイントラデイ時間ウィンドウ。 |
| `ClosePositionsOnMinEquity` | 資産ガードが発動したときにすべてのポジションをクローズ。 |
| `MinimumEquityPercent` | 資産ガードで許可される初期ポートフォリオ価値の最小パーセンテージ。 |

## リスク管理

- 戦略はストップロス、テイクプロフィット、トレーリングレベルを内部で計算し、成行注文を通じて決済します。これは MQL スクリプトの隠れた保護ロジックを模倣しています。
- `MinimumEquityPercent` は起動時の初期ポートフォリオ価値を保存し、資産がしきい値を下回った場合に強制的な平坦化を引き起こす可能性があります。
- ポジションサイズは基本の `Strategy.Volume` プロパティを通じて制御されます。デフォルトのボリュームは `1` に設定されています。

## 使用上の注意

- 戦略は両方の設定された時間軸のローソク足データを必要とします。関連するコネクタが要求された時間軸をサポートしていることを確認してください。
- 両方の移動平均が同じ時間軸を使用している場合でも、戦略はロジックを対称に保つために 2 つのストリームを購読します。
- ストップとテイクプロフィットの決済は成行注文で実行されるため、注文帳に保護注文が残りません。
- パラメーターは元の MQL エキスパートアドバイザーの主要な入力に対応していますが、ブローカー固有の関数に依存するリスク/マージン管理機能（ヘッジ、アベレージング）は意図的に省略されています。

## MQL バージョンとの違い

- アベレージング機能（`Average_Up`、`Average_Down`）とヘッジ設定は、ロジックを StockSharp 高レベル API と互換性を保つために実装されていません。
- 資産ガードは、フリーマージン固有の計算の代わりに StockSharp のポートフォリオ価値を使用します。
- リスク決済はローソク足クローズイベントで成行注文によって実行されるため、常に注文帳から隠されます。
